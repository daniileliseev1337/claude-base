# 1С:ERP integration tools — harvest 2026-05-26

## Контекст

Строительная фирма использует **1С:ERP** (платформа 1С:Предприятие 8.3).
Задача — поднять **read-only** аналитические дашборды для руководства:
финансы (выручка, ДЗ/КЗ, прибыль), проекты (статусы договоров, КС-2/КС-3,
план/факт), сотрудники (зарплаты, KPI).

Цель harvest — найти готовые инструменты под эту задачу, чтобы не писать
с нуля. Бюджет — 30 минут, фокус на 6 категориях: open-source библиотеки,
BI-коннекторы, SaaS, расширения конфигурации, ETL, «под ключ».

Главный вывод: **рынок зрелый**. Есть и open-source обвязки OData, и
коммерческие BI-коннекторы (АТК, Денвик, BI Consult, Bitrix-Бит),
и встроенная 1С:Аналитика, и недавно появилась мощная MCP-экосистема
(40+ серверов в каталоге `Untru/1c-mcp`, 94★, активный 2026).

## Найденные инструменты

### Категория 1. Open-source Python/JS библиотеки

#### belov38/1c-odata

- **URL:** https://github.com/belov38/1c-odata
- **License:** Unlicense (= public domain, безопасно для использования)
- **Активность:** ★ 8, обновления редкие, но базовая функциональность стабильна
- **Что делает:** Python-обёртка над OData v8 1С. Классы `Infobase`,
  `Document`, `Catalog`; CRUD по справочникам и документам, фильтры,
  пагинация, выбор полей.
- **Подходит для нас:** **частично.** Для прототипа дашборда —да:
  быстро вытащить справочники/документы и сложить в pandas. Для production
  на 1С:ERP — слабо: 8★ и редкие коммиты значат отсутствие сообщества;
  на больших таблицах ERP (регистры накопления, ТЧ документов) могут
  быть проблемы с производительностью.
- **Verdict:** **candidate для прототипа**, не для production.

#### tuomur/python-odata

- **URL:** https://github.com/tuomur/python-odata
- **License:** MIT
- **Активность:** generic OData v4 клиент, не 1С-специфичный
- **Что делает:** read/write OData v3/v4. Не заточен под 1С, но 1С OData
  v3.0 — стандарт, должно работать.
- **Подходит для нас:** **да, как универсальная альтернатива** к
  `belov38/1c-odata` если нужен более «взрослый» код.
- **Verdict:** **candidate**.

#### SAP/python-pyodata

- **URL:** https://github.com/SAP/python-pyodata
- **License:** Apache 2.0
- **Активность:** активный, корпоративный, ★ много
- **Что делает:** Enterprise OData client от SAP. Generic, не 1С.
- **Подходит для нас:** избыточно для 1С; SAP-специфичные фичи (CSRF,
  batch) могут не дружить с 1С OData v3.
- **Verdict:** **reject** — overkill, лучше `python-odata` или прямой
  `requests`.

#### Infactum/onec_dtools

- **URL:** https://github.com/Infactum/onec_dtools
- **License:** MIT
- **Активность:** ★ 103, **последний коммит 2017** (мёртвый)
- **Что делает:** парсит бинарные файлы 1С (1CD, cf, epf, efd) без
  платформы. Чисто для file-based баз и распаковки конфигураций.
- **Подходит для нас:** **нет.** Современная 1С:ERP это SQL/PostgreSQL,
  файловые .1CD не используются. Утилита из другой эпохи.
- **Verdict:** **reject**.

#### kilylabs/odata-1c (PHP, упомянуть для полноты)

- **URL:** https://github.com/kilylabs/odata-1c
- **License:** не уточнена на странице
- **Активность:** ★ 77, last commit 2022 (стагнация)
- **Что делает:** PHP OData клиент для 1С 8.3.
- **Подходит для нас:** **нет.** Мы Python-стек, PHP не наш.
- **Verdict:** **reject** для нашего стека.

### Категория 2. BI-коннекторы (Power BI / Tableau / Superset / DataLens)

#### ATK BiView (АТК Консалтинг)

- **URL:** https://biview.atkcg.ru/
- **License:** коммерческая
- **Цена:** Корпоративная версия (неограниченное число баз) — **412 500 ₽**
  (с обслуживанием ПО). Есть варианты на 1 базу подешевле.
- **Что делает:** автоматически генерит SQL-представления (views) над

  Visiology) подключается к этим view, не к сырому 1С-SQL.
- **Подходит для нас:** **да, флагман-вариант.** Совместим с любыми
  конфигурациями (включая 1С:ERP), не лезет в исходный код 1С,
  не нагружает базу как OData. Подтверждена совместимость с Visiology
  (российская BI, аналог Tableau).
- **Verdict:** **candidate (топ-1 для коммерческой опции)**. Дорого, но

#### Денвик «1С Connector»

- **URL:** https://denvic.tech/ , https://bi.denvic.ru/
- **License:** коммерческий SaaS / on-premise
- **Цена:** настройка 25 000 ₽ (один источник + один отчёт) +

- **Подходит для нас:** **да, бюджетный вариант.** Дешевле АТК на порядок,
  но и менее «толстый» (один отчёт за раз). Окей для пилота.
- **Verdict:** **candidate (топ для пилота)**.

- **URL:** https://infostart.ru/marketplace/1270915/
- **License:** коммерческая разовая (Infostart)
- **Цена:** **233 834 ₽** (MS SQL или PostgreSQL версия)

- **URL:** https://skyvia.com/data-integration/analyze-odata-with-superset
- **License:** SaaS, подписка

## Что НЕ нашлось

1. **Зрелого open-source Python-фреймворка под 1С:ERP с поддержкой
   регистров накопления/сведений.** Все Python OData библиотеки —
   тонкие обёртки общего назначения, не понимают семантику ERP
   (период, измерения, ресурсы регистров).

2. **Готового Airflow / NiFi operator-а под 1С.** Всё ad-hoc.

3. **Open-source альтернативы АТК BiView / Денвик** (т.е. свободного
   генератора SQL view над 1С). Это узкая ниша, монетизированная
   коммерческими игроками.

4. **Бесплатных коннекторов к Yandex DataLens.** DataLens **не**
   поддерживает OData; нужен промежуточный слой (PostgreSQL/файл).

5. **Готовых MCP-серверов с явным read-only режимом** под 1С:ERP.
   Все найденные (1c-mcp-toolkit, onec-mcp) имеют `execute_code` —
   нужны ограничения на уровне 1С-прав.

### Путь B — Встроенная 1С:Аналитика + готовый комплект для ERP

  sb-vnedr.ru / RDV-IT.

### Путь C — Своё решение через MCP (для тех, кто инвестирует в AI-стек)

  `Nikolay-Shirokov/cc-1c-skills` (наша Claude-методология) +
  read-only роль в 1С на уровне прав.

- **Плюс:** работает с нашей экосистемой `~/.claude/skills/`, MIT-лицензия

- **Минус:** новизна (2026-й год — это всё свежее). Production-надёжность

**Лично рекомендую начать с пути A** (Денвик пилот) — минимум затрат и

## Risks / открытые вопросы

1. **License проверка для `1c-mcp-toolkit`** — без LICENSE-файла код не
   копируем, нужно явно списаться с автором перед production.

2. **OData-производительность на регистрах накопления** — для 1С:ERP
   с большим объёмом операций OData может «висеть» (известная проблема).
   Все коммерческие коннекторы (АТК, Денвик) обходят это через SQL view —
   значит и нам нужно это, чистый OData для production не годится.

3. **Read-only гарантия** — на стороне 1С это делается через ограничение
   прав роли (RLS, права на чтение конкретных объектов), а не через
   ограничения инструмента. Нужно настроить service-учётку с
   read-only ролью прежде чем подключать любой инструмент к боевой базе.

4. **Где живёт DBA на проекте** — все SQL-view подходы (АТК, Денвик,
   View Generator) требуют доступа к SQL Server / PostgreSQL базе 1С.
   Если базой управляет ЦОД 1С-облака или франчайзинговый партнёр —
   нужно согласование.

5. **Yandex DataLens** — если кто-то в фирме хочет DataLens, готовых
   путей кроме «выгрузить в PostgreSQL» нет. Это удорожает интеграцию.

6. **MCP-стек ещё молодой** — 40+ серверов в каталоге Untru/1c-mcp,
   но почти все 2025-2026 годов. Зрелость отдельных серверов разная;
   нужно тестировать прежде чем класть в production.
