- Описание: Высокопроизводительная Python-библиотека для извлечения, анализа, конвертации и манипуляции PDF (и других форматов).

## Зачем смотрели
Region rendering (clip), text-in-curves через `page.get_drawings()`, A2 страницы, мощнее текущего pdf-mcp (который основан на PyMuPDF, но с лимитом 2000x2000).

## Оценка
- Подходит для нашей проблемы? **Да** (для region rendering и анализа кривых) / **Под условием** (для текста в кривых нужна доп. логика).
- Сильные стороны:
  - `page.get_text(clip=Rect(...))` — извлечение по региону.
  - `page.get_pixmap(clip=Rect(...), dpi=300)` — render региона на высоком dpi без лимита 2000×2000.
  - `page.get_drawings()` — возвращает список path-объектов с lines/curves/безье, можно анализировать формы глифов.
  - Поддержка больших страниц (A2/A1) без проблем — формат не ограничен.
  - Активный maintainer (Artifex), быстрые релизы.
- Слабые стороны / риски:
  - **AGPL-3.0** — для коммерческого использования внутри проектов нужно платить или быть OSS. Для внутреннего инструмента в нашей компании — серый вопрос.
  - При custom font encodings без CMap — выдаёт «garbled characters». Это **наш кейс** (текст в кривых ≈ нет CMap). Сам по себе текст не вытащит.
- Решение: **используем** как основу для region-rendering обхода лимита 2000×2000. Для text-in-curves — комбо с PaddleOCR/EasyOCR/Surya.
