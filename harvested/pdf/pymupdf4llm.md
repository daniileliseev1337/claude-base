# pymupdf4llm
- URL: https://github.com/pymupdf/pymupdf4llm
- Stars: 1695
- Last commit: 2026-05-11
- License: AGPL-3.0 (как и PyMuPDF)
- Описание: Обёртка над PyMuPDF, конвертирующая PDF в Markdown с сохранением структуры (заголовки, таблицы, многоколоночность).

## Зачем смотрели
Layout-aware extraction → markdown «из коробки» для LLM. Multi-column + tables в одном пайплайне.

## Оценка
- Подходит для нашей проблемы? **Да** для текстовых PDF / **Нет** для нашего конкретного кейса с текстом в кривых.
- Сильные стороны:
  - Многоколоночные layouts → правильный reading order.
  - Таблицы → GitHub-совместимый Markdown.
  - Прямой выход для RAG/LLM.
  - Активная разработка от Artifex.
- Слабые стороны / риски:
  - AGPL-3.0 — те же ограничения, что и PyMuPDF.
  - Russian OCR не в списке примеров (eng+fra только в доке) — нужна проверка.
  - На PDF с текстом в кривых выдаст ту же ерунду что и базовый PyMuPDF — обёртка не решает корневую проблему.
- Решение: **держим в уме** для нормальных текстовых PDF (ПД пояснительная записка, ТЗ). На текст в кривых не поможет.
