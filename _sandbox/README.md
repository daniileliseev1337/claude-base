# _sandbox/

**Локальная** изолированная папка для тестирования внешних инструментов из harvest-workflow.

## В .gitignore — НЕ пушится

Содержимое `_sandbox/` **никогда** не попадает в git. Это локальная зона для:

- Клонирования чужих репо для проверки (`git clone <repo> _sandbox/<repo-name>`).
- Запуска кода в Windows Sandbox или временном venv.
- Промежуточных артефактов теста.

## Дисциплина

- **Никогда** не запускать код из `_sandbox/` с доступом к credentials или production-данным.
- Использовать **только тестовые** входные данные (можно синтетические, обезличенные).
- После теста — папка очищается (через `git clean` не получится, потому что в .gitignore; вручную `Remove-Item _sandbox/<name> -Recurse`).
- Идеально — выполнять тесты внутри **Windows Sandbox** (компонент Windows 10/11 Pro): запустил `WindowsSandbox.exe`, скопировал `_sandbox/` внутрь, запустил, закрыл — все следы исчезли.

## Что попадает в git

Только **отчёт о тесте** (в `sessions/<...>/report.md` или `harvested/<...>.md`). Сами файлы из `_sandbox/` — нет.
