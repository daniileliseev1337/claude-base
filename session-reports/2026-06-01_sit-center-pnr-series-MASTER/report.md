## Методические профили ПНР (по типам систем)

### 1. Слаботочка (СКС, ОС, СОТ, СКУД, СПС, СОУЭ, АУГПТ)
- Единица ПНР — **адресное устройство**, функции поштучно: `кол-во итого = функц./ед. × кол-во устройств из ВОР`.
- Функций на устройство 2–12 в зависимости от типа (контроллер/прибор/извещатель/…).
- **ЗИП:** в ВОР полное кол-во + пометка «X к монтажу + Y ЗИП» в колонке «Тип монтажа»; в ПНР — без ЗИП (ЗИП не пусконаладится). [[feedback-pnr-zip-methodology]]
- Объединение одинаковых по составу ПНР разных исполнений (розетки 5 форм-факторов, табло 3 типов) — в одну строку ПНР.
- **СКС** — самый большой (3300 функц.): 3 подсистемы (UTP кат.6 / кат.6А / оптика). ТШ, магистраль и активное оборуд. — в разделе ММС ИОС4.2, не в СКС.
- **СПС** — ЗКПС 73 зоны × 2 = 146 функц. отдельной строкой; интеграции СКУД↔СПС↔СПЗ↔СОУЭ↔АУГПТ — отдельные пункты ПНР центральных приборов.
- **ОС** — ГЭСНп Сб.2 (АСУ) исключает ОС/ОПС → метод аналогии с СПС с оговоркой о нормативной лакуне.

### 2. Инженерные ОВ/ВК (ОВ1 отопление, ОВ2 вентиляция, ОВ3 кондиц., ВК)
- Единица ПНР — **система** (или этаж+блоки), не адресное устройство.
- **ОВ2 вентиляция:** 8 функц./система по СП 73.13330.2016 (осмотр→опробование→замер расходов→балансировка→герметичность→шум→автоматика→сдача) + отдельные группы клапанов: КЛОП-3/КЛОП-2 (3 функц., СП 7.13130.2013), балансировочные (2 функц.). v1 был 216 → расширен до 401 по запросу пользователя.
- **ОВ3 кондиц.:** единица = система (VRF 10+4×внутр.блок, сплит 11, прецизионный 12, дренаж 3×система). Спека из PDF через PyMuPDF find_tables.
- **ОВ1 отопление:** единица = этаж+блоки (наладка 6 + балансировка 1×клапан + коллектор 2× + прибор 1× + электро 3×). СП 73.13330.2016 разд.7 (опрессовка п.7.3.1) + СП 60.13330.2020 п.6.2.12.
- **ВК:** 2 системы + блоки. Водоснаб.: наладка 8 + арматура 1×N + санприборы 2×N + оборуд. 4×N. Канализация 6. СП 73.13330.2016 п.7.2/7.5.

### 3. Электрика (ЭОМ ИОС1.1)
- ⚠ Единица ПНР = **ЛИНИЯ и АППАРАТ**, НЕ «щит» (укрупнение «на щит» = 196 функц. забраковано пользователем как заниженное).
- Щиты — только общие операции 4 ф/щит; аппараты защиты 1×N; кабельные линии 2×N (изоляция + «фаза-ноль», **N точно из кабельного журнала** — здесь 459 линий); освещение 21; заземление 5. Итог 1507.
- ГЭСНп 81-05-01-2022; DALI → Сб.2 (АСУ). ВРУ вне раздела. Источник спеки часто .xls (xlrd). [[reference-pnr-eom-method]]

### 4. Водяное пожаротушение (ВПВ + АУВПТ, ИОС2.2)

- ГЭСНп 81-05-02-2020 Сб.2; насосная и ПКА — вне раздела. [[reference-pnr-firefighting-method]]

### 5. Оснащение (ТХ1)

## Стабильный pipeline (применялся ко всем разделам)

1. **Разведка** папки (`Get-ChildItem`) + `pdf_info`. «(выверенная спека)» → искать готовый xlsx-источник DataLink; «(для загрузки в МГЭ)» → спека в PDF.
2. **Чтение источника:** готовый xlsx через **openpyxl** ИЛИ спека из PDF через **PyMuPDF `find_tables()`** (НЕ pdf-mcp — глючит на граничных страницах).
3. **Подсчёт элементов** по типам/этажам/системам (для удельных чисел ПНР).
4. **AskUserQuestion** — 4 вопроса: охват + принцип подсчёта ПНР + удельные числа + ЗИП/специфика. **Профиль ПНР для каждого типа систем РАЗНЫЙ — не выдумывать, согласовывать.**
5. **norm-lookup ДО генерации docx** (правило #7). Локальная библиотека `~/Yandex.Disk\Claude_Library\spds\` богаче чем казалось (СП 73, СП 60, СП 30, ГОСТ Р 59972, ГЭСНп/ГЭСНм). cntd.ru недоступен (WEBFETCH_BLOCKED).
6. **Сборка xlsx:** копия шаблона СКУД (`5_ИОС4.5_СКУД…\ПНР и ВОРы\ПНР и ВОРы СКУД.xlsx`) → парс источника + маппинг работ + наследование стилей + перезапись.
7. **Сборка docx обоснования** (python-docx): 5 разделов (общие сведения / нормативная база / методика / сводная таблица с ИТОГО / исключения) + подписи.
8. **Parallel-валидация:** excel-validator + word-checker через Task tool в одном сообщении. До PASSED — не двигаться.
9. **Memory** — обновить если узнал новое, без дублей.

## Технические находки и инструменты

- **openpyxl вместо excel MCP** на больших таблицах — MCP раздувает JSON метаданными валидации/формул и упирается в лимит токенов (64–74K на лист). Для маленьких (<50×10) MCP ок.
- **PyMuPDF (fitz) вместо pdf-mcp** для чтения PDF — pdf-mcp глючит на граничных/одиночных страницах. `find_tables()` для извлечения спецификаций; обрабатывать ВСЕ таблицы страницы (штамп-таблица бывает крупнее контентной).
- **Парсинг СО:** наименование = первая непустая ячейка после кода (не max-len); включать единицы м.п. (портьеры теряются иначе); стоп-лист поставщиков.
- **AutoCAD MCP (backend file_ipc):** `execute_lisp` не возвращает stdout (писать в файл); открывать через прямой `vla-Open` + отдельный `vla-Activate`; результат читать `Get-Content -Encoding default` (AutoLISP пишет CP1251).
- **Связь DWG ↔ xlsx — AcDbTable + ACAD_DATALINK**, правка только в xlsx, обновление `DATALINKUPDATE`.

## Итерации и где сломалось

- **ОВ2 объём (216 → 401).** Пользователь усомнился «не мало ли 216?» — анализ показал нижнюю границу (не учтены КЛОП и балансировочные клапаны). Расширено по запросу. Урок: при сомнении в объёме — анализ + расширение, не защита первой версии.
- **ЭОМ «на щит» (196) забраковано.** Укрупнение оказалось несопоставимо мало. Перешли на поштучный учёт линий и аппаратов (1507). Урок: для электрики единица ПНР — линия/аппарат, число линий — точно из КЖ.
- **ЗИП-развилка у excel-validator.** Валидатор давал формальный NOT PASSED, ожидая «C = B × N(с ЗИП)». Это не ошибка файла — методически ПНР без ЗИП. Исправлено формулировкой prompt'а валидатору. [[feedback-pnr-zip-methodology]]
- **Дубль ИВЭПР в ОС** — оказался не дублем, а 3 разными местами установки (сверка с PDF). Урок: не объединять дубли позиций без сверки с проектом.
- **Склейка названия ГОСТ по памяти** (ГОСТ Р 59636 vs 59638) — поймана word-checker'ом → norm-lookup. Урок: нормы только через norm-lookup. [[feedback-norms-via-norm-lookup]]
- **DataLink блокер (СОТ и далее):** AutoCAD не видит Excel из-за пустого `HKLM\…\Excel\InstallRoot\Path` (Office LTSC 2024 + AutoCAD 2024). Фикс требует прав админа — отдан главному ПК/админу.

5. Не дублировать в ПНР строки разных исполнений с идентичным составом функций.
6. Не выдумывать профиль ПНР нового типа систем — согласовывать через AskUserQuestion.
7. Не пытаться создавать DataLink через MCP — перепривязку оставлять пользователю в AutoCAD UI.
8. Не сохранять/не править эталоны после открытия в AutoCAD — работать с копиями.
