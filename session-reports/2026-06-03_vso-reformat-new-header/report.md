# Session report — переформатирование актов ВСО под новую шапку заказчика

Дата: 2026-06-03. Раздел: ВиВ (вода + канализация), 5 зданий. Обезличено.

## Задача
Заказчик прислал новую шапку таблицы ВСО (Ведомость смонтированного оборудования):
`№ п/п | Наименование | Тип, марка | Завод-изготовитель | Кол-во | Примечание (с указанием номеров помещений)`.
Нужно переделать существующие акты ВСO (.docx) под неё: разнести единую графу «Наименование работ»
на три (Наименование / Тип-марка / Завод), перенести помещения в Примечание, заполнить «ничего не выдумывая».

## Что сделал
- 9 актов переформатированы (по 2 на здание, минус 1 — у одного здания нет оборудования по воде).
- Решение через **код в tools/** (Python + python-docx): детерминированная трансформация таблицы +
  выверенный вручную словарь `vso_map.json` (≈100 позиций) для разнесения наименование→{наим/тип-марка/завод}.
- Серийники насосов: где были впечатаны в исходник — сохранены (перенёс в Примечание «зав. № …»);
  где «б/н» — искал на исполнительных чертежах (рукопись), читал увеличением фото.

## Ключевые уроки (забрать в скилл/память)
1. **Таблица ВСО может быть ВЛОЖЕНА в ячейку другой таблицы.** `doc.tables` (python-docx) её не видит →
   парсер «не нашёл таблицу», хотя визуально она есть. Лечится рекурсией по `cell.tables`
   (см. `_iter_all_tables`). Это же объясняет «пустой» на вид акт, который у пользователя заполнен.
2. **PowerShell `>` пишет UTF-16** → кириллица «ломается» при последующем чтении. Скрипты должны писать
   вывод в файл сами через `open(path, 'w', encoding='utf-8')`, не через пайп/редирект PS.
3. **Yandex.Disk online-only (ReparsePoint)**: файл может читаться скриптом как старая версия, пока не
   «доступен на этом устройстве». Расхождение «я вижу пусто / пользователь видит 55 строк» — отсюда.
   Проверять `LastWriteTime` бесполезно: синхронизация затирает дату у всех файлов одним числом.
4. **Серийники оборудования — только на исполнительных чертежах от руки.** В реестрах/паспортах их нет
   (паспорт = «б/н»). Рукопись на фото низкого разрешения → часть цифр неоднозначна (3↔5, 7↔1, 4↔9);
   для официальной ИД угадывать нельзя — помечать «?» и «(проверить)», отдавать на сверку.
5. **Разнесение свободного текста на колонки** надёжнее делать выверенным словарём (наименование→поля),
   а не регэкспами: словарь применяется детерминированно, правки точечные, ревью прозрачное.
6. Завод-изготовитель проставлять только из факта (бренд в тексте, ПВ, паспорт, торговая марка
   типа SANICUBIC=SFA, Duofix=Geberit), иначе «—». Не присваивать наугад.

## Где сломался / что осталось
- Серийники одного здания — фото исполнит. чертежа низкого разрешения, прочитаны best-effort с пометками.
- У 2 зданий фото раздела вообще нет → серийники взять негде.
- Расхождение модели насоса акт↔чертёж — отдал на проверку человеку.
