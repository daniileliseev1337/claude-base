# Сессия 2026-07-01/02 — строки-работ над позициями СВОР и ВОР (16 файлов)

## TL;DR
Над каждой позицией оборудования в ведомостях СВОР (сопоставительная, 12 кол, с формулами)
и ВОР (простая, 7 кол, без формул) вставлена строка-работа (наименование работы) с
наследованием ед./объёма/формул и объединением №/произв./модель по блоку. 16 файлов
(8 СВОР + 8 ВОР, объект `<объект>`, шифр `<шифр>`-ИОСx.x/ТХ.15). Готово, отревьюено,
разложено по папкам. Открыт 1 опциональный пункт (Монтаж↔Пусконаладка).

## Что сделано
- **СВОР (8 файлов, 3361 позиция):** sot 298, ahp 71, sks 2314, sech 65, stm 493, skpt 30,
  mgn 90, ММ(ТХ.15) 471. Строки-работы + наследование E:L (ед/объём до-после/формулы H,I) +
  объединение A/B/C. 2-строчные (замена модели) зеркалятся.
- **ВОР (8 файлов, 2048 позиций):** те же разделы, форма проще (7 кол, БЕЗ формул, один объём F).
  Наименования переиспользованы из СВОР по имени оборудования (кэш name→work, sks совпал 100%),
  66 уникальных несовпавших доименованы отдельным агентом.
- **Новые правила пользователя:** компьютерная/АРМ-техника (ноутбук/АРМ/ПК/мини-ПК/моноблок/
  монитор/планшет/сенс.панель/ТВ/рабочая станция) → «Настройка и запуск»; камеры → «Монтаж и
  пусконаладка»; кабель(м) → «Прокладка»; активное программируемое → «Пусконаладка»; ПО/лицензии →
  «Установка и настройка ПО»; сервисные сертификаты → «Проверка комплектности и передача сертификата».
  ТВ с отдельным креплением → ТВ «Настройка и запуск», крепление «Монтаж».
- **Ревью (Workflow, 14+16 агентов auditor+excel-validator).** Найдено 2 FAIL — исправлены в
  обеих папках: телехирургическая стойка (Advantech AVAS-60, рабочая станция) Монтаж→Настройка и
  запуск; встраиваемый ПК Lumien Пусконаладка→Настройка и запуск (retro-sync-регэксп пропустил).
- **Унификация Монтаж/Прокладка** (запрос пользователя): 5 номенклатур приведены к канону по
  единице (кабель/шнур шт→Монтаж, труба м→Прокладка), 80 D-ячеек в 16 файлах.
- **Раскладка по папкам:** `<...>\Балашиха 01.07\СВОР\{Оригиналы|С работами}\` и `\ВОР\{...}\`,
  файлы «С работами» с чистыми именами (без «_с работами»).

## Метод (детали — в [[reference_svor_add_works_win32com]] и [[feedback_svor_work_naming_computers]])
1. extract позиций openpyxl→JSON (fullwidth A:lastcol заголовки пропускать; two_row по вертикальному
   A-merge). 2. Наименование: pto-engineer fan-out (Workflow при 20+ чанках) по единым правилам;
   сборка **по индексу чанка** (works[i]↔chunk[i].anchor; num-echo сверять 100% — в исходниках дубли №).
   3. win32com вставка (openpyxl-insert ломает относит. формулы СВОР) + merge A/B/C по блоку
   (2-строчность **по факту A-merge**, не по two_by_num — дубли num схлопывают словарь). 4. Самопроверка
   openpyxl (покрытие, дрейф формул=0, work-text по ПОРЯДКУ а не по num). 5. Ревью excel-validator+auditor.
- ВОР: наследовать E:G (не E:L), формул нет. Кэш из СВОР по mode(work_text per name).
- Правки D (сервис/классиф/канон): по ТЕКСТУ D (ловит обе строки 2-строчных), не по A==num.

## Грабли
- openpyxl insert_rows ломает относит. формулы H/I СВОР → win32com.
- Excel COM: Calculation ставить ПОСЛЕ Open; не писать Value в только что merged ячейку (сначала
  верхняя→очистить нижнюю→merge); стрея-merge при insert снимать guard'ом.
- Дубли № в исходниках → сборка/сверка по ИНДЕКСу/ПОРЯДКу, не по num.
- Файл `inspect.py` в cwd затеняет stdlib inspect (numpy падает) — не называть скрипты как stdlib.
- Naming-ловушка: «встраиваемый ПК»/«рабочая станция»/телехирург.стойка = «Настройка и запуск»,
  регэксп по «мини-пк/системный блок» их пропускает.

## Верификация (финал, все 16)
СВОР: покрытие 100%, формулы без дрейфа (sks 9248/0, ММ 2040/0 и т.д.), work-text mismatch=0.
ВОР: покрытие 100%, формул 0, mismatch=0. Оба FAIL закрыты. Монтаж/Прокладка расхождений=0.
