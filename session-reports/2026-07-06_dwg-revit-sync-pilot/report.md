## Что делали

Пилот двухэтапного конвейера «модель Revit → актуализация по DWG-эталону» на одной зоне
(блок/этаж): покомнатный diff DWG↔модель, подтверждение через виджет, внесение в модель.

## Что закрыли

- Мост координат: оси сетки имя-в-имя (18 верт. + 9 гориз.), чистый сдвиг, невязка 0.0 мм.
- Маппинг DWG-блоков → типы приборов: кросс-таб позиционных меток + верификация эталонными
  помещениями (алфавитная интуиция дала бы крест-матчинг — эмпирика обязательна).
- Diff: венгерка по типогруппам с калибровкой символьных смещений per (тип, facing, rot);
  53 SAME / 54 MOVED / 7 смен типа / 1 MISSING / 0 EXTRA. Аудит субагентом: PASSED
  (блокер room-привязки на границе полигона — facing-пробы).
- Внесение: 61 move + 7 ChangeTypeId + 1 NewFamilyInstance; затем 3 итерации исправлений
  (все три бага ловил ПОЛЬЗОВАТЕЛЬ по модели — см. уроки); финал: независимая диагностика
  чистая, 7 зон проверены рендерами реальных видов, состав = DWG.

## Где сломался (уроки — главная ценность)

1. **Координатный перенос DWG-целей на переехавшие приборы = брак**: символьный офсет
   калиброван по facing СТАРОЙ позиции — на новой стене повёрнут неверно (промахи до метров,
   чужие помещения). Внесение только правилами домена: DWG даёт СОСТАВ+ПОМЕЩЕНИЕ (pip по
   сырой вставке), координаты — детерминированные правила (у сантехники / ряды живых ламп).
2. **Конвенция facing семейств**: у вызывных панелей facing направлен В СТЕНУ (геометрия
   кнопки рисуется против facing). Задал «нормалью в комнату» — 45 приборов на 180° задом.
   Конвенцию ВСЕГДА снимать с нетронутых выверенных приборов, «пары ±1» = зеркальные
   спина-к-спине, dot-тест бесполезен.
3. **Эталонный зазор**: точка вставки ровно на грани стены (gap 0); ИНСЕТ 12 мм в легаси-
   скрипте — ловушка. Зазор мерить по осям+толщинам стен линка (грань = ось ± W/2);
   ReferenceIntersector непригоден (слепнет со старта на грани, не видит сантехперегородки).
4. **«Аудит против своих же целей» = самообман**: моё «62/62 OK» сверяло факт с моими же
   неверными целями. Верификация — только независимыми метриками (расстояние до профильной
   сантехники, фактическая комната, зазор до стены) + рендеры реальных видов глазами.
5. Сводку пользователю писать ТОЛЬКО из финального артефакта — фраза из промежуточной
   итерации прогона уползла в итоговый текст и дала ложный флаг.
6. AutoCAD MCP мультидок: IPC следует за активным документом, активация асинхронна (2 вызова),
   «Чертёж занят» на close/DBX, xref re-path на локальную копию, кириллица слоёв по индексу.
   (Дополнено в memory: reference-autocad-mcp-gotchas, reference-dwg-revit-sync-method.)
