## 2. РАЗБОР ЭТАЛОНА (`Пример Claude.dwg`)
- Единицы — мм (INSUNITS=4). AutoCAD 2024 (file_ipc), backend подтверждать `system status/init`.
- **24 слоя**, ОВ-набор, префикс `!` (чтобы всплывали наверх), все Continuous:
  - `!Приток` `!Вытяжка` `!Рециркуляция` (lw 0.18) — **цвета: Приток=1 КРАСНЫЙ, Вытяжка=5 СИНИЙ, Рециркуляция=6**
  - `!Дренаж` (0.25), `!Конд_оборудование` `!Вент_оборудование` `!Фасонка` `!Люки` `!Запорная_арматура_ХС` (0.09)
  - `!Трассы_кондиционирвоания` (опечатка в имени — так в эталоне!), `!Выводы питания`, `!Кабель`
  - `!Подложка` (lw 0.05, цвет 253 серый), тексты `!Текст`/`_в`/`_к`/`_д`/`_р`/`_в_эл.подкл.` (0.15), `!Размеры`
  - `!Не печатается` (plot=N), `0`, `Defpoints` (plot=N)
- **Динблоки (библиотека, dyn=Y):** Воздуховод, Пр_воздуховод, Воздуховод_гибкий, Отвод 90/45 (+гибкие),
  Тройник, Тройник_В, Редукция, Переход_гибкий, Диффузор, ДК, решётки, оборудование
  (Breezart_550_Lux_SB, ZSK-250 с приводом, AD25S2SS1FA, ОЗК_В, РЭД-ЛУК…). **Рамка_со_штампом — тоже динблок** (Выбор1=тип штампа).
- **Текстовые стили:** Standard(arial), RUSS3(SIMPLEX.SHX w0.6), ГОСТ(wwcade.shx w0.77), ROMANS08, GOST 2.304 и др.
  Размерные стили: Размерный_основной, ЗАСЕЧКИ-1X100, ISO-25, Аннотативный. **Использовать стиль фирмы, не создавать свой.**
- **Масштабы:** листы 1-4 — типовые таблицы/общие данные (1:100, меняются только данные).
  Листы планов: VP1 = рамка-штамп 1:100 (стандарт), VP2 = сам план **произвольный масштаб** под читабельность на А3.
- **Компоновка:** в МОДЕЛИ фрагменты (планы приток/вытяжка/аксонометрия/развёртки) разложены В РЯД по горизонтали
  (extents эталона ~668×140 м), каждый ловится своим viewport на своём из 11 листов. `Model` + 11 листов.

## 3. PIPELINE (8 этапов)
- **Этап 0. Каркас:** копия эталона КАК ЕСТЬ (модель не чистить). Рабочая копия в ASCII-пути `C:\temp\*.dwg`.
- **Этап 1. Подложка** ✅ (см. рецепт): PDFIMPORT → калибровка → чистка → монохром+блок на `!Подложка`.
- **Этап 2. Калибровка/привязки** (слита в Э1).
- **Этап 3. Вентиляция:** воздуховоды приток(красн)/вытяжка(синь)/рециркуляция, фасонка, вентоборудование.
- **Этап 4. Кондиционирование:** трассы, внутр./внеш. блоки, дренаж.
- **Этап 5. Воздухораспределители и люки:** диффузоры, решётки, люки.
- **Этап 6. Аннотации:** тексты (СТИЛЬ ФИРМЫ), размеры, выводы питания, подписи ИС.
- **Этап 7. Компоновка листов:** видовые, масштаб под А3, штампы, общие данные; УДАЛИТЬ старую квартиру; навести видовые.
- **Этап 8. Верификация:** печать PDF, сверка с исходником, нормоконтроль.

**Рабочий процесс пользователя:** старую квартиру ОСТАВЛЯЕМ как референс/источник копирования типового,
новую строим РЯДОМ (свободная зона модели, X≈750000), старую удаляем и наводим видовые ТОЛЬКО В ФИНАЛЕ.

## 4. РАБОЧИЕ РЕЦЕПТЫ (отлажены на этой сессии)

### 4.1 Кириллица в autocad-mcp (file_ipc) — АСИММЕТРИЯ
- **ВХОД (запись): кириллицу писать ПРЯМО в коде LISP** (доходит как Unicode). `(vla-InsertBlock ms pt "Воздуховод" …)`,
  `(vla-put-Layer ref "!Приток")`, имена dyn-свойств кириллицей — работают. НЕ реконструировать через `(chr N)` cp1251
  («В»=1042, не 194; `(chr 194)`=`Â` → InsertBlock ищет внешний .dwg → «Ошибка файлера»).
- **ВЫХОД (чтение): русские имена приходят в cp1251** (в JSON как `Ïð…`). Длинные списки: LISP пишет дамп в файл
  `(open "C:/temp/x.txt" "w")…(write-line …)…(close)`, читать PowerShell:
  `$enc=[Text.Encoding]::GetEncoding(1251);[IO.File]::ReadAllText("C:\temp\x.txt",$enc)`.
- **`drawing open` через MCP НЕ открывает файл** (врёт «opened», активный док не меняется) — ни кириллица, ни ASCII.
  Решение: COM `(vla-Open (vla-get-Documents (vlax-get-acad-object)) "C:\\temp\\x.dwg")`, затем отдельным вызовом
  `(vla-put-ActiveDocument acad tgt)` (переключение применяется ПОСЛЕ выхода из LISP). Проверять `(getvar "DWGNAME")`.
  ASCII-путь обязателен (обход кириллицы в путях). В финале копировать в проектную папку PowerShell-ом.

### 4.2 PDFIMPORT (вектор-PDF)
- `(setvar "FILEDIA" 0)`, `PDFIMPORTMODE` (7 = вектор+заливки+TrueType-текст).
- `(command "_.-PDFIMPORT" "_F" "C:\\temp\\podloga.pdf" "1" "750000,0" "1" "0")` (файл, стр, точка, масштаб, поворот).
- Ложится ОДНИМ блоком (имя `podloga`/`podloga1`… автоинкремент) → `(vla-Explode ref)` + `(vla-Delete ref)`.
- **OCG-слои PDF сохраняются** как слои `PDF_<имя>` (с пробелом-подчёркиванием) → чистка фильтром по слоям.

### 4.3 Калибровка масштаба (план печатался «по размеру листа», круглого масштаба НЕТ)
- Импорт ~+3.6% крупнее реала. **Коэффициент 0.9644** для этого объекта.
- Метод: найти 2 одинаковые размерные подписи соседних равных сегментов (две «3765»), измерить расстояние центров
  в DWG = реальному размеру; либо длинная база (4135→5230 центры = сумме сегментов). units_per_mm = DWG/мм.
- Проверка: ширина по стенам должна = сумме внутр. цепочки + толщины стен (на Y703 = 17422 мм).

### 4.4 Чистка подложки (ОТЛАЖЕНО, после 3 итераций + 3 правки пользователя)
- **Слои целиком в выброс:** `размер`, `DIST`, `NET`, `вспомог.`, `вспомогательные`, `плитка`, `LINE`
  (на LINE — размерные линии + рамка таблицы экспликации).
- **Двери (`def`): НЕ удалять слой целиком!** 237 чёрных (ByLayer) = дверные полотна (ОСТАВИТЬ),
  182 ОРАНЖЕВЫХ (color 30) = распашонки/область открытия (УДАЛИТЬ по ЦВЕТУ 30). Правило: убрать только область открытия.
- **`PDF_0` (смешанный):** эллипс/дуги = ванна/сантехника (ОСТАВИТЬ); прямые 2-точечные полилинии (≤2 вершин) =
  размерные линии/засечки (УДАЛИТЬ).
- **`PDF_Текст`:** удалять размерные числа (чистое целое >16) + экспликацию (содержит `,`/`.`/`Специфик`/`Итог`/=«Столовая»);
  ОСТАВИТЬ номера помещений 1-16 + подписи мебели.
- **Цветные хвосты (по цвету, кроме окон):** color 1/4/6/32 = рамки-выделения/метки → удалить.
  ОСТАВИТЬ c8/c252 (серая мебель/заливка), c5 синий на `заливка` (фоны окон), штриховку стен.
- **⚠ Чистить ДО слияния слоёв в `!Подложка`** (иначе теряется деление по слоям).
- **⚠ Фильтр по слою НЕДОСТАТОЧЕН:** дизайнер кладёт размеры/рамки/внешние подписи на слои архитектуры (мебель/стены).
  Добавлять: (а) удаление текста/линий ВНЕ контура стен (bbox слоя `стены`), (б) длинные 2-точечные линии на любых слоях,
  (в) ОБЯЗАТЕЛЬНЫЙ визуальный обход скрином по зонам.
- Засечки бывают И ВНУТРИ квартиры, не только по периметру.

### 4.5 Финал подложки: сначала БЛОК, потом масштаб
- Монохром: все import-объекты → слой `!Подложка` + цвет ByLayer(256).
- Собрать в ОДИН блок (имя ОТДЕЛЬНОЕ, напр. `Подложка_Y703` — в эталоне уже есть `Подложка`/`!Подложка`).
- **Масштабировать БЛОК как единый объект** (`vla-ScaleEntity ref base 0.9644`).
- ⚠ НЕ масштабировать россыпь через `ssget "_C"`+SCALE: `ssget "_C/_W"` берёт ТОЛЬКО видимое в текущем виде →
  часть вне экрана не масштабируется → РАССИНХРОН (наступал: 12195 из 14314). Перед ssget — ZOOM на габарит,
  проверять `sslength` = ожидаемому. Но надёжнее — блок.

### 4.6 Параметризация воздуховода (динблок `Воздуховод`, круглый)
- Диаметр НАПРЯМУЮ: dyn-props `Диаметр воздуховода`=N, `Радиус`=N/2, `Расстояние1`=длина (double, `(vlax-make-variant N 5)`).
- НЕ через lookup `Значение`/`Таблица блоков1` (не резолвится: Таблица=-1, диаметр уходит в дефолт 100).
- Слой — `CLAYER` перед вставкой. Ориентация: rot 0 = ось +Y (вверх), rot 1.5π = влево (−X) и т.д. (rotation в РАДИАНАХ).
- В плане круглый воздуховод = 2 параллельные линии на расстоянии = диаметр.

### 4.7 Рендер DWG для проверки
- `plot_pdf` (MCP) и COM `PlotToFile` дают ПУСТОЙ PDF (модельные plot-настройки) — НЕ использовать для превью.
- Рабочий способ: `(setvar "FILEDIA" 0)(command "_.ZOOM" "_W" p1 p2)(command "_.PNGOUT" "C:\\temp\\x.png" "")` → читать через Read.
- **Скрины: zoom+снимок ЗОНЫ, не весь план** (весь план мельчит — можно спутать/не разглядеть). Весь план — только обзор.

### 4.8 Откат и зависания
- Откат командных правок: `(command "_.U")` (Ctrl+Z). Reopen-from-save — для запутанного состояния (много COM-циклов).
- Зависание (таймаут на тривиальном запросе = модальный диалог): PowerShell `WScript.Shell.AppActivate($pid)` + `SendKeys("{ESC}")` (+ `{ENTER}`).
- REGENALL на большой геометрии может дать таймаут — избегать, PNGOUT сам регенит вид.

## 7. ПРЕДЛОЖЕНИЕ ПО АРХИТЕКТУРЕ АГЕНТА (для «идеально справляться»)
Цель пользователя — агент(ы) под эту задачу. Рекомендация:
- **Главный агент-оркестратор** `dwg-recreator` (pipeline 8 этапов, чтение FACTS/scheme, делегация).
- **Слой tools/ (детерминированный код, НЕ модель):** готовые LISP-сниппеты в файлах —
  `cyrillic_dump.lsp` (дамп слоёв/блоков/листов), `pdf_import_calibrate` (импорт+поиск размерных подписей+коэффициент),
  `backdrop_cleanup` (правила 4.4 параметрически: drop-слои, color 30 двери, PDF_0 2-вершинные, текст-фильтр, цветные),
  `block_and_scale`, `place_duct` (вставка Воздуховод с диаметром/длиной/ориентацией/слоем), `pngout_zone` (рендер зоны).
  Python-helper: PyMuPDF probe (OCG-слои, размерные подписи, вектор/растр), cp1251-reader.
- **Узкий ревьюер** `dwg-backdrop-reviewer` (read-only): после чистки — обязательный визуальный обход по зонам (PNGOUT),
  проверка: нет размеров/засечек/экспликации/распашонок, есть стены/окна/двери/мебель/сантех; вне контура стен пусто.
  Failure-mode строгий: без явного списка проверенных зон — НЕ пройдено.
- **Ключевые инварианты-гварды (зашить в код):** (1) ВХОД кириллица в коде / ВЫХОД cp1251 через файл; (2) ASCII рабочий путь;
  (3) фильтр по слою + позиция + геометрия + визуальный обход; (4) блок→масштаб; (5) скрин ЗОНЫ не весь план;
  (6) сверка скрином после каждого деструктивного шага; (7) ничего не удалять огулом.
- **Что отдавать человеку (не автоматизировать слепо):** финальную приёмку стиля, разводку по растру без вектора (приблизительно),
  спорные «оставить/удалить» элементы (подсветка цветом → решение человека).
