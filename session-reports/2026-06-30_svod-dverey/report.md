# Отчёт сессии: унификация 4 блоков КП дверей в один свод

**Дата:** 2026-06-30
**Задача:** объединить 4 Excel-блока (КП на двери от разных поставщиков) в один
унифицированный читаемый файл, блоки по возрастанию, БЕЗ объединения одинаковых позиций.
Папку КП не трогать.

## Что сделано
- Детейлинг 4 исходников через openpyxl (excel MCP захлёбывается — подтверждена ловушка из
  handoff-отчёта предыдущей сессии).
- Установлено: **раскладки колонок РАЗНЫЕ у всех 4 блоков** (главная сложность). Блок-источник
  с другой структурой имел Размер в C, этажи D..N, цену в P, обозначение+наименование слиты в B;
  остальные — Размер D, этажи E..O, цены в T/U. Приведено к единому эталону (21 колонка).
- Скрытых строк/столбцов НЕТ ни в одном файле (проверено hidden + нулевая ширина + outline),
  хотя пользователь предупреждал о возможных — фактически чисто.
- Собран свод детерминированным Python-скриптом (НЕ fan-out LLM — точный матчинг колонок,
  0 галлюцинаций). Один лист, блоки 1→4 подряд, заголовок-поставщик + шапка + разделы 1/2
  у каждого; Сумма = живая формула Цена×Всего; «Комплектующие» отдельным листом.
- Нейтральное оформление (серые шапки, тонкие границы — без декоративных цветов).

## Верификация и ревью
- Программная попозиционная сверка свод↔источник: строк-дверей 173/164/164/141 (Σ=642),
  Σ «Всего» 1411/1415/1416/540 — сошлось.
- auditor (sonnet) → PASSED, 10/10 свойств.
- excel-validator (sonnet) → FAIL: 6 пустых цен, 8 «точных дублей», кэш формул пуст.
- **Расхождение ревьюеров разрешено ДЕТЕРМИНИРОВАННО** (не резолвером): все «дефекты»
  excel-validator оказались СВОЙСТВАМИ ИСТОЧНИКОВ — пустые цены на доп-позиции (поставщик
  не дал), точные дубли уже есть в исходных КП. По ТЗ «не объединять» — обязаны сохранить.
- Единственная реальная находка (кэш формул openpyxl=None) исправлена пересчётом через Excel COM.
- Денежные итоги двух блоков совпали с итоговыми строками источников ДО КОПЕЙКИ — сильное
  подтверждение корректности.

## Уроки / ловушки (для базы)
- **Расхождение ревьюеров не всегда требует резолвера-LLM.** Если находки — проверяемые ФАКТЫ
  (есть ли дубль/пустая цена в источнике), дешевле и надёжнее проверить детерминированно
  Python-скриптом. Резолвер нужен для расхождения СУЖДЕНИЙ, не фактов.
- **excel-validator проверяет форму файла В ВАКУУМЕ** и не знает ТЗ — «дубли» и «пустые цены»
  он штатно метит как дефект. При задаче «не объединять» это ложные срабатывания; их надо
  отбивать сверкой с источником, а не правкой файла.
- openpyxl пишет формулы без кэша значений → data_only=True даёт None. Для «живого + читаемого
  программно» файла: после сборки прогнать Excel COM (Open→CalculateFull→Save). Excel COM на
  этой машине доступен (Office 16.0).
- Контрольные суммы (кол-во строк + Σ количеств + денежный итог vs итог-строка источника) —
  обязательная проверка переноса; ловит и ошибки сборки, и подтверждает корректность.

## Этап 2: применение ответов поставщиков на замечания (тот же день)
- Два docx (АРМАДА/Блок 4, РПГ/Блок 2) — НЕ простые прайсы, а переписка-разбор расхождений.
  Формат ячейки цены: «<старая> <новая> <комментарий>». Блоки 1 и 3 пользователь отметил
  корректными — не трогать.
- Стратегия против ошибок на деньгах: извлечь пары (Поз, старая, новая) вручную (надёжнее
  регулярок на «грязном» тексте) → сверить СТАРУЮ цену с фактической в своде (все 35 совпали →
  подтверждение верного прочтения и сопоставления) → применить с ТРОЙНОЙ защитой (row+poz+old,
  иначе стоп без записи).
- Конфликты разрешены сверкой старой цены со сводом: напр. Д5 в docx фигурировал с двумя
  «старыми» (161669 и 152679) — применилась та запись, где старая совпала со сводом.
- 4 развилки эскалированы пользователю (л-версии; Доп4л 256764 vs 291690; Д59/Д59С без цены —
  поставщик просит уточнить ТЗ; Д59.1 пометка «С» vs факт «л»). Решения получены и применены.
- Итог: 52 ячейки цены изменены. Δ Блок 2 = +53 ₽ (округления), Δ Блок 4 = +273 520 ₽ (гл.
  Д46.1/Д46.1л — поставщик исправил «по ошибке учтённую стальную дверь»). Кол-во строк/количеств
  не менялось. Суммы пересчитаны Excel COM.
- Независимый auditor (sonnet) сам перечитал docx, сверил 57 позиций побитово → PASSED 9/9.

## Урок (этап 2, для базы)
- Ответы поставщиков на замечания приходят как ПЕРЕПИСКА с форматом «старая новая» + коммент,
  а не готовый прайс. Ключ к безопасному применению: сверять СТАРУЮ цену из документа с
  фактической в целевом файле — это (а) подтверждает корректность прочтения, (б) разрешает
  внутренние конфликты документа, (в) ловит несовпадение позиций. Применять только с защитой
  по совпадению старого значения.
- Зеркальные «л»-версии в КП = та же дверь/цена; поставщик часто правит лишь базовую марку,
  подразумевая л. Это решение пользователя (спросить), но дефолт — синхронизировать л с базовой.

## Этап 3: индексация абсолютно идентичных дверей (новый столбец)
- Задача: найти идентичные двери через 4 блока по ключу 4 полей (Поз+Наименование+Размер+
  Описание) и проставить групповой индекс ИД-NNN в новый столбец V.
- Перед стартом — understanding-map виджет (show_widget) на примере Д2: показал ключ, обработку
  Блока 3, присвоение индекса, 3 развилки. Согласованы: критерий = совпадение в ≥2 блоках;
  Блок 3 наименование извлекается из Обозначения; сравнение с нормализацией пробелов/регистра.
- Подвох Блока 3: колонка Наименование пустая, имя «зашито» в Обозначение после ГОСТ-префикса →
  извлекается regex Д[А-ЯA-Z].*$ (от первого «Д»+заглавная). Без этого Блок 3 не матчился бы.
- Реализация: dry-run (статистика групп) → запись столбца → пересчёт Excel COM. Итог: 138 групп
  (ИД-001..138), 440 строк с индексом (из 642), распределение по блокам {2:56, 3:36, 4:46}.
- Независимый auditor (OPUS) перестроил группировку с нуля → PASSED 7/7, 0 расхождений на 642
  строках; извлечение имени Блока 3 верно (107/107 совпали с Блоком 1).

## Урок (этап 3)
- PermissionError при сохранении = файл открыт у пользователя в Excel. Безопасный путь: НЕ убивать
  EXCEL.EXE (риск потери чужих данных) — сохранить в *_tmp.xlsx + os.replace; при неудаче оставить
  tmp и попросить закрыть. Для пересчёта tmp использовать win32com DispatchEx (ОТДЕЛЬНЫЙ инстанс),
  иначе Dispatch подключится к пользовательскому Excel и Quit закроет его окна.
- openpyxl: StyleProxy (cell.fill) нельзя присвоить другой ячейке напрямую → copy.copy().
- Понимание-карта (understanding-map) перед неоднозначной задачей на 4-польном ключе сняла 3
  развилки до старта — дешевле, чем переделывать индексацию.

## Этап 4: разбиение Обозначения/Наименования Блока 3 + плоский лист для фильтра
- Проблема пользователя: автофильтр Excel на читаемом листе фильтрует только первый блок (разрывы
  между блоками). Решение: отдельный лист «Все двери (фильтр)» — 642 строки подряд, колонка
  «Блок», автофильтр, сортировка по индексу. Итоги совпали с основным до копейки.
- Блок 3 имел слитые Обозначение+Наименование в колонке B (C пустая). Разбито: наименование от
  первого «Д»+заглавная в C, обозначение-префикс в B. Самосверка наименования с близнецами по
  (Поз+Размер+Описание) дала 96/96 совпадений → извлечение верное (и индексация на нём верна).
- Обозначения Б3 были УСЕЧЕНЫ в исходнике («ГОСТ 475-20» вместо «ГОСТ 475-2016»). По решению
  пользователя восстановлены полные: 96 точно из близнеца, 60 по правилу (усечённый префикс+тип
  двери → мода полного обозначения из близнецов того же типа), 8 (противопожарные без близнеца) —
  по базовому префиксу (наименее уверенные, отмечено пользователю). Самосверка обозначений 96/96.

## Уроки (этап 4)
- Восстановление повреждённого поля: точное значение брать из идентичного близнеца (через
  ключ/индекс), а где близнеца нет — выводить правило из тех же близнецов (префикс+тип→мода), не
  угадывать; неуверенный остаток (здесь 8 строк) явно называть пользователю, не маскировать.
- Самосверка «разрезал поле → совпало ли с независимым источником» (наименование Б3 vs близнец,
  96/96) — дешёвая и сильная валидация, которую запросил сам пользователь.
- При повторной правке xlsx с формулами: цикл openpyxl-save (кэш слетает) → Excel COM DispatchEx
  CalculateFull (кэш восстановлен); плоский-лист Сумму считать в Python (vsego×cena), не из кэша.

## Этап 5: лист «Свод по макс. цене»
- Задача: для каждой группы идентичных дверей (один ИД-NNN) присвоить всем строкам максимальную
  цену среди блоков; уникальные — текущая цена. Перед стартом — understanding-map виджет на группе
  ИД-037 (Д15Сл) с реальными ценами 4 блоков.
- Решение пользователя: ОТДЕЛЬНЫЙ лист (не новый столбец), цена перезаписана на макс, сумма от макс.
- Реализация: openpyxl copy_worksheet («Свод дверей»→«Свод по макс. цене»), добить ширины/freeze/
  merge (copy_worksheet переносит не всё); цена групп = max по индексу (440 строк), уникальные (202)
  не тронуты; Сумма-формула =Q*P пересчитана Excel COM.
- Верификация: 0 групп с неединой ценой, 0 расхождений Сумма, 0 изменённых уникальных, исходный лист
  «Свод дверей» итоги не изменились. Итоги макс-листа выше исходных (ожидаемо).

## Артефакт
`<папка-задачи>\Сводная_4_блока_двери.xlsx` — лист «Свод дверей» (642 строки, 22 колонки) +
«Комплектующие» (7 поз.). Колонки: номенклатура + этажи + Цена/Сумма + Помещение/Описание/
Комментарий + Индекс идентичности (V). Цены Блоков 2 и 4 обновлены по ответам поставщиков
(52 правки); столбец «Индекс идентичности» — 138 групп идентичных дверей. Блоки 1 и 3 по ценам
не тронуты. Исходные блоки не изменялись. Папка КП не затронута. Все этапы прошли независимый
ревью (auditor/excel-validator).
