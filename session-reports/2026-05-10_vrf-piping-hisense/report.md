# Сессия 2026-05-09…2026-05-10: VRF Hi-FLEXi S5 — спецификация и разводка фреонопровода

## Контекст

Объект — обезличено (далее `<объект>`), мультифункциональное здание под съёмочный
кинокомплекс. Два независимых VRF-контура Hisense Hi-FLEXi S5 на крыше, всего
24 внутренних блока (12+12). Подбор оборудования по прайс-листу дилера Breez.

## Тема сессии

Две большие задачи, выполнены последовательно:

1. **Актуализация спецификации VRF** под текущий каталог Hisense + сокращение
   количества кассет («многовато» по замечанию ГИП из прошлой сессии).
2. **Разводка фреонопровода** — определение диаметров на каждом сегменте
   магистрали; верификация плана разводки, который параллельно рисовал
   пользователь в DWG/DXF.

## Хронология и развилки

### Задача 1 — спецификация

Старт: была спецификация Ред.3 от другого Claude. Замечания ГИП «многовато,
актуализировать по каталогам». Каталоги в папке: Hi-FLEXi S5 global
(`TCY12024017C`), Hisense Indoor Units TC R410A V17 (07.2025).

- **Ред.4** — снизил 27→24 кассеты (минус 3 шт в Холле центр., Морге, Кор-3
  через укрупнение AVBC-24→AVBC-30). Обновил серии: НБ `…FKFSA→…HKF5`,
  кассеты `AVC-…UR4SAA→AVBC-…HJFKA`. Σ Q НБ = 117.5 кВт (цель ГИП 100–120 ✓).
- **Ред.5** — пользователь дал важное уточнение: прайс Breez делит серии по
  типоразмеру — блоки **<19 kBtu/h** в серии AVC (Mini 4-way, 570×570,
  панель HPE-D-NK), **≥19 kBtu/h** — серия AVBC (стандарт 840×840,
  HP-G-NK). Замены: AVBC-09/12/15 → AVC-09/12/15HJFA. Σ ВБ без изменений.
- **Финальный артикул НБ** — пользователь подтвердил, что по прайсу Breez
  региональный артикул HKFSXA (не HKF5 из global-каталога).

### Задача 2 — разводка

**Первая попытка (древо с двумя ветками от 1st refnet):** отвергнута
пользователем. Принцип фирмы — одна магистраль от НБ, рефнеты отщепляют ВБ.

**Вторая итерация (линейная цепочка):** правильная архитектура, но я
синтетически отсортировал ВБ по убыванию мощности. Тоже отвергнуто:
«ВБ разных мощностей идут вперемешку». Вернул естественный порядок
Cas-X.1…X.12 (как в спеке) с явной нотой «при привязке к трассе порядок
пересмотреть».

**Параллельно пользователь рисовал DXF плана разводки** — сначала прислал
«Правую часть.pdf», затем единый DXF (`Планэтажа-В_Этаж01_Кондиционирование.dxf`)
с двумя крыльями. Я извлёк из DXF координаты:
- 2 НБ, 24 ВБ (с маркировкой К1.X / К2.X)
- 21 refnet (Y-1/Y-2/Y-3 = HFQ-102F/162F/242F)
- 94 текстовых подписи диаметров

По геометрии восстановил порядок отщепления (по расстоянию от НБ).
Сделал поучастковую сверку «фактический диаметр на плане vs требуемый
по сумме нижестоящих ВБ из каталога».

### Найденные ошибки и итерации

**Правая часть (НБ-2):** 2 BLOCKER на типоразмер refnet
- Y-2 на главной магистрали где Σ=184.7 (нужен Y-3) — пользователь исправил
- Y-2 на главной магистрали где Σ=113.7 (нужен Y-1) — пользователь исправил

**Левая часть (НБ-1):** после уточнения «К2.3+К2.4 (Операционные)
отщепляются одной общей веткой» — отсутствие Φ25.4 на плане
оказалось корректным (Σ перескакивает диапазон 114-153 за один Y-tee).
Φ19.05 на отщепе ветки заменён пользователем на Φ15.88 (Σ=48.4 < 54,
по каталогу нужно Φ15.88/Φ9.53). Финальный план соответствует каталогу.

## Где промахнулся и куда смотреть

1. **Изначальная двух-веточная топология вместо линейной.** Не уточнил
   стандарт фирмы до выдачи артефакта. Линейная магистраль с
   последовательными refnet'ами — внутренний стандарт пользователя.

2. **Синтетическая сортировка ВБ по мощности.** Технически валидно, но не
   совпадает с практикой («ВБ разных мощностей идут вперемешку по трассе»).
   Правильное по умолчанию — естественный порядок из спеки или по геометрии.

3. **Парность газ/жидкость vs поучастковая сверка.** Когда пользователь
   попросил проверить план — проверил только парность газ↔жидкость на
   refnet'ах, не сделал поучастковую сверку «Σ нижестоящих → требуемый
   диаметр». Пользователь справедливо указал. Поправил после получения DXF.

4. **«Регион→прайс vs global-каталог».** Для НБ артикул в global-каталоге =
   HKF5, у дилера = HKFSXA. Нужно сразу спрашивать «какая нотация по прайсу
   у вашего поставщика», чтобы спека и план не расходились в индексах.

## Что узнал

- **Hisense Hi-FLEXi S5 (TCY12024017C 06.2025):**
  - НБ AVWT-XHKF5 базовые 8…42 HP, комбо до 120 HP
  - Подбор магистрали по мощности НБ (таблица 3.1)
  - Подбор пайпа после refnet по Σ нижестоящих ВБ (таблица 3.2 ④)
  - Refnet'ы HFQ-102F/162F/242F/302F/462F/682F
  - Ограничения: L1≤220m, L1eq≤260m, L2≤90m, L3≤40m (рек.15), H1≤110m
- **Hisense R410A V17 (07.2025):**
  - AVBC-…HJFKA (стандарт 840×840) — 4-way Cassette
  - AVC-…HJFA (mini 570×570) — Mini 4-way
  - AVS-…HJFDJD — настенные
- **Прайс Breez (важно):** деление AVC/AVBC по границе 19 kBtu/h
- **Hi-FLEXi S5 газ/жидк строго парны:** Φ22.2 газ ↔ Φ9.53 жидк, и т.д.
  Несочетающиеся пары вроде Φ22.2/Φ15.88 на схеме — индикатор ошибки.

## Workflow что работало

- **Парсинг DXF через ezdxf** — быстро вытащил координаты всех меток
  (НБ, ВБ, refnet'ы, диаметры) и линий. По геометрии восстановил порядок
  отщепления.
- **Рендеринг плана в PNG через pdf-mcp `render_pages` dpi=300** —
  PNG-проверка читается лучше чем экстракция текста (особенно для меток).
- **Парные ревьюеры `auditor` + `excel-validator`** — после каждой
  крупной правки спеки. Auditor нашёл BLOCKER «индексы НБ HKF5 в спеке
  vs HKFSXA в разводке» — это привело к Ред.6 (после закрытия файла).

## Что НЕ работало

- `Read` PDF напрямую — не годится, нужен `pdf-mcp` или `markitdown`.
- Бинарные `.xlsx` через `read_data_from_excel` MCP — превышение лимита
  токенов даже на маленькой шапке. Спас `markitdown` для конвертации в md.
- Запись в `.xlsx` пока файл открыт в Excel у пользователя —
  PermissionError. Пришлось делать отдельный файл и предупредить
  пользователя что финал обновлю когда закроет.
