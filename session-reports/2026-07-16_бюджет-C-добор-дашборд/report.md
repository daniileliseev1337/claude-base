# Session report — добор ценового бюджета C + дашборд + реестр замен

**Дата:** 2026-07-16 · **Тема:** бюджет проекта, замена изготовителей на вендоров подрядчика (обезличено)

## Что делали
Продолжение ценовой фазы бюджета инженерных систем объекта. Три бюджета: A (по смете), B (бренды
проектной документации, готов), C (те же позиции с заменой изготовителя на вендоров подрядчика).
Задача сессии — добор бюджета C (58 непокрытых позиций) + фиксация состояния + дашборд.

## Сделано
- **Разбор pending** детерминированным скриптом → публичные (веб) / непубличные (КП).
- **Веб-добыча цен** через прямой вызов Exa API из main (чат-ключ, последовательно) — обход исчерпанного
  MCP-exa у субагентов, не жжёт лимит параллельного fan-out. Добыто ~9 цен аналогов.
- **КП-список** непубличных вендоров (xlsx, сгруппирован) + разделение «запросить КП» / «вне ассортимента вендора».
- **fill-скрипт** переоценки колонки C, бюджет B неприкосновенен (полный дифф колонок = 0).
- **Ревью auditor: PASSED** (арифметика Δ сведена построчно до копейки, класс аналогов, неприкосновенность B).
- **Дашборд** (self-contained HTML Artifact, обе темы): достоверность, разделы, реестр вендоров, движения цен,
  реестр замен, УГО-легенда. Генератор: data-скрипт → template + впаивание JSON.
- **Реестр замен** изготовитель ПД ↔ вендор (xlsx, обе стороны: заменено / не выпускает / ждёт КП).

## Ключевые находки / уроки
- **Достоверность ценового бюджета по замене**: подтверждено цен ~18% суммы, ~52% — ориентир «C=B»
  (непубличные вендоры ждут КП), ~30% — законно C=B. Фиксировать долю подтверждённого явно — это честность бюджета.
- **Класс аналога решает цену**: варифокал-моторзум vs фикс-объектив камеры = ×2; активная Hi-Fi АС vs пассивная
  инсталляционная; бытовой ТВ vs коммерческий. Неверный класс = искажение бюджета.
- **Не занижать при неясной конфигурации источника**: где разрыв цены большой и спецификация не детализирована —
  оставлять ориентир + флаг, а не подставлять правдоподобную цену. Порог нерелевантности замены: цена > исходной ×3 → оставить исходную.
- **Завышения из-за премиального аналога**: замена generic-компонента на премиального вендора даёт рост в разы
  (органайзер +818%, кросс +441%, заземление +388%) — помечать флагом «проверить эквивалентность класса».
- **fill-скрипт DRY не отражает итоги** (читает старые ячейки) — судить по статистике применения.
- **Матчинг результата к таблице** — по (нормализованное наименование, марка) из ячеек; новые батчи писать
  поля дословно из классификации, иначе промах (баг прошлых прогонов: часть «не собрано» = провал матчинга).
- **Exa API дёшев** (~$0.007/запрос) и надёжнее MCP при исчерпанном ключе — прямой вызов из main.
