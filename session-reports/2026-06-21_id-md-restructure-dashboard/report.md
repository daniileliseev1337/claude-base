## Блок 1 — актуализация реестра-описи документов качества (xlsx)
- Опись документов качества живёт синхронно в ДВУХ xlsx: отдельная опись-приложение к акту (лист со своими позициями) + сводный «монолит» (сквозная опись всего тома с накопительной формулой листажа `G[n]=F[n-1]+G[n-1]`).
- Правка: убрать N позиций (чужие системы/узел/дубли), исправить реквизиты, добавить добытые документы, перенумеровать. Делегировано доменному агенту (id-engineer, sonnet) детерминированным python+openpyxl; ревью excel-validator + auditor.
- **Урок (важный):** реквизиты КАЖДОЙ правки брать с РЕАЛЬНЫХ сканов/PDF, не из отчётов-сверки — OCR-проходы (haiku/sonnet) ошибаются в номерах/датах. Поймано 2 ошибки отчёта (номер+год отказного письма; номер+срок сертификата). Низкое разрешение Read большого PNG тоже искажает — crop+zoom шапки. → memory `feedback_id_reestr_requisites_from_scan`.
- **Грабля openpyxl:** при правке книги с формулами кэш значений убирается (data_only=None до открытия в Excel) — финализировать открытием/пересохранением в Excel.

## Блок 2 — свод открытых хвостов (многоагентно)
- Workflow: 4 haiku читают зоны источников (реестр замечаний / трекер+журнал / handoff+ответы / комплектность) → sonnet синтез единого перечня (блокеры/важное/мелочь + противоречия).
- **Урок:** субагенты-читатели оценивают файлы в ИЗОЛЯЦИИ — дают ложные «блокеры» по остаточным фразам. Оркестратор обязан выверить свод против актуальных закрытий, прежде чем отдавать.

## Блок 3 — реструктуризация MD (главное)
Проблема: на каждый том ~5 трекеров-дублёров про одно («что осталось») → рассинхрон, теряются задачи (пример: правка ВОР в DWG не была в блокерах).

**Целевая структура (внедрена) — паттерн «3 живых файла на том»:**
- `01_СТАТУС <том>.md` — главный трекер (замечания, блокеры, развилки, ответы, история). Читать первым, обновлять каждую сессию.
- `02_СОСТАВ и пересборка <том>.md` — состав (удалить/заменить/добавить/перенести), каскад, ⚠ таблицы DWG к перевставке, план пересборки.
- `03_СПРАВКА <том>.md` — структура тома, инвентарь документов, сверки, источники, расчёты.
- Общее: `Методология\` (3 стабильных файла) + `README.md` + навигатор.

**Как делалось:**
1. Workflow: 7 haiku-карточек на КАЖДЫЙ из ~40 MD (назначение/что несёт/актуальность/дубли/ценность/рекомендация).
2. Оркестратор спроектировал структуру, согласовал глубину слияния с пользователем.
3. Workflow: по 3 sonnet-агента (general-purpose) на том собирают целевые файлы из источников с сохранением ВСЕГО содержания дословно (правило: дедуп только точных повторов, статусы по самой свежей дате).
4. Верификация: объём целевых vs источников (77%/101% — здоровая дедупликация), spot-check критичных маркеров.
5. Исходники → `_archive\консолидация-<дата>\` (НЕ удалять — обратимо).
- Итог: ~36 активных MD → 10 живых. Навигатор/README/память обновлены.

**Уроки реструктуризации:**
- НЕ удалять рабочие файлы — архивировать (обратимо), удалять только по явному подтверждению.
- Верифицировать полноту слияния объёмом + spot-check маркерами, не доверять отчётам агентов.
- Сравнить «дубль-копию» построчным diff перед архивацией (была устаревшая копия, отстающая на N правок).

## Блок 4 — xlsx-дашборд + хук синхронизации
- **Принцип:** MD (`01_СТАТУС`) — источник истины, xlsx — производный обзор.
- `tools/generate_id_dashboard.py` — парсит таблицы замечаний/блокеров из обоих `01_СТАТУС` ПО ИМЕНАМ КОЛОНОК (переживает правки формата), нормализует статусы по эмодзи, генерит xlsx (4 листа: Сводка с %готовности, Замечания на том, Блокеры) с цветовой заливкой.
- **Хук** `tools/regen_hook.py` + project `.claude/settings.local.json` (PostToolUse, matcher Edit|Write): при правке файла `01_СТАТУС*.md` авто-перегенерирует дашборд.
- **Грабля кириллицы:** в команде хука использовать ОТНОСИТЕЛЬНЫЙ ASCII-путь (`python tools/regen_hook.py`) — кириллица остаётся в cwd (его задаёт харнесс), а не в парсинге команды shell. Тестировать хук подачей UTF-8 байтов через subprocess (PowerShell pipe искажает кириллицу в stdin — даёт ложный «не сработал»).
