# Session report — закрытие блокер-гейта О3-3 (сверка реестра замечаний с первоисточниками)

**Дата:** 2026-06-24
**Блок:** ПТО / ось 3 Графика R-слой. Том ИД раздела ВиВ. Обезличено (методология; объект/шифры/ФИО — плейсхолдеры).

## Контекст
Открытый системный блокер-гейт: пересказы замечаний в трекерах ненадёжны (доказано ранее — пометка
«Расход воды» по видео = «расход/расчёт ПЕНЫ»). Регламент: до работы по любому замечанию открыть
первоисточник и сверить дословно; переход к след. оси при открытом кросс-блокере запрещён.
Действие к закрытию: пройти весь реестр, у каждого пункта проставить первоисточник + флаг сверки,
исправить расхождения, распространить на смежный том.

## Что сделано (read-only по первоисточникам + правки md-трекеров; DWG не трогался)

  - 13 пунктов ТН (фото печатного списка) — дословно, все совпадают.
  - 11 фото карандашных пометок P1-01…P1-11 — стикеры прочитаны плотными кропами с поворотом 180°.
  - 5 видео V1…V5 — кадры извлечены cv2 (ROTATE_180 + детектор синей ручки по площади синего HSV,
    кадры «остановки» камеры, плотный вырез).
- **Артефакт-сводка** «О3-3 — сверка реестра замечаний с первоисточниками» (таблицы пункт →
  первоисточник → дословно → статус → расхождение).
- **Флаги/исправления внесены** в трекеры тома (§ статус, § состав, § блокеры): фантом вычищен,
  расхождения помечены. Гейт переведён в ✅ ЗАКРЫТ.
- **Распространение на смежный том:** добавлен блок-предупреждение «гейт сверки первоисточников»
  в его трекер состава.

## Находки сверки
1. **Систематического искажения НЕТ.** Подавляющее большинство пунктов сверено дословно и совпадает.
2. **Единственный фантом-действие** (вода→ПЕНА) — устранён ранее; теперь **подтверждён на чётком
   видеокадре** («Расчёт пены / 1 и 2 этаже? / узел устройства прокладки»).
3. **Дубль-фантома** «расход воды» в стикере P1-09 (схема дезинфекции): на стикере ТОЛЬКО
   «расход материалов для дезинфекции» — «вода» приклеилась из видео-пометки. Вычищено (дезинфекция
   к тому же снята пользователем ранее).
4. **Расхождение идентификации источника:** фото, подписанное в трекере/имени файла как сертификат
   «Пирон/Pyron», по факту = сертификат другого изготовителя (тип изделия совпадает, срок до той же
   даты). Текст замечания («недействителен, заменить») верен; на действие не влияет (замена сделана).
5. **Неоднозначность:** вторая пометка одного видео (левый стикер) дословно не совпадает с пересказом
   «Объёмы см. лист 2» — читается иначе, смысл неясен; на действие не влияет (основная пометка точна).
6. **Побочное для оси 4:** дата приложения к одному из актов в видео = 21.12.2025 (связано с ранее
   зарегистрированным блокером про даты актов).
7. Прочее — мелкие текстовые отличия (синонимы, орфография оригинала), правок не требуют.

## Где сломался / уроки
1. **cv2.imwrite молча НЕ пишет в кириллический путь на Windows** (вернул False, файлы «создались»
   с искажённым именем → Read не нашёл). Решение: сохранять кадры в **ASCII-путь** (`C:\v_frames`)
   через `cv2.imencode('.jpg', img)[1].tofile(path)`. При этом `cv2.VideoCapture` кириллический путь
   ВХОДА открывает нормально — ломается только запись. (Совпадает с правилом «PNG на ASCII-путь».)
2. **Имя файла первоисточника — это тоже пересказ.** Фото, названное «…Пирон…», оказалось другим
   сертификатом. Гейт сверки ловит именно это: открывать содержимое, не доверять имени.
3. **Метод чтения телефонных фото стикеров:** exif_transpose (часть фото с EXIF-поворотом) + upscale
   ×2 для восприятия, затем плотный кроп зоны стикера с поворотом 180° ×4-6 — рукопись читается
   уверенно. Полный кадр в уменьшении — недостаточно (ловушка «не доверять Read уменьшенного»).
4. **Детектор синей ручки по площади синего работает**, но кадр макс-синего бывает смазан (камера
   в движении) — для дословного чтения брать чёткий кадр (max Лапласиан) среди кадров с синим в
   нужном временном окне.

## Статус осей (Р-1)
Гейт О3-3 ✅ ЗАКРЫТ → работа по R-слою разрешена. Остаётся R-слой: **B** (Т4 в ИТП + аксонометрия
3/3), **V2** (подвод к пож. крану — на листе уже есть «Подъём Ф25 к пл.кр.»/«Пл.кр.-1 Ф25»),
**V3** (узлы учёта М + легенда арматуры — РД-факты готовы). **A** — простановка за исполнителем
(гибрид). Графику Revit-DWG вести гибридом; ревьюер графики = opus; бэкап DWG до правок.

## Харвест-кандидаты
- Ловушка `cv2.imwrite` + кириллица → в `image-text-replace` / `local-video-digest`: всегда писать
  кадры через `imencode+tofile` в ASCII-путь.
- Паттерн «сверка реестра замечаний с первоисточниками» (фото-стикеры + видеопометки) — кандидат
  в отдельный мини-скилл/чек-лист для блока ИД (гейт первоисточника).
