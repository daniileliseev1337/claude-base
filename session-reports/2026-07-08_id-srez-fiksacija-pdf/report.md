# Сессия 2026-07-08 — конвейер ИД по срезам: шаги 7(хвост)–11 + фиксация PDF тома

**Контекст:** полигон-копия тома ИД (<объект>, срез «оборудование водоснабжения»), конвейер
11 шагов по DESIGN блока pto. Продолжение сессий 07.07 (шаги 0–7).

## Что сделано

1. **Хвост шага 7 (аксонометрия):** текст-слой плота оказался живым (не SHX-битым, в отличие
   от планов) → счёт выносок детерминированным скриптом (regex «N шт» + суммирование по
   артикулам). Все ключевые позиции ≡ ВОР. Урок: сначала пробовать текст-слой, vision — fallback.
2. **Шаг 8:** сверка «файлы ↔ позиции реестра» скриптом (35/35), унификация формата
   (дата-datetime → текст), финишный автофит Excel COM + self-check.
3. **Шаг 9:** дозапись реестра-описи через COM (вставка строк сохраняет формулы соседей,
   в отличие от openpyxl insert_rows); инвариант книг пересчётом: построчный (ВОР↔ПВ↔ПВ ИД)
   + агрегация по типам позиций через классификатор-RULES (маркеры моделей) — обе PASSED.
4. **Шаг 10 (фиксация PDF):** карта живого тома текст-маркерами + dhash-опознание сканов
   (все 32 скана опознаны, порог d≤40 на 16×16). НАХОДКА МАСШТАБА: сданный том массово
   расходился с реестром (8 позиций физически отсутствовали, 32 стр. мусора вне реестра,
   схема отставала от факта). Три развилки владельцу (способ фиксации / листажи / вложения)
   → пересборка блока по реестру, листажи по факту, обоснования вшить. Сборка fitz по
   манифесту, 466 стр., листажи сошлись до страницы (238 = реестр).
5. **Шаг 11:** гейт (папки, полнота, виджет) + параллельное ревью pdf-reviewer ∥ auditor.
6. **ТЗ на спец-агентов** конвейера + карта параллелизма — из практики прогона
   (`blocks/pto/ТЗ АГЕНТЫ — конвейер ИД по срезам (2026-07-08).md`): 4 агента + жирный
   tools-слой (8 обкатанных скриптов на вынос в skill).

## Уроки / грабли

- **Текст-слой плотов НЕОДНОРОДЕН в одном комплекте:** планы SHX-битые, аксонометрия живая.
  Проверять per-лист (len(get_text)), а не решать за весь комплект.
- **dhash 16×16 на 50 dpi надёжно опознаёт сканы** против файлов-эталонов (d=0–13 у
  совпадений, d≥40 у чужих) — дешёвая альтернатива vision для «страница тома = какой файл?».
- **Построчный zip ВСО↔ВОР ложно фейлит:** внутри блоков помещений порядок позиций различается
  (перестановка), правильный инвариант — агрегация по каноническому типу позиции.
- **Классификатор типов: правила с артикулами (39424000) ставить ВЫШЕ словесных**, длинные
  имена ВОР содержат чужие подстроки («гибкая подводка» внутри имени смесителя).
- **«Кол-во листов» по типам:** формы (акт/ВСО/реестр) — двусторонка (1 лист = 2 стр.),
  копии докквалити — 1 стр. = 1 лист. Опись живёт по этой модели, G[n]=F[n-1]+G[n-1].
- **Move файла может упасть «Device or resource busy»** (открыт в браузере/синке) — не
  форсировать, оставить с пометкой в журнал.
