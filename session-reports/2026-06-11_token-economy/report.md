2. **Телеметрия восстановлена.** PostToolUse-хук был потерян (tool-usage.jsonl не
   писался с 10.06, хука не было ни в одном конфиге). Новый скрипт
   `scripts/tool-usage-track.ps1`: пишет jsonl + каждые 75 вызовов сессии подсказывает
   Claude предложить handoff (additionalContext). Зарегистрирован в **settings.local.json**
   (обкатка на хабе, в shared не уезжает). Ротация лога на 5 МБ.

3. **Мониторинг: ccusage** (npx ccusage daily / blocks). Разбирает локальные
   транскрипты, показывает расход по дням/моделям/5ч-окнам, cache отдельно,
   у активного окна — прогноз выгорания. `ccusage blocks --live` — живой монитор окна.
   Замер подтвердил диагноз: 08-10.06 — по 100-140M в день, **cache reads = 91%**
   (315M из 345M за 4 дня); лимит 5ч-окна ≈ 69M токенов (окно 09.06 06:00 = 100%).

4. **Ревизия плагинов/MCP.** CLI: из 11 установленных плагинов включены только 2
   (superpowers, claude-md-management) — выключенные веса не дают, удалять не стал
   (Karpathy #3). Глобальные mcpServers = эталон манифеста (11 core + autocad +
   firecrawl) — не трогал. Тяжесть desktop-сессий — от коннекторов приложения:
   дубль Desktop Commander (MCP + плагин), Windows-MCP при наличии встроенного
   computer-use, Autodesk Knowledge MCP. Файлово не правится — клик-лист пользователю
   (Settings → Extensions/Connectors в desktop-приложении).
