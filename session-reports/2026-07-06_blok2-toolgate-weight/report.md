# Сессия 2026-07-06 — Блок 2 реворка: сессионный гейт по весу (log-tool-usage.ps1)

## Что сделано

PostToolUse-хук `scripts/log-tool-usage.ps1` переписан: сессионный гейт срабатывает по
ВЕСУ tool-результатов, вернувшихся в основной контекст, вместо счётчика 75 вызовов.
Направление владельца (BASE-FIX 2026-07-01) выполнено полностью: (1) вес, (2) один раз,
(3) вес — на хуке, смена темы — на модели.

Механика: вес вызова = длина `ConvertTo-Json(tool_response)`; копится в
`.local-state/toolgate/<session_id>.json` под именованным mutex; выстрел ровно на первом
пересечении порога (512 КБ, override env `CLAUDE_TOOLGATE_BYTES`), fired=true исключает
повторы; субагентские вызовы (`transcript_path` с `\subagents\`) в вес не идут, в
телеметрии помечаются `sub=1`; телеметрия обратно-совместима (+поля bytes/sub);
append лога и ротация — под вторым mutex (гонка теряла строки). UTF-8 In/Out.

3. Зонд stdin PostToolUse: поля session_id, transcript_path, cwd, tool_name, tool_input,

   результатов: сегодняшняя сессия не заорала бы, монстры ловятся на середине.

## TDD и ловушки PowerShell 5.1 (все пойманы тестами, задокументированы)

Тесты: `scripts/tests/test-toolgate.ps1` — 22 assert'а чёрным ящиком (изолированный
USERPROFILE, синтетический stdin, Start-Job-гонка). RED → GREEN честный.

- **BOM в `$OutputEncoding` ломает stdin-пайп**: `[Text.Encoding]::UTF8` подсовывает
  преамбулу EF BB BF в пайп к дочернему powershell → ConvertFrom-Json тихо падает.
  Лечение: `New-Object UTF8Encoding($false)` + защитный `Trim([char]0xFEFF)` в хуке.
- **PSCustomObject не имеет intrinsic .Count в 5.1** → счётные ассерты на скалярах лгут;
  всегда оборачивать в `@()`.
- **Comma-return (`return ,$arr`) скармливает Where-Object весь массив одним объектом**
  (member-enumeration маскирует: часть ассертов «проходила случайно»). Функции-читалки —
  обычный enumerate-return + `@()` на вызывающей стороне.
- **Start-Job требует живой USERPROFILE родителя** (persistence path) — песочницу
  передавать аргументом, не подменой env родителя.
- Ассерты долговечности параллельных append'ов — с retry-окном (мгновенная видимость
  не гарантируется).

## v2.5 (2026-07-07) — гейт по РЕАЛЬНОМУ token-fill

Вопрос владельца «может ли сессия узнать реальный контекст?» → да: транскрипт сессии
пишет usage каждого api-хода; последняя запись = фактический контекст
(input+cache_creation+cache_read). Хук получает transcript_path в stdin → читает хвост
(~256 КБ с конца, парс последней usage-строки) и гейтит по реальным токенам:
порог 700000 (≈70% окна 1M — вилка владельца 65–75%), env CLAUDE_TOOLGATE_CTX_TOKENS.
Байт-прокси 512 КБ — только fallback при недоступном транскрипте (fail-open сохранён).
+5 ассертов (27/27; старые 22 с fake-путями = доказательство fallback-ветки).
Живой smoke на сессии разработки: `{bytes:375856, fired:false, ctx:447168}` —
байт-прокси подползал к ложному срабатыванию, реальный fill честно молчит (45% окна).
Бонус-знания: реальный fill «видит» компакцию (байты — нет); `totalTokensReminder:
"countdown"` (@internal, settings) может показывать остаток окна самой модели —
кандидат-пилот; поправка владельца: автокомпакт наступает при переполнении 1M
(мой «сессия пережила компакт» был домыслом); ccusage-статуслайн владельцем не
используется (терминальный) — кандидат Блока 4 на пересмотр в shared.
Ревью: облегчённое по памяти light-review (полный auditor — на ядре блока 06.07).
