# OpenCV line/geometry stack (УЖЕ УСТАНОВЛЕН — 0 установки)

- **URL:** https://github.com/opencv/opencv (+ opencv-contrib `ximgproc`)
- **Stars:** 80k+
- **License:** Apache-2.0
- **Описание:** не «находка», а наличный инструмент; ключевое для «точно».

## Зачем смотрели

Реальный workhorse точной трассировки по растру — не один magic-репозиторий,
а связка примитивов CV, которыми мы УЖЕ владеем (opencv в окружении):
- `cv2.ximgproc.createFastLineDetector` / LSD / `HoughLinesP` — точные
  координаты прямых сегментов (стенки воздуховодов).
- `cv2.findHomography` / 4-точечный perspective transform — выпрямление
  **фото** чертежа (когда присылают снимок под углом).
- `cv2.matchTemplate` — повтор символов (диффузоры/решётки) одного типа.
- `skimage.morphology.skeletonize` (BSD) — осевые без GPL-autotrace.

## Оценка

- Подходит? **Да — это база, к которой vtracer/autotrace идут добавкой.**
- Сильные стороны: 0 установки, лицензия чистая, полный контроль координат →
  можно **снапить** детектированные сегменты к строгим углам/осям и точным мм
  (вот это и даёт «без неточностей», а не slepoe тыкание по пикселям).
- Слабые стороны: нужен наш код-обвязка (детект → фильтр → snap → transform→мм →
  выгрузка координат для LISP-вставки динблоков). Это пишем сами в tools/.
- Решение: **ядро пайплайна**; внешние репо — вокруг него.
