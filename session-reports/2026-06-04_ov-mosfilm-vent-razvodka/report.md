# Сессия: ОВ-проект квартиры — инструменты и метод трассировки вентиляции по растру

**Дата:** 2026-06-04 … 2026-06-05
**Объект:** <объект-Москва-ЗАО>, квартира ~188 м² (обезличено)
**Задача:** этап «разводка вентиляции» проекта воссоздания ОВ-DWG из растровых источников.

## Что сделано

1. **Разрешён блокер прошлой сессии.** Схема вентиляции — растровая (вектора нет).
   Пользователь: в таких проектах вектора/DWG часто НЕТ (бывают фото) → нужно научиться
   чертить по растру. Вызван `/harvest`.

2. **Harvest: стек инструментов под трассировку по растру** (верифицировано через GitHub API):
   - **OpenCV + skimage.skeletonize** (есть, Apache/BSD) — ядро: цветомаска, контуры, снап, центрлайн.
   - **vtracer** (visioncortex, 6.1k⭐, MIT) — установлен `pip install vtracer`. Общий растр→SVG.
   - **autotrace** -centerline (707⭐, GPL-2.0) — установлен portable БЕЗ админа.
   - В резерве: Deep-Vectorization-of-Technical-Drawings (MPL), PID_Symbol_Detection (MIT).
   - Заметки: `harvested/*.md`.

3. **autotrace без админ-прав** (ПК без админа): NSIS-`setup.exe` через UAC не ставится →
   распаковка 7-Zip (`7z x setup.exe`), win32-сборка, закреплено `…\Tools\autotrace32\`.
   **Блокер:** нет `libssp-0.dll` (i686) → exit 127. Диагностировано `pefile`. Современный
   MSYS2 gcc-libs её уже не содержит — добирать из архивной версии. Пока заменяет skimage.skeletonize.

4. **Метод трассировки установлен и проверен на реальной схеме** (A1, plot MEP, @300dpi 9934×7017):
   - Цветоразделение HSV приток(красный)/вытяжка(синий) — **работает чисто**.
   - Воздуховоды = залитые полосы с двойным контуром → **контуры стенок** (findContours+approxPolyDP),
     а НЕ глобальный скелет (skeletonize+Hough рвётся — проверено, тупик).
   - vtracer «в лоб» по всему листу = 68k path (шум) → нужен таргет (сначала цветомаска).

5. **Привязка к подложке согласована:** схема = тот же план квартиры → аффинно по углам стен.
   Подтверждено выделением архитектуры со схемы (совпадает). Целевая рамка — блок
   `Подложка_Y703`, модельные X 751479–768901, Y 1548–13933, габарит 17422×12385 мм.

## Где сломался / уроки

- **autotrace — DLL-погоня съела много времени.** Урок: при portable-MinGW-сборке СНАЧАЛА
  `pefile`-разбор импорта (точный список недостающих DLL), а не угадывание/перебор. И сразу
  оценивать, есть ли уже установленный эквивалент (skimage.skeletonize) — не перфекционировать
  редундантный инструмент в ущерб основной задаче (Karpathy #2).
- **Порог архитектуры:** screened-underlay стены светло-серые → ловить по низкой S (S<50),
  а не по тёмной V. Первый порог V≤130 поймал 2459 px (пусто).
- Дважды прерывался интернет — фоновые web-команды отваливались по таймауту.

## Следующий шаг (новая сессия)

Аффинная регистрация 2-3 пар углов стен (схема px ↔ подложка мм) → трансформ контуров притока
в координаты подложки → импорт направляющей → разводка динблоками `!Приток` (цвет 1).
