## Что сделано

1. **STOP-процедура** — MCP 11/11 core (+autocad/firecrawl/Revit), агенты 16/16, install-долга нет. Роль — хаб.

2. **Разрешены 4 развилки артикулов оконечного оборудования** (главный блокер тома), все по фактам, без догадок:
   - Источники в порядке приоритета: **СВОД ОКОНЕЧКИ (факт закупки/монтажа) > РД-спецификация (проект) > datasheet/каталог**.
   - Клавиша смыва: проект+факт = одна модель, арт. подтверждён datasheet'ом дословно (наименование/цвет совпали) — ВСО верна.
   - Блок питания (низковольтный монтажный комплект): легитимен — есть в РД и своде «по проекту», сосуществует с сетевой системой смыва. ВСО верна. Прежняя запись «в проекте нет» — ошибка инвентаря, снята.
   - Душевой поддон: проект = бренд A, **факт (свод) = бренд B того же размера** — добытый документ подходит, ВСО верна.
   - Раковина МГН: **проект = арт. ...660, факт (свод) = арт. ...659**. ВСО (659) ВЕРНА. Едва не внесли ошибочную правку «659→660» по РД — свод предотвратил. Паспорт ...660 принят как унификация линейки.

3. **Найден ранее неизвестный источник факта** — главная таблица «что покупалось-монтировалось» по разделу (4 строения). Не был в навигаторе/трекерах. Колонки: проект / ЗАМЕНА(факт) / купили. Внесён в навигатор + сохранён в memory как приоритетный источник для разрешения артикулов в любом томе раздела.

## Где сломался / поучительное

- `.doc` (старый бинарный OLE2) не читается ни markitdown, ни word-MCP; document-loader заперт в рабочей папке. Решение: конвертация `.doc`→`.docx` через Word COM (PowerShell), затем word-MCP.
- excel-MCP `read_data` на широком листе выдаёт по-ячеечный JSON → переполнение токенов. Решение: pandas-дамп в UTF-8 (консоль Windows искажает кириллицу — писать в файл, не в stdout).
- Гонка с hook'ом дашборда: одна вставка строки в трекер «потерялась» (хук перегенерил), хотя Edit вернул success. Урок: после правки трекера, на который висит хук, **проверять grep'ом, что правка устояла**.
- Главный урок: **проект (РД) ≠ факт**. Источник факта монтажа (свод закупки) приоритетнее РД. Подтверждает память про «реквизиты с реального скана, не из отчёта».
