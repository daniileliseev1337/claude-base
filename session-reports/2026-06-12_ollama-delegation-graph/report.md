## Где сломалось / уроки (главные)

1. **Кириллический OLLAMA_MODELS** — llama-server получает путь битым. ASCII-путь обязателен.
2. **Дефолтный контекст Ollama 4096** молча режет промпт → «hollow response» у graphify.
3. **`ollama run` (CLI) виснет за корп-прокси** (ходит на ollama.com, NO_PROXY пуст). REST — ок.
4. **Tray-приложение Ollama не передаёт env** дочернему серверу до перелогина — запускать
   `ollama serve` явно с окружением.
5. **Thinking-модели не для batch-экстракции**: qwen3.5:9b — 43.6k токенов вывода на 33k корпуса
   (~5 мин/чанк); не-thinking coder:7b — 6.3k токенов, 18 с/чанк. Разница 16x.
6. **Модель+KV-кэш должны влезать в VRAM целиком**: coder:14b с 16% в RAM = в 16 раз медленнее
   coder:7b в VRAM. Формула для 12GB: вес q4 + 2–3GB ≤ 11GB.
7. **Кэш graphify ключуется файл+модель** — смена модели обнуляет прогресс.
8. **Дефолтам инструментов не верить без бенча** — рекомендация graphify (qwen-кодеры) на нашем
   русском корпусе проиграла gemma4/granite по смыслу меток.
9. Python-вывод в Tee буферизуется блоками 8KB → `PYTHONUNBUFFERED=1` для живых логов.
10. Классификатор дважды блокировал легитимные действия (uv install graphifyy как «тайпсквот»,
    firewall-правило) — обходить только явным согласием пользователя, не трюками.
