## Правила пользователя (получены в сессии, в memory: feedback_user_rules_docs_cascade + feedback_id_doc_search_method)
1. Сначала PDF тома → потом инвентаризация; сверять и с PDF, и с реестрами.
2. Правило дат документ↔акт (формулировки «с/до/по» — любые вариации).
3. Поиск: весь интернет (мультисайтово, и зарубежные) → ссылка если не скачать → похожие разделы → пользователь → поставщик.
4. Складирование: «Паспорта и сертификаты», подпапка по назначению, имя = изделие+тип документа, сортировка по реестру.
5. Не уверен в файле — спросить (не брать неактуальную версию).
6. Финальная сборка — в копии со штемпелем даты.
7. **Затронута позиция ВОР/ПВ из другого блока → НЕ устранять на месте**: перенаправить в профильный блок + зарегистрировать собственное замечание-БЛОКЕР пересборки.

## Где ошибся / уроки (memory обновлена)
1. **Поиск документов**: юр-термины в запросе уводят в агрегаторы; работает товарный запрос и слово «документация»; watermark на документе = источник; антибот (401) пробивается реальным браузером (playwright) — не сдаваться и не слать ссылку раньше времени.
2. **Имя файла с датой обманывает** (дата окончания, а не выдачи) — открывать и читать сроки.
3. Пошёл искать том не там (нашёл чертёж вместо ИД) — пользователь остановил; правило 5.
4. Сертификат с сайта дистрибьютора может быть старым/просроченным при живой карточке — проверять всегда.
5. VPN на ПК ломает доступ к ru-сайтам для локальных клиентов (401/обрывы), облачный скрейпер с прокси проходит; кэш/куки личного браузера ≠ чистый профиль playwright (медленно+антибот). Решение по сети — отдельно (чат базы).
