# Сессия: валидация личной ПТО-базы на открытых вопросах vault

**Дата:** 2026-06-20. **Модель:** Opus 4.8. **Канал vault:** MCP Obsidian.

Продолжение сессии «открытые вопросы ПТО» (см. handoff `2026-06-20_pto-open-questions-daniil`).
Цель: проверить, верно ли личная ПТО-база (доменный агент `id-engineer` + граф/методичка ИД +
реальные тома объекта) отвечает на доменные вопросы из vault. Совпадение с практикой
customer zero = и ответ на вопрос, и валидация базы.

## Что сделано

Обработаны 3 целевых вопроса (flow: база отвечает → практик подтверждает/правит → запись в ноту):

- **SD-116** (перенумерация выпущенного акта) — закрыт. Q1 сценарии (вставка пропущенного,
  исправление ошибки); Q2 порядковый + шифр раздела; Q3 только каскад; Q4 подписанный заморожен
  по номеру, кроме переформирования ИД с переподписанием.
- **SD-100 Q3** (мандаторность ВОР/ВСО) — Q3 отвечен, нота осталась open (Q1/Q2 — продукт).
  Таблица ВСО/ВОР по типам актов + правило pre-flight warning, дифференцированное по типу акта.
- **SD-102** (раздел РД: проектная докум. + добивки) — закрыт. Стадия П как `kind: project`;
  отдельный ordinal «Поз.» для движка было-стало; датированная редакция; экспорт+импорт
  спецификации в Excel на бэке.

По каждому — спавн `id-engineer` (sonnet) с чтением методички ИД, регламента и реальных томов;
черновик ответа базы; правки практика; оформление `[x]` + блок «### дата — <практик>» в
«Обсуждении» + резолюция (для закрытых).

## Источники

## Где сломалось / грабли

- **Кириллица в путях через Bash:** `python <кирилл-путь>` даёт mojibake в argv (cwd-независимый
  скрипт графа не запустился). Решение — PowerShell с `chcp 65001` + `PYTHONUTF8=1` + call-оператор
  `&`. Read/Glob/Edit с кириллицей работают штатно (ломается именно argv для native exe в Bash).
- **SendMessage недоступен** в этой среде (нет в deferred tools) — продолжить того же субагента
  нельзя, каждый вопрос = свежий спавн id-engineer (перечитывает методичку).
- Граф методички (BFS) не содержит семантики перенумерации (это NOT IN DOCS) — выводили из
  принципов (каскад, сквозная нумерация, «факт первичен», ADR про скан-истину).

## Уроки (валидация базы — главный результат)

База как каркас рабочая, но customer zero выявил **2 содержательные неточности** + 1 пробел:
1. **ВОР сужена до АОСР** — на деле ВОР у КАЖДОГО акта, закрывающего работы (вкл. АОМР, рядом
   с ВСО). ВСО только у АОМР. АОУСИТО/испытания — без ВОР/ВСО. → правка графа ИД в backlog.
2. **«Поз.» РД** база видела как навигацию — на деле это движок «было-стало» РД↔ИД (diff
   позиций), которого в базе нет. Номера РД и ПВ не совпадают — норма.
3. Подписанный акт: база переоценила стоимость переподписания — при принятых работах
   переформирование ИД (разбить/добавить акт) даёт лёгкое переподписание.

Вывод: доменный агент даёт качественный каркасный ответ из методички, но финальная сверка с
практиком обязательна — он ловит сужения модели и продуктовую специфику, отсутствующую в графе.
Уточнения сохранены в память; правку графа по мандаторности ВОР внести при следующей актуализации.
