# Тест Conductor — supervised-слой над Claude-сессиями (2026-07-01)

Программа реворка базы, ветка «оркестрация/надзор сессий». Решение прошлой сессии: тестируем
`rmindgh/Conductor` в изоляции → при успехе в базу. Критерии владельца: **не локально · бесплатно · просто.**

## Что такое Conductor
PoC (MIT, автор один, последний коммит 2026-05-11): одна Claude-сессия надзирает/оркеструет другие.
2 слоя мониторинга:
- **Path 1** — чтение JSONL-транскриптов + PreToolUse guard-хук (блок опасных команд) + Telegram-алерты. Документированные примитивы.
- **Path 2** — **Remote Control**: WebSocket `wss://api.anthropic.com/v1/sessions/ws/{id}/subscribe`, авто-approve инструментов (сессия не ждёт «Y»), inject задач между сессиями, чейнинг. Reverse-engineered недокументированный протокол.

## Метод — 2 этапа (verifiable-first, от 0 риска к решению)

### Этап A — изолированная разведка (0 риска)

### Этап B — живой изолированный тест (по согласованию владельца; трогает конфиг+аккаунт)
Изоляция: тест-папка под Desktop + временная trust-запись в `~/.claude.json` (бэкап снят, откачено после); НЕ ставили глобальный хук/MCP.
- **B1 ✓**: `claude remote-control` (persistent server-mode, скрытая subcommand) поднят в фоне (обойдён интерактивный промпт `Enable Remote Control? (y/n)` подачей `y` + удержанием stdin). Сессия зарегистрирована на аккаунте: bridge-pointer.json (`session_*`), env/cse ID, сервер «Connected», доступ с claude.ai/code.
- **B2 ✓ РЕШАЮЩИЙ**: точечный probe строго по протоколу Conductor к живой сессии →
  **WS-endpoint ПРИНИМАЕТ подключение** (WS_OPEN, OAuth-токен авторизует), **НО сервер НЕ шлёт `initialize`** (полное молчание 12 с). В протоколе Conductor сервер обязан инициировать handshake сразу. На CC 2.1.195 официальный клиент работает через **work-poll** (`GET /v1/code/sessions/.../work/poll` — видно в debug-логе самого сервера), а не push-subscribe-initialize.
  → Прикладной протокол Path 2 **устарел**: WS открывается, но `initialize→can_use_tool→auto-approve` не запускается.
- Подтверждено **поведением реального** `remote-control.py`: discovery (GET /v1/code/sessions → 200) находит все RC-сессии аккаунта и открывает WS ко всем, но это лишь открытые сокеты без рабочего handshake.
- **B3**: авто-approve недостижим (заблокирован B2 — протокол не доходит до can_use_tool).

## Оценка по критериям владельца (финал)
- **② Бесплатно — ДА.** По подписке, без API-ключа/платного SaaS (Telegram-бот отдельно, бесплатно).
- **① Не локально — наполовину.** Модель-надзиратель облачная (Claude по подписке — плюс vs Overseer/NeMo с локальной моделью), НО локально ставятся скрипты + **персистентный сервис** + Node-хук + правки 2 конфигов → «ставить нечего» не выполняется.
- **③ Просто — НЕТ.** Многоточечная интеграция в ОСНОВНУЮ систему (MCP в ~/.claude.json, глобальный PreToolUse-хук на каждый Bash/Write/Edit/Agent, персистентный сервис, Telegram) + хрупкость.

## Вердикт: Conductor как ПАКЕТ в базу — НЕ брать
1. Несущая ценность (Path 2 — авто-approve живых сессий + inject) **не работает на CC 2.1.195** и будет ломаться впредь (undocumented, сервер ушёл на work-poll). Хрупкость, предсказанная на Этапе A, подтверждена фактом.
2. Полноценный Conductor требует вторжения в основную конфигурацию → против правила harvest «чужой код не в основную систему».
3. Не проходит «просто» и «ставить нечего».
4. Command-analysis — сам автор: «speed bump, не sandbox» (обходится bash -c/base64/переменными).

## Находка (приватность/безопасность)
