## Итерации и где сломалось

- **CID-encoding PDF.** `pdf_read_all` дал текст-кашу. Решение: рендер 180dpi и визуальное чтение. Урок: если PDF создан через PScript+Distiller — сразу на рендер.

- **MCP excel max-tokens overflow.** Чтение 1 листа `Спецификация` через `mcp__excel__read_data_from_excel` отдало 64K символов JSON из-за валидационных метаданных в каждой ячейке. Решение: openpyxl напрямую.

- **PowerShell cp1251 + python stdout.** UnicodeEncodeError на русских символах. Решение: `sys.stdout.reconfigure(encoding="utf-8")` + `$env:PYTHONIOENCODING = "utf-8"` + `Out-File -Encoding utf8`.

- **openpyxl MergedCell.** При записи формул в ячейки merge-range — `AttributeError: 'MergedCell' object attribute 'value' is read-only`. Решение: try/except — формулы в merge применяются через corner-ячейку, остальные read-only, это нормально.

- **insert_rows vs INDIRECT.** Когда добавлял коробку Блока С в .С.П — нужно было встроить **внутрь** документа (между Блоком С и заголовком Блок П), а не дописывать в хвост. `insert_rows(85)` сдвинул всё ниже на +1, но формулы `На печать` со статичными AD продолжают работать корректно: AD=85 теперь показывает новую коробку, AD=86 — заголовок "Блок П", и т.д. Архитектура с INDIRECT идеально пережила сдвиг.

- **Hard-coded поверх формул.** Кто-то ранее правил Excel вручную, ввёл значения прямо в ячейки `На печать` поверх формул. openpyxl видит их как строки/числа, а не формулы. Мои правки в `Данные` не влияли на эти ячейки. Решение: donor-файл-эталон + автоматическое сравнение и замена.

- **Решение заказчика менялось 3 раза за 36 минут.** По коробкам в Блоке С .С.П. Сохранил в memory с историей колебаний — урок: «не записывать в memory правило по одному письму, ждать финального ответа». Также пометил что **первая моя версия memory была ошибочной** (записал «не добавлять») и заменил после второго письма.

- **Малый штамп ≠ зеркало большого.** Изначально я записал B15=B16=B17 как полное зеркало (3 строки). Реально малый штамп использует **только B16 и B17** (2 строки) — пользователь напомнил про СПДС-правило «на 2-й странице только 2 последних изм». B15 чистил.

## Технические заметки на будущее

- **Donor-pattern для восстановления формул** — когда формулы перебиты hard-coded значениями в Excel, использовать другой файл с той же архитектурой как эталон. Работает только если файлы построены по одному шаблону (как наши 4 СО Р-варианта).
- **`pdf_render_pages` 180dpi** — оптимальный компромисс читаемости/размера для штампов СПДС.
- **Лимит на чтение через mcp__excel__read_data_from_excel** — на сложных файлах с валидацией легко превышается 64K символов. Сразу openpyxl.
