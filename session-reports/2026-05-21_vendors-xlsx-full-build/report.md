## Технические рецепты (для повторения)

### Запись в xlsx без потери drawings
**НЕ через MCP excel** (любая запись выкидывает `xl/drawings/` слой). Только ZIP-хирургия:
1. `[System.IO.Compression.ZipFile]::ExtractToDirectory`
2. правка `xl/worksheets/sheetN.xml`, `xl/sharedStrings.xml`, `xl/drawings/drawing1.xml`
3. `[System.IO.Compression.ZipFile]::CreateFromDirectory`

### Вставка нового логотипа
1. PNG в `xl/media/imageN.png`
2. Добавить `<Relationship Id="rIdN" Type=".../image" Target="../media/imageN.png"/>` в `xl/drawings/_rels/drawing1.xml.rels`
3. Добавить `<xdr:twoCellAnchor>` в `xl/drawings/drawing1.xml` с координатами `<xdr:from>/<xdr:to>` (col/row 0-indexed) и `<a:blip r:embed="rIdN"/>`

### Координаты для logo-row в листе 1
Блок «Логотип» каждой системы (XML row = Excel row − 1):
- СКС row=2 (Excel row 3)
- СОТ-камеры row=13 (Excel row 14)
- СОТ-ПО row=24 (Excel row 25)
- Часофикация row=35 (Excel row 36)
- СКУД row=46 (Excel row 47)
- ЛВС row=56 (Excel row 57)
- IP-телефония row=67 (Excel row 68)
- Палатная row=78 (Excel row 79)
- **Телевидение row=89** (Excel row 90) — новый блок
- **ИБП row=100** (Excel row 101) — новый
- **AV-коммутация row=111** (Excel row 112) — новый

Excel col: A=0, B=1, C=2, D=3, E=4, F=5, G=6, H=7. Производитель N = col N+1.

### Стили эталонов листа 2

- Child row (outlineLevel=1): из row LINDEX-Черепанова
- ht="35.1" customHeight="1"

### Стили эталонов листа 1
- Параметр-row → строится по эталонному блоку Палатная (rows 78-87):
  - Название row 78, Логотип row 79, Страна row 80, Сегмент row 81, Скидка row 82, ВЭД row 83, Другая инф. row 84, Референс row 85, Контакт вендор row 86, Контакт дистр. row 87
- При записи через мою функцию `BuildBlock` стили подбираются из эталонной row автоматически.

### Поиск лого
1. Clearbit Logo API (`https://logo.clearbit.com/<domain>`) — лучший источник, работает для известных брендов
2. fetch HTML → парсинг og:image / apple-touch-icon — fallback для нишевых
3. DuckDuckGo favicon (`https://icons.duckduckgo.com/ip3/<domain>.ico`) — последний fallback, **только если апскейл не нужен** (32x32 при увеличении мутный)
4. Bitrix-сайты: попробовать `/local/templates/<theme>/images/logo.{png,svg}` или `/img/logo.svg`
5. SVG → PNG: Edge headless `--screenshot` с временной HTML-обёрткой
6. ICO → PNG: `System.Drawing.Image.FromFile` (НЕ `Icon` — он не парсит «фейковые» ico). Fallback на PNG-magic detection (header 0x89 0x50 0x4E 0x47).
7. Из буклета: pdf_render_pages 300 DPI → ротация при необходимости → нарезка по координатам с автообрезкой белых полей (threshold 240).

## Цитаты пользователя (важные правила)

- «нужны контакты по определенному оборудованию можем быстро раскидать, но вторую тоже нужно сделать более читаемой, разбить поставщиков по разделам» → переделка листа 2 в блоки 10+1.
- «Пусто + список открытых вопросов» — правило для незаполненных полей.
- «Обновить + журналировать» — для дублирующихся вендоров.
- «Вставляй что есть визуально оценю» — для лого. После того как пользователь не одобрил мутные favicon (КИПОД/Макроскоп/RVI) — пересмотрел стратегию (Clearbit + офсайт + буклет SNK).
- «Сразу всё» — пользователь не любит спрашивать дробно, делать пакетно.
- «Не видишь общего оформления? Почему новые позиции просто закинуты без оформления» — критический фидбек по стилям. С тех пор всегда копирую style-indices эталона при вставке новых строк.
