## Методы / что сработало
- **Workflow (fan-out)** для batch: распознавание 30/11 фото и 5 видео (1 агент на единицу), картирование 392-стр PDF (16 агентов по ~25 стр). Структурированный вывод (schema) → синтез у главного. По явному запросу «на качество, токены не важны».
- **OpenCV** — извлечение кадров из видео (самый чёткий кадр в секунду по Laplacian), обход отсутствия ffmpeg.
- **structured-artifacts** — всё знание на диск (карта тома, методичка, память, реестры), handoff без потери и без повтора Workflow.
- **pdf-mcp** — постраничное чтение/рендер большого PDF.

## Где сломался / грабли
1. **Yandex.Disk + массовые rename online-only плейсхолдеров** (Mode `-a---l`): серия быстрых rename (усугублённая багом OrderedDictionary) утащила 29 из 30 фото Р-4 в облачную корзину. 3. **Workflow-агенты** сохраняли временные зум-png в рабочую папку (Yandex) — чистить после.

## Ключевое знание (устройство ИД)
Том = комплекты вокруг актов + 3 прошивки (Предъявит. ведомость / общий Реестр-опись / Общий журнал работ). Типы актов: АОСР (скрытые) / АОМР (оборудование) / АОУСИТО (главный акт тома). Цепочка: журнал→акт→(п.3 материалы или Реестр №X→паспорта)→(п.4 схемы)→ВСО/ПВ. Нумерация листов = двусторонняя печать (акт/ВСО/АОСР = 1 лист = 2 стр PDF). Состав диктует заказчик. Замечания ТН = разрывы требование↔факт. Подробно — memory `reference_id_assembly_method` + методичка разд. 18-26.
