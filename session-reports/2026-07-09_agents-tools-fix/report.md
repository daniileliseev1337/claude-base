# Отчёт сессии 2026-07-09 (вечер) — боль агентов: диагноз и фикс tools-надёжности

**Машина:** ноутбук (hub). **Тема:** фокус-сессия оси #1 «ревьюеры спотыкаются
„нет доступа к инструментам"» (реворк базы). Метод: systematic-debugging —
сначала verifiable-диагноз на реальных транскриптах, потом хирургический фикс.

## Что сделал

1. **Сбор улик.** Grep по session-reports + всем локальным транскриптам
   (`~/.claude/projects/**/*.jsonl`, включая subagents/workflows). Грабля по пути:
   whitelist-`.gitignore` базы (`*`) заставляет ripgrep молча пропускать
   негитованные папки — искать в projects/ только с `--no-ignore`.
2. **Диагноз (гипотеза «холодный office-MCP» НЕ подтвердилась как главный корень):**
   - **C. Tools-gap во frontmatter — главный корень.** «No such tool available»:
     `mcp__word__search_and_replace` ×6 (21.06, воркфлоу заполнения комплекта актов —
     у word-писателей не было replace/find в списке), `mcp__pdf-mcp__pdf_render_pages`
     ×5 (в т.ч. auditor 03.07 на реестрах тома ИД — pdf-ревьюеры не могли РЕНДЕРИТЬ
     сканы). У всех упавших СОСЕДНИЕ вызовы тех же серверов работали → сервер жив,
     инструмента нет в списке. Детерминированная сверка frontmatter всех 16 агентов
     с живым инвентарём (167 инструментов) подтвердила оба гэпа поимённо.
   - **B. Fallback-рефлекс не закреплён:** у 9/16 агентов в теле ни слова про
     Bash+python-обход. Сильные прогоны выкручивались сами (auditor 03.07 ушёл в
     fitz и довёл аудит до PASSED), слабые капитулировали («не смог проверить»).
   - **E. Ревьюер в вакууме:** без ТЗ/источников штампует ложные замечания
     (кейс свода дверей 30.06: excel-validator метил «дубли/пустые цены» как дефект
     при ТЗ «не объединять»).
   - Усилитель: правило auditor'а «не смог проверить = NOT PASSED» (правильное,
     анти-имитация) при недоданных инструментах серийно генерит ложные NOT PASSED —
     то самое «из 8 ошибок 6» владельца.
   - A (холодный MCP): ✗-старты в логах есть, но ни один отказ ревьюера к ним не
     трассируется — второстепенный фактор, автопрогрев НЕ делали (Karpathy #2).
3. **Фикс (11 файлов `agents/`, патчер с dry-run):**
   - +`pdf_render_pages` пяти pdf-читающим: auditor, audit-rd-section,
     rd-coordinator, снабженец, expertiza-responder;
   - +`find_text_in_document` + `search_and_replace` шести word-писателям:
     id-engineer, letter-writer, kp-writer, designer, pto-engineer, expertiza-responder;

     агентам без fallback-правила: упал/нет инструмента → Bash+python
     (openpyxl/python-docx/fitz), «не смог проверить» = граница проверки (НЕ дефект),
     вердикт о нечитаемости — только после провала ОБОИХ путей;
   - строка истории всем правленым; letter-writer: сняты два ложных утверждения
     тела («Нет Bash», «Frontmatter разрешает ровно Read/Write/Edit/Glob/Grep»).
4. **CLAUDE.md п.3 (по явному «ок» владельца):** спавн ревьюера обязан включать

   НЕ дефект артефакта.
5. **Верификация:** re-run чекера — все целевые гэпы закрыты, fallback-слова у 16/16;
   структурная проверка (один блок на файл, tools-строки целы); ревью auditor —
