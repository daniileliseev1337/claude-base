# Кандидаты, отброшенные фильтром harvest

Короткие пометки — не отдельные файлы, чтобы не плодить мусор.

## danii1/realty-parser

- 3 stars, last commit **2015-10-23** (≈11 лет назад), **archived**, ISC.
- Парсеры CIAN+Avito. Архив. Не годится.

## bmwx591/yrlparser, van-mronov/yrl-feed-parser

- Парсят **YRL (Yandex Realty Language)** — это XML-формат, в котором
  **сами агентства** выгружают свои объявления для индексации Яндексом.
  Это **не** парсинг страниц realty.yandex.ru с точки зрения покупателя.
- Подходит только если у пользователя есть доступ к YRL-выгрузке агентства
  (т.е. он сам агент и публикует объявления). У нас — покупатель/арендатор,
  не подходит.

## chuprik/avito-realty-notifier

- 12 stars, last commit **2016-01**, **нет лицензии**, Go.
- Slack-бот для Avito. Мёртв 10 лет, лицензия неясна → можно прочитать
  идеи, но **код не копировать**.

## ejay34/01_real_estate_market

- Учебный проект на данных Yandex Realty (анализ Saint Petersburg).
- Не парсер и не трекер — статистический анализ готового датасета.
  Полезен только как пример скоринга/фичей.

## flathunters/flathunter

- Активный, популярный (>1k stars), Python, MIT.
- Поддерживает **только** ImmoScout24, Immowelt, WG-Gesucht, Kleinanzeigen
  (Германия) + Idealista, Subito, Immobiliare.it (Италия).
- **НЕ поддерживает** Yandex Realty / CIAN / Avito.
- Полезен как **архитектурный референс** для своего проекта (как они
  строят полинг, дедупликацию, уведомления) — но интегрировать нельзя.

## CreatmanCEO/cian-parser-showcase

- Описание звучит идеально (price history + Telegram + фильтрация), но
- метаданные через GitHub API получить не удалось (rate limit 403).
- Слово «showcase» в имени → подозрение что это демо-репо для портфолио,
  не production-tool.
- **Требует ручной проверки** на ПК пользователя если интересен.
