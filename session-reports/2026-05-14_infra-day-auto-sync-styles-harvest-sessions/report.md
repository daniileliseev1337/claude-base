## Итерации, ошибки, что переделывал (честно, без приукрашивания)

### Мои промахи в коде (4 бага подряд)

1. **2>&1 ловушка** — в `b81a5d4` я добавил
   `$pullOutput = & git ... 2>&1 | Out-String` с regex-классификацией
   ошибок pull. Прошёл `-DryRun`, но **не прошёл `-Yes`** — потому
   что DryRun не триггерил эту ветку. На реальном прогоне упал на
   `NativeCommandError` от прогресс-сообщений git fetch. Урок:
   `-DryRun` ≠ полное покрытие путей кода. Полный `-Yes` на
   изолированной копии обязателен.
2. **`.Split('\','/')` ловушка** — в `4ed1426` (Uninstall.ps1) для
   cleanup empty dirs использовал `.Split('\','/')[0]`. PowerShell
   парсит это как `Split(char separator, int count)` — `/` пытался
   стать числом → `MethodArgumentConversionInvalidCastArgument`.
   Поправил через regex `-split '[\\/]'`. Опять — `-DryRun` не
   триггерил эту секцию (она после `if ($DryRun) { exit }`).
3. **Ошибочный диагноз 12-й ловушки** — увидел 403 на Deliseev,
   быстро сработавший `cmdkey /delete`, обобщил «PAT истёк» как
   универсальный ответ. Ivan-овский session-report показал что на
   его машине корень другой — wrong account, не PAT. Правильно
   было сразу записать оба сценария 12a и 12b с указанием как их
   различить.
4. **Нарушил принцип «диагностика по слоям»** — увидел
   `Proxy CONNECT aborted`, побежал писать «прокси режет git» как
   причину. Без снятия прокси-слоя реальная auth-проблема была
   невидима. Ivan честно прошёл все слои и нашёл корень — wrong
   account. Это **методический урок 13**, который у нас теперь в
   базе.
