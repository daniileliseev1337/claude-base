- Описание: Добавляет OCR-текстовый слой в скан/векторный PDF, делая его поисковым и копируемым.

## Зачем смотрели
**Прямое решение** проблемы «текст в кривых»: OCRmyPDF рендерит страницу в bitmap, прогоняет OCR, накладывает текстовый слой → дальше любой PDF-инструмент видит нормальный текст.

## Оценка
- Подходит для нашей проблемы? **Да — primary candidate** для нашего кейса с текстом в кривых.
- Сильные стороны:
  - **MPL-2.0** — либеральная лицензия.
  - Самый прямой workflow: input PDF → output PDF с OCR-слоем → дальше pdf-mcp работает как обычно.
  - Поддерживает 100+ языков через Tesseract.
  - Многопоточный CPU.
  - Сохраняет оригинальный layout (OCR-текст под изображением).
- Слабые стороны / риски:
  - **Зависит от Tesseract** — наш текущий блокер. Но: на Windows есть portable Tesseract (через pip-пакет `tesseract` или conda) без админ-прав. Также можно использовать pypi-обёртку с bundled binaries.
  - Альтернатива: ждать апстрим-поддержку других engine (есть тикеты, но не релизнуто).
  - Для русского нужен `rus` lang-pack (~10 МБ).
  - На A2 страницах ОЗУ потребление высокое.
- Решение: **используем как primary**. Если найдём portable Tesseract без админ-прав — это самое прямое решение. Иначе — пайплайн «pypdfium2 render → PaddleOCR → склейка обратно в PDF» (своя реализация).
