# pdfplumber
- URL: https://github.com/jsvine/pdfplumber
- Stars: 10271
- Last commit: 2026-01-28 (v0.11.9)
- License: MIT
- Описание: Pythonовский PDF-инструмент для детального доступа к chars/lines/rects/curves, извлечения текста и таблиц с визуальной отладкой.

## Зачем смотрели
Layout-aware extraction, table extraction, доступ к примитивам curves/lines (нужно для анализа PDF с текстом в кривых).

## Оценка
- Подходит для нашей проблемы? **Да** (для tables) / **Под условием** (для text-in-curves — даёт сырые `.curves`, но без распознавания).
- Сильные стороны:
  - **MIT-лицензия** — никаких ограничений.
  - `page.chars`, `page.lines`, `page.rects`, `page.curves` — прямой доступ ко всем графическим примитивам.
  - `page.crop(bbox)` — clip к региону.
  - Table extraction по сетке (lattice через `lines` strategy, или `text` для безграничных таблиц).
  - Visual debugging — рисует bbox-ы поверх PDF для отладки.
  - Активная разработка, 10к ★.
- Слабые стороны / риски:
  - Не работает со сканами — нужен OCR.
  - Текст в кривых = кривые → goes в `.curves`, но нужно своё распознавание формы.
  - Multi-column не явный — нужна постобработка по координатам.
- Решение: **используем** как основной инструмент для таблиц и layout-анализа на векторных PDF. Для текста в кривых — даёт сырые данные, дальше OCR на отрендеренном регионе.
