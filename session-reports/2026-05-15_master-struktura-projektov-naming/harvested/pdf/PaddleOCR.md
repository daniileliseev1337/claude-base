# PaddleOCR
- URL: https://github.com/PaddlePaddle/PaddleOCR
- Stars: 77894
- Last commit: 2026-05-14 (v3.5.0)
- License: Apache-2.0
- Описание: Мощный OCR-toolkit (100+ языков, включая русский), превращающий PDF/изображения в структурированные данные для LLM.

## Зачем смотрели
Альтернатива Tesseract для русского+английского, мощное распознавание шрифтов после рендера PDF с текстом в кривых. Один из топовых OCR (78к ★).

## Оценка
- Подходит для нашей проблемы? **Да** — топ-1 кандидат для распознавания отрендеренного текста-в-кривых.
- Сильные стороны:
  - **Apache 2.0**.
  - PP-OCRv5 — state-of-the-art точность на 2026.
  - Поддерживает русский (Cyrillic) + English одновременно.
  - CPU-режим работает (медленнее GPU, но без админ-прав).
  - pip install → Windows-совместим.
  - PP-Structure: layout-analysis + tables + формулы + графики.
  - 78к ★ — крупнейшее OCR-комьюнити.
- Слабые стороны / риски:
  - Тяжёлый install (paddlepaddle ≈ 500 МБ); первый запуск качает модели (~100-300 МБ).
  - Нет специальных моделей для инженерных чертежей — стандартный OCR.
  - Для текста в **сильно** искажённых кривых (выделение мышью обвода) может быть менее надёжен — нужен тест.
- Решение: **используем** как primary OCR для нашего пайплайна «render → OCR» на векторных PDF с текстом в кривых.
