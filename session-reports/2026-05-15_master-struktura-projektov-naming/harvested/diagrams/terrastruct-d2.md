## Применимость к нашей задаче

- **Плюсы:**
  - Современная альтернатива Graphviz/PlantUML с понятным синтаксисом.
  - Layouts: **dagre + ELK + TALA** — ELK даёт **лучшие иерархические layouts** для крупных диаграмм (наши 200 узлов!).
  - Container syntax: вложенные группы — `cluster.floor1.cabinet01` — идеально для «объект → этаж → шкаф».
  - Icons: можно подгружать SVG-иконки (Cisco, etc.) через `icon:` свойство.
  - SVG/PNG/PDF.
  - Themes из коробки.
- **Минусы:**
  - TALA (премиум-layout) — платный/проприетарный движок (флаг!). Бесплатные dagre/ELK — норм, но TALA лучше.
  - Сообщество меньше PlantUML.
- **Сценарий:**
  - Из xlsx генерим `.d2` текст с группировками `building → floor → cabinet`.
  - `d2 --layout=elk schema.d2 schema.svg` → красивая иерархическая схема.
  - Для штампа ГОСТ — постпроцессинг (вставка SVG-рамки).

## Вердикт

**Сильный кандидат №2** после `mingrammer/diagrams`. Возможно даже №1 для крупных схем — ELK-layout на 200 узлов даст лучший автоматический результат.
