**Файл-эталон:** `C:\Users\Deliseev\Desktop\Здадчака\Мастер структура_проектов.xlsx`

**Состав 7 листов:**

1. `01_Мастер_Структура` — реестр шкафов с 13 колонками (Проект, Объект, Блок, Система, Сегмент_сети, Уровень_сети, Тип_шкафа, Человекочитаемое_имя, System_ID, Помещение, Функция, Статус, Комментарий)
2. `02_Правила_Naming` — паттерн `[Объект]-[Сегмент]-[Тип][Номер]` с примерами
3. `03_Классификатор_Шкафов` — ШГК / ШР / ШРД / ШД
4. `04_Классификатор_Оборудования` — КЯ / КР / КД / МЭ / ОК / ПП / ИБП
5. `05_Алгоритм_П_Р` — 9-этапный процесс от ЧТЗ до отчёта изменений
6. `06_BIM_Параметры` — 8 параметров для Revit
7. `07_Статусы_Изменений` — SAME / NEW / REMOVED / MODEL_CHANGED / QTY_CHANGED / ARCH_CHANGED / MOVED

**Когда обращаться:** при старте любого инженерного проекта (СКС, СС,
ЭО, ОВ, ВК) — этот файл задаёт схему именования и BIM-привязки.
Методика разобрана и зафиксирована в [[master-structure-naming]].

**Ограничения файла:**
- Данные на листе 01 — 4 строки демо-примеров (1 строка с расхождением:
  `ГК-Л2-ШГК01` без блока, не соответствует правилу «всегда с блоком»).
- Правило naming в файле упрощено (`[Объект]-[Сегмент]-[Тип][Номер]`),
  актуальное правило по решению пользователя 2026-05-15 расширено до
  `[Объект]-[Блок]-[Сегмент]-[Тип][Этаж][№]`.
- Файл — стартовый эталон, а не полная спецификация. Уточнения собраны
  в [[master-structure-naming]] (раздел «Открытые вопросы»).
