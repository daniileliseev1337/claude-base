Дата: 2026-06-26. Метаданные — через api.github.com (stargazers_count / pushed_at / license.spdx_id), не из README.

## Контекст (зачем смотрели)

Виджет «как Claude понял задачу + план» после грилинга. Рендер в ЛОКАЛЬНОМ HTML-файле
(Claude генерит → пользователь открывает в браузере/VS Code). Данные эмитит LLM. Два уровня:
- **Сверка** (лёгкий) — декларативный рендер из текста/JSON, без написания механики.
- **Карта сессии** (тяжёлый) — интерактивный холст, узлы/рёбра/zoom/подсветка.

**РЕШАЮЩИЙ критерий:** автономный HTML по CDN без сборки/npm. MIT/пермиссив. Живой проект. Контроль визуала узла.

## Главный фильтр: «React или нет»

Критерий «автономный HTML без сборки» ВЫБИВАЕТ всю React-экосистему (React Flow / Excalidraw /
tldraw): они требуют React в рантайме; UMD-по-CDN возможен, но это уже не «один чистый файл».
→ Моя прежняя похвала React Flow («золотая середина») отменяется ФАКТОМ критерия автономности.
React Flow вернётся в гонку ТОЛЬКО если откажемся от «без сборки» (согласимся на React-CDN/bundler).

tldraw вдобавок: лицензия NOASSERTION (своя tldraw-license с watermark, не OSI) → не зелёный свет.

## Оценка под 2 уровня

### Лёгкий «Сверка» → mermaid (победитель, бесспорно)
- 88k stars, MIT, чистый JS, один `<script>` с CDN. Live (push сегодня).
- Claude нативно отлично генерит mermaid-синтаксис (flowchart/mindmap/graph).
- `classDef` → цветовая подсветка узлов = наша «уверенность» (допущение vs точно). ✅
- `click`-директива → базовая кликабельность узла. ✅
- Минус: статичные диаграммы, ограниченный визуальный лоск и живой интерактив (нет drag-узлов).
- Вердикт: **берём для лёгкого уровня.** Покрывает 80% случаев «просто свериться».

### Тяжёлый «Карта сессии» → развилка (решение пользователя)
- **Fabric.js** (31k, MIT, чистый JS/CDN) — фаворит владельца (выбирал в Revit-сессии).
  Макс. контроль визуала и интерактива. НО граф-механику (рёбра за узлом, layout, сворачивание,
  hit-test) пишем САМИ = «наш слой» = реальная работа, но и есть «свой инструмент под нас».
- **Cytoscape.js** (11k, MIT, чистый JS/CDN) — граф из коробки: layout-движки, zoom, события,
  стилизация через селекторы. Зрелый (open issues 14). НО навязывает рамки (стиль рёбер/layout),
  узлы стилизуются, но это не «свои HTML-узлы» → меньше визуальной свободы (чего владелец избегал).
- **vis-network** (3.6k, Apache-2.0) — проще Cytoscape, красивый физический layout из коробки.
  Минус: меньше community, больше открытых issues (344).
- **React Flow** — лучший по «свои узлы-компоненты + механика даром», НО ценой React-зависимости
  (см. главный фильтр). Опция, если откажемся от автономности.

## Итоговая рекомендация harvest

- **Лёгкий уровень: mermaid** — однозначно.
- **Тяжёлый уровень: Fabric.js** (под приоритет владельца «визуал + свой инструмент»), приняв цену
  «пишем граф-слой сами». Альтернатива при желании «граф из коробки ценой рамок» — Cytoscape.js.
- **Прагматика для ПЕРВОГО прототипа:** начать целиком на mermaid (даже «карту» можно
  прототипировать mermaid-графом) → доказать мост дёшево → Fabric-уровень строить, когда mermaid
  упрётся в потолок интерактива. Карпатски №2 (простота) + №4 (узкий первый шаг).
