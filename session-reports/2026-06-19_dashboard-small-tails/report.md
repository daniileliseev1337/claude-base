### M2-путь2 / C-путь2 — self-take в команду
- Разведка живой БД: RPC `take_project` УЖЕ добавлял взявшего в `project_members(editor)`,
  но НЕ заполнял `executors` jsonb. Клиент писал только текстовый `executor` →
  при следующем сохранении формы владельцем `projectJsToDb` пересобирал `executor`
  из ПУСТОГО `executors` → имя исполнителя слетало. (RLS `projects_update` пускает
  взявшего как `is_project_editor`, поэтому клиентский `update(executor)` проходил —
  но `executors` оставался пуст.)
- `take_project` существовал ТОЛЬКО в живой БД (вне репо-миграций — невоспроизводимость).
- Фикс: миграция `20260619_0001` — полное определение `take_project` зафиксировано в
  репо + расширено: атомарно заполняет `executors`+`executor`, `INSERT project_members(editor)`,
  `SET search_path = public, pg_temp` (заодно I-2 hardening для этой функции).
  Имя берётся `COALESCE(NULLIF(name,''), email)` из `profiles` (= клиентский fallback).
- Клиент (App.jsx, кнопка «Взять в работу»): убран дублирующий `update({executor})`
  (RPC сам всё ставит), `executors` добавлен в optimistic-апдейт.
- Верификация: транзакция `BEGIN…ROLLBACK` на живой БД (эмуляция self-take под 2-м
  юзером через `request.jwt.claims`) — `executors`/`executor`/`project_members(editor)`
  заполняются верно, повторное взятие → `ERROR Project is not available` (защита цела).
  Боевые данные не тронуты.

## Грабли / уроки
- **`replace_all` промах (важно):** замена `gridTemplateColumns: "1fr 1fr"` по gap:12
  затронула не только ProjectForm, но и `ClientForm` (строка вне scope), где `isMobile`
  ещё не был определён → был бы runtime `ReferenceError`. Поймал Grep'ом всех заменённых
  вхождений, откатил лишнюю. Урок: `replace_all` — только когда old_string гарантированно
  уникален в нужном scope; иначе точечные Edit с уникальным якорем (имя поля/Label рядом).
- Деплой БД: миграции копировал F:→`C:\temp` (кириллица «Сайт» в bash-аргументе через
  WSL теряется), подавал `docker exec -i supabase-db psql < /mnt/c/temp/*.sql`.
- `deploy-web.sh` сам использует абсолютный glob `/mnt/f/*/redesign-v2-fresh/dist` —
  не cwd-чувствителен, запуск `bash -c "bash /mnt/f/*/.../deploy-web.sh"`.
- git на F: с `core.fsyncMethod=writeout-only` + ретраи (в этот раз add/commit/push
  прошли с 1-й попытки — диск вёл себя стабильно).
- Контроль числовой: распределение проектов по стадиям читал `GROUP BY stage` —
  дало факты для решения №11 (0 проектов в кандидатах → безопасное удаление без миграции).

## Баг прав editor (доли/платежи) — исправлен, на проде

Замечание владельца из эксплуатации: editor чужого проекта при сохранении формы получал
`not_project_owner`, часть данных не сохранялась. **Root cause:** доли/платежи owner-only в RPC
(`set_project_shares`/`set_project_payments`) И в RLS (shares/payments select+write), а `projects`
UPDATE пускает editor. `set_project_shares` зовётся при каждом сохранении (replace-all) → editor
всегда видел ошибку, успев сохранить поля проекта (updateProject прошёл раньше throw). **Фикс:**
миграция `20260619_0003` — `owner → owner OR is_project_editor OR is_admin` в 2 RPC + 4 RLS.
select тоже (иначе editor грузит пустые и replace-all стирает). viewer не затронут. Деплой
`a0ba4a5` (только БД), верифицировано транзакцией + post-deploy probe. **Урок: owner-only гейт на
дочерней сущности рассинхронен с editor-доступом родителя → проверять все дочерние RPC/RLS;
replace-all + узкий select = тихая потеря данных.**

## Фиксы «Моя доля» на чужом проекте (продолжение)

Симптом владельца: на чужом проекте (он editor/admin, не владелец) назначил доли — в форме
«Я · остаток» писалось, а на карточке «Моя доля» нет. Разведка на БД (имена не выношу):
владелец проекта — другой человек, текущий юзер admin+editor, и долю он назначил... владельцу
(а не себе). **Root cause (2):** (1) автокомплит участников долей звал `searchApprovedUsers`,
исключающий self → editor не мог добавить СЕБЯ с привязкой к userId (только внешним именем) →
`viewerShareOnProject` (ищет долю по `participantUserId===viewerId`) не находил; (2) строка
«Я · остаток» в форме показывалась всем, включая не-владельца, хотя остаток принадлежит ВЛАДЕЛЬЦУ.
**Фикс** (`fbb4ec2`, бандл `index-qOBlVIEZ.js`): (1) при `!isOwner` юзер добавляется в результаты
автокомплита долей; (2) «Я · остаток» только при `isOwner`. Модель долей: владелец=остаток (в доли
явно не добавлять), участники=явные доли по userId. **Урок: при отладке UI-индикатора сверять с
реальными данными БД — гипотеза «кэш не обновился» оказалась «данные введены не в ту сущность
из-за UX-ограничения (self исключён)».**
