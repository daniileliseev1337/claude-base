## Метод (3 слоя, все детерминированные)
1. **Экстрактор СО из PDF** — `verify_vor.py: extract_co()`. `find_tables(strategy="lines")` строит сетку по векторным линиям таблицы → каждое слово в правильную ячейку (решает проблему «сырой текст путает колонки», из-за которой прошлая сессия ушла в рендер). Учтены ловушки: страницы СО **повёрнуты на 90°** (rotation=90, координаты в неповёрнутом пространстве — find_tables справляется сам); заголовки колонок с **переносами** («Коли-\nчество», «Единица измере-\nния») — дегифенизация; разные шапки («Поз.» vs «Позиция»); **название в данных уезжает на 1 колонку** относительно шапки → название = склейка ячеек от позиции до тип/код; **безномерные строки** в PDF (напр. Pigtail с пустым номером) — берём по наличию кол-ва.
2. **Сверщик СО(PDF)↔json** — гибрид: матч по (СО,позиция) + добор по нормализованному названию (нормализация гомоглифов лат/кир: `4x3`↔`4х3`, `M8`↔`М8`, `c`↔`с`). Снимает ложные тревоги от разных схем нумерации и опечаток.
3. **Генератор xlsx==json** — `check_xlsx.py`. Подтвердил: генератор переносит данные 1:1 (0 расхождений по всем 12).

**Ключевая дисциплина:** детерминированный прогон сужает 546 строк до горстки расхождений → каждое подтверждается визуально по конкретной странице PDF. Где экстрактор и json совпали — оба верны; где разошлись — смотрим источник. Так пойман и системный недосмотр, и не наплодили ложных срабатываний.

## Ловушки-уроки
- **Опечатка в самом PDF:** в СО3 узле 3 патч-корд напечатан с номером «2.11» вместо «3.11» (стр.75) — экстрактор честно прочитал кривой ярлык, дал ложный DUP/EXTRA. json разнёс верно. Вывод: номер позиции в PDF ненадёжен (опечатки, пустые, мульти-СО) — матчить гибридно (позиция+название), не только по номеру.
- **Глаз ошибается, text-layer — нет:** при визуальной сверке я один раз прочитал «16»/«25» там, где text-layer (find_tables) и оба ревьюера дают «10»/«20». Для ВЕКТОРНОГО PDF извлечение из text-layer авторитетнее визуального чтения рендера (рендер — лишь растеризация того же слоя; ошибается человек). Рендер нужен только для адъюдикации единичных спорных ячеек.
- **Виды работ keyword-правилом не проверяются детерминированно** — слова «винт/кабель/труба» внутри описаний оборудования дают массу ложных флагов. Нужен семантический ревьюер, если потребуется.
- **5.12 нюанс (НЕ трогали):** наш ВОР = «Система **вызывной** палатной сигнализации» (совпадает с папкой СВПС), штамп исходного PDF = «Система палатной сигнализации» (без «вызывной»). Источник сам себе противоречит — оставлено на решение пользователя.
