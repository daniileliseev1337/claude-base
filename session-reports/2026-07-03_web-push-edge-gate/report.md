# Сессия 2026-07-03 (глубокая ночь) — edge-гейт функции уведомлений (self-hosted Supabase)

## Что сделал
- Закрыл интернет-открытую edge-функцию web-push (Deno, verify_jwt off) dual-гейтом:
  валидный user-JWT (валидация через GoTrue `GET /auth/v1/user` — отсекает anon/service-токены,
  у них нет пользователя) ИЛИ заголовок `X-Push-Secret` из config.json функции (не в git).
- Секрет для pg_cron — в supabase vault (`vault.create_secret` / чтение `vault.decrypted_secrets`
  прямо в команде cron-джоба): секрет НЕ попадает ни в миграцию, ни в `cron.job.command`, ни в репо.
- Удалил мёртвую edge-функцию (телеграм-канал уведомлений, вызовов не было) с хоста и из репо.
- Верификация скриптом: 401 (no-auth, anon-JWT) / 200 (секрет, реальный user-JWT через temp-юзера
  с password grant + trap-cleanup) / cron-джоб содержит заголовок. Прогнано трижды, включая после рестартов.
- Ревью диффа субагентом (sonnet) до merge: PASSED, 2 MINOR + 2 INFO — применены
  (gitignore-страховка на config.json, trap для temp-юзера, устаревший комментарий в старой миграции).

## Источники / паттерны, которые пригодились
- Паттерн верификации соседней функции проекта (hand-signed JWT + password grant + cleanup) —
  скопирован почти дословно, сэкономил час.
- Порядок внедрения БЕЗ окна поломки: (1) секрет в vault + config → (2) cron с заголовком
  (старая функция игнорирует лишний заголовок) → (3) деплой гейченной функции.

## Где сломался / грабли
- Git Bash (Bash-тул) манглит абсолютные WSL-пути: `wsl ls /srv/...` превращается в
  `C:/Program Files/Git/srv/...` (MSYS path conversion). Обход: host-команды wsl гонять через
  PowerShell-тул (аргументы передаёт verbatim); docker-команды без путей не страдают.

## Уроки
- «Секрет из настройки при применении» на Postgres — это vault: cron-команда читает секрет
  при КАЖДОМ запуске, значит ротация = обновить vault-строку + config.json, миграции не трогать.
- Пустой vault (coalesce → '') безопасен со стороны функции: пустая строка falsy → путь секрета
  не срабатывает, отдаёт 401, а не пускает всех.
- Тип-«пустышка» (`__gate_check` → ветка unknown type) — удобный zero-side-effect зонд для
  проверки авторизации живой функции.
