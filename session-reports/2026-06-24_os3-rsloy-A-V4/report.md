## Что сделано (DWG НЕ изменялся — только read-only + смена активной вкладки; DBMOD=0)

### Пункт A (п.10 ТН: длины / переходы Ø ►◄ / арматура) — метод пересмотрен на ГИБРИД
- **Пилот выявил:** слепая MCP-простановка на этом чертеже ненадёжна. Чертёж — Revit-экспорт высокой плотности (~150 слоёв, 19287 объектов, планы этажей продублированы в Model по Y, viewport с поворотом/нестандартной DCS). Координатная привязка «лист↔Model» сбоит: трубы воды не находятся по вычисленной зоне листа (в «зоне» оказалась только канализация). Даже локализовать нужную трубу для одного размера — проблема.
- **Решение (согласовано с пользователем):** гибрид — аналитика моя (ВОР/РД/геометрия), физическая простановка исполнителем курсором в живом AutoCAD, верификация моя.
- **Разведка (факт DWG):** размеры воды уже стоят — `!Размеры <сист>` 229 + привязки 129 (канализация на отдельных слоях); арматура — 63 блока «задв-фронт» на слое кранов; трубы воды = LINE на системных слоях (не полилинии).
- **Выдано задание-разметка** (md): длины (229 уже стоят → проверка полноты + ориентир Σ метража по Ду из ВОР), переходы Ø (цепочки диаметров + 5 типов из ВОР), арматура (ведомость ВОР 101 ед. = 89 кранов + 5 воздухоотв + 4 задвижки + 1 обр.клапан + 2 компенс. vs факт схемы 63 знака → дефицит, где ставить). Все числа — из прямого дампа ВОР.

### Пункт V4 (ведомость материалов / расход воды / охват этажей)
- **Ведомость материалов** — ✅ УЖЕ на исп.схеме трубопровода (нижняя полоса, OLE — полная ВОР мт). Ответ ТН-разъяснение.
- **Охват этажей** — ✅ подтверждён (магистрали на нижней отметке + стояки/приборы на верхних; планы всех этажей в комплекте). Ответ ТН-разъяснение.
- **Расход воды** — числа извлечены КООРДИНАТНО из РД (pdfplumber + визуальная сверка кропа таблицы). Ключевая находка (по наводящему вопросу пользователя): таблица РД — для ВСЕГО КОМПЛЕКСА; для конкретного строения релевантна отдельная строка (не общая). «при пож.» большое число проверено визуально — корректно (комплекс + спринклер. пожаротушение), не артефакт извлечения. Нанесение таблицы — отложено (выбор строк не решён, пользователь отвлёкся).

## Где сломался / уроки
1. **Viewport-локализация на сложном Revit-DWG ненадёжна** (twist/DCS + многоэтажные дубли в Model). Вычисление «зоны листа» из ViewCenter/ViewHeight дало неверную область. Урок: на таких чертежах слепая координатная простановка через MCP — плохой инструмент; гибрид (аналитика → человек ставит) честнее. Вставка готовой ТАБЛИЦЫ в полосу листа (как W-слой) — остаётся выполнимой.
2. **Кодировка кириллицы:** имена слоёв/листов/ячеек надёжно читать через cp1251-файл → PowerShell ReadAllText(1251) или python-вывод в UTF-8 файл; git bash и python-stdout в консоль ломают кириллицу. Эвристика выбора листа по подстроке поймала канализацию вместо воды — выбирать точно.
3. **Числа в чертёж — только после визуальной сверки** (правило подтвердилось): pdfplumber на объединённых ячейках может сместить значения; кроп+глаза разрешают. Плюс семантика: проектная таблица «весь комплекс» ≠ показатели одного строения.
4. **Диспетчер autocad-mcp** (mcp_dispatch.lsp) нужно подгружать вручную в новом сеансе AutoCAD (сервер ✓Connected, но backend init падает «LT detected, dispatch not loaded»). Команда `(load ".../lisp-code/mcp_dispatch.lsp")` в командной строке AutoCAD.

## Развязка «расхода» (важно)
Пометка V4, пересказанная в трекере как «**Расход воды**», по первоисточнику (видео, развёрнутое на 180° через cv2 и прочитанное с чёткого кадра) оказалась «**Расход ПЕНЫ**» (заделка узлов прохода) — это **фантом пересказа**. Проектная таблица расхода воды из РД (которую я успел извлечь) — к делу не относится, отброшена. - **Чтение рукописных пометок с видео (cv2, без ffmpeg/whisper):** (1) развернуть кадры `ROTATE_180`; (2) детектор синей ручки — HSV-маска, выбор кадра по ПЛОЩАДИ синего, НЕ по резкости (Лапласиан ловит печатные таблицы/цифры, а не размытую рукопись); (3) камера в движении смазывает крупный план — брать кадры «остановки» (конец панорамы); (4) cv2.VideoWriter('mp4v') разворачивает видео целиком (звук теряется).
