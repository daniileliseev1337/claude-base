## Зачем смотрели

Усилить самописный виджет-карту (standalone HTML, генерируется build_widget.py, vanilla Canvas+JS)
под интерактивную расстановку точечных приборов на плане этажа Revit-проекта: drag&drop точек,
зум/пан, привязки, измерения, экспорт изменённых координат обратно в JSON.

## Оценка

- **Подходит? ДА — лучший под сценарий.**
- Сильные стороны:
  - Это ДВИЖОК, не фреймворк-приложение → встраивается в standalone HTML одним `<script>`
    (можно инлайнить в генерируемый Python файл — single-file сохраняется).
  - Из коробки снимает большую часть самописного кода build_widget.py:
    drag&drop (`draggable:true`), **слои** (Layer — у нас они уже есть), зум/пан
    (stage scale + drag), Transformer (перемещение/поворот/ресайз с ручками),
    события клика/наведения с bubbling, hit-detection.
  - **Сериализация `stage.toJSON()` / `Konva.Node.create(json)`** = готовый механизм
    экспорта изменённых координат, который потом применяет Python → Revit.
  - 10+ лет один мейнтейнер (Anton Lavrenov), консистентный API, ревью каждого PR.
  - Автор прямо позиционирует под «design editors / annotation tools / seat maps» — наш класс.
- Слабые стороны / риски:
  - «Привязки к стене/двери» и «измерения» придётся дописывать самим (любая либа не знает
    геометрию Revit) — но Konva даёт snap-хуки в `dragmove`.
  - Перевод виджета с голого Canvas на Konva — это переписывание рендера (объём работы).
- **Решение: РЕКОМЕНДУЮ как основу виджета.**
