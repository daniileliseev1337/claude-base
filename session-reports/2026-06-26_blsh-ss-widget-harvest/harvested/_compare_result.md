# Konva vs Fabric — сравнение на реальном виджете-карте

Задача: усилить интерактивный canvas-виджет (standalone HTML, генерится Python) для
расстановки точечных приборов на плане этажа из Revit-модели. Прототипы собраны на ОДНИХ
данных: ~3387 объектов геометрии (стены/окна/двери/мебель/сантех/колонны) + 227 приборов.

## Вес (факт, чистый Python, без браузера)

| | min.js | gzip | standalone HTML |
|---|---|---|---|
| Konva 9 | 171.2 KB | 50.0 KB | 300.5 KB |
| Fabric 5.3 | 304.9 KB | 85.1 KB | 434.5 KB |
| Fabric тяжелее | ×1.78 | ×1.70 | ×1.45 |

## Производительность (playwright Chromium 1280×800, СИНХРОННЫЙ замер: 30 полных перерисовок)

| метрика | Konva | Fabric |
|---|---|---|
| init | 40.5 ms | 45.9–53 ms |
| full-scene redraw 3614 об. | **168 FPS** (5.96 ms) | **160 FPS** (6.27 ms) |
| только 227 приборов | 1442 FPS | 2000 FPS |
| heap | **15 MB** | 27 MB |

Производительность **практически равна** — оба >>60 FPS, в реальной архитектуре (статика-фон +
227 интерактивных приборов) оба мгновенны. FPS не различитель.

## Методологическая ловушка (важно на будущее)

Первый замер дал «Fabric 61 / Konva 2–4 FPS» — это БЫЛ АРТЕФАКТ. Konva `batchDraw()`
**асинхронный** (планирует отрисовку на свой rAF), а rAF-счётчик считал свои кадры → мерялись
разные вещи. Корректно: синхронный цикл `for(N){ move; draw()/renderAll() }`, FPS=N·1000/Σ.
После фикса Konva и Fabric сравнялись (168 vs 160).

## Удобство под наш кейс (из кода прототипов)

| | Konva | Fabric |
|---|---|---|
| Пан | `draggable:true` на stage (0 строк) | руками mouse:down/move/up + relativePan (~5 строк) |
| Слои (центральная фича виджета) | нативные `Layer`, `.visible(false)` | слоёв нет — прятать перебором objects |
| Drag приборов | из коробки | из коробки |
| Экспорт координат | `dragend`→x()/y(); + `stage.toJSON()` | `object:modified`→left/top |
| Интеграция в single-file HTML | отрисовался сразу | DOM-обёртка `.canvas-container` конфликтует с layout (потребовала бы возни) |

## Вывод

Производительность равна. **Konva выигрывает по весу (×1.45–1.78), памяти (×1.8), старту и
удобству под наш сценарий** (нативные слои + пан из коробки + чистая интеграция в single-file).
Козырь Fabric (SVG import/export) нам не нужен — план рендерит Python/Revit.
