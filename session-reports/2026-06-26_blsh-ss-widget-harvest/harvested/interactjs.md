- **Описание:** Drag&drop, resize, multi-touch жесты с инерцией и **snapping** для современных браузеров.

## Зачем смотрели

Если делать виджет на SVG/DOM (а не Canvas) — interact.js навешивает drag&drop и привязки
поверх любых DOM/SVG-элементов, не требуя переписывать рендер на canvas-движок.

## Оценка

- **Подходит? ПОД УСЛОВИЕМ (если виджет останется на SVG/DOM).**
- Сильные стороны:
  - **Snapping** (snap to grid / targets / anchors) из коробки — прямо «привязки к стене/двери».
  - Vanilla, zero-deps, ~зрелый и стабильный, MIT.
  - Не требует менять архитектуру рендера — навешивается на готовые DOM-узлы приборов.
- Слабые стороны / риски:
  - 2+ года без коммитов — развития нет (хотя API стабилен, браузеры не ломали).
  - Только взаимодействие (drag/snap/resize) — рендер/зум/слои/экспорт делаешь сам.
- **Решение: держим в уме.** Хорош, если виджет переведём на SVG-узлы вместо Canvas;
  тогда interact.js = только слой взаимодействий, остальное — свой код. Для Canvas-пути
  предпочтительнее Konva (всё в одном).
