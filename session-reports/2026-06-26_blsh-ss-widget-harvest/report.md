## Что сделано

### 1. Harvest JS-движков (заметки — harvested/)
Кандидаты (метрики верифицированы через api.github.com, не из README):

### 2. Сравнение Konva vs Fabric на реальных данных (~3387 объектов геометрии + 227 приборов)
- **Вес:** Konva min 171/gzip 50/HTML 300 KB; Fabric min 305/gzip 85/HTML 434 KB (Fabric ×1.45–1.78 тяжелее).
- **Производительность РАВНА** (синхронный замер): full-redraw Konva 168 / Fabric 160 FPS; heap 15 vs 27 MB.
  ⚠ Первый замер «Fabric в 15× быстрее» был АРТЕФАКТОМ (Konva batchDraw асинхронный) — переделал на
  синхронный цикл draw/renderAll. Урок — в [[reference_js_canvas_engine_choice]].
- **Выбор пользователя — Fabric** (после ручной проверки прототипов): важнее богатый интерактив-редактор
  из коробки (controls с поворотом, выделение группой), чем вес/память. Решение пользователя > замеров.

### 3. Изучены офиц. доки Fabric v5 (после замечания пользователя «не прочитал доки»)
Грабли вынесены в [[reference_fabric_gotchas]]: fireMiddleClick (default false до v7 — без него пан средней
кнопкой мёртв), офиц. паттерн пана (vpt[4]/[5]+setViewportTransform), zoomToPoint+renderAll,
absolutePointer для координат, баг textBaseline='alphabetical' (~200 warnings → монкей-патч),
два canvas (фон на контейнер, не на canvas).

### 4. Production-виджет — widget_fabric_prod.html
Генератор: `C:\Temp\blsh_sps\build_widget_fabric.py` (читает blk1_f6.json + снимки, инлайнит fabric.min.js).
Фичи (все проверены в браузере через playwright + loopback-сервер):
- слои плана + **легенда приборов по типам** (контраст-цвета, счётчики, вкл/выкл по типу);
- **схематичные иконки по типу** (форма по назначению: круг-лампа / квадрат-вызывное / прямоугольник-терминал)
  + стрелка-нос направления; цвета разведены контрастно;
- зум(колесо)/пан(средняя кнопка)/поворот ручкой/выделение группой;
- **undo** (Ctrl+Z / кнопка) — откат перемещений и поворотов, чистит экспорт;
- **привязки по типу** — лампы(LUM/LUS)/RCU→центр проёма двери, вызывные(PCU/BCU/WCCU)→сантехника;
- координаты курсора (модельные мм), **двойной экспорт** (JSON для Revit + список для контроля), warnings убраны.
- Типы приборов в данных: BCU 22, LUM 28, LUS 4, NCP 4, PAT 114, PCU 20, RCU 31, WCCU 4 = 227 (6 этаж целиком).

## Метод работы (важно для следующей фазы)
Виджет = **снимок** живой модели + общий язык координации. Цикл: оператор двигает/крутит приборы в
браузере → копирует JSON → присылает Claude → Claude применяет в Revit через MCP execute_revit_code
(по id найти прибор, переместить Location.Point, повернуть по dir/углу). После изменений модели —
пересобрать devs2.json + rebuild виджета (как старый build_widget.py).
