## Инструмент (в `ГЧ_прототип/`, кандидат в `skills/id-scheme-toolkit/tools/`)
Двухслойная архитектура (Karpathy «код в tools/»):
1. **`xlsx2acadtable.py`** — режет лист по строкам «Ведомость объёмов работ» на таблицы,
   классифицирует строки (title/header/section/data), опц. нормализует нумерацию (№ + 1..N),
   пишет `C:/temp/table_data.txt` (cp1251, TAB-разделители). Параметры argv: layout, ins x/y,
   cols csv, style, thmax, margin, target_h, renumber.
2. **`gen_table.lsp`** — статичный ASCII-ридер: парсит data-файл, строит ACAD_TABLE
   (vla-AddTable в paperspace листа), SetText/MergeCells/SetCellTextStyle/SetCellTextHeight/
   SetRowHeight/SetCellAlignment, **бинпоиск textheight под target_h** (c7:build).
   Кириллица приходит из cp1251-файла, не из кода → кавычки/спецсимволы безопасны.

## Анатомия целевого xlsx (лист «М об»)
Содержит ДВЕ таблицы ВОР монтажа оборудования: r1–r96 (этаж −1, пом. −1-1-XX, ~88 поз → лист
«Об -1») и r97–r114 (этаж 0, пом. 0-1-XX, 15 поз → лист «Об 1»). Колонки: A №(узк), B
Наименование(шир), C Ед.изм, D Всего. Секции-подзаголовки = merged A:D. Аномалия источника:
в шапке колонка № = «1», нумерация позиций сдвинута (с 2) — по решению пользователя нормализуем.

## Грабли / уроки (КЛЮЧЕВЫЕ — в память)
1. **`vla-get-PaperSpace` ненадёжен** для AddTable/перебора → Automation error. Брать paperspace
   через перебор `(vlax-for lo (vla-get-Layouts doc))` → `(vla-get-Block lo)`. (То же с vla-Item.)
2. **`vl-catch-all-apply` требует ВТОРОЙ аргумент-список**: `(vl-catch-all-apply 'fn (list a b))`,
   НЕ `(vl-catch-all-apply '(lambda()...))`. Иначе ошибка эскалирует мимо catch.
3. **Высота строки ACAD_TABLE не сжимается под мелкий текст сама** — начальная rowHeight из
   AddTable фиксируется. Явно `vla-SetRowHeight` (минимум = th*1.4+2*margin); перенос растит выше.
4. **Adaptive auto-fit = бинпоиск** th, не линейная коррекция (высота нелинейна из-за переносов
   → линейная проскакивает вниз, недоиспользует зону).
5. **Кавычки/дюймы в данных** ломают LISP-литералы → данные через cp1251-ФАЙЛ, ридер на ASCII.
6. **`(open file)` в LISP при падении виснет дескриптором** → следующий open того же файла = nil
   → "bad list". Собирать результат в payload, не писать файл LISP'ом в цикле отладки.
7. **`vla-HandleToObject` на невалидный handle = Automation error ВНЕ catch** (если в списке
   аргументов другого catch). Оборачивать сам HandleToObject (см. c7:safe-del).
8. **Копировать .lsp в ASCII-temp ПОСЛЕ КАЖДОЙ правки** — load кэша нет, но забытая копия =
   отладка старой версии (потерял ~3 итерации на "numberp nil"/"no function").
9. execute_lisp контекст (defun/setq) сохраняется между вызовами в рамках сессии file_ipc.
10. PNGOUT всего листа → мелкий текст рендерится чёрными блоками; для оценки текста — ZOOM Window
    на bbox таблицы, для оценки вмещения — ZOOM Extents.

## Второй бой — «Об -1» (multi-column) — PASSED
Та же сессия. Доработан multi-режим: `c7:build-multi` + `c7:build-list` (рефактор build-once на
произвольное подмножество строк). Алгоритм: измерить высоты строк пробной таблицей →
`c7:layout-cols` (жадная раскладка по колонкам target=высота зоны, повтор title+header,
перенос висячей секции) → бинпоиск шрифта чтобы колонок ≤ MAXCOLS → построить K таблиц рядом.
Результат: 88 позиций → **3 колонки, шрифт 2.47 мм** (зона 517×315 на А1, maxcols=4 — запас).
Верификация: все **88/88** количеств совпали с xlsx (`C:/temp/verify_obm1.py`). 5 OLE заменены,
DWG сохранён. build-multi отработал быстро (таймаута НЕ было, опасения не подтвердились).
Косметика: подколонку «Всего» расширить (9→12), иначе «Всег/о» переносится в шапке.
