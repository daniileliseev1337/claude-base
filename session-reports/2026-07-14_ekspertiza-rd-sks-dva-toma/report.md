# Сессия 2026-07-14 — экспертиза двух томов РД СКС-ЛВС-ТФ (конвейером)

## Что сделал
- Экспертиза двух томов РД марки СКС-ЛВС-ТФ (<объект>, 43 и 37 стр.) по отработанному конвейеру:
  `extract (PyMuPDF) → fan-out ×3 субагентов (auditor / audit-rd-section / general-purpose-артикулы,
  все claude-sonnet-5) → собственный визуальный ГОСТ-аудит рендеров → findings_*.py → make_shots →
  build docx → ревью word-checker + auditor → правки → точечная переверификация через SendMessage`.
- По каждому тому — Word-ведомость 38 замечаний (3 критических), сквозная нумерация, «сухой» стиль
  по калибровке ГИП. Оба ревью закрыты: auditor PASSED (после правок), word-checker PASS/WARN.
- norm-lookup пропущен обоснованно: перечень ссылочных норм в томах серии идентичен ранее проверенным
  (тот же шаблон) — выводы перенесены, статусы подтверждены в сводной таблице.

## Источники
- PDF томов + постраничная выгрузка text/tables (PyMuPDF) в `Claude/tools/obNN/`.
- База вердиктов по артикулам из прошлых томов (`ob05_artikuly.md`) — переносы без повторного веба.
- Локальная библиотека СПДС-норм на Я.Диске (цитаты пунктов).

## Где сломался / уроки
1. **make_shots + rot90:** добавил «улучшение» — свап размеров display для повёрнутых страниц. Ошибка:
   `page.rect` у PyMuPDF УЖЕ повёрнут. Итог — часть скринов кропалась из неверных зон (в т.ч. в уже
   собранном docx). Поймал на второй ведомости по ошибке crop; перегенерил и глазами проверил оба тома.
   Урок: не «улучшать» рабочий эталонный скрипт без теста на повёрнутой странице.
2. **Ревьюер-auditor окупился дважды:** на обоих томах нашёл по 2 фактических неточности в МОИХ ведомостях
   (неполный охват ссылки на чужой том; смещение страницы цитаты; счётная ошибка «24» vs 22+П.1; неверный
   диапазон листов, взятый из отчёта субагента без пересчёта). Урок: числа и диапазоны из отчётов
   субагентов пересчитывать перед вставкой в findings, особенно «N листов».
3. **Текстовый слой врёт не только в штампах:** «ООО "К-<N>"» без цифры — артефакт извлечения (глиф рядом
   с векторным логотипом), рендер показывает полное имя. Отсутствие символа устройства на плане тоже
   проверять рендером зоны по координатам осей (нашёлся у оси, grep не видел).
4. **Точечная переверификация через SendMessage** (вместо полного повторного аудита) — быстро (1–2 мин)
   и дёшево; ревьюер сам перечитывает пересобранный docx.

## Наблюдения по серии (для следующих томов)
- Сквозные дефекты шаблона кочуют, но НЕ все: в одном томе нет задвоения наименования, но есть дубль листа
  и отсутствующие листы; в другом — задвоение есть, комплектность в порядке. Каждый том проверять полным
  конвейером, не полагаясь на «в прошлый раз было».
- Новые классы: заглушка «Смените формат…» может стоять на нескольких листах; метаданные PDF хранят имя
  исходного файла другого тома (прямое доказательство клонирования); привязка уличного оборудования
  к опорам на генплане может быть зеркально перепутана против схемы и КЖ.
