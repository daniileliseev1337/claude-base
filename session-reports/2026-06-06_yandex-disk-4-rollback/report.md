# Сессия 2026-06-06 — Откат Яндекс.Диск 4.0 → 3.x, восстановление синхронизации папок ПК

## Запрос
Пользователь запутался в настройке синхронизации Яндекс.Диска: «где главный источник, кто с чем синхронизируется». По ходу выяснилось — сломалась синхронизация рабочих папок.

## Что выяснили (диагностика)
- На ПК сосуществовали ДВА клиента Яндекс.Диска: старый 3.x (`YandexDisk2.exe`) и новый 4.0 (`YandexDisk4.exe`), оба стартовали в один день.
- Рабочая схема пользователя: рабочие папки физически на `F:`, синхронизировались «на месте» через функцию 3.x «Синхронизация папок компьютера» → облачный раздел «Компьютеры». Папка Диска была пустой — это норма для функции.
- Клиент 4.0 эту функцию НЕ поддерживает: только качает копию облака в свою папку на `C:`. По логам (`Disk3SettingsWatcher`) 4.0 прочитал список внешних папок старого клиента, но в синхронизацию их не взял; за сессию — тысячи операций скачивания и 0 заливок с диска.
- В день сессии автообновление удалило старый клиент 3.x (`disk3 installed = false`) → синхронизация `F:` оборвалась.
- Junction-обход на 4.0 НЕ работает — клиент игнорирует reparse points (`empty symLink … will be ignored`), проверено отдельным безопасным тестом.
- В GUI 4.0 штатной функции «папки компьютера» нет (подтверждено скриншотом настроек и поддержкой).

## Решение
Откат на клиент 3.0 по официальной инструкции поддержки Яндекс 360 (поддержка подтвердила: общий доступ сохранится):
1. Пользователь удалил 4.0 через Панель управления.
2. Удалили папку-дубль на `C:` и пустой хвост второй папки Диска (после остановки клиента — безопасно для облака).
3. Пользователь установил 3.0; клиент подхватил сохранённый конфиг внешних папок и восстановил синхронизацию `F:` (подтверждено логами `LOCAL`/`SHADOW`/`DIGEST`).
4. Отключили автообновление: в `settings.xml` клиента `AutoUpdate=false` + файл сделан read-only (защита от перезаписи). Клиент перезапущен, синхронизация активна.

## Источники
- Справка Яндекс 360 (клиент 4.0): synchronization/status, synchronization/folder, sync-folders.
- Ответ поддержки Яндекс 360 (инструкция отката на 3.0).

## Где ошибался (уроки)
1. **Поспешный вывод.** Сначала заявил «синхронизация встала месяц назад» по датам папок — оказалось, что Яндекс при синхронизации перебивает время модификации файлов. **Урок:** копии сравнивать по РАЗМЕРУ/контенту, а не по дате; не делать выводов до проверки механизма.
2. **Неверная оценка объёма.** Назвал дубль «169 ГБ» по логическому размеру файлов — физически это были облачные заглушки (files-on-demand), места почти не занимали. **Урок:** для cloud-папок логический размер ≠ занятое на диске.
3. Пользователь дважды поправлял («стоп», «всё работает») — оба раза был прав. **Урок:** при противоречии данных и слов пользователя — перепроверять факты глубже (логи клиента), а не настаивать.
