# Harvest анализ 7 GitHub-репо для Claude Code базы

## vex-tui
**Факты:** 906 звёзд | MIT | Апр 2026

Go-приложение — терминальный редактор Excel/CSV с vim-биндингом, фильтрацией строк, профилированием, поддержкой формул и экспортом графиков. Активно разрабатывается, готово к боевой работе.

**Пересечение:** Excel-просмотр в тармиале конкурирует с нашим excel-MCP (программное чтение) + встроенный Excel для пользователей. Для CLI-сценариев инженеров может быть полезен, но требует Go (у нас Python-стек). Собирает файлы, не интегрируется с агентами.

**Вердикт:** ПОПРОБОВАТЬ. Если team CLI-инженеров часто режет данные в тармиале — пригодится как подспорье. Не критичное; низкий приоритет.

## Matrix
**Факты:** 2 звезды | нет лицензии | Июн 2026 (свежее)

Python CLI для cross-device session jumping — переноса окружения, файлов, clipboard между машинами по WiFi/Bluetooth/WebSocket/облаку с Signal-spec шифрованием.

**Пересечение:** Наш hub-and-spoke git-синк базы + handoff-скилл уже решают мобильность сессий через git + cloud. Matrix — криптированный канал для живого переноса состояния (env, clipboard) между машинами; мы синхронизируем артефакты (доки, агенты). Параллельные задачи, не взаимно исключающие.

**Вердикт:** НЕ НАДО. Наша архитектура — async git-шаринг (масштабируемо для команды). Matrix — point-to-point криптосессии (для личного use-case). Если кто-то просит live-clipboard-sync между ПК — направить на встроенные облачные синхи (OneDrive, Yandex.Disk), а не внешний инструмент.

## rightmind
**Факты:** 5 звёзд | нет лицензии | Июн 2026 (сегодня)

Python-приложение: мультиагентные дебаты решений. Задача идёт на panel агентов (Claude, GPT, Gemini, DeepSeek), они аргументируют, стресс-тестируют, синтезируют результат. Поддерживает web-поиск у каждого агента.

**Пересечение:** Наш Workflow (judge-panel паттерн) + agenthub-плагин уже реализуют мультиагентную оркестрацию с ревью. rightmind — перепакованная оракул-архитектура с кросс-моделевым (Claude+GPT+Gemini) дебатом; у нас агенты work внутри 1-2 моделей (Fable/Haiku). Для нас — обогащение, но не архитектурный сдвиг.

**Вердикт:** ПОПРОБОВАТЬ. Если команде нужна видимая дискуссия мнений (для stake-holder'ов /审批委 процесс), rightmind пример хорошего UX. Скопировать pattern в наш Workflow, не тащить зависимость на GPT/Gemini (у нас только Claude).

## opra.ai
**Факты:** 1 звезда | Apache 2.0 | Июн 2026

GitHub-native governance слой для бизнес-workflow. Записывает решения как readable files, валидирует локально, пропускает мутации через RBAC + approval policy, эмитит audit evidence. Демо CLI, API, браузер-UI, GitHub PR, Skills.

**Пересечение:** Наша feedback-архитектура (roles developer/consumer, pull-feedback, feedback-канал) + PR-базированное управление версиями — почти то же. opra.ai — более формальный RBAC (Approval policy engine). Можно тянуть паттерны governance (кто может коммитить что, audit log).

**Вердикт:** ПОПРОБОВАТЬ. Проконсультироваться с hub-администратором: нужна ли нам явная approval policy (e.g. 2-of-3 ревью перед merge в agents/)? Если да — посмотреть opra.ai как синий слой. Если сейчас trust-based — отложить.

## peer-review
**Факты:** 4 звезды | нет лицензии | Июн 2026

CLI-приложение: panel AI-агентов peer-review'ит черновик (проектный доклад, записку, план, историю — что угодно), спецьализированные ревьюеры аргументируют, редактор резолвит разногласия, выдаёт улучшенный текст + полную историю дебатов.

**Пересечение:** Наша архитектура ревьюеров 2-го уровня (auditor, excel-validator, word-checker, pdf-reviewer) уже делает это для специфических типов артефактов (pdf/excel/word). peer-review — обобщённый текстовый review-agent с синтезом мнений.

**Вердикт:** ВЗЯТЬ. Заимствовать pattern "panel consensus + editor synthesizer" для наших ревьюеров: если 3 ревьюера расходятся (один PASSED, двое BLOCKER) — спавнить editor-агента (Fable) для резолюции вместо fail. Это upgrade к текущему (fail fast). Видимые ревью-дебаты добавляют доверие.

## GitMo
**Факты:** 1 звезда | GPLv3 | Июн 2026

Linux desktop-приложение: автоматический GitHub-синк для локальных папок. Автосейв-коммиты, синхронизация personal workflows без терминала. v0.1.2, не production-ready.

**Пересечение:** Наши auto-pull/auto-push hooks (settings.json) уже делают это программно (не UI). GitMo — UI обёртка для Git для non-CLI users. Для нас не подходит (мы CLI-first, Linux-only, ранняя версия).

**Вердикт:** НЕ НАДО. Мы автосинк решили через hooks + PowerShell-скрипты (Windows-friendly). GitMo — альтернативный stack. Если в команде появятся non-dev пользователи (невалидный use-case для нас) — вспомнить.

## MetaPurge
**Факты:** 3 звезды | MIT | Янв 2026 (застарелое)

Windows 10/11 утилита: удаление метаданных из изображений (EXIF, device fingerprint, timestamps) и PDF (title, author, subject, keywords, coordinates).

**Пересечение:** Наш pikepdf (программный API для PDF-манипуляции) + встроенная Python-обработка изображений умеют то же самое. MetaPurge — UI обёртка (скачал, кликнул, готово). Для скриптовых batch-операций нашей системе Python-API достаточно.

**Вердикт:** НЕ НАДО. Если user явно попросит "удалить EXIF из пакета фото перед заливкой в отчёт" — дам Python-сценарий (pikepdf + PIL). MetaPurge — для конечных пользователей, не для инженерной системы.
