## Уроки сессии (How to apply)
1. **Прод-деплой — только по ЯВНОЙ команде владельца.** «Устраивает» относится к проверке, не к выкатке;
   auto-классификатор правильно блокирует деплой без явного «деплой» (зафиксировано в памяти проекта).
2. **«Не обновилось» при успешном деплое = клиентский SW-кэш PWA**, не сервер/туннель. Сначала холодный
   перезапуск, проверять серверные ассеты (диск/nginx) — они почти всегда уже новые. Не дёргать туннель/редеплой.
3. **Временный фикс, который скоро переделывать — не делать** (#8 отложен в #1/#2). Karpathy: не плодить выбрасываемое.
4. **subagent-driven**: связные задачи в одном файле группировать в юниты (token economy), two-stage review на юнит;
   для механических удалений — один субагент с точными локациями + build + мой просмотр диффа (без полного ревью).
