# Session report — ПНР+ВОР: ВПВ/АУВПТ и ЭОМ (2026-06-02)

Продолжение серии «ПНР+ВОР» для инженерного объекта (<объект>, шифр <шифр>-ИОСx.x).
За сессию закрыто 2 раздела (9-й и 10-й в серии). Pipeline тот же:
разведка → norm-lookup (ГЭСНп) → AskUserQuestion → xlsx (копия шаблона) → docx → валидация parallel.

## Сделано

### 1. Раздел ВПВ + АУВПТ (противопожарный водопровод + автоматическое водяное пожаротушение)
- ВОР 49 позиций, ПНР **123 функции**. Обе валидации PASSED.
- Методика пожаротушения: функции = испытания (гидростатика/водоотдача/промывка) + наладка
  полевых устройств (пожарный кран 4, СПЖ 4, задвижка с контролем положения 3, затвор 1,
  трубопроводная сеть 2-3, комплексное опробование 1). Оросители/воздухоотводчики — укрупнённо.
- Норма: **ГЭСНп 81-05-02-2020 Сб.2** «АСУ» п.1.2.4 (нормирует по системам/каналам, не пономерно;
  исключает пожарную сигнализацию → ГЭСНм Сб.10). Насосная и приборы АУП — в базовой системе
  здания, вне раздела.
- Без ЗИП (решение заказчика). Память: создан `reference_pnr_firefighting_method.md`.

### 2. Раздел ЭОМ (силовое электрооборудование + электроосвещение) — самый крупный
- ВОР **103 позиции** (укрупнено из ~380 спеки: агрегация однотипных по ключевым словам +
  сумма по 3 этажам; светотехника с 80 моделей сведена в 10 функциональных категорий).
- ПНР **1507 функций** (после пересчёта — см. «Где сломался»).
- Структура ПНР (детальный метод): щиты общие 4 ф/щит ×26 = 104; **аппараты защиты 1×459 = 459**;
  **кабельные линии 2×459 = 918** (изоляция + «фаза-нуль»); освещение (рабочее/аварийное/DALI
  по этажам) 21; заземление 5.
- Норма: **ГЭСНп 81-05-01-2022 Сб.1** «Электротехнические устройства» (аппарат — на штуку,
  измерения — на линию). DALI → ГЭСНп 81-05-02-2020 Сб.2 АСУ. ВРУ в раздел не входит.
- Память: создан и затем **исправлен** `reference_pnr_eom_method.md`.

## Где сломался (главный урок сессии)

ЭОМ сначала посчитал ПНР **укрупнённо «на щит»** (26 щитов × ~7 = **196**). Пользователь
сделал push back: 196 несопоставимо мало с другими разделами серии (у слаботочки ПНР 1000–3300
при десятках позиций ВОР — потому что считалось ПОШТУЧНО устройство×функции).

**Корень ошибки:** для ЭОМ единица ПНР — это **линия и аппарат защиты** (их сотни), а не «щит».
По ГЭСНп 81-05-01 измерители именно поэлементные. Распарсил кабельный журнал (21 лист PDF,
стр. 72–92) → **459 линий** (ЩР 314, ЩЭ 58, ЩАО 45, ЩО 16, питающие от ВРУ 26). Пересчёт дал
**1507** — сопоставимо с серией. Аппараты приняты = числу линий (допущение, помечено в docx;
уточняется по схемам щитов).

Урок зафиксирован в памяти жирным предупреждением. Применять ко всем будущим ЭОМ-разделам.

## Замечание по валидатору (на будущее)

Повторная excel-валидация ЭОМ дала ложный NOT PASSED: валидатор интерпретировал мою формулировку
«стало 1507 поэлементно» как требование 1507 физических строк, тогда как формат ПНР во всех
разделах агрегированный (строка=группа, столбцы «функций/ед × кол-во = итого»; 1507 — сумма).
Урок: формулировать задание валидатору точнее — «1507 это контрольная сумма функций, формат
агрегированный N строк-групп». (Перекликается с feedback-pnr-zip-methodology про точность
prompt'а валидатору.)
