## Что сделано

### Волна багфиксов #2 (4 из 11 замечаний), в проде
Подход: systematic-debugging (root cause до фикса).
- **№3** «Финансы не подтягивают деньги за проекты» — оказался НЕ багом: вкладка Финансы = ручной кошелёк
  (таблица transactions без project_id), проектные деньги (paid_amount) by design отдельны. Развилка трактовок →
  спросил владельца → выбран read-only блок-сводка «По проектам» (переиспользует расчёты долей).
- **№6** уведомления неинформативны + нет перехода — root cause: edge формировал жёсткий body и `url:"/"`,
  ИГНОРИРУЯ уже присылаемые фронтом projectName/actorName/customText/projectId. Фикс: edge подставляет имя проекта
  в body + ставит url на проект/задачи; фронт шлёт projectId; навигация открывает карточку.
- **№5** self-notif при само-назначении — `initiatorId` не передавался, edge не отсекал инициатора. Фикс:
  `initiatorId` во все sendPush.
- **№8** вкладки уезжают под шапку при скролле — магический `top:64` ≠ переменной высоте шапки. Фикс: единый
  sticky-контейнер `top:0` для шапки+вкладок.

### История платежей + сводная панель №2 (XL-фича), в проде

- Финансы: «Доходы» = ручные + моя доля проектных платежей по дате (на лету, только мои проекты).
- Панель №2: режим выбора проектов + сводка (получено/остаток/договор/разбивка).
- Чистые функции + vitest (61). БД верифицирована исполнением (RLS_OK, MISMATCH=0).

## Уроки (в скилл/память)
- При жалобе «X не работает» сначала проверять, БАГ ли это или дизайн — root cause может оказаться «так задумано»
  + развилка трактовок (тогда спрашивать владельца, не чинить вслепую). Сэкономило неверную реализацию №3.
- Когда фронт уже шлёт данные, а они «не видны» — искать сторону-потребителя (edge игнорировал поля) прежде чем
  менять отправителя. Это был корень №6.
- Денормализованное поле (paid_amount) при вводе истории-источника: единственный писатель = триггер; убрать
  поле из всех клиентских записей в БД (projectJsToDb), иначе сохранение затирает триггерное значение.
- subagent-driven в среде с граблями (git Windows-сторона, кириллица, нет SendMessage): детерминированный/
  средо-зависимый код и git/деплой — на контроллере; UI-компоненты — субагентам; ревью diff контроллером.
