## Где сломалось / грабли (новые, в копилку)
- **fsync-сбои диска (EUNKNOWN) у Edit/Write-тулов** на проектном диске: 5 отказов подряд, зависший
  npm build. Обход: контент писать на системный диск, переносить python-скриптом (io.open без os.fsync).
  Git: `-c core.fsyncMethod=writeout-only` как и раньше.
- **PowerShell → wsl bash -c инлайн ломается** на `2>`, `$?`, вложенных кавычках (даже в одинарных).
  Надёжно ТОЛЬКО файлом: `Get-Content -Raw x.sh | wsl bash -c 'tr -d "\r" > /tmp/x; bash /tmp/x'`.
- **git push зависит от сетевого режима**: при активном VPN (TUN) push работает с ДЕФОЛТНЫМ окружением,
  а привычный «обход прокси» наоборот даёт connect timeout. Правило «всегда обходи прокси» неверно —
  пробовать оба варианта.
- **/dev/tcp «порт открыт» — ложный позитив при TUN-VPN** (SYN/ACK отдаёт сам VPN-клиент). Проверять
  доступность curl'ом с оценкой время/код, не голым TCP-коннектом.
- **pg_dump из docker → /tmp → cp на виндовый диск** — рабочий путь для БД→репо (drvfs-cp стабилен там,
  где Windows-fsync падает).
- **Explore-субагент может зависнуть на npm build** (тот же fsync): output 0 байт 20+ мин → TaskStop
  и прогнать детерминированную часть самому.
