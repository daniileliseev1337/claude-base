## Ключевые знания/исправления (важно не потерять)
1. **graphify НЕ требует Ollama/GPU** — штатный LLM-бэкенд = Claude Code субагенты (или Gemini опц.). Старый вывод «docs-граф упёрся в 16 ГБ» был тупиком Ollama-эксперимента. Построение графа доков = облако; query готового — оффлайн, 0 токенов.
2. **Запечатанная модель ИД** (грилинг пользователя, 54+6 пунктов): ПВ — master-список; ВСО/ВОР — проекции; **инвариант ПВ≡ВСО≡ВОР≡ИС**; методичка `Музей\Обучение составления ИД.md` §26 **ошибочна** про шаринг ИС между актами (ИС 1:1 с актом, у АОУСИТО свои «общие схемы») — граф содержит исправленную модель; подсчёт позиций — по PDF, метод НЕ финализирован (DWG-COM не канон).
3. **Метод «практика строит базу»** (решение пользователя): не строить агентов спекулятивно — гонять реальные замечания как «золотые пути» по runbook'ам, харвестить переиспользуемое в базу после каждого. Уже работает: золотой путь №1 (каскад состава п.6 Молот: ВСО→ВОР→ПВ + учёт листов) — ЗАКРЫТ в отдельном чате; **docs-каскад** (документы качества) — В ПРОЦЕССЕ в отдельном чате; первый урок оттуда (playwright-антибот) уже поднят в memory базы.
4. **Hand-authoring графов в graphify**: пишем курированный extraction-JSON (nodes/edges/hyperedges) → `build_from_json`+`cluster`+`to_json` → graph.json; RU-relations сохраняются; query (подстрока+IDF) — основной интерфейс, `path`/`explain` требуют точную метку (спецсимволы типа «≡» в метках ломают матч — избегать).
