## Ловушка дня — субагент word-checker без MCP

**Дефект архитектуры**, на который пользователь явно попросил обратить внимание.

При спавне `Agent(subagent_type="word-checker", ...)` субагент получает базовый toolset (Read, Bash, Grep, Glob) **без** mcp__word__* инструментов. Read на .docx (бинарный ZIP) возвращает мусор, Grep тоже не помогает (deflate-сжатие). Субагент честно ответил «NOT PASSED — не смог проверить» и предложил 3 варианта решения.

Я обошёл это, прочитав документ через `mcp__word__get_document_text` в основной сессии и проверив 8 пунктов чек-листа вручную. Но это **компромисс**: word-checker как ревьюер для docx-формата сейчас не работает по назначению.

**Что нужно по-хорошему:**
- Либо передавать MCP в субагент через настройку доступных tools у агента (в `~/.claude/agents/word-checker.md` явно прописать `mcp__word__*`).
- Либо main-агент перед вызовом word-checker конвертирует docx → md/txt через markitdown и передаёт текстовый дамп — тогда субагенту хватит обычного Read.
- Либо отказаться от субагента для docx-ревью и проверять самим — но тогда теряем независимое мнение.

Аналогичная проблема может быть у `excel-validator` и `pdf-reviewer`. Стоит проверить и эти агенты.

## Итерации и что сломалось

1. `Word.Application` через PowerShell New-Object — упал на QueryInterface (TYPE_E_ELEMENTNOTFOUND). Перешёл на pywin32 — тоже упал на gen_py-кеше (`AttributeError: ... has no attribute 'CLSIDToClassMap'`). Помогло: очистка gen_py + переход на `Dispatch` (late-binding) вместо `EnsureDispatch`.
2. Пропуск пункта 3.1.1 — типичная ошибка генерации по памяти без TodoList. Цена — отдельная правка через insert.
3. Таблица графика попала в самый конец после подписи (особенность последовательного add_table в python-docx). Решил через переоформление как «Приложение № 1» с обратной ссылкой из п. 8.3.
