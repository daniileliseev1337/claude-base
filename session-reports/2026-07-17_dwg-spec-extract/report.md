# Отчёт сессии — выгрузка спецификаций оборудования из DWG в Excel+Word

Дата: 2026-07-17. Объект: «<объект>» (продолжение сессии по спецификациям).

## Задача
По разделам проекта выгрузить спецификацию оборудования из DWG в Excel+Word там, где выгруженной Excel-спеки ещё нет. Извлечение — через ЖИВОЙ AutoCAD (autocad-mcp, backend file_ipc), не из PDF. Формат — по образцу ранее сданного раздела (Word с рамкой+штампом ГОСТ 21.101 + Excel, таблица ГОСТ 21.110).

## Сделано
- Скриптом пересобран список кандидатов по всем 24 разделам (критерий: есть DWG-таблица спеки И нет xlsx-спеки; АРХИВ исключён).
- **2 раздела закрыты** (СПС и СОТ): спеки извлечены из DWG через AutoCAD (AcDbTable), оформлены Excel+Word по образцу, разложены в папки DWG разделов.
  - СПС: 47 позиций (оборудование + кабельная продукция + электромонтаж), 5 листов Word.
  - СОТ: 27 позиций (камеры/PoE/АРМ + кабель + электромонтаж), 3 листа Word.
- Исключены с обоснованием: АР (облицовка — 0 таблиц), ПОС/ПОД (стройгенплан — 0 таблиц), ЛВС (DWG отсутствует — «из DWG» невозможно, только PDF).

## Метод извлечения (детали — в памяти проекта [[acad-table-extract-method]])
DWG → локальная ASCII-копия (`C:\temp`) → `vla-Open` без активации документа → `SelectionSet.Select` по неактивному документу с фильтром `ACAD_TABLE` (C-уровень, без итерации модели) → определение спеки по 9 столбцам → `vla-GetText` ячеек в cp1251-файл → Python-парсинг (декод `\U+`, `%%`, снятие MTEXT-кодов, NFC).

## Где споткнулся (ловушки — важно для базы)
1. **Открытие DWG с Я.Диска/кириллического пути** через `drawing open` возвращает `ok`, но не открывает. Решение — локальная ASCII-копия.
2. **Зависание AutoCAD**: активный документ не переключался (`vla-put-ActiveDocument` молча не работает в этом AutoCAD) + proxy-объекты АПС/СС вешают модальным окном (лечится `PROXYNOTICE=0` до открытия). Работать с неактивным документом через его SelectionSet.
3. **Устаревший hwnd в MCP** после рестарта AutoCAD → все `execute_lisp` таймаутят при живом `health`. Решение — `system init` (переподхват).
4. **ODA File Converter** (запасной путь DWG→DXF+ezdxf) НЕ ставится через winget — 403 на opendesign.com/guestfiles (нужна ручная регистрация). Запасной конвертер — сам живой AutoCAD (`vla-SaveAs ... 61`).
5. **Баг очистки MTEXT**: экранированный `\\` (литеральный обратный слэш в тексте ячейки) очиститель принял за коды `\W`/`\L` и вырезал вместе с буквами — исказил одну ячейку. Поймал auditor (BLOCKER), устранён защитой `\\`/`\{`/`\}` плейсхолдерами ДО снятия кодов. Урок: MTEXT-эскейпы литералов защищать первыми.

## Ревью (архитектурный гейт)
Каждый артефакт — через ревьюеров (claude-sonnet-5): excel-validator + word-checker + auditor.
- auditor поймал BLOCKER (искажение текста ячейки) — сверка по визуальному рендеру исходного чертежа + сырому дампу. Устранено, повторный auditor — PASSED.
- word-checker поймал узкую колонку «Кол.» (5-значные числа переносились) — расширена за счёт пустой колонки-донора. Проверено рендером.
- Все финалы прошли PASSED (данные сверены с источником: 0 расхождений по позициям/количествам).

## Правило заказчика (в памяти [[client-yandex-folders-clean]])
Папки на Я.Диске — заказчика, там чисто. Вся обработка (копии, DXF, дампы, скрипты, рендеры) — локально; в папку раздела — только финальный проверенный xlsx+docx.
