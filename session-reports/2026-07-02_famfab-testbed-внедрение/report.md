## Что сделал
По своду разбора двух AI-инструментов (обезличенный свод + техразбор кода FamFab,
12 находок) исполнены 4 решения куратора:

1. **(а) Скилл `revit-family-generator` принят AS-IS** (MIT, из репо FamFab/ArchSmarter,
   v1.0) в `skills/` — байт-в-байт копия (подтверждено sha256 аудитором) + рядом
   `README-provenance.md`: источник/лицензия/5 главных ловушек (кириллица в формулах,
   мёртвая constraints, молчаливый 0, нет shared params/MEP/materials, plain-text ключ)
   + план форков A/B/C. Решение куратора: RU/мм-форк — при первом использовании;
   живой прогон end-to-end — в отдельном локальном Revit-чате (не в этой сессии).
2. **(б) Роль «Revit-testbed»**: новый скилл `revit-testbed` (протокол приёмки
   5 пунктов: копия/изоляция → до/после → TransactionStatus, не верить return →
   отчёт в memory → гарантированный откат; + чек-лист end-to-end) + режим
   «приёмка стороннего инструмента» в существующем агенте `pyrevit-engineer`
   (frontmatter-триггер, When to invoke, секция режима, Related). Новый агент
   НЕ создавался — роль легла в существующую архитектуру.
3. **(в) Anti-patterns**: новая Категория 11 «Проектирование AI-конвейеров» —
   A11.1 «AI напрямую водит целевой API без инспектируемого артефакта»,
   A11.2 «Молчаливый 0 / дефолт вместо ошибки — тихая порча данных».
4. **(г) Методичка** `memory/reference_hybrid_ai_pipeline.md` — паттерн гибридного
   конвейера («код держит порядок, модель держит насмотренность»): каноническая
   схема + чек-лист 8 приёмов (6 из таблицы свода + 2 из его же разделов:
   инспектируемый артефакт, fail fast) + зрелые промпт-практики FamFab.

## Уроки
- Приём стороннего скилла «as-is + README-provenance рядом» — рабочая форма: оригинал
  не тронут (апстрим-совместимость, MIT), а наши ловушки/планы форков живут рядом
  и не потеряются.
- Роль в базе не обязана быть агентом: скилл-протокол + режим в существующем агенте
  дешевле и не плодит сущности (архитектура main → domain agent → auditor сохранена).
