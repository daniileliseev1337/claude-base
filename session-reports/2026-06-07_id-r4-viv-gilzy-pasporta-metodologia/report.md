## Ключевые уроки (вынесены в feedback-pending, запушить)
1. **Факт первичен.** Разбор замечания ТН: сначала смотреть собранный PDF тома + исп.схему (актуально ли замечание), потом править. Количества — по чертежу, не по xlsx (ВОР устаревает).
2. **Как строится ИС:** схема под акт; позиции акта — в ЦВЕТЕ, контекст (трубы/оборуд.) — серым. Гильзы красные, выноски к ним чёрные. Подсчёт по схеме = считать цветное с чёрными выносками. Это надёжнее xlsx и DWG-парсинга.
3. **AutoCAD — через COM** `win32com.Dispatch("AutoCAD.Application")`, не ezdxf (ezdxf не читает DWG; правит лишь render-cache таблиц). pywin32 ставится pip. Кириллица — в файл UTF-8 (консоль cp1251 рушит вывод). Revit-экспортный DWG для подсчёта позиций не парсить (каша на слоях).
4. **Паспорта/СС — проактивно скачивать** из интернета (open-source, производители публикуют). Нет инструмента — найти/поставить, а не отговариваться.
5. **Расхождение артикула** — сверять: проект + аналог. разделы строений + каталог производителя; не менять вслепую.

## Грабли инструментов
- Windows + mingw/консоль cp1251: кириллица/символ ∅ в argv `python -c` и в stdout рушатся → писать .py файлом (UTF-8) и вывод в файл UTF-8.
- MCP excel/word read на больших листах/документах превышает лимит токенов → читать прицельно или дампить компактно скриптом в файл.
- ezdxf-бэкенд autocad-mcp не открывает DWG («not a DXF file») → нужен живой AutoCAD через COM.
