# Session report — 2026-06-03 — правка наложения видеонаблюдения на AutoCAD-PDF

## Что сделал
Переработал «халтурное» наложение схемы видеонаблюдения на СПОЗУ-PDF `<объект>` (печать из AutoCAD):
- толстые красные лучи → тонкие синие;
- красные кружки → блок «видеокамера» (корпус+объектив);
- красные квадраты → блок «купольная камера»;
- камеры развёрнуты по направлению обзора (биссектриса лучей);
- сделано 2 варианта: (1) лучи+значки камер; (2) **секторы (конусы) обзора** + значки, без длинных лучей, 5 дальних камер без зоны.
Выход: 2× (DWG 2018 + PDF) на рабочем столе пользователя. Векторное качество подложки сохранено.

## Источники / инструменты
- `autocad-mcp` (живой AutoCAD 2024 через file_ipc, execute_lisp/AutoLISP+VLA).
- `pdf-mcp` (render для верификации), PyMuPDF (идентификация наложения, расчёт геометрии/углов/секторов).

## Где сломался / что заняло время
- Долго искал способ выделить наложение в AutoCAD после PDFIMPORT — оказалось, импорт тесселлирует толстые штрихи и теряет цвет (тупик). Решение — гибрид: идентификация в PyMuPDF, построение заново в AutoCAD по трансформу.
- Калибровка page→model: ложные промахи при проверке «по красному» (камеры импортируются не-красными). Верный трансформ — из extents (k≈0.10579, Y-flip).
- Печать: `plot_pdf` (MCP) не создаёт файл; `EXPORTPDF` повесил IPC модальным диалогом (вытащил через SendKeys ESC). Рабочее — `vla-PlotToFile`.
- `-INSERT` искажает масштаб (unit-factor ~1/25) — масштаб ставил через entmod.
- Прозрачная заливка зон не печатается через DWG→PDF — оставил контуры секторов.
