# Сессия 2026-07-01 — добавление строк-работ в СВОР (раздел ТХ, слаботочка/мультимедиа)

## Задача
В существующей сопоставительной ведомости объёмов работ (СВОР, `<шифр>-ТХ.15`, объект `<объект>`)
над каждой позицией оборудования добавить одну строку-работу (наименование работы) в колонку
«Наименование работ и затрат», тип работы по классу оборудования, сверяясь с нормативами
(ГОСТ/ГЭСН/ТЕР/ФЕР). Строка-работа наследует ед.изм.+объёмы до/после+формулы; позиции на 2 строки
(замена модели) зеркалятся.

## Что сделано
- Итог: копия `<шифр>-ТХ.15_ПД_МС_с работами.xlsx`, оригинал не тронут.

## Метод (детерминированно-first + доменная доводка)
1. openpyxl-экстракт позиций → JSON (якорь, ед., наимен., произв., модель, подсистема, two_row).
2. 5 параллельных `pto-engineer` (sonnet) по единому своду правил → work_text каждой позиции (файлы),
   покрытие сверил сам (471/471, 0 пропусков/дублей).
3. Вставка через **win32com** (живой Excel): `Rows.Insert` сам сдвигает формулы и merge; наследование
   E:L через `Copy→PasteSpecial`, D=работа, A:C очистить; обработка снизу вверх.
4. Самопроверка openpyxl + ревью `excel-validator` (форма) + `auditor` (содержание).

## Грабли / уроки
- **openpyxl insert_rows ломает относительные формулы** H/I (`=IF(...G{r}-F{r}...)` — 1020 шт.) — не
  пересчитывает ссылки. Решение: **win32com**. (Родственно уроку про INDIRECT/win32com.)
- Заголовки разделов = merge `A{r}:L{r}` во всю ширину (однострочные!) — в extract нельзя ставить
  условие `top!=bottom`, иначе не ловятся → контекст подсистем теряется.
- Ловушка классификации: ЖК-панель с Android, мини-ПК/ноутбук/моноблок с Windows — это монтаж/ПНР
  ОБОРУДОВАНИЯ, а НЕ «установка ПО» (детерминированный keyword-классификатор ошибался — нужен
  домен-агент для наименований и типов).
- Сервисные сертификаты (гарантия/подмена/консультации) — реально немонтажные, работу не выдумывать,
  вынести на решение пользователя.
- Файл `inspect.py` в рабочей папке затеняет stdlib `inspect` → numpy/openpyxl падают. Не называть
  скрипты именами stdlib-модулей.
