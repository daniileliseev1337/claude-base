# Session report — ось 3 Графика (R-слой): пункт C — зачеканка гильз + расход пены

**Дата:** 2026-06-23
**Блок:** pto / ось 3 Графика, R-слой (каскад `id-scheme-cascade`, toolkit `id-scheme-toolkit`)
**Контекст:** том ИД (раздел ВиВ, водоснабжение), продолжение поэтапной проработки осей после закрытия W-слоя. Обезличено (методология; объект/шифры/ФИО — плейсхолдеры).

## Что сделано
- **Догрилка вводных** (skill `domain-grilling`, выбор пользователя «полная догрилка сейчас»). Разрешены ветки: источник R-данных = DWG-факт + указания пользователя (отдельного РД ВК в папке тома нет, РД-проект — в общей папке проектов объекта); методика пены — формат эталона соседнего тома, но зазор по факту расчётного листа; данные гильз — из конкретного листа расчётной книги (правильные).
- **Осмотр DWG** (live AutoCAD, file_ipc): карта layouts через `vlax-for Layouts`+TabOrder в файл (cp1251→utf8). Лист заделки узлов прохода **водоснабжения** = 26 узлов; листы заделки `Г1/Г2/Г3` — **канализация** (приложение к АОСР К-тома), не наша зона. Идентификация объектов листа: штамп = нативный ACAD_TABLE; таблица пены = **OLE2FRAME** (грабля `gch_is_anatomy` подтвердилась снова); блок «Параметры» (выход баллона) — вторая OLE.
- **Library-first**: эталон соседнего тома (лист зачеканки в собранном PDF — **формат = 2 компактные таблицы**: «ВОР зачеканки» + «Расчёт пены», НЕ 13-столбцовый расчёт); файл расчётов гр/пена (правильные гильзы); скан сертификата пены (подтверждает наименование).
- **Замена OLE→2 нативные ACAD_TABLE** через переиспользование `gen_table.lsp` toolkit (cp1251 data-файл, `c7:build` auto-fit): таблица 1 «ВОР по зачеканке гильз» (4 кол) + таблица 2 «Расчёт расхода пены» (5 кол). Удалены неправильная OLE-таблица (45 узлов, выход 0,7 л→22 балл) и блок «Параметры». Стиль текста — под штамп (`GOST Common_B`).
- **QA**: deter-verify (дамп ячеек ACAD_TABLE через entget, контрольные суммы Σшт/ΣV/баллоны) + ревьюер `auditor` (opus) 6/6 PASSED. Сверка узлов↔план: **программный подсчёт красных кластеров-отметок** на плане (PIL+scipy) ≈ 26 (подтвердил таблицу, опроверг 45). DWG сохранён (DBMOD=0), бэкап R-слоя снят.
- **Кросс-блокеры в ось состава** (НЕ чинил сам): объём пены в ПВ от неправильного листа; гильзы как позиция материала отсутствуют в ВОР/ПВ.

## Источники / опора
- governance осей `blocks/pto/axes/`, каскад `id-scheme-cascade.md`, toolkit `id-scheme-toolkit` (`gen_table.lsp` оказался переиспользуем для любых компактных таблиц, не только ВОР оборудования).
- Эталон соседнего тома (формат таблиц зачеканки + методика выхода баллона) — повтор единообразно.
- Источник данных гильз — расчётный лист книги (факт первичен; toolkit ничего не сочиняет).
- Память `feedback_foam_calc_and_handoff` (расход = V÷выход вспененной, НЕ ÷объём баллона) — корень ошибки чернового расчёта.

## Где сломался (и как починил)
1. **Нарушил архитектуру осей** — предложил пользователю править ПВ (объём пены) самому из блока графики. Пользователь поправил: **«ГЧ только фиксирует изменение; синхронизацию ПВ делает ось состава»** (правило навигатора п.3 + протокол §1а). → переоформил как **кросс-блокеры** в `02_СОСТАВ`, своё (графику) сделал, чужое (ПВ) отдал профильной оси. УРОК закреплён.
2. **Методика пены в черновом файле неверна** (делал сторонний LLM): «выход 1 баллона 0,7 л» трактовался как объём баллона → раздутые баллоны (×37). Правильно — выход **вспененной** пены (по техлисту, 45 л/баллон) → 1 баллон. Типовая грабля foam_calc.
3. **Имена листов в расчётной книге перепутаны с фактом на чертеже**: лист с суффиксом «…В» (предполагалось водоснабжение, 26 узлов) на чертеже оказался у канализационного листа, а водоснабженческий лист нёс 45-узловой набор. → **доверять штампу чертежа (entget ACAD_TABLE штампа), не имени листа книги.**
4. **CTAB остаётся на последнем листе** после серии PNGOUT по другим layouts → `ssget` с фильтром `(410 . layout)` по «текущему» вернул пусто. → перед выборкой явно вернуть CTAB на целевой лист через объект layout по TabOrder.
5. **Ревьюер графики просил сверить с сертификатом** (по моему же промпту) — это **зона оси документов качества**, не графики; сертификат не передавался ревьюеру → NOT PASSED по непроверенному пункту. Разрешено предъявлением реального скана (документ найден в папке тома) + указанием, что подлинность серта — чужая ось (уже PASSED ранее). УРОК: не нагружать ревьюера графики проверкой документов качества; фасировать артефакт DWG в PNG+дамп (ревьюер DWG не читает).

## Уроки / харвест
- **НОВЫЙ пункт R-слоя (от пользователя): актуализация выносок оборудования на схемах под ВОР** — нигде в каскаде/памяти не прописано; в соседнем томе это поймала только финальная проверка и правила в DWG. Кандидат в `id-scheme-cascade` (R-слой) + память. Добавить в скоуп каждой правки состава: после изменения ВОР — сверить и обновить выноски-маркировку позиций на самих схемах.
- **`gen_table.lsp` toolkit — generic-построитель ACAD_TABLE**: подать произвольный cp1251 data-файл (LAYOUT/INS/COLS/NROWS/STYLE/TARGETH + ROW type cell…), `c7:build` строит и auto-fit'ит любую компактную таблицу (не только ВОР оборудования). Кириллица через файл, не argv. cp1251 не держит `→`/`π`/`²`/`−` — заменять ASCII.
- **Сверка таблица↔графика на плане**: программный подсчёт цветовых кластеров-отметок узлов (PIL маска + scipy.ndimage.label) — дешёвый объективный контроль кол-ва против выдуманного/ошибочного.
- **Бинарный артефакт перед ревьюером**: рендеры PNGOUT + дамп ячеек entget в текст (DWG ревьюер не открывает) — anti-холодный-кэш сработал.
