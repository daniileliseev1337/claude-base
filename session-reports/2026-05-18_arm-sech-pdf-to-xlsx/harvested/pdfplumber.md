# jsvine/pdfplumber

- **URL:** https://github.com/jsvine/pdfplumber
- **Stars:** ~7k
- **Last commit:** активный
- **License:** MIT
- **Описание:** Python lib для plumb'инга PDF — char-level и rect-level доступ. Хорош для извлечения **таблиц с координатами** и аннотаций (`.annots`).

## Зачем смотрели

Альтернатива нашему `pdf-mcp`, которая даёт **координаты каждого символа/линии/прямоугольника**. Полезно когда нужно понять «эта аннотация на каком ряду таблицы лежит» — у нас была такая нужда (PDF-пометки = разрозненные числа без явной привязки к позициям; визуальный рендер показал привязку, но программно был бы pdfplumber).

## Оценка

- **Подходит?** Да, для будущих похожих задач.
- **Сильные стороны:**
  - `.annots` — прямой доступ к аннотациям (Section 8.4 PDF spec).
  - `.chars` / `.lines` / `.rects` с координатами — можно автоматически класть пометки по таблице.
  - `.extract_tables()` — таблицы по линиям.
- **Слабые стороны:**
  - Не обрабатывает «нарисованный поверх» текст (textlayer без аннотаций) специальным образом — просто читает как chars.
  - Тяжелее в API чем `pdfplumber.open().pages[0].extract_text()`.
- **Решение:** держим как fallback. В этой задаче не использовали — pdf-mcp pdf_read_all + pdf_render_pages дали всё нужное (текст + визуал) без программной координатной логики.

## Заметка про наш pdf-mcp

Наш установленный `pdf-mcp` под капотом скорее всего обёртывает pdfplumber или pypdfium2/pikepdf. Его `pdf_read_all` и `pdf_render_pages` в этой сессии справились со всем. Plumber понадобился бы только если: (а) нужна структурированная таблица с координатами; (б) нужно matched mapping аннотации→таблица в коде без визуального осмотра.
