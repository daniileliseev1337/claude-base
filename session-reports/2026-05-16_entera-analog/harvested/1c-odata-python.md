# 1c-odata (Python wrapper)

- **URL:** https://github.com/belov38/1c-odata
- **Категория:** Интеграция с 1С (Python → 1C через OData)
- **Stars:** ~8 (мало!)
- **Last commit:** не указан явно, ~45 коммитов в истории
- **License:** Unlicense (public domain)
- **Описание:** Python-обёртка для работы с 1С v8 через OData-интерфейс. Классы `Infobase`, `Document`, `Catalog`. Фильтрация, пагинация, создание/редактирование записей, выбор полей.

## Зачем смотрели
Готовый Python-клиент специально для 1С OData. Альтернатива писать «сырые HTTP-запросы к /odata/standard.odata».

## Оценка
- **Подходит?** Под условием (как точка отсчёта, не как dependency)
- **Сильные стороны:**
  - **Unlicense** — максимально свободная
  - 100% Python
  - Специально под 1С — учитывает специфику типов и URL-схемы
- **Слабые стороны / риски:**
  - **Только 8 звёзд** — комьюнити фактически отсутствует
  - Статус поддержки неясен (нет явной даты последнего коммита)
  - Скорее всего покрывает базовые операции, на edge cases (составные ключи, табличные части документов) — придётся допиливать
- **Решение:** Используем как **референс/стартовую точку**. Возможно проще написать свой тонкий wrapper на `httpx` (50-100 строк) и подсмотреть в этом репо обработку специфических 1С-кейсов. Не подключать как dependency.
