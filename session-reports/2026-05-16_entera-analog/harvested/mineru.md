# MinerU

- **URL:** https://github.com/opendatalab/MinerU
- **Категория:** OCR / Document Understanding
- **Stars:** ~63.5k
- **Last commit:** 2026-05-15 (mineru-3.1.14)
- **License:** MinerU Open Source License (Apache-2.0 + поправки; недавно сменили AGPL → этот формат, чтобы упростить commercial use)
- **Описание:** «PDF/Office docs → LLM-ready Markdown/JSON». VLM+OCR двухдвижковая архитектура, 109 языков OCR.

## Зачем смотрели
Универсальный пайплайн «PDF УПД → структурированный JSON» без необходимости собирать самим связку OCR+layout+tables. Главный конкурент Surya/marker.

## Оценка
- **Подходит?** Да
- **Сильные стороны:**
  - Огромная база (63k звёзд), активность ультра-высокая (релизы каждые недели)
  - Уже даёт markdown + JSON с reading order, таблицы выделены отдельно
  - **Лицензия Apache-2.0 базированная** — намного лучше GPL Surya для внутреннего продукта
  - Свой VLM (MinerU2.5-Pro-2604-1.2B) + OCR fallback — двойной движок
  - 109 языков OCR (русский в комплекте)
  - Работает на CPU, ускоряется GPU/MPS
- **Слабые стороны / риски:**
  - Custom-лицензия (не стандарт MIT/Apache) — стоит однажды прочитать diff с Apache; вряд ли есть подводные камни для внутреннего PRO, но это требует 15 минут юриста
  - Большой моноблок — тащит много зависимостей; контейнеризация почти обязательна
  - Качество на специфике русского УПД (рукописные пометки, печати, факсимиле подписей) — тест на реальных образцах
- **Решение:** Кандидат №1 для категории OCR. Берём как основной движок. Лицензия дружелюбнее Surya, качество выше «голого» PaddleOCR.
