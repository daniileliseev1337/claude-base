# Категория 4 (Интеграция с 1С) — итог поиска

## Что искали
Open-source расширения конфигурации 1С (cfe-файлы) для приёма JSON-документов и автоматического создания УПД/Поступлений в БП 3.0 / УНФ. Готовые шаблоны HTTP-сервиса. Python/JS-обвязки OData.

## Что нашли в open source

### Зрелые кандидаты (есть карточки)
1. **vbondarevsky/Connector** — HTTP-клиент для 1С (для архитектуры «1С опрашивает наш сервис»). Apache-2.0, 786 звёзд, актуальный.
2. **belov38/1c-odata** — тонкий Python-клиент 1С OData. Unlicense, 8 звёзд, как референс.

### Что НЕ нашли в open source
- **Готового расширения конфигурации БП 3.0 / УНФ** под приём JSON-УПД из внешней системы — не существует в открытом доступе. Все коммерческие интеграторы держат такие расширения как платный продукт (Entera, Tessa, EnDocs, 1С-Гэндальф). На GitHub лежат только утилиты для разработки (1c-edt-issues, 1CFilesConverter, ExternalModulesConverterFor1C) и BSL Language Server — не наш кейс.
- **Шаблона HTTP-сервиса БП-под-приём-первички** — нет.
- **Python-клиента к стандартному 1С OData с поддержкой ВСЕХ типов** (документ + табличная часть + регистры + журнал) — все существующие тонкие.

## Вывод
Категория 4 — **слабая в open-source**. Готовых «кирпичей» под архитектуру «внешний сервис пишет УПД в 1С автоматически» нет. Будем писать обвязку сами:

### Рекомендуемая архитектура MVP
**Pull-режим (предпочтительный):**
- Наш сервис кладёт распознанные УПД в локальную HTTP-очередь (FastAPI + SQLite).
- Расширение конфигурации в 1С (написать с нуля, ~200-400 строк 1С-кода) на регламентном задании каждые 5 минут опрашивает наш сервис через `Connector`, забирает новые JSON-документы и создаёт «Поступление товаров и услуг» / «УПД полученный».
- Плюсы: нет внешнего доступа к 1С, нет нужды держать 1С web-доступной.

**Push-режим (альтернатива):**
- Включаем стандартный 1С OData (`/odata/standard.odata`).
- Наш Python-сервис создаёт документы напрямую POST-запросами (берём `belov38/1c-odata` как референс, дописываем под Документ.ПоступлениеТоваровУслуг.).
- Минусы: придётся настраивать аутентификацию, открывать 1С наружу, разбираться с составными типами (контрагенты по ИНН + табличные части).

### Что НЕ делать
- Не пытаться найти готовое open-source расширение БП — его нет, потратишь время впустую.
- Не использовать SOAP/Web-сервисы 1С — устарели; OData проще.
- Не писать обмен через xml/dt-файлы — синхронность ниже, ошибок больше.
