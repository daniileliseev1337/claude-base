# PaddleOCR

- **URL:** https://github.com/PaddlePaddle/PaddleOCR
- **Категория:** OCR / Document Understanding
- **Stars:** ~78k
- **Last commit:** 2026-04-21 (v3.5.0)
- **License:** Apache-2.0
- **Описание:** OCR-фреймворк от Baidu/PaddlePaddle. В v3.x появилась PaddleOCR-VL-1.5 (0.9B VLM) для document parsing с 94.5% на OmniDocBench.

## Зачем смотрели
Промышленно зрелый OCR с явной поддержкой кириллицы (русский в списке восьми основных переводов README — значит, поддержка модели делалась осознанно). Альтернатива MinerU с чисто Apache-лицензией.

## Оценка
- **Подходит?** Да
- **Сильные стороны:**
  - **Чистый Apache-2.0** — самая дружелюбная лицензия из всех топ-кандидатов
  - Огромная база (78k звёзд), очень зрелый проект
  - PaddleOCR-VL-1.5 — отдельный лёгкий 0.9B VLM специально для документов
  - PP-StructureV3 — table-aware конвертация в markdown/JSON с координатами
  - 111 языков, русский официально
  - Sealed text spotting (печати) — критично для УПД
- **Слабые стороны / риски:**
  - Привязка к PaddlePaddle (не PyTorch) — деплоймент сложнее, чем у альтернатив на torch. Хотя есть ONNX-конвертация (см. RapidOCR)
  - Меньше готовых обвязок «PDF in → markdown out», больше «голый OCR + структурный модуль»
  - Документация местами китайская; английская версия отстаёт
- **Решение:** Используем PaddleOCR-VL-1.5 как основной OCR-движок, если MinerU не подойдёт по лицензии. Либо как параллельный fallback. Apache-2.0 побеждает по чистоте.
