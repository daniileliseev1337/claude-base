# Connector (1C:Enterprise HTTP client)

- **URL:** https://github.com/vbondarevsky/Connector
- **Категория:** Интеграция с 1С (исходящие запросы из 1С)
- **Stars:** ~786
- **Last commit:** 2026-03-16 (v2.6.1)
- **License:** Apache-2.0
- **Описание:** HTTP-клиент для 1С:Предприятие 8, вдохновлённый Python Requests. `GetJson/PostJson/PutJson/DeleteJson` методы. Basic/Digest/Bearer/AWS4 авторизация. Платформа 8.3.10+.

## Зачем смотрели
Если архитектура «1С тянет данные от нашего сервиса» (а не «сервис пушит в 1С») — это лучший готовый клиент. Минимизирует ботстрэп-код в 1С.

## Оценка
- **Подходит?** Да
- **Сильные стороны:**
  - **Apache-2.0**
  - Зрелый (786 звёзд для нишевого 1С-проекта — это много), стабильные релизы
  - Идиоматичный «requests-style» API делает 1С-код читаемым
  - Полная JSON-обвязка, sessions, retry с exponential backoff, proxy
- **Слабые стороны / риски:**
  - Это **клиент**, не сервер — для архитектуры «push в 1С» нужно своё HTTP-сервис в конфигурации
  - Нужно встраивать в конфигурацию (расширение или общий модуль) — не plug-and-play для типовых БП/УНФ
- **Решение:** Используем для архитектуры «1С опрашивает наш сервис каждые N минут на предмет новых распознанных документов и тянет их сама». Чище чем «снаружи пушить через OData».
