# NuExtract 2.0

- **URL:** https://github.com/numindai/nuextract
- **Категория:** Invoice/Receipt parsing (структурное извлечение через LLM)
- **Stars:** ~121 (свежий репо, но автор — Numind, известный в structured extraction)
- **Last commit:** не указан явно в README, но модели NuExtract 2.0 опубликованы в 2026
- **License:** MIT (для 2B и 8B), Qwen Research License (для 4B)
- **Описание:** Семейство моделей специально для structured information extraction. Multimodal + multilingual. JSON template на входе → JSON на выходе.

## Зачем смотрели
Семейство моделей, разработанных именно под задачу «дай JSON-шаблон → получи заполненный JSON из текста/картинки». Идеальная архитектура для УПД.

## Оценка
- **Подходит?** Да
- **Сильные стороны:**
  - **MIT** для 2B/8B вариантов — отлично
  - Идеальная концепция для нашей задачи: «вот формат УПД в JSON Schema, вот картинка — заполни»
  - Multimodal — можно подавать сразу картинку УПД, без отдельного OCR
  - Multilingual (русский явно не указан, но архитектура унаследована от мультиязычных моделей)
  - Маленькие размеры (2B запускается на CPU за разумное время)
- **Слабые стороны / риски:**
  - Очень мало звёзд (121) — комьюнити маленькое
  - Русский в документации не подтверждён — тест на УПД обязателен
  - 4B-вариант на Qwen Research License — несвободная, использовать только 2B/8B
- **Решение:** Кандидат №1 в категории «Invoice parsing» — концептуально идеален для задачи. Прототипировать первым: NuExtract-2B на UTF-кириллице с JSON-схемой УПД.
