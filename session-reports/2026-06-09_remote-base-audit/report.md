# 2026-06-09 — Аудит базы из удалённой сессии (телефон → Claude Code on the web)

## Контекст
Первая рабочая сессия с базой из облачного контейнера (не dev-ноутбук): свежий клон,
без локального окружения (~/.claude рантайма, MCP, hooks). Подтверждено: контент-разработка
базы из облака полноценна; непроверяемо отсюда — исполнение PowerShell/hooks/MCP.

## Что сделал
1. **Аудит трёх хронических симптомов** (агенты не вызываются; MCP-инциденты Excel/Word;
   consumer ушёл в WebFetch). Метод: прямое чтение + 3 параллельных Explore-агента.
2. **Диагноз — системный:** работают только правила с жёстким гейтом в always-loaded
   CLAUDE.md (norm-lookup — правило 7; ревьюер — пункт 3 архитектуры). Cascade-память и
   тела агентов невидимы в момент решения. Enforcement-слой (permissions deny, hooks)
   не использовался вовсе. Frontmatter'ы 16 агентов валидны, description после прошлой
   починки хорошие → «чинили не тот слой».
3. **Пакет правок** (детально — CHANGELOG 2026-06-09): роутинг-гейт; лестница веб-доступа;
   ловушки чтения xlsx (формулы без кэша ≠ пусто) и docx (merged-ячейки ×N) + независимый
   пересчёт денег в валидаторах; единый /sync-base (tier/needs_admin/needs_key в манифесте,
   declined-механика с постоянной видимостью); flag-файл install-долга + тихая строка в STOP;
   личный слой CLAUDE.user.md (@import); граф базы как версионируемый навигатор.

## Решения и их мотивировка
- **Граф ≠ CLAUDE.md**: граф — pull-навигатор (query → source_location → файл),
  поведенческие правила — только push (CLAUDE.md). Зафиксировано в CLAUDE.md.
- **deny WebFetch НЕ включён** сознательно: norm-lookup использует его как документированный
  fallback. Сначала обкатка лестницы, потом замена fallback, потом deny (см. docs/чек-лист).
- **Отказ от LLM-merge личного и общего**: граница структурная (отдельные файлы), не умный merge.

## Где сломался / ограничения
- Графа базы в репо не оказалось (`.gitignore: graphify-out/`) — построен только локально
  на хабе. Решение: исключение в .gitignore, граф пушится с ноутбука.
- PowerShell-правки (auto-pull flag) из облака не исполнить — верификация на ноутбуке
  по чек-листу `docs/2026-06-09-laptop-after-merge.md`.
- setup-extras.ps1 пока не понимает tier и method=claude-mcp-add (безопасно: unknown
  method = skip); интерпретация — в /sync-base, доработка скрипта — на dev-ПК.

## Уроки (кандидаты в скиллы/память)
- «Записано» ≠ «загружено» ≠ «принудительно» — проверять, в каком из трёх слоёв живёт
  правило, прежде чем чинить его текст.
- Инциденты "тихо посчитал не то" (Excel/Word) — дети невызова ревьюеров, т.е. дети
  отсутствия гейта: чинить причину, не симптом.
- Удалённые сессии: коммит/пуш сразу, контейнер эфемерен; граф и любые деливераблы
  с локальной машины должны быть в git, иначе для остальных сессий их не существует.
