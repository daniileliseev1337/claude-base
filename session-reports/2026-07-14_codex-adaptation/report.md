# 2026-07-14 · Мульти-LLM адаптация базы под Codex (фазы 1–2)

## Задача
Владелец: база создана под Claude, адаптировать под любую LLM (сейчас — Codex),
«полный паритет» — оба агента одинаково понимают и используют базу. У Codex свои
инструменты документов (excel/word/pdf) — не переучивать где родное лучше.

## Процесс (по пайплайну базы)
brainstorming → 2 фоновых research (механика Codex + промптинг GPT-5.x) + локальная
разведка → грилинг (роль Codex, профили Plus/Pro x5) → дизайн вариант B (утверждён) →
план 13 задач → субагентный конвейер (implementer haiku/sonnet → task-review sonnet →
фиксы → приёмка) → финальный обзор opus → фиксация вехи.

## Результат: фазы 1–2 обкатаны
- **Ядро** `core/AGENTS.core.md` — vendor-нейтральные правила, нейтральный стиль
  (без КАПСа/«ЗАПРЕЩЕНО» — GPT-5.x жжёт reasoning на противоречиях), зоны
  вопрос/действие разведены. CLAUDE.md = `@import` ядра + Claude-слой; auditor-гейт
  эквивалентности PASSED (131 ссылка, ни одно правило не потеряно).
- **Генератор** `scripts/codex_sync.py` (19 тестов, идемпотентен) собирает `~/.codex`:
  AGENTS.md (LF, ≤32768), managed-блок config.toml (чужое приложение цело),
  hooks.json, 16 agents/*.toml (модель-маппинг, read-only sandbox ревьюерам),
  скиллы через junctions `~/.agents/skills`.
- **Живой смоук в Codex:** правила пересказаны по-русски с Done-when, 10 скиллов
  видны (наши + родные, Codex сам разложил по категориям), план работает, MCP чисто.

## Где сломался / грабли (документированы в память codex_multi_llm_adaptation)
- Тяжёлые MCP (Revit без живого Revit → exit 1; autocad-mcp — кириллица пути
  uv-python) валили старт приложения + тормоза → дефолт MCP сужен до `time`.
- `skills.config` с произвольными путями в десктоп-сборке не работает → junctions.
- `hooks.json` в десктоп-Codex не исполняется → синк иным механизмом (фаза 3).
- Ревьюеры по ходу поймали реальные баги ПЛАНОВОГО кода: regex-escape на
  Windows-путях в managed-блоке; невалидное TOML-экранирование апострофа; CRLF во
  фронтматтере ронял конвертер на 5/16 агентов; block-scalar description; гейт-32K
  мерил LF а не CRLF. Все закрыты TDD.
- Разъезд base-репо (auto-push отставал) — rebase+push.

## Отложено (осознанно)
