### Методология (накопленные параметры)

- **Объём**: всегда ВОР+ПНР в одной книге.
- **Группировка**: 1-уровневая по типам оборудования (как СОТ/ЛВС эталоны).
- **Шифр**: `25/6729-ИОС4.X.ВОР`/`.ПНР` с точкой как разделителем.
- **Папка**: подпапка «ПНР и ВОРы» рядом с PDF раздела.
- **Шаблон xlsx**: копируем `ПНР и ВОРы СКУД.xlsx` как образец стилей (он самый «свежий» эталон).
- **ЗИП**: в ВОР указывается в колонке «Тип монтажа» как «N к монтажу + N ЗИП», в ПНР — кол-во без ЗИП.
- **Световые табло 3 типов** (как ОСТ в СПС или ЛЮКС-12 в АУГПТ): в ВОР раздельно по исполнениям, в ПНР объединены в одну строку.
- **Расхождение ТЧ vs Спецификация**: использовать данные из Спецификации (это окончательная версия для DataLink).

## Что узнал за сессию (новая методология)

### 1. xlsx как DataLink-источник

В нескольких разделах нашёлся **готовый xlsx-источник Спецификации** в папке DWG (СОУЭ_СО, АУГПТ.СО, СПС). Это AutoCAD DataLink — рабочий источник Спецификации. Полезно читать раньше PDF — данные те же, но структурированно.

### 2. ЗИП — стандартная практика для пожарных систем

В СПС 9 позиций имеют ЗИП (29 ДИП, 2 ИПР, 2 ИПДЛ, 3+3 СП4/220, 2 УДП, 2+1+1 ОСТ, 25 МК-2). Стандартная политика:
- ВОР: указать полное кол-во (с ЗИПом) + пометка «N к монтажу + N ЗИП» в Типе монтажа.
- ПНР: использовать кол-во **без ЗИПа** (ЗИП не монтируется и не пусконаладится).

### 3. ЗКПС в СПС — отдельная единица ПНР

73 зоны ЗКПС в Сит-Центре. Каждая зона = программирование привязки извещателей + алгоритм A/C + проверка. 2 функции × 73 зоны = 146 функций отдельной строкой ПНР.

### 4. Интеграции между подсистемами

Сложная сеть взаимосвязей:
- СКУД ↔ СПС: при пожаре С2000-СП2 отключает замки СКУД (через Sigur E210 UPS).
- СПС ↔ СПЗ: Сириус СПС → Сириус СПЗ по RS-485 кольцу, отключение вентиляции, закрытие клапанов.
- СПЗ ↔ СОУЭ: пуск речевого оповещения LPA-Duplex-1 от Сириус.
- АУГПТ ↔ СПС: передача «Пожар» и подтверждение пуска.
- АУГПТ ↔ СПЗ: отключение вентиляции в защищаемом помещении при пуске ГОТВ.

Эти интеграции — отдельные пункты в ПНР центральных приборов (Сириус СПС, С2000-АСПТ).

### 5. Норматив СОУЭ изменился 01.06.2026

СП 3.13130.2009 утратил силу с 01.06.2026, действует СП 3.13130.2026 (Приказ МЧС № 133 от 26.02.2026). При ПНР после смены норматива — ссылаться на новый, даже если в ТЧ старый. См. memory `reference_sp_3_13130_2026.md`.

### 6. Нормы через norm-lookup — не формальность

Я процитировал ГОСТ Р 59636-2021 по памяти как «Установки автоматические газового пожаротушения. Руководство...». Word-checker заподозрил склейку → норм-lookup подтвердил искажение. Правильное название: «**Установки пожаротушения автоматические**. Руководство по проектированию, монтажу, техническому обслуживанию, ремонту и методам испытаний на работоспособность» (это **общий** норматив для всех АУП, не только газового). См. memory `feedback_norms_via_norm_lookup.md`.

## Что НЕ делать

1. **Не цитировать нормы по памяти** — всегда через `norm-lookup` агент.
2. **Не дублировать строки ВОР для световых табло разных исполнений в ПНР** — там ПНР идентична → 1 строка с общим кол-вом.
3. **Не использовать excel MCP для больших xlsx** — openpyxl напрямую.
4. **Не вычитать ЗИП из ВОР** — он часть Спецификации, должен быть. Только в ПНР используется кол-во без ЗИП.
5. **Не сохранять обоснование в session-report или в xlsx** — отдельный docx рядом с xlsx.

## Pipeline для следующего раздела (готовая схема)

1. **Разведка** — `Get-ChildItem` папки + `pdf_info` + проверить наличие xlsx-источника Спецификации в DWG.
2. **Чтение PDF** — PyMuPDF (fitz) для ТЧ (Общие данные) и Спецификации. **Не pdf-mcp** — глючит на граничных страницах.
3. **AskUserQuestion** — стандартные 4 вопроса:
   - Параметры (стандарт)
   - Кол-во функций на 1-2 ключевые позиции
   - Учёт ЗИП (если есть)
   - Объединять ли разные исполнения в одну строку ПНР
4. **norm-lookup** — если в обосновании нужны точные ГОСТ/СП.
5. **Сборка xlsx** — копия СКУД-шаблона + Python скрипт.
6. **Сборка docx** — Python скрипт по той же структуре.
7. **Валидация parallel** — excel-validator + word-checker через Task tool.
8. **Memory обновление** — если узнал что-то новое.
