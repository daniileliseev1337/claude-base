# Revit MCP коннектор: диагностика «модель не отвечает» + разведка модели

## Задача
Изучить (read-only) большую Revit-модель раздела СС (слаботочка) медцентра, разделы которой
разбиты по рабочим наборам. Перед разведкой — коннектор `Revit-Connector` стабильно не отвечал.

## Что сломалось и как чинили (главное — «камни преткновения»)
Все MCP-команды к Revit висели/падали, хотя Revit был открыт и исправен. Диагностика по слоям
(а НЕ перезапуск Revit вслепую) вскрыла **тройную** причину:

1. **Корп-прокси (корень).** `httpx` в MCP-клиенте (`trust_env=True` по умолчанию) гнал даже
   `http://127.0.0.1:48884` через `HTTP_PROXY` (на ПК не задан `NO_PROXY`). Прокси → **пустой 503**.
   Симптомы маскировались под «таймаут»/«Server disconnected». Репро: `httpx.get` 503 / `trust_env=False` 200.
   Фикс: `trust_env=False` обоим `httpx.AsyncClient` в `main.py`.
2. **localhost→IPv6.** Routes слушает `0.0.0.0:48884` (IPv4), `localhost` на Windows резолвится в `::1`.
   Фикс: `REVIT_HOST = "127.0.0.1"`.
3. **Home-экран Revit** = `ActiveUIDocument=None` → 503 «No active document». Лечение: открыть вид модели.

Доп. грабли: **две копии `main.py`** (правил сначала серверную в pyRevit Extensions, а Claude
запускает клиента из `~/.claude/mcp-servers/revit-mcp-python/`) — reload не помогал, пока не нашёл
запускаемый файл по CommandLine процесса.

## Метод диагностики (переиспользуемый)
- `Get-NetTCPConnection -State Listen` → порт 48884 слушается?
- прямой `Invoke-WebRequest http://127.0.0.1:48884/revit_mcp/status/` (200) vs MCP-команда (висит)
  → изолирует «обёртка vs Revit»;
- чтение тела 500/503 через `[Net.HttpWebRequest]`;
- репро httpx с `trust_env` True/False;
- `Get-CimInstance Win32_Process` → какой именно `main.py` запущен.

## Результат разведки модели
Открыт раздел **СС** (слаботочка) медцентра. 15 рабочих наборов двух типов: собственные данные
`СС2_*` (СПС ~7038 элем — пож. сигнализация; СКПТ ~722; Оси/уровни; СТМ пустой) и контейнеры
RVT-линков с префиксом `#` (АР×4 блока, ВК×4, ТХ, МГ, Базовый файл — 11 линков). 19 уровней
(9 надземных этажей + техэтажи/кровли + 2 подземных). 10 листов, 101 вид. 0 предупреждений.
Расшифровки СКПТ/СТМ не подтверждены — оставлены на уточнение (правило «не выдумывать»).

## Открытые рекомендации (не выполнено, требует решения владельца)
- Внести фикс (`trust_env=False` + `REVIT_HOST=127.0.0.1`) в **дистрибутив** revit-mcp-python
  (setup-extras), иначе у каждого ПК за прокси коннектор будет висеть. `mcp-servers/` не в git базы.
- Добавить `NO_PROXY=127.0.0.1,localhost,::1` в Set-Proxy.ps1 / setup-extras Step 0.
- На этом ПК отсутствует `~/.claude/.developer-marker` (хотя владелец — разработчик): авто-режим
  считает ПК consumer'ом. Создать маркер при необходимости.
