## TL;DR
- Развернули парадигму закрытия тома ИД: с **горизонтальных слоёв-осей** («все схемы по тому → все акты → вся ПВ») на **вертикальные срезы** (акт от и до → фиксация → следующий). Инициатива пользователя, не моя.
- Собрали целостную рамку **вход → конвейер срезов → выход**; оси Р→6 не выброшены, а переупакованы.

- Агентов НЕ создавали (HARD-GATE соблюдён): состав выкристаллизуется из первого боевого прогона.

## Что решено (ключевые развилки)
1. **Единица = срез** (акт). Типы: ВиВ‑N (закрывают материалы, свои приложения) + АОУСИТО (общий, объёмов нет, схемы+испытания, ведётся сквозняком).
2. **Конвейер последовательный, не fan-out** (инсайт пользователя): 11 действий цепочкой, срезы по одному (накопители сквозные). Параллель только внутри шага.
3. **Разметка природы**: агент (суждение/веб) / скрипт (детерминизм: агрегация реестра, сверки дат, пагинация) / гейт (проверка). Deterministic-first.
4. **Замечания ТН — три канала** (общий список / стикеры / рукописные поверх листа) + тип `наша находка`. Рукописные — только vision (текст-слой слеп → раньше «терялись»). Дословно = вырезка изображения + расшифровка рядом.
5. **Анализ существующего тома — гибрид** (лёгкий скан всего + глубоко по срезам; находки сразу в реестр).
6. **Учёт вне PDF**: журнал JSON → автономный виджет HTML (3 вкладки: Статус тома / Карта акта / Замечания ТН с перекрёстными метками). PDF чистый.
7. **Папки по актам** (4 подпапки: Акт/Схемы/Документы качества/Реестр) — зеркалит срезы, лечит бардак; гейт держит структуру.
8. **Пагинация** — скрипт-нумератор по живому листу (двусторонка), не по PDF.
9. **Полигон** — копия Молот/Треугольник (оригинал не трогаем).

## Где сломался / уроки
- **Полировка виджета в вакууме** — 4 хода крутили форму на выдуманных данных, пользователь устал («топорно», «порядок не тот»). Урок: **форму (виджет) не доводить на демо-данных, только на боевых**; спеки достаточно.
- Дважды навязал непроверенное допущение (единица=«система»; общие схемы «висят») — пользователь поправил. Урок: **спрашивать факт до постройки решения** (общие схемы → под АОУСИТО, ответ был на поверхности).
- Пользователь трижды подряд ловил забытые ямы (замечания ТН, анализ графики, вход/выход осей). Урок: **рамку осей держать перед глазами** — при развороте парадигмы легко растерять этапы.
