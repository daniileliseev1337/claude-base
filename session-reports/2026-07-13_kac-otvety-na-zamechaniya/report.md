# 2026-07-13 — Ответы на замечания эксперта по КАЦ (цены «завышено»)

## Задача
Заказчик прислал 7 xlsx «Сводная таблица результатов конъюнктурного анализа»
(форма 421/пр) с замечаниями эксперта: жёлтые позиции = «завышено» + ссылка на
розничный источник. Нужно: по каждой позиции мотивированное обоснование
сохранения цены (минимизировать снижение) и 3 новых столбца в этих же файлах:
согласие / предлагаемая цена / комментарий с источником.

## Что сделал (пайплайн)
1. **Инвентаризация** (openpyxl): вердикт эксперта в колонке G или F (два
   варианта раскладки формы!), жёлтая заливка = завышено; 593 позиции, 84 «завышено».
2. **Ядро проекта** (project-memory bootstrap) + FACTS.md + дашборд-Artifact
   (палитра по dataviz-валидатору, статусы = семантика заливок исходника).
3. **Workflow ×34 sonnet**: 17 чекеров (проверка ссылки эксперта: тот ли товар,
   НДС, наличие, тип продавца + 1-3 своих цены) → pipeline → 17 адверсариальных
   верификаторов (вердикт по умолчанию REJECTED). 14 отклонено → 11 поправил по
   фактам из verify_note (ревьюер сам приносит фактическую цену со страницы!),
   3 добрал агент-снабженец.
4. **Запись win32com** в оригиналы (бэкап обязателен): 3 колонки после последней
   графы, формат копией соседней графы, NumberFormatLocal "# ##0,00".
5. **Ревью-гейт**: excel-validator (PASS 7/7, исходные зоны побайтово = бэкапу)
   + auditor (сплошная 84/84; 3 блокера «метка не согласны при сниженной цене» —
   исправил на «частично»). Формулы до/после — идентичны (отдельный дифф).
6. **Правки пользователя на лету**: (а) позиция-«сенсорный стол» = индивидуальное
   исполнение (адаптация для незрячих, спецПО) → «не согласны», цена сохранена,
   аргумент «у производителя цена по запросу»; (б) статус «частично согласны»
   признан некорректным («перекидывание ответственности») → второй вариант
   файлов в подпапке: только «согласны»/«не согласны»/«уточнить у ГИП» (8 позиций
   с индивидуальным исполнением или родовым описанием без модели).

## Уроки
- **Верификатор с note = золото**: адверсариальный ревьюер, который пишет
  фактическую цену со страницы, позволяет чинить отклонённое детерминированно,
  без повторного веб-захода.
- **Политика цены в промпте** («предлагаемая никогда не ниже цены эксперта;
  не согласны → цена = наша») ловится аудитором как инвариант — 3 нарушения
  нашлись именно так.
- **Args Workflow приходит строкой** — защитный `typeof args === 'string' ?
  JSON.parse(args) : args` обязателен (повторная грабля).
- **Формы КАЦ неоднородны**: вердикт-колонка гуляет (F/G) — автоопределение по
  словарю вердиктов, не хардкод.
- **Статус «частично согласны» в официальном ответе** пользователь считает
  перекидыванием ответственности — для спорных позиций лучше «уточнить у ГИП»,
  для остальных бинарно «согласны / не согласны» (несогласие + скорректированная
  цена в колонке цены).
- **Индивидуальное исполнение** (адаптация для МГН, спецПО, «в составе»,
  родовое описание без модели) — сильный аргумент против сравнения с розницей;
  маркер: у производителя «цена по запросу».
- Магазины с антиботом (dns-shop, vseinstrumenti, lunda): web_get ступени
  direct/jina часто 401 — фиксировать как «источник эксперта не верифицируется»,
  а цену подтверждать другим источником.
