## Что сделано

1. **STOP-процедура** — MCP 11/11 core (+autocad/firecrawl/Revit), агенты 16/16, install-долга нет. Роль хаб.

2. **Разведка (без правок).** Прочитаны 3 трекера + RUNBOOK + отчёт прошлой сессии + ВСО (эталон факта) + 3 реестра
   (приложение к АОМР, сквозной «монолит», приложение к АОСР) + старый бэкап монолита (карта физ.PDF, кэш листажа цел)
   + листажи всех добытых _NEW PDF (pikepdf). Источник-PDF 392 стр.

3. **Карта синхронизации (артефакт).** Сведён единый эталонный список документов качества оборудования по факту ВСО
   + доборы _NEW. Выявлено: реестр-приложение и блок монолита структурно разошлись (разное число строк на прибор,
   пропущенная позиция) → приведены к одному эталону.

4. **Развилки согласованы с пользователем** (AskUserQuestion): замена устаревшего наименования прибора на фактический
   с физ. заменой документа; объём руководства (datasheet+installation); исправление чужого шифра тома в приложении.

5. **Адверсариальная находка по датам.** Один добытый _NEW (добровольный СС) просрочен на ДАТУ АКТА (срок истёк ранее
   даты подписания акта) → ИСКЛЮЧЁН из добора. Закрывается бессрочным экспертным заключением. Реквизиты 4 сканов
   (без OCR-слоя) сняты vision самостоятельно (не haiku — он врёт в номерах).

6. **Правки реестров (скрипты openpyxl, бэкапы, самопроверка):**
   - Приложение-оборудование: 25→33 поз. (замены на месте + удаления + доборы), исправлен шифр тома и тип акта
     в заголовке (АОСР→АОМР); сохранение формата копированием стилей поколоночно.
   - Монолит: 102→115 поз. (блок оборудования = эталон + 5 новых паспортов арматуры после смежного блока),
     перенумерация, формула сквозного листажа. Перестройка области данных (надёжнее insert_rows, не ломает merge).

7. **Верификация — многоуровневая:**
   - Программная кросс-сверка (посимвольно наименование + листаж): приложение ↔ блок монолита = 0 расхождений;
     второе приложение ↔ блок монолита = 0 расхождений; нумерация непрерывна.
   - Workflow adversarial-ревью (3 субагента sonnet параллельно): excel-validator (форма) PASS;
     auditor (ВСО-покрытие + даты) WARN→исправлено; auditor (кросс-согласованность) WARN→разобрано.
   - Оба WARN закрыты: заголовок исправлен; «укороченное наименование» = формат листа-приложения (не дефект).

8. **План физ. пересборки PDF** (раздел артефакта): удалить/заменить/вставить по наименованию+листу тома,
   листажи _NEW, методика перенумерации сквозного листажа после сборки.

9. **Трекеры + память + отчёт** обновлены.

## Где сломался / поучительное

- **Excel-MCP read_data отдаёт мегабайты** (метаданные валидации по каждой ячейке) → переполнение. Перешёл на
  компактный openpyxl-дамп в utf-8 файл.
- **TOC скан-PDF — мусорная** (закладки сборки, page=-1) — не карта документов.
- **haiku-fan-out для локализации страниц в скан-PDF БЕЗ OCR провалился** — агенты путали физ. расположение
  документа с упоминанием в содержании/реестре (стр. оглавления 8/31/218). Точные страницы отброшены; основа
  плана удаления — лист тома из старого бэкапа монолита + наименование. Урок: для сканов локализация — vision, не haiku.
- **Листаж реестра ≠ страницы PDF** (документ «18 листов» = ~6 физ. страниц). Не вычислять страницы линейно из листажа.
- **Консоль Windows cp1251** калечит кириллицу в stdout → весь вывод проверок писать в utf-8 файл (повтор урока).
- **Workflow tool не принимает run_in_background** (он и так фоновый) — убрать параметр.
