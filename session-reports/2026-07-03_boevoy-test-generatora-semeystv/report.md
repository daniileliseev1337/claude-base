# Session report: боевой тест генератора семейств (И1) на реальном оборудовании

Дата: 2026-07-03. Стенд: Revit 2025 + pyRevit Routes MCP. Протокол: revit-testbed.

## Что сделал

- По запросу владельца провёл боевой тест умного генератора семейств: построил
  семейство реального оборудования (канальный вентилятор, 6 типоразмеров) «как
  BIM-менеджер», стандарт = чек-лист 8/8 — ПРОЙДЕН с первого железа, без образцов.
- Данные добыл сам из веба: официальный паспорт производителя (PDF валидирован,
  таблицы через pdf-mcp, чертёж прочитан рендером страницы). Свод и «чего в паспорте
  НЕТ» зафиксированы файлом до стройки.
- Полный стек: revolve-геометрия с живыми диаметрами, живой Coarse (победа над
  caveat эталона), lookup-CSV вшита в rfa, минимальный каталог типоразмеров,
  16 shared по GUID из корпоративного ФОП, 2 duct-коннектора с диаметром от
  параметра, RCP; в тест-проекте: 6 типов из каталога, врезка в воздуховод от
  коннектора (Ø из параметра), смена типоразмера под живым соединением.
- Откат стенда по протоколу (тест-проект закрыт без сохранения, эталоны не тронуты).

## Источники

- Паспорт производителя (веб, зеркало дилера; сверен с 2 дилерскими карточками).
- Корпоративный ФОП (GUID-сверка), lookup-правила из исследования статьи Муратова,
  образцы каталогов из библиотеки базы (идентификаторы ##other## / нижний регистр).

## Где сломался / что поймал (15 находок, полный список в отчёте проекта)

- Diameter/radial dims на РЁБРАХ тел — мёртвые лейблы; живой канал — dims на
  ЭСКИЗНЫЕ линии (`Revolution.Sketch`, `.SketchId` отсутствует в 2025).
- NewExtrusion идёт вдоль нормали sketch-плоскости; нормали новых RP мерить фактом.
- DuctSystemType в DB.Mechanical; CreateDuctConnector требует ConnectorProfileType;
  Duct.Create от коннектора — перегрузка без systemTypeId.
- Формулы: «А» (ампер) только кириллицей; приведение `number * 1 <ед>` работает;
  Revit пересохраняет строку формулы в display units с округлением (вычисление
  точное — риск ручного редактирования в UI).
- Конкатенации строк в формулах нет → Наименование/Марка per-type через каталог.
- Снова кириллица в литералах (u'ADSK_…'→None) — доступ к shared по GUID.

## Уроки

- Паттерн «минимальный каталог (ключ+тексты) + lookup-CSV внутри rfa» — рекомендуемый
  стандарт: типы в проекте пересчитываются сами, CSV рядом не нужна.
- Воркфлоу добычи данных проработан (что достаю сам / что просить у пользователя) —
  раздел в отчёте проекта; ключевое правило: ни одна цифра в rfa мимо файла данных.
- Ловушки вынесены в memory (revit_family_editor_api_traps) + отчёт в папке проекта.

## Продолжение сессии (вторая половина)

- **Провал-урок:** закрыл тест-проект отката, пока владелец смотрел результат
  («смотрел, крутил — исчез»). Новое правило: показ → подтверждение → откат
  (memory feedback-revit-stend-etiquette). Второй урок: оборудование для семейства
  выбирает владелец, не разработчик.
- **Галочка «считать в спеке» внедрена в боевой шаблон**: shared Yes/No параметр
  (instance) на 13 MEP-категорий + скрытое поле и фильтр «отлично от Нет» в 14
  спецификациях изделий. Семантика доказана чистым экспериментом: не задано/Да —
  элемент в спеке, снял галочку — исчез. Бэкап прежнего шаблона рядом.
- **Форма заявки на семейства расширена** 4 колонками (Категория Revit / Подключения /
  Источник размеров / Параметрика) — на согласовании у BIM-менеджера.
- **Кейс «изделие под заказ»**: заявка на интерактивный стол — габаритов/веса/мощности
  НЕТ в открытом доступе (22+ источника: производитель+архив Wayback+микроразметка,
  6 дилеров, тендеры, маркетплейсы; производитель пишет «изготовим по вашим размерам»).
  Найден ребрендинг линеек производителя (старые имена в файлах картинок — ключ к
  архивному поиску). Семейство припарковано до паспорта от заявителя — легитимный
  сценарий воркфлоу, ради него в форме появилась колонка «Источник размеров».
- Техника: скрытые JS-вкладки «Характеристики» дилерских карточек читаются через
  exa web_fetch (r.jina даёт только статику); Wayback CDX API работает с иностранного
  egress напрямую.
