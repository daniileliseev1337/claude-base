# Сессия 2026-06-23 — Узел под балконом: 6 труб + канальный блок (AutoCAD via MCP)

## Задача
Техническое решение + чертёж (DWG для монтажников): как проложить под балконом-антресолью
6 магистральных стальных труб ТС/ХС/ГВС Ø76 (без изоляции, 2 ряда) и канальный внутренний
блок кондиционера TL-PD170-FR4 (+пленум всаса h=200), прижав и трубы, и блок к низу балок.
Объект — <объект> (раздел ВиК, 37 листов).

## Что сделал
1. Грилинг вводных (по выбору пользователя): трубы = голая сталь Ø76; компоновка A (трубы
   2 ряда×3 у стены, блок к краю балкона); высота балки ~200 (допущение).
2. Разбор источников: проект ВиК (стр.10/27 — кондиционирование DX-фреон, блоки рядом),
   каталог Thaicon стр.22 (габариты TL-PD170-FR4: 1200×760×245, F=1022, H=149), видео с
   объекта (раскадровка ffmpeg — балкон стальной, пучок труб уже смонтирован у низа балок).
3. Чертёж: разрез 1-1 + план (вид снизу) + примечания. Сначала ezdxf→DXF→DWG (SaveAs COM),
   затем по просьбе пользователя переделал НАТИВНО через autocad-mcp (execute_lisp load
   cp1251-LISP). Auditor (opus) — PASSED (габариты сверены с каталогом, 445=245+200, зазор
   блок↔пучок +72, согласованность разрез↔план). PDF (белый фон) через ezdxf-рендер.
4. По фидбэку «всё налезает» — переверстал: единый источник build_uzel.py → два эмиттера
   (LISP→DWG + ezdxf→DXF/PDF), чистые зоны, размеры вынесенными цепочками.

## Источники
Проект ВиК <объект> (Клуб.pdf, 37 л.); каталог Thaicon TL-PD-FR стр.22; видео/фото с объекта.

## Где сломался / уроки (→ память)
- ffmpeg в системе нет → curl с gyan.dev (`--ssl-no-revoke` через прокси). [[reference_ffmpeg_video_frames]]
- autocad-mcp: `plot_pdf`/`save_as_dxf`/`-DXFOUT` НЕ работают; кириллица только через
  `(load)` cp1251-LISP + arial.ttf style + `%%c`; рендер только PNGOUT; DWG→PDF через
  ezdxf (явный целочисленный текст размеров, иначе «.00»); один IPC-канал — по одной
  операции. [[reference_autocad_mcp_workflow]]
- Упустил АУДИО видео (контекст был в речи «детально объясняется») — взял только кадры.
  → решение: скилл local-video-digest (ffmpeg keyframes + faster-whisper) в отдельном чате.
- Первая раскладка чертежа — наложения подписей; вынос аннотаций в «чистые зоны» обязателен.
