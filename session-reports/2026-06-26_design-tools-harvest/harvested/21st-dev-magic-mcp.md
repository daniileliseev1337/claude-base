## Зачем смотрели
Кандидат на анимации/glassmorphism/лейауты под смену дизайн-подхода klimat-pro.

## Что это технически
- **Облачный MCP-сервер.** `/ui create a navbar...` → промпт+контекст уходят в **облако 21st.dev** → возвращается React/Tailwind-компонент. **Нужен API-ключ** (21st.dev/magic/console).
- **Прайсинг:** сейчас «free during beta», но FAQ прямо про «monthly generation limit» и «upgrade your plan» → **freemium, в будущем платно + лимиты**.
- Интеграция: `npx @21st-dev/cli install claude --api-key <key>` (поддержка client `claude`), либо MCP-конфиг вручную.
- SVGL-интеграция (логотипы брендов), TS-компоненты, «inspired by 21st.dev library».

## Оценка
- **Подходит? Под условием / опционально.**
- Сильное: быстрый генератор полированных компонентов (как v0), хорош для разовых «дай красивый компонент».
- Слабое/риск:
  - **Это ТОЖЕ облако** → не приближает к цели «уйти от облачного дизайна», а меняет одно облако (Claude Design) на другое (21st.dev) + **утечка prompt/контекста** наружу (для клиентского проекта — минус).
  - Будущая платность + лимиты; beta-нестабильность; лицензия неоднозначна; активность ниже (4 мес).
  - **Стек-несовпадение:** klimat-pro = React + инлайн-стили (частично Tailwind). Magic отдаёт Tailwind/shadcn → нужна адаптация, не drop-in.
  - Перекрывается с собственной генерацией Claude + Claude Design.
- **Решение: держим в уме (опционально), НЕ ядро смены подхода. Пробовать позже, если перейдём на Tailwind/shadcn и примем облако-egress.**
