## Открытые вопросы / pending

### 🔴 Блокер: Data Link не работает

**Симптом:** AutoCAD при `DATALINK` → выбрать xlsx показывает «Невозможно создать связь с данными, так как программа Excel не найдена».

**Корень проблемы (диагностировано):**
- Office LTSC 2024 Click-to-Run **установлен** (`C:\Program Files\Microsoft Office\root\Office16\EXCEL.EXE`).
- COM-класс `Excel.Application` зарегистрирован, но **Type Library повреждена** (`TYPE_E_ELEMENTNOTFOUND` при вызове из PowerShell COM).
- В реестре ключ `HKLM\SOFTWARE\Microsoft\Office\16.0\Excel\InstallRoot` есть, **но свойство `Path` пустое** — AutoCAD читает именно его.

**Решение (требует админа):**
```powershell
Set-ItemProperty -Path "HKLM:\SOFTWARE\Microsoft\Office\16.0\Excel\InstallRoot" -Name "Path" -Value "C:\Program Files\Microsoft Office\root\Office16\"
```

**Пользователь** отправил админу запрос. **Ждём фикса.**

## Что узнал за сессию (методология)

### 1. Структура тома (по двум эталонам)

Том проектной/рабочей документации содержит:
- **Содержание тома** (`-С`)
- **Текстовая часть** (`-ТЧ`)
- **Графическая часть** (`-ГЧ`) — планы, схемы
- **Спецификация оборудования** (`-СО`) — модели/артикулы/заводы
- **Задания смежникам** (`.Д1` ... `.Дn`) — на электроснабжение, в СКС, в АПС, на отведение теплоизбытков и т.д.
- **Ведомость объёмов работ** (`.ВОР` или `-ВОР`) — монтажные работы с количествами
- **Программа пуско-наладочных работ** (`.ПНР` или `-ПНР`) — функции ПНР с количествами

Шифр документа: `<номер>-<раздел><подраздел>.<тип>` или `<номер>-<раздел><подраздел>-<тип>`. **Разделитель (точка/дефис) непоследователен** между разделами — смотреть на шифр СО раздела как образец.

### 2. Структура таблиц ПНР и ВОР

**ВОР** (9 колонок): `Поз. | № в ЛСР | Наименование работ | Ед.изм | Кол-во | Ссылка | Вес одной ед., кг | Высота установки, м | Тип монтажа`

**ПНР** (3 колонки): `Функции | Кол-во функций на единицу | Кол-во функций итого`

**Иерархия групп**:
- 1 уровень (как ЛВС-эталон): тип оборудования («Ядро ЛВС», «Точка беспроводного доступа»...)
- 2 уровня (как ТХ2-эталон): помещение → подсистема («Операционный зал №107» → «Подсистема отображения информации»)

Группа-заголовок = строка где заполнена ТОЛЬКО колонка A/C, остальные пусты.

**Формула чисел в ПНР**: `Кол-во итого = Кол-во на ед. × Кол-во устройств этого типа из ВОР`.

### 3. Откуда брать число функций

**Единого эталонного источника нет** — выводить по:
1. Логике типа оборудования (что технологически нужно делать для запуска)
2. Аналогам в похожих ПНР (особенно ЛВС/ТХ2-эталонах)
3. ГЭСНп/ФЕРп на пусконаладку (если есть прямые расценки)
4. Документации вендора

При отсутствии источника — **спрашивать** пользователя, не выдумывать. Это правило `feedback-pnr-functions-source` в memory.

### 4. Связь с DWG: Data Link

Таблицы в DWG — это **AcDbTable со стилем «КРК_С границами»** + связь `ACAD_DATALINK` на xlsx. Редактирование только в xlsx, DWG обновляется через `DATALINKUPDATE`.

Структура шаблона DWG для одного листа ПНР (ЛВС-эталон, ~82 entities в Model):
- Главный штамп: блок `*U16` (анонимный) с атрибутами `ПРОЕКТБ`, `ДОБАВКАБ`, `ДОБАВКАМ`, `ПРОЕКТМ`, `СИСТЕМА`, `ЛИСТМ`, `ЛИСТБ`, `ЛИСТОВБ`. Внутри блока — `XREF.FIO.dwg` (название системы + ФИО) и `Форма 6 ГОСТ Р 21.101—2020`.
- Шапка таблицы: блок `Шапка_СП` (×2).
- Сама таблица: AcDbTable rows×cols (для ЛВС-ПНР 8×3) с Data Link.
- MText «Программа пуско-наладочных работ» — над таблицей.
- 30+ блоков `*U17`/`*U8` на слое Defpoints — шаблоны для подшивки/малого штампа, **визуально невидимы**, не трогать.

В **ВОР** главный штамп — блок `*U19` с теми же атрибутами, но `ПРОЕКТБ` хранится **без** разделителя в конце (`25/6729-ИОС4.2`), а `ДОБАВКАБ` — с разделителем в начале (`-ВОР`). Несимметрично с ПНР.

## Технические заметки

### AutoCAD MCP (важно для следующего чата)

**Backend:** `file_ipc`. Это значит:
- `execute_lisp` **не возвращает stdout** — нужно писать результат в файл и читать.
- `drawing.open` команда МСС вернула ok, но НЕ открыла файл. **Открывать через прямой LISP**: `(vla-Open (vla-get-Documents (vlax-get-acad-object)) "C:/path/to.dwg")`.
- После `vla-Open` нужно **отдельным вызовом** активировать документ через `vla-Activate` — внутри одного LISP не сработает (MDI асинхронно).
- Закрывать активный документ нельзя — сначала переключиться на другой, потом закрыть.

**Пример паттерна (работающий):**
```lisp
; Открыть файл с русскими символами в пути (только через прямой vla-Open):
(setq r (vl-catch-all-apply 'vla-Open
  (list (vla-get-Documents (vlax-get-acad-object))
    "C:/Users/<разработчик>/Desktop/.../Программа ПНР.dwg")))

; Отдельным вызовом — активировать:
(vlax-for d (vla-get-Documents (vlax-get-acad-object))
  (if (and (= (vla-get-Name d) "Программа ПНР.dwg")
           (vl-string-search "нужное_подстроки_пути" (vla-get-FullName d)))
    (vla-Activate d)))

; Запись результата для возврата (stdout не работает в file_ipc):
(setq f (open "C:/temp/result.txt" "w"))
(write-line "result data" f)
(close f)
```

**Чтение результата**: PowerShell `Get-Content -Encoding default` (AutoLISP пишет в CP1251).

### Excel MCP vs openpyxl

Excel MCP (`mcp__excel__read_data_from_excel`) на больших таблицах раздувает JSON метаданными валидации/формул и упирается в лимит токенов. **Для больших xlsx использовать openpyxl напрямую через Python:**

```powershell
$env:PYTHONIOENCODING="utf-8"; python -c @"
import sys; sys.stdout.reconfigure(encoding='utf-8')
from openpyxl import load_workbook
wb = load_workbook(r'path.xlsx', data_only=False)
ws = wb['ВОР']
for r, row in enumerate(ws.iter_rows(values_only=True), 1):
    if any(c is not None for c in row):
        print(f'{r}:', row)
"@
```

Для маленьких (< 50 строк × 10 колонок) Excel MCP работает нормально и даёт удобный JSON.

### PDF-MCP подводный камень

`pdf_read_pages` глючит на **граничных страницах** (1 и последняя) и на single-page запросах. Для надёжного чтения PDF используется PyMuPDF (`fitz`) напрямую через Python.

**Пример:**
```powershell
$env:PYTHONIOENCODING="utf-8"; python -c @"
import fitz, sys; sys.stdout.reconfigure(encoding='utf-8')
d = fitz.open(r'path.pdf')
for i in [0, 4, 41]:  # 0-indexed
    print('=== PAGE', i+1, '===')
    print(d[i].get_text()[:5000])
"@
```
