## Где сломался (и как починил)
1. **Внутренний `\n` в ячейке xlsx ломает data-файл toolkit** → теряется объём позиции →
   verify FAILED (1 поз. из 33). Причина: `xlsx2acadtable.sanitize()` чистит символы вне
   cp1251, но не переносы строк; TAB/построчный data-файл рвётся. Обход: монки-патч
   `sanitize` (\n,\r→пробел) в UTF-8 обёртке. Записано в feedback-pending (кандидат в
   правку базового toolkit). → [[feedback_id_priemka_technique]] рядом по теме.
2. **Кириллица в argv** (имя листа/layout/стиля) бьётся при запуске из Git Bash → запускать
   toolkit через UTF-8 обёртку с хардкодом `sys.argv`, не передавать кириллицу в argv.
3. **plot_pdf не пишет файл** (плоттер не сконфигурирован) → для визуала использовать PNGOUT
   (FILEDIA=0), не plot. `get_screenshot` окна на layout даёт чёрный кадр (тёмный фон) —
   тоже PNGOUT (даёт белый фон, читаемо).

  не вводим вручную); визуал — PNGOUT + Read.
- **Anti-холодный-кэш ревьюера:** office-MCP (excel) был холодным → ревьюеру явно предписано
  читать xlsx через Bash+openpyxl, не через MCP (сработало, PASSED с реальным чтением).
- **Кандидат в харвест toolkit:** strip `\n`/`\r` в `sanitize()`; возможно — авто-обёртка
  запуска с UTF-8 argv.
