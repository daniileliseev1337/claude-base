# xlcompare (ravi-chandran)

- **URL:** https://github.com/ravi-chandran/xlcompare
- **Звёзды:** ~5
- **Последний коммит:** не виден явно из README, но релизы есть; репо считается малопопулярным, но рабочим
- **Лицензия:** MIT (зелёный свет)
- **Язык:** Python (openpyxl + difflib)

## Что делает

CLI-утилита: на вход даёт два xlsx с обязательной колонкой-идентификатором
(по умолчанию `ID`, переопределяется флагом `--id`). На выход — `diff.xlsx`:

- автофильтр сверху для быстрой выборки изменений
- удалённые ячейки — красным со зачёркиванием
- добавленные — синим
- удалённые строки — внизу красным
- настраиваемая макс. ширина столбца

Сравниваются **только общие колонки**, остальные игнорируются.

## Чем подходит под нашу задачу

- Прямо в README перечислены кейсы: **«requirements, bill of materials,
  invoices, exported databases»** — то есть автор задумывал инструмент
  под наш сценарий «было-стало в табличном виде».
- Логика «уникальный ID + общие колонки» = ровно то, что нужно для
  спецификаций (артикул/позиция как ключ).
- MIT — можно копировать код в скилл/installer без согласований.
- Простой, читаемый Python — легко форкнуть и адаптировать под наши
  колонки (производитель/модель/ед.изм./кол-во).

## Чем НЕ подходит / риски

- **Звёзд почти нет** (~5) — нарушает фильтр CLAUDE.md «≥50 stars ИЛИ
  явная репутация». Репутации автора тоже нет.
- Требует **уникальный ключ ID** во входе. В реальных спецификациях
  П-стадии часто нет стабильного артикула — позиции нумеруются заново
  в каждой версии. Придётся либо вручную проставить ID, либо вытащить
  его эвристикой (например, по «обозначение»/«тип»).
- Сравниваются только первые листы — для много-листовой спецификации
  нужен будет цикл-обёртка.
- Нет fuzzy matching — если в П указан «Hikvision DS-2CD2143G2-I», а
  в Р «HIKVISION DS2CD2143G2I», утилита посчитает это разными строками.

## Оценка: **MAYBE**

Хороший шаблон логики и идеальный кейс позиционирования, но без
fuzzy-уровня и при отсутствии стабильного ID в реальных спецификациях
П→Р в чистом виде не сработает. Брать **как референс архитектуры**
(идея «ID + общие колонки + красно-синий xlsx»), не как готовый
инструмент.
