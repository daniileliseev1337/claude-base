## Что делает

«data diff» — библиотека сравнения **таблиц** (не файлов).
Специфицирует свой собственный
[tabular diff format](https://specs.frictionlessdata.io/tabular-diff/):
формальное описание insert/delete/update на уровне ячеек с
авто-выравниванием строк и колонок.

- Поддерживает primary key (`--id`).
- Делает 3-way merge, патчи между версиями данных.
- Выходы: CSV-diff, **HTML визуализация**, цветной терминал.
- Интегрируется как diff-driver для git на csv-файлы.

## Чем подходит под нашу задачу

- 904 звезды, активный, MIT — все фильтры пройдены.
- Идея «row alignment by key + cell-level diff» = ровно про
  спецификации.
- HTML-отчёт можно сохранить как «было-стало» web-страницу для
  отправки заказчику/смежникам.
- Кросс-язык: если будем переписывать пайплайн на JS/Python — daff
  есть везде.

## Чем НЕ подходит / риски

- Формат выхода — **HTML / CSV-diff**, не xlsx. Конверсия в xlsx
  потребует дополнительной обвязки (HTML → openpyxl с цветами, либо
  парсинг diff-CSV → ручная отрисовка).
- Нет fuzzy matching (как и у всех остальных).
- Документация местами скудная, основной канал — спецификация формата.
- Питон-обёртка через pip есть, но менее обкатана чем родная Haxe/JS
  версия.

## Оценка: **MAYBE** (как альтернатива datacompy)

Сильный кандидат если приоритет — **HTML-отчёт** (можно показать
смежникам в браузере). Если приоритет — xlsx бок-о-бок, datacompy
проще встроить. Брать либо/или, не оба.
