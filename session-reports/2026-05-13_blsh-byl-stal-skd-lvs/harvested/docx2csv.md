# docx2csv (ivbeg)

- **URL:** https://github.com/ivbeg/docx2csv
- **Звёзды:** 65
- **Последний коммит:** не виден явно, признаки активности слабые
  (TravisCI в репо, нет релизов) — **под вопросом**
- **Лицензия:** BSD-3-Clause (зелёный свет)
- **Язык:** Python (python-docx + openpyxl)

## Что делает

Узкая утилита: вытаскивает **все таблицы** из docx-файла и сохраняет
их либо как несколько CSV (по одной таблице на файл), либо как один
**xlsx с несколькими листами**.

API:
- `extract_tables(filename)` → список таблиц как Python-структур;
- `extract(filename, format='xlsx'|'csv'|'xls')` → файл на диске.

## Чем подходит под нашу задачу

- Закрывает **этап парсинга**: docx-спецификация → структурированные
  данные. Это то, что у пользователя сейчас написано на PowerShell.
- BSD-3 — спокойно копируется в installer как зависимость.
- 65 звёзд — пограничный, но фильтр CLAUDE.md (≥50) пройден.
- Сосуществует с datacompy: docx2csv делает извлечение, datacompy —
  сравнение.

## Чем НЕ подходит / риски

- **Не сравнивает** ничего — это только парсер. Нужна связка с другим
  инструментом для diff'а.
- Активность под вопросом (нет релизов на PyPI новее ~2020-х). Может
  плохо работать с современными версиями python-docx.
- Не различает таблицы по семантике (где спецификация, где титульный
  блок) — всё свалится в один xlsx, нужна логика выбора нужной
  таблицы по заголовкам.
- Сложные кейсы python-docx (объединённые ячейки, вложенные таблицы)
  парсятся не всегда корректно — придётся проверять на реальной БЛШ-
  спецификации.

## Оценка: **MAYBE** (как кирпич пайплайна)

Полезен **только в связке** с движком сравнения (datacompy/daff/
xlcompare). Сам по себе задачу не решает. Если связка покажет
проблемы парсинга — заменить на ручной python-docx (15-30 строк) и
не тащить зависимость.
