# Сессия 2026-05-13 — Балансировка VRF Royal Clima + таблица ХОВС

## Тема
Анализ проекта кондиционирования общественного здания (стадия Р). Подбор баланса холодопроизводительности по двум контурам VRF Royal Clima (= Hisense Hi-FLEXi S OEM). Создание Excel-таблицы характеристик систем кондиционирования (ХОВС) по эталонному формату.

## Контекст
- Объект: общественное здание со зрительскими трибунами (≈301 место) + офисно-бытовая зона на 1+2 этажах.
- Система: 2 контура VRF Royal Clima:
  - К1: одиночный наружный блок (20 HP) на 18 внутренних блоков (1 этаж + левая часть 2 этажа — офисы, медпункт, раздевалки).
  - К2: составной наружный блок (32 HP, 2 модуля 16+18 HP) на 9 внутренних блоков (трибуны + смежные помещения).
- Хладагент R410A.

## Что было сделано
1. **Изучение PDF альбома РД.** Распознавание планов кондиционирования, схем К1/К2, дренажа, электрики. Извлечение нумерации блоков `Кx.y` со страниц «План электрических подключений» (это оказалось самый надёжный источник нумерации, потому что метки блоков подписаны рядом с проводкой 220-240 В).
2. **Экспликация помещений.** Считана с PDF AutoCAD (стадия раннего согласования, 2024 г.) — там подписаны площади и назначения помещений; в финальном альбоме РД этих данных нет.
3. **Каталоги оборудования.** Использованы:
   - `TCY12024017-Technical Catalog-Hi FLEXi S5(Global)-V2.pdf` — наружные блоки.
   - `2025-07_Hisense Indoor Units TC (R410A)-V17.pdf` — внутренние.
4. **Excel-файл «Баланс холода по помещениям.xlsx»** — 5 листов:
   - **Сводка** — баланс по контурам (CR%), статус OK/перегруз, заключение.
   - **Помещения** — детальная таблица 27 ВБ с VLOOKUP-привязкой к экспликации (площадь + назначение), нормативом, требуемой/фактической нагрузкой, запасом.
   - **Каталог Hisense** — справочник характеристик.
   - **Экспликация** — полная ведомость помещений 1+2 этажа.
   - **ХОВС** — характеристика систем кондиционирования (по эталонному формату от пользователя), наружные блоки разделены на отдельные строки для каждого модуля составного блока.
5. **Опорные рамы для НБ** — проектное предложение с расчётом несущих балок (швеллер) и виброопор. Расчёт на снеговую нагрузку III снегового района.
6. **Долговременная память:**
   - `reference_royal_clima_hisense_oem.md` — соответствие префиксов RCWT/RCS/RCC ↔ AVWT/AVS/AVC.
   - `feedback_royal_clima_composite_outdoor.md` — правило распознавания составного НБ на схемах (по магистралям и вентиляторам, не по подписи).

- WebSearch: характеристики наружных блоков AVWT-154/170/190/250/272/308FKFSA, FKF5, HKFSA.

## Итерации и ошибки

### Итерация 1: первичная привязка блоков
- **Что делал:** считал блоки по схемам К1/К2 (стр.7-8 PDF), сделал предположительную привязку к помещениям по визуальному анализу плана при 220 DPI.
- **Ошибка:** привязал блок 2.2 кВт в помещение «Галерея 309 м²» — удельная нагрузка 7 Вт/м², явный абсурд.
- **Найдено пользователем:** «Ты не верно определил помещения».
- **Исправление:** отрендерил план в 350 DPI, прочитал номера помещений у блоков точно. Обновил привязку.

### Итерация 2: расхождение количества блоков
- **Что делал:** считал 16 ВБ на К1 и 9 на К2 (всего 25) — по схеме К1.
- **Ошибка:** не учёл новую модель RCC-12CFD, добавленную в обновлённой версии PDF (10 стр), и пропустил один настенный блок на 1 этаже.
- **Найдено:** при чтении плана электрики (стр.13-14) увидел нумерацию К1.1...К1.18 (18 блоков) и новую модель RCC-12CFD (К1.14 в Судейской).
- **Исправление:** перешёл к таблице 18+9 = 27 ВБ. Добавил RCC-12CFD в Каталог.

### Итерация 3: ошибка в номерах ячеек
- **Что делал:** записал нормализованные номера помещений в неправильные строки C18-C20 вместо C25-C27.
- **Найдено:** валидатором excel-validator (CRIT-1).
- **Исправление:** восстановил исходные значения, записал в правильные позиции.

### Итерация 4: висячая мусорная строка
- **Что делал:** при первичной записи Экспликации одна строка осталась без B/C значений (потеря данных при пакетной записи openpyxl).
- **Найдено:** валидатором (CRIT-2).
- **Исправление:** unmerge ячеек, дописал значения.

### Итерация 5: дублированная площадь
- **Что делал:** в Экспликации помещению «Холл 216.1» проставил площадь = площади «Трибуны 2» (375.88 м²) — копипаст ошибка.
- **Найдено:** валидатором (CRIT-3) при проверке суммы площадей этажа (расхождение с эталоном).
- **Исправление:** оценочно поставил 91.35 м² по балансу площадей этажа (2553.10 − Σ остальные).
- **Заметка о выдумывании:** значение оценочное; в файле явно помечено «(площадь оценочная по балансу этажа, не подписана на плане)».

### Итерация 6: неправильные марки наружных блоков
- **Что делал:** взял `RCWT-154STFG` и `RCWT-250STFG` из первой версии PDF (6 страниц).
- **Ошибка:** в обновлённой версии PDF (10 страниц, дата 2026-05-13) марки изменились: К1 = `RCWT-190STFG`, К2 = `RCWT-154STFG` (подпись головного модуля).
- **Найдено пользователем:** «С чего ты взял что там именно эти наружные блоки?»
- **Исправление:** прочитал схемы К1 (стр.7) и К2 (стр.8) повторно.

### Итерация 7 (КЛЮЧЕВАЯ): составной наружный блок
- **Что делал:** прочитав «RCWT-154STFG» на схеме К2, подсчитал ΣВБ = 70.4 кВт vs НБ 45 кВт → CR = 156% → пометил «перегруз».
- **Ошибка:** не распознал, что подпись «RCWT-154STFG» относится только к головному модулю **составного** блока. Реальный НБ — `RCWT-308STFG` = 154 + 170 (32 HP, 85 кВт).
- **Признаки, которые я не использовал:**
  - Магистраль на схеме ⌀38.1 мм (для 28+ HP, а не 16 HP).
  - На рендере у НБ К2 видно 4 вентилятора (а не 2 у одиночного).
- **Найдено пользователем:** «Там два блока а вместе они идут как RCWT-308STFG».
- **Исправление:**
  - Помещения!D26:D34 → RCWT-308STFG.
  - ХОВС: вставлена новая строка, два модуля показаны отдельно с разными характеристиками (Qхол 45/50.4, патрубки 12.70/15.88).
  - В долговременную память записано правило-фидбек (`feedback_royal_clima_composite_outdoor.md`).

## Что выдумывал явно (флаги)
1. **Площадь Холла 216.1 = 91.35 м²** — оценка по балансу площадей этажа. На плане площадь этого помещения не подписана, исходный архитектурный план в папке проекта отсутствует. Если найдётся реальная площадь — пересчитать L21/L33 на листе «Сводка».
2. **Привязка блоков 1 этажа к помещениям 038, 039, 039а, 045, 047, 050** — по плану электрики; на ранних рендерах путал с 041 (Галерея). Подтверждение получено в итерации 2 после пересчёта при 350 DPI.

## Что нового нашёл/проверил
- Hisense Hi-FLEXi S5 — модельный ряд:
  - Одиночные: 8/10/12/14/16/18/20/22 HP (76/96/114/136/154/170/190/212 HKF5).
  - Двухмодульные комбинации: 24/26/28 HP (232/250/272 HKF5, 2 компрессора в одном корпусе) и составные через Y-joint (например, 308 = 154+170).
- Префикс Royal Clima `RCWT-XXX-STFG` = Hisense `AVWT-XXX-FKFSA/HKF5`. ТТХ идентичны (OEM).
- Для составного блока на схеме типично подписан **только головной модуль** — недостаточно для определения, обязательно сверять магистрали и количество вентиляторов.
- Межблочный кабель H-LINK II для Royal Clima: 2-жильный экранированный, 0.75 мм², неполярный, daisy-chain до 1000 м. Для общественных зданий — КПСЭнг(А)-FRLS 1×2×0.75.

## Инструменты, которые сработали
- `mcp__pdf-mcp__pdf_read_pages` / `pdf_render_pages` (DPI 350) — для чтения нумерации блоков и марок на схемах.
- `mcp__excel__*` — для построения многолистового файла с формулами VLOOKUP + COUNTIF-дедупликацией.
- `excel-validator` агент — 4 ре-валидации, каждый раз ловил новые ошибки: расхождения сумм, висячие строки, формулы со сдвинутыми ссылками после insert_rows.
- WebSearch — для подтверждения характеристик моделей Hisense.

## Где сломался workflow
- **`insert_rows` не сдвигает формулы автоматически.** После вставки 2 строк перед позицией 21 формулы H/I/K/L строк 26-34 остались ссылающимися на старые C24-C32 вместо C26-C34. Пришлось переписывать вручную. **На будущее:** после `insert_rows` всегда проверять формулы в сдвинутых строках валидатором.
- **`insert_rows` сдвигает merge-диапазоны нестандартно.** Заголовок «Внутренние блоки» с merge `A7:Q7` после `insert_rows(start=7)` остался в `A7:Q7`, но содержимое уехало в `A8`. Пришлось unmerge + re-merge.
- **Каталоги в папке проекта появились в середине сессии.** Сначала использовал данные из WebSearch (старый Hi-FLEXi S, серия FKFSA), потом обнаружил локальные каталоги (новая S5 HKF5) — пришлось пересчитать характеристики (RCS-28DJ: 8.0 → 8.4 кВт). **На будущее:** в начале сессии — `Glob` всей папки проекта на предмет каталогов оборудования.

## Ключевые выводы для будущих сессий
1. **Не делать поспешных выводов о перегрузе VRF-системы** при формальном CR>130%. Сначала проверить:
   - Диаметры магистралей (если ⌀38.1 при подписи «154» → составной).
   - Количество вентиляторов на 3D-рендере НБ.
   - Спросить пользователя про артикул НБ.
2. **План электрики — основной источник нумерации блоков** (с подписями К1.X, К2.X), поскольку на плане кондиционирования может не быть точных позиций.
3. **Каталоги оборудования читать с проверкой версии** (S vs S5, FKFSA vs HKF5) — параметры одинаковых моделей в разных каталогах могут отличаться (например, RCS-28DJ: 8.0 кВт в старом S, 8.4 кВт в S5).
4. **При составной таблице НБ — отдельные строки в ХОВС** для каждого модуля с разными Qхол, патрубками, шумом.
