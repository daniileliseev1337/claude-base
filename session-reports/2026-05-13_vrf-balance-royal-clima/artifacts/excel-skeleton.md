# Скелет xlsx-файла «Баланс холода + ХОВС» (шаблон)

Структура 5 листов для VRF-проекта с балансом холода по контурам и таблицей ХОВС.

## Лист «Сводка»
- Заголовок проекта + шифр + дата
- Таблица **Баланс по контурам**: Контур / Qхол НБ / Σ Qхол ВБ / CR%
- Таблица **Баланс по площади**: Контур / Σ площадь / Уд.факт / Статус
- Справочник нормативных удельных нагрузок (СП 60.13330)
- Заключение текстом

## Лист «Помещения»
Колонки A..M:
- A: № строки
- B: Этаж
- C: № помещения (ключ для VLOOKUP)
- D: Контур (наружный блок)
- E: Марка ВБ
- F: Тип ВБ
- G: Qхол, кВт
- H: Площадь, м² — формула `=IFERROR(IF(COUNTIF(C$start:Cn,Cn)>1,0,VLOOKUP(Cn,Экспликация!A:C,3,FALSE)),"")`
- I: Уд.факт., Вт/м² — `=IFERROR(G*1000/H,"")`
- J: Норматив, Вт/м² — ввод вручную (по умолчанию 100)
- K: Требуемо, кВт — `=IF(H="","",H*J/1000)`
- L: Запас, кВт — `=IF(H="","",G-K)`
- M: Назначение — `=IFERROR(VLOOKUP(Cn,Экспликация!A:B,2,FALSE),"")`

Группировка: контур 1 → итог → контур 2 → итог → общий итог.

**Особенность COUNTIF-якоря:** для каждого контура — отдельный якорь начала (`C$<начало контура>`), чтобы дубль номера помещения в одном контуре давал 0 (а не задваивал площадь), но при этом одинаковые помещения в разных контурах считались как разные.

## Лист «Каталог»
Справочные характеристики оборудования (отдельно НБ и ВБ).
- Модель / Тип / Индекс мощности / Qхол / Qтепл / N / Размер / Патрубки / Вес / Шум

## Лист «Экспликация»
- A: № помещения (текст, ключ)
- B: Назначение
- C: Площадь, м²

Заголовки этажей объединены через ширину таблицы.

## Лист «ХОВС» (Характеристика систем кондиционирования воздуха)
Эталонная структура (17 колонок):
1. Обозначение системы (К1.X / К2.X)
2. Наименование обслуживаемого помещения
3. № пом.
4. Модель
5. Тип
6. N, кВт
7. U, В/ф
8. Qхол, кВт
9. Qтепл, кВт
10. Вес, кг
11. Размер (ВхШхГ), мм
12. Диаметры присоединительных патрубков
13. L, м³/ч
14. P, Па
15. Уровень звук. давления, дБ(А)
16. Кол., шт.
17. Примечание

Группировка строк:
- Заголовок проекта (merge A1:Q1)
- Подзаголовок «Системы кондиционирования» (merge A2:Q2)
- Шапка row 3
- Подзаголовок «Наружные блоки» (merge A4:Q4)
- Строки по каждому НБ (для составного блока — отдельная строка на каждый модуль)
- Подзаголовок «Внутренние блоки» (merge)
- Строки по модели ВБ с агрегацией количества (например, RCS-07DJ — 9 шт)

## Грабли при работе через openpyxl/excel-mcp
1. После `insert_rows` формулы НЕ обновляют относительные ссылки автоматически — переписывать вручную.
2. `insert_rows` оставляет merge-диапазоны на прежних координатах — нужно unmerge + re-merge.
3. `data_only=True` возвращает None для всех формул, пока файл не открыт в Excel и не сохранён (Ctrl+S).
4. Пакетная запись массива может оставлять пустые ячейки, если строки массива не дотягиваются до количества колонок — проверять валидатором.
