## Что исполнено механически (приёмка НЕ пройдена, кроме п.1 — не проверен)
1. **Кухонные зонты v9 → v9.1 (3 семейства, стенд):** duct-коннекторы на невидимых
   хостах (Revit 2025: `ConnectorElement.CreateDuctConnector`, старого `NewDuctConnector`
   НЕТ), bbox ужат до габарита (парковка патрубков внутрь, служебные RP → not-a-ref),
   галочка «Показывать жироуловитель», controls-стрелки, un-shared вложения. Геометрия
   при флексе ВСЕХ осей + ручном смещении чинена: панели пришиты к RP, EQ-центровки
   заменены locked-парами (EQ растягивает эскиз!), низ/верх хостов формулами. Верификация
   фактами в тест-проектах: IsConnected=True, Flow-раздача, флекс-детектор ПО ТЕЛАМ.
2. **Виды по системам (боевой док):** 36 видов (18 планов + 18 3D) + 18 фильтров по
   «Имя системы» (границы списка, тёзки В1/В1с), 36 шаблонов (планы: только Filters;
   3D: + скрытие всего кроме вентиляции, аннотации-марки видимы).
3. **Альбом ИД:** 30 листов (комплект 32), формат рамки ADSK параметрами инстанса,
   планы А2 1:120, схемы А3 (каскад масштабов), порядок «планы→схемы, П→В→МО→ВЕ»,
   пачки схем без смешения групп, перенумерация двухфазная.
4. **ВОР по системам:** 14 мультикатегорийных спек (Наименование/Марка/Ед/Кол-во);
   у Multi-category НЕТ builtin «Имя системы» → зеркало в shared «ИмяСистемы» (766 эл.)
   + фильтры Contains/NotContains. Формат целых: ТОЛЬКО UnitTypeId.Fixed+Accuracy 1.0.
5. **Эталон-репликация:** владелец довёл один лист руками → все настройки (префикс имени,
   кроп, центр, выравнивания/ширины/позиция таблицы) считаны фактами и применены ко всем.
6. **Размеры длин участков** воздуховодов (132 шт; торцевые PlanarFace, <200мм скип).

## Ключевые ловушки (полные — в ZONTY_data.md v8/v9 и журнале проекта)
- DimensionType с одним именем живёт в линейном И диаметральном стиле — фильтровать
  StyleType==Linear, иначе «non-linear dimension type» (глотался молча — 3 пустых прогона).
- CopyElements view-specific аннотаций: с залоченного 3D — отказ; на планы — теги умирают
  без основы (344 «скопировано»/0 живых). Рабочий путь: IndependentTag.Create по индексу
  эталона. В UI «вставить с выравниванием» работает (слова владельца).
- Категория «Камеры» (сами 3D-виды) раздувает bbox всех 3D; CanCategoryBeHidden=false —
  при подборе масштабов мерить только нужные категории.
- MCP-таймаут 60с на тяжёлых прогонах: вывод в файл + ожидание файла циклом; версии
  скриптов сверять по mtime (был прогон старой версии при свежем Write).
- Кириллица в execute_revit_code ломается ТОЛЬКО инлайн — execfile UTF-8-файла чист.
- Орфанная транзакция от упавшего exec неубиваема GC — на плохой ветке t.Start() строго
  внутри try, восстановление только рестартом Revit.

## Процесс/уроки (главное)
- **Замечание владельца «то как проектировщик, то лишь бы отстали» — справедливо:**
  вставил ведомости на листы не проверив, что легли поверх плана; «оптимальные» пачки
  смешали группы систем; центровку делал прикидкой вместо эталона. Правило: раскладка,
  имена, порядок листов проверяются ГЛАЗАМИ с той же строгостью, что геометрия/фильтры.
- «Расставь размеры на воздуховоды» ≠ очевидно: сначала посмотреть образец/спросить
  (владелец имел в виду ДЛИНЫ участков, я поставил ширины сечений — переделка).
- Эталон-репликация (владелец доводит один лист → снять фактами → применить всем) —
  сильный паттерн совместной работы, закрепить.
- Данные ведомостей: пустые строки = незаполненный ADSK_Наименование в модели — данные
  заполняет владелец, не выдумывать.
