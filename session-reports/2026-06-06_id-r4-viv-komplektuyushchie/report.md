# Session report — ИД, том водоснабжения, устранение замечаний ТН (продолжение)

Дата: 2026-06-06. Тема: каскадная правка исполнительной документации тома внутреннего водоснабжения по замечанию технадзора (п.6 — недостающие позиции в ИД). Обезличено.

## Что сделано
1. **Закрыт п.6 ТН** (недостающие комплектующие санприборов). В каскад сдаточных документов ИД (ведомость смонтированного оборудования docx → лист ВОР по монтажу оборудования xlsx → предъявительская ведомость xlsx → сравнительный лист «было/стало») добавлены, **разнесённые по 6 помещениям**: сифоны 3 типов (10+3+4), гибкие подводки (20), угловые краны 1/2″ (26). 2. **Узел «раковины под сушилку-смеситель»** — по факту разобрано: раковин под совмещённые сушилки-смесители в томе водоснабжения нет (приёмники = мойки в столешницах, смежный раздел ПД). Решение — не добавлять в ВС, подготовить письменное пояснение технадзору.

## Метод (что сработало)
- **domain-grilling + library-first ПЕРЕД правкой.** Сверил спецификацию проекта (.doc через antiword) с текущей ведомостью — выявил, что именно отсутствует, и 3 расхождения (производитель крана проект↔паспорт; сушилка проект↔факт; два типа сифонов). Не правил вслепую — грилинг развилок с рекомендациями, числа подтвердил контрольной суммой (число приёмников = число донных клапанов).
- **Тройная сверка вводных** по требованию пользователя: исп. схема (PDF), фото пометок ТН (зум), норма/практика (делегировал norm-lookup). Норма: жёсткого норматива «2+2+1 на прибор» нет — это монтажно-комплектовочная практика (ГЭСН/ФЕР сб.17 прил.17.1 «состав комплектов санприборов»); количество по проекту. Для ответа ТН формулировать «по проекту/составу комплекта», не «норма требует N».
- **Делегирование каскада доменному агенту (id-engineer) по детальному ТЗ-артефакту**, затем независимый QA двумя ревьюерами (word-checker + excel-validator). Сработало с первого прохода (прошлый аналогичный узел делался в 2 итерации) — потому что ТЗ содержало точные точки вставки, грабли и контрольные суммы.

## Грабли / уроки
- **Windows + mingw bash + кириллица в argv Python**: `python -c "...путь с кириллицей..."` приходит «крякозябрами» (FileNotFoundError на мусорном пути). Решение — записать .py файлом (UTF-8) и запускать `python file.py`, пути хардкодить в файле (raw-string). Не передавать кириллические пути аргументами/в -c.
- **MCP excel read_data_from_excel** на больших листах превышает лимит токенов (выгрузка в файл). Читать прицельными диапазонами (заголовок + хвост), а не лист целиком.
- **merge + перенумерация в xlsx (openpyxl)**: снимать merge заголовков ДО insert, восстанавливать после со сдвигом — иначе ломается. В docx-таблице новые run наследуют 12pt от Normal — ставить 9pt явно. Эти грабли заранее прописаны в ТЗ → агент не наступил.
- **Не каждое «добавить позицию» = добавление.** Замечание «2 раковины» по факту оказалось вопросом разграничения разделов (мойки в столешницах — не наш том) → ответ пояснением, а не правкой. Факт-сверка до действия сэкономила ошибочную правку официального документа.
