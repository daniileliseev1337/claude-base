# 2026-07-13 · Восстановление сессий десктоп-приложения после смены аккаунта

## Задача
Пользователь не видел прошлые сессии в «Recents» десктоп-приложения Claude.
Требование: вернуть видимость всех сессий.

## Диагноз (путь с ложными следами — важно для будущего)
1. Транскрипты `~/.claude/projects/*.jsonl` целы (152 сессии, 19 проектов) —
   потери данных не было ни в какой момент.
2. **Ложный след №1:** карта `projects` в `~/.claude.json` (9 корней проектов
   отсутствовали — починил, но на Recents это не влияет; влияет только на
   регистрацию проектов).
3. **Ложный след №2:** «серверный список, привязанный к аккаунту» — смена
   аккаунта действительно коррелировала, но механизм локальный.
4. **Верный ответ нашёлся grep'ом по СВОИМ транскриптам** (эпизод 09.07 в
   journal-записи прошлой сессии): реестр Recents =
   `%APPDATA%\Claude\claude-code-sessions\<UUID аккаунта>\<UUID организации>\local_*.json`,
   по папке на аккаунт. При ротации аккаунтов список пустеет.

## Фикс
- Слияние файлов реестра из папок двух прежних аккаунтов в папку текущего
  (103 уникальных, при дублях — свежайший, бэкап `.bak-2026-07-13`): список
  вернулся после перезапуска приложения (подтверждено пользователем).
- Закреплено скриптом `~/.claude/scripts/restore-ccd-sessions.ps1`:
  автоопределение аккаунта из `~/.claude.json`, идемпотентен, бэкап только
  при реальном копировании. Проверен боевым и повторным запуском.
- `cleanupPeriodDays` в `~/.claude/settings.json`: 365 → 3650 (транскрипты
  старше лимита удаляются на старте — защита истории).
- Memory-заметка `ccd_recents_registry_account_switch.md` + строка в MEMORY.md.
- Пользователю выдан «Каталог сессий (восстановление).md» (все 152 сессии:
  дата/название/ID для `claude --resume`).

## Где сломался / уроки
- Три ложных следа до верного ответа. Урок: при «уже чинили такое» —
  ПЕРВЫМ делом grep по своим транскриптам/журналам (`~/.claude/projects`,
  session-reports), а не диагностика с нуля.
- Write-tool пишет .ps1 в UTF-8 без BOM → PowerShell 5.1 читает кириллицу как
  ANSI и падает парсером. Фикс: пересохранить с BOM.
- Справочный агент по докам подтвердил: у десктопа/веба/VS Code РАЗДЕЛЬНАЯ
  история сессий; `claude --resume` работает по локальным файлам независимо
  от аккаунта (Ctrl+A — все проекты).
