## Где сломался / уроки
- **Связь id→комната по координатам НЕ работает** (coord-match дал 82/114; лампы в расчётном
  файле звались `LAMP` ≠ `LUM`/`LUS`). Сработала связь по ПОРЯДКУ постановки
  (`placed_ids[k]` ↔ `placements[k].room`, verify 114/114). Урок: для связи факт↔расчёт
  использовать стабильный порядок, не координаты (приборы двигались/округлялись).
- **`doc.Regenerate()` вне транзакции бросает «Modification forbidden»** — убрать, commit сам
  регенерирует. Паттерн: t1 повороты→commit→t2 прижатие по bbox (bbox уже актуальный).
- **Origin семейств РАЗНЫЙ** (RCU задняя грань, лампа верхняя, вызывное центр) → прижимать
  ПО get_BoundingBox, не по origin.
- **Снап к ближайшей стене не разносит наложения** (приборы из одной точки v3) → пост-обработка
  «веер вдоль стены 700мм» по группам (комната, стена).
- **Правило «наибольший сегмент = стена с дверью» ломается на кабинетах/комнатах с 2+ дверями**
  (RCU кабинета сел не туда) → для не-палат нужен явный выбор входной двери.
- **Karpathy #5 в деле:** поспешил назвать размещение RCU «корректным» — пользователь поправил
  (на косяке, не на цельной стене). Визуальная проверка глазами + правки пользователя по ходу
  ловят то, что алгоритм считает «ок».
