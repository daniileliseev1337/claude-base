# Сессия 2026-07-09 (ночь) — зонты v9: MEP-обвязка трёх семейств

## Что сделал
Подхватил незавершённую сессию по кухонным вытяжным зонтам (Revit-семейства, 3 шт.)
по замечаниям владельца с теста в живой модели боевого проекта <объект>:
1. **Коннекторы к родителю.** Duct-коннекторы (4×ExhaustAir/Round) сидели в shared-вложении
   «патрубок» — у родителя ConnectorManager=None, воздуховод не цеплялся. Построены
   невидимые столбики-хосты 60×60 в родителе (EQ-димы к RP патрубков — позиции живые при
   флексе), на торцы — `ConnectorElement.CreateDuctConnector`, Ø/Flow ассоциированы.
   Доказано API-подключением воздуховода: `IsConnected=True`.
2. **Вложение un-shared** (FAMILY_SHARED=0 из родителя): субкомпоненты 0, из спецификации
   исчезло.
3. **bbox = ровно корпус.** Две причины раздува: парковка невидимых патрубков ЗА торцом
   (формула переписана — парковка внутрь, (2i−1)/8·L) и анкерные анти-флип RP со «Слабой»
   ссылкой за габаритом (ELEM_REFERENCE_NAME=12). 2400→1500 у пристенного.
4. **Островной:** торцевые стенки юбки и Fine-обвязка ЖУ не следовали за длиной
   (найдено флекс-детектором: снапшот bbox тел до/после +400) — пришиты align + locked dim.
5. **Новое:** instance Yes/No «Показывать жироуловитель» (29/36 тел начинки), controls-стрелки
   (DoubleHorizontal/VerticalArrow).
6. Верификация каждого семейства в фоновом тест-проекте: 4 коннектора / Flow при N=2 /
   bbox / флекс +1000 / галочка / duct-connect. Рендеры глазами. Продукция перезаписана,
   перезагружена в боевой проект, осиротевшее семейство патрубка удалено.

## Где сломался / уроки
1. **Оборванный субагент** (API error на старте) → второй заход упёрся в орфанную
   транзакцию: `t.Start()` вне try + AttributeError (в Revit 2025 удалён
   `FamilyCreate.NewDuctConnector`). GC-recovery такой лик НЕ снимает (нативный стек держит
   GCHandle) — только ручное закрытие дока/рестарт Revit. Правило: Start строго внутри try,
   скрипты самодостаточные (commit/rollback + save + close в одном execfile).
2. **Канал против кириллицы-ловушки:** писать .py в UTF-8 на диск, в exec слать одну
   ASCII-строку `execfile(...)`, результаты — в UTF-8 файл (переживает 60с-таймаут
   транспорта; таймаут ≠ смерть — Revit дорабатывает, проверять app.Documents).
   Инлайн-кириллица один раз стрельнула даже у меня (найм файла не сматчился).
3. **RollBack транзакции с подключённой MEP-сетью в фоновом доке поднимает модальный
   диалог у пользователя** («семейство подключено к сети») — одноразовые тест-доки
   коммитить и Close(False), не откатывать.
4. **Формула instance-параметров не садится на type-параметр** (SetFormula молча
   отклоняется по очереди вариантов юнитов — казалось, дело в юнитах, а дело в instance).
5. **bbox инстанса растягивают named-RP со Strong/Weak Reference** — неочевидный источник,
   нашёлся сверкой «в семействе за габарит никто не выходит, а bbox больше».
6. ControlShape enum: только *Arrow-формы (никаких Flip/Rotate из старых доков).
7. IronPython диспатчит перегрузку Duct.Create в (XYZ,XYZ) — соединять ConnectTo.
8. Дубли параметров на инстансах («ИмяСистемы», взрывозащита) оказались параметрами
   ПРОЕКТА (ADSK-шаблон, биндинг на категорию) — диагноз важнее правки: семейства чистые.

(пользователь видит каждый шаг, рендер-итерации). Все правки на стейдж-копиях C:\rvt_stage,
продакшн перезаписан только после полной верификации. Активный док пользователя не трогался
(кроме финальной перезагрузки семейств по мандату «реализуй тут»).
