# Сессия 2026-07-07 — экспертиза раздела РД СКС/ЛВС/ТФ (жёсткий нормоконтроль)

## Задача
Выступить в роли строгой экспертизы по одному тому РД <шифр> (СКС/ЛВС/ТФ, подземный паркинг + КПП, Изм.1): проверить нормативные ссылки на актуальность/полноту, изучить ТЧ/ГЧ/СО/КЖ, найти расхождения, проверить оформление по ГОСТ (штампы/форматы/рамки), этап 2 — Word-ведомость замечаний со скринами.

## Что сделал
- PDF 31 стр.: постраничная выгрузка текста (PyMuPDF) в scratchpad + карта листов.
- Fan-out ×5 (sonnet): norm-lookup ×2 (ФЗ/СП + все ГОСТ), auditor (внутренние сверки), audit-rd-section (СПДС), general-purpose (пробив артикулов через exa).
- Сам: рендеры штампов (текстовый слой штампов ВРЁТ про порядок фамилий — подписантов сверять только по рендерам!), пиксельный промер рамок/штампов (185×55 проверка), grep-сверки (запись об Изм. на каждом листе, шифры смежных томов, редакции ГОСТ в тексте), подъём точных пунктов ГОСТ Р 21.101-2020 из локальной библиотеки.
- Итог: ведомость 63 замечания (8 критических) → Word с 40 скринами (fitz-кропы по search/clip) → ревью word-checker + auditor.

## Ключевые находки по тому (обезличенно)
- «Листов 9» при 19 листах; запись «Зам. <разрешение>» отсутствует на 14 листах при «замене всех листов» (п.7.3.13 требует на каждом листе при автоматизированном способе).
- Дефицит кабеля UTP против КЖ: −50% в одной спецификации, −19% в другой; блок розеток из задания ЭОМ отсутствует в СО; «1 категория» vs «первая особая» (ТЧ↔приложение).
- Служебный текст «Смените формат на A0А» в графе «Формат»; нестандартный размер листа 1642×1189 (≠А0×2).
- Ссылки на спецификации ЧУЖОГО тома (.С2/.С3/.С4 корпуса 01) на фасадах; закладки PDF из тома 03; примеры маркировки из главного корпуса.
- Изготовители в СО недостоверны: климатический шкаф ТША511 — не Yealink (реально «АМАДОН»); SFP FH-* — не Элтекс (Fang Hang); «Камертон-Н» — не «Телда» (Коралл-Телеком).
- Нормативка: ГОСТ Р 21.101-2020 заменён ГОСТ Р 21.101-2026 с 01.04.2026 (том выпущен 04.2026!); ГОСТ Р 53246-2025 существует (с 01.02.2026), но в томе указан со старым названием 2008 и в указаниях листов — ссылка на 53246-2008; «Правила ТБ при эксплуатации электроустановок потребителей» давно отменены (ПОТЭУ 903н).

## Уроки / грабли
1. **Текстовый слой штампов AutoCAD-PDF перемешивает порядок строк** — роли подписантов из extraction НЕ выводить; только рендер. (Ранняя гипотеза «разные ГИП» оказалась ложной.)
2. **PyMuPDF и /Rotate:** в этом PDF get_pixmap(matrix=Matrix(zoom)) даёт читаемый (displayed) вид, а search_for/get_text возвращают координаты в системе MediaBox — для кропа по найденному тексту на повёрнутых страницах надо rect * page.rotation_matrix. Диагноз ставить эмпирически (4 кандидатных преобразования + мини-кропы), теория по докам путает.
3. **Порядок умножения матриц fitz:** Matrix(zoom)*rotation_matrix ≠ rotation_matrix*Matrix(zoom) — translate в rotation рассчитан на немасштабированную страницу.
4. **Grep-детектор изменений**: наличие «<номер разрешения>» в page_NN.txt — дешёвый сплошной контроль внесения изменений по всем листам; «Зам» ловит «Взам.» (регистронезависимость PowerShell -match).
5. **Схемы с «…»-свёрткой**: grep-отсутствие маркировки ≠ отсутствие устройства — сначала рендер зоны (замечание про «пропущенные .02» смягчено после визуальной проверки).
6. **Вектор-замер рамок эвристикой «длинные линии» ломается** на AutoCAD-пучках штрихов; пиксельные проекции бинаризованного рендера надёжнее.
7. Локальная библиотека СПДС окупилась: точные цитаты 21.101 (4.2.7 «.СО», 4.3.3 разделы ведомости, 7.3.13 «Зам. на каждом листе», прил. Л подпись нормоконтролёра, табл. Г.1 марки по 21.1703/21.703) — сняли одно ложное замечание агента (ведомость спецификаций не нужна для самостоятельных СО — стр.30 ГОСТ) и укрепили остальные.
8. Субагенты-ревьюеры без Write — план «пиши в файл» не работает; итоговые отчёты агентов сохранял сам из их ответов.
