# Сессия 2026-07-06 — сверка папки сертификатов со спецификацией, добыча недостающих документов

## Задача
Спецификация сценического оборудования (<объект>, 178 позиций: звук/свет/механика сцены/видео/кабель) + папка с 51 файлом документов качества. Требовалось: проанализировать имеющиеся документы, привязать к позициям, переименовать файлы по содержимому, найти недостающие документы в вебе, выдать дашборд-сводку.

## Что сделано
1. Спецификация .doc → docx (Word COM) → CSV (python-docx), 178 позиций с ID.
2. MD5-хэши: 51 файл, 8 групп побайтовых дубликатов (имена файлов массово не соответствовали содержимому).
3. Workflow-анализ 42 уникальных файлов (агент-анализатор + скептик-верификатор на файл, пакетами по 8) + 19 перекрёстных повторов. Выявлены: просроченные документы (отказное 2017, письмо ВНИИПО до 12.2025, декларация 2024), 5 писем о замене, файл-подмена (письмо ИЭК под именем «Коробка У-995»).
4. Переименование 51/51 по шаблону «Тип документа - продукция (марка)», дубли с пометкой «(копия N)».
5. Веб-поиск по 36 бренд-группам: фаза без браузера (exa/fetch/WebSearch + curl) параллельно, браузерная фаза (реестр ФСА) строго последовательно. Скачано и проверено 37 PDF (декларации ЕАЭС с выписками ФСА, сертификаты, отказные).
6. Ревью auditor: NOT PASSED → 22 корректировки (ложные привязки метизов к декларациям на электронику, просрочки без флагов, расхождение артикулов) → все применены → повторный контроль PASSED.
7. Дашборд (Artifact + локальная HTML-копия в папке): 64 «в папке» / 75 «найдено в вебе» / 39 «не найдено» (у каждой — причина и куда идти), секция замечаний для ИД.

## Источники добычи
Реестр ФСА pub.fsa.gov.ru (официальные выписки; API отдаёт PDF по цепочке unloading→jobId, но поиск за WAF — нужен браузер), сайты производителей (equivalent.pro, nhk-aricon.ru, dkc.ru), дистрибьюторы (Сантехкомплект, ВсеИнструменты, ЭТМ, Аудиомания и др. — по подсказке пользователя), агрегатор декларации-соответствия.рус.

## Где сломалось / уроки
- **Workflow-инструмент терял результаты** (output 0 байт, journal обрывался, транскрипты поздних пакетов отсутствовали) — 3 раза. Лечение: обязательный Write JSON-записи каждым агентом на диск + восстановление из транскриптов (tool_use StructuredOutput). → memory `workflow_results_lost_bug`.
- **Одновременный старт 40+ агентов** ловил серверный rate-limit — пакеты по 6-8 с ретраем.
- **Playwright MCP один на всех**: параллельные агенты закрывали друг другу браузер, скачивания падали GUID-файлами в `.playwright-mcp`/`Downloads` — браузерная фаза только последовательно. → memory `reference_playwright_mcp_single_browser`.
- **Фидбек пользователя**: документы качества почти всегда в открытом доступе у крупных дистрибьюторов — не списывать «партийные» позиции в «запрос поставщику» без поиска. → memory `feedback_supplier_sites_have_docs`.
- 5-часовой лимит подписки убил один прогон целиком — диск-записи агентов позволили продолжить без потерь.
- Аудитор поймал классику: декларация на акустику ≠ документ на металлоконструкции подвеса; РЭ ≠ декларация; общебрендовая декларация на мебель — натяжка при позиции «под заказ».
