## Методики (3 новых профиля)

### ИОС3.3 ОВ3 — Кондиционирование воздуха (Часть 3 ОВиК)
- Отопление/вентиляция в томе НЕ разрабатываются. Только кондиционирование (К1–К12: VRF LG Multi V + сплит + прецизионные NED) + дренаж конденсата (Д1–Д11).
- **Нет готового xlsx-DataLink** — спецификация (284 поз.) извлечена из PDF `25.6729-ИОС3.3.pdf` через PyMuPDF `find_tables()`.
- **Единица учёта ПНР — система.** VRF: 10 на систему + 4×внутр.блок. Сплит: 11. Прецизионный: 12. Дренаж: 3×система. Итого 308+22+24+33 = **387**.
- Нормы: СП 60.13330.2020 разд.15 + ГОСТ Р 59972-2021 разд.9.9.3. Испытания фреоновых контуров — по докам LG/NED.

### ИОС3.1 ОВ1 — Отопление (Часть 1 ОВиК)
- Готовый xlsx `2026.05.27_ОВ1.xlsx` (Лист1 — спецификация по этажам). 183 позиции, 3 этажа (1/2/2А).
- Водяная двухтрубная + лучевая, поэтажные коллекторы, 90-70°C, + электронагреватели.
- **Единица учёта ПНР — этаж + блоки.** Общая наладка 6 + балансировка 1×клапан + коллектор 2× + прибор 1× + электро 3×. По этажам: 53+97+85 = **235**.
- Нормы: СП 73.13330.2016 разд.7 (опрессовка п.7.3.1: 1,5×раб., ≥0,2 МПа; тепловое испытание п.7.3.5; промывка п.6.1.13) + СП 60.13330.2020 п.6.2.12 (балансировка). ⚠ Разд.15 СП 60 — это ВК, НЕ отопление.

### ИОС2.1 ВК — Водоснабжение и канализация

- **Единица учёта ПНР — 2 системы + блоки.** Водоснабж.: наладка 8 + арматура 1×35 + санприборы с автоматикой 2×60 + оборудование 4×8 = 195. Канализация: 6. Итого **201**.

## Общий pipeline (стабилен, отшлифован)

1. Разведка папки (Get-ChildItem). Если есть «(выверенная спека)» → искать готовый xlsx-источник (DataLink); если «(для загрузки в МГЭ)» → спека в PDF.
2. Чтение источника: готовый xlsx через openpyxl ИЛИ спека из PDF через PyMuPDF `find_tables()` (НЕ pdf-mcp). Карта PDF через `ov3_map.py` (env PDFPATH).
3. Подсчёт элементов по типам/этажам/системам (для удельных чисел ПНР).
4. AskUserQuestion: охват + принцип подсчёта ПНР + удельные числа + шифр + детализация ВОР. **Профиль ПНР для каждого типа систем РАЗНЫЙ — не выдумывать, согласовывать.**
5. norm-lookup ДО генерации docx (правило #7). Локальная библиотека `Claude_Library\spds\` богаче чем казалось — СП 73, СП 60, СП 30, ГОСТ Р 59972 все нашлись. cntd.ru недоступен (WEBFETCH_BLOCKED).
6. Сборка xlsx: копия шаблона СКУД (`5_ИОС4.5_СКУД...\ПНР и ВОРы\ПНР и ВОРы СКУД.xlsx`) через Copy-Item → парс источника + маппинг работ + snap стилей из строк 2/3 + delete_rows + перезапись с наследованием стилей.
7. Сборка docx обоснования через python-docx (5 разделов: общие сведения / нормативная база / методика / сводная таблица / исключения + подписи).
8. Валидация parallel: excel-validator + word-checker (Task tool, в одном сообщении).

Скрипты-шаблоны в `~\AppData\Local\Temp\`: `ov3_map.py`, `ov3_read.py`, `ov3_tables.py`, `build_ov3.py`+`_docx.py`, `build_ov1.py`+`_docx.py`, `build_vk.py`+`_docx.py`, `count_ov1.py`, `count_vk.py`, `inspect_skud.py`, `inspect_ov1_xlsx.py`.
