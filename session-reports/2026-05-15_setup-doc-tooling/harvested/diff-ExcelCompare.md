## Что делает
CLI-утилита для cell-by-cell сравнения двух Excel/ODS-книг. Выдаёт список
изменённых/добавленных/удалённых ячеек с координатами и значениями. По
умолчанию сравнивает **формулы**, не вычисленные значения — это редкое и
ценное поведение.

## Какой именно diff
- **Cell-level** с указанием Sheet!Cell.
- **Formula-aware:** по умолчанию сравнивает текст формулы (`=A1+B2`), флаг
  `--diff_ignore_formulas` переключает на сравнение результатов.
- Игнорирует форматирование, макросы, условное форматирование.
- Кросс-формат (xlsx vs ods, xls vs xlsx) — полезно при миграциях.

## Как подключить
```
brew install excel-compare              # macOS
# или скачать JAR с GitHub Releases:
java -jar excel_cmp.jar v1.xlsx v2.xlsx
```
На Windows — JAR + JRE 1.8+.

## Подводные камни
- **Последний коммит 2022-04** — флаг по нашему фильтру (>12 мес от
  2026-05-15). Проект «зрелый, но замороженный».
- Требует Java 1.8+ на Windows-машине пользователя. У нас Java может
  отсутствовать → потенциальная boilerplate-установка.
- Нет MCP-обёртки, нет Python API — только CLI.
- Не показывает дельту условного форматирования, цветов, charts — только
  значения/формулы ячеек.
- Альтернатива чисто на Python: наш `excel-helper` скилл уже умеет это через
  openpyxl (см. CLAUDE.md). Возможно проще доточить скилл, чем тащить Java.

## Вердикт
ФЛАГ. Брать только если задача — сравнить две большие книги с формулами и
формат вывода CLI устроит. Иначе — расширить `excel-helper` скилл
openpyxl-логикой (это и без харвеста дешевле). Java-зависимость — минус
на Windows без админ-прав.
