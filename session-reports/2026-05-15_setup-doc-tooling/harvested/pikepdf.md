## Что делает

Прямой Python-биндинг к **qpdf** — индустриальному C++ инструменту работы с PDF на уровне структуры (objects, streams, references). Не «генератор PDF», а именно редактор существующих файлов.

- **Merge/Split/Page reorder** — на уровне объектов, без рендеринга
- **AcroForm fields** — чтение и запись fillable полей
- **Annotations** — манипуляция объектами аннотаций (комментарии, штампы, polygon clouds)
- **Metadata** — title, author, XMP
- **Encryption/Decryption** — пароли, права доступа
- **Linearization** — fast-web-view оптимизация
- **Object inspection** — низкоуровневая работа с дерева объектов PDF

## Почему подходит нам

Пользователь уже упомянул, что pikepdf «мы знаем». Я включаю как **базовый fallback** — если ни один MCP-сервер из списка не пройдёт тест, минимальный жизнеспособный путь:

1. Установить `pikepdf` в общий venv для скриптов (`~/scripts/.venv`).
2. Написать 3-5 атомарных функций (merge / split / fill_form / add_watermark / replace_page).
3. Обернуть их в **наш собственный** MCP-сервер через FastMCP (один файл, ~150 строк) и подключить как `pdf-edit`.

Это путь Karpathy-простоты: контролируем код, нет внешних зависимостей кроме одной библиотеки, без cloud, без Docker, без Java.

## Как подключить

```powershell
pip install pikepdf
```

Минимальный пример (merge):
```python
import pikepdf
with pikepdf.Pdf.new() as out:
    for src in ["a.pdf", "b.pdf"]:
        with pikepdf.open(src) as pdf:
            out.pages.extend(pdf.pages)
    out.save("merged.pdf")
```

## Подводные камни

- **MPL-2.0** — лицензия "weak copyleft": если мы модифицируем сам pikepdf и распространяем — модификации остаются под MPL. Использование как зависимости (import) — никаких обязательств, можно в любых проектах включая closed-source.
- **Не умеет рендерить контент** — добавить текст/изображение «поверх страницы» через pikepdf неудобно. Для рендеринга нового контента нужен **reportlab** (генерация overlay-PDF) + pikepdf (наложение). Это известный паттерн.
- **AcroForm fill** работает, но **не flatten'ит** автоматически — после заполнения форма остаётся редактируемой. Для «зафризить» поля — отдельные манипуляции с `/Ff` флагами.
- На Windows ставится через wheel без проблем, дополнительных DLL не нужно.

## Вердикт

✅ **Базовый кирпич** для собственного MCP, если внешние варианты не зайдут. Зрелая, активно поддерживаемая, лицензионно чистая библиотека. Стандарт индустрии вместе с PyMuPDF (но PyMuPDF — AGPL, для нас токсично).
