## Что делает
Redlining-движок для AI-агентов: переводит DOCX в Markdown, принимает правки от LLM
и записывает их обратно в DOCX как Native Word Track Changes (настоящие
`<w:ins>`/`<w:del>` в XML), не ломая форматирование, таблицы, нумерацию.
Содержит CLI-команду `adeu diff v1.docx v2.docx` — генерирует визуальный
diff между двумя версиями документа как третий DOCX с трек-чейнджами.

## Какой именно diff
- **Структурный + текстовый** через Markdown-репрезентацию документа.
- **Track Changes для docx:** ДА, нативные — открываются в MS Word как обычные
  правки автора (Accept/Reject).
- Не предназначен для xlsx/pdf.
- Сохраняет сложное форматирование (стили, таблицы, нумерация) — это его ключевая
  фишка vs. наивный python-docx diff.

## Как подключить
MCP (Claude Desktop / Claude Code):
```
uvx adeu init
```
Python SDK:
```
pip install uv
pip install adeu
```
CLI:
```
adeu diff v1.docx v2.docx -o redline.docx
```

## Подводные камни
- Требует Python 3.12+ (Node.js 20+ как альтернатива).
- Только DOCX, никаких xlsx/pdf.
- 75 stars — не топовый, но автор активен (релиз сегодня, 2026-05-15) и
  позиционирует именно как MCP для AI-агентов — это редкое попадание в нишу.
- Markdown-промежуточный слой = возможна потеря экзотических конструкций
  (текстовые рамки, сложные формулы Word).
