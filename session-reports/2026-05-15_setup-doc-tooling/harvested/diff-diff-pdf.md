## Что делает
Де-факто стандарт визуального сравнения PDF. Рендерит обе версии через Poppler
+ Cairo, сравнивает попиксельно, помечает различия цветом. Может сохранить
diff обратно как PDF (`--output-diff`) с подсвеченными областями каждой
страницы.

## Какой именно diff
- **Визуальный, попиксельный**, page-by-page.
- Не делает текстовый diff (для текста — pdf-mcp + redlines/стандартный diff).
- Не различает «текст переместился на 1px» от «текст изменился» — это и
  плюс (ловит всё), и минус (шумный на разной растеризации).

## Как подключить
Windows:
```
choco install diff-pdf
```
Скачать ZIP с релизов:
https://github.com/vslavik/diff-pdf/releases
Использование:
```
diff-pdf --output-diff=diff.pdf v1.pdf v2.pdf
diff-pdf --view v1.pdf v2.pdf
```
Возвращает exit code 0/1 — пригодно для скриптов в Claude Code.

## Подводные камни
- **GPL-2.0** — вирусность лицензии. Использовать как **внешний бинарь**
  (через subprocess) можно свободно, **встраивать код** в наши скиллы/
  агенты — нельзя без перехода в GPL. Пользоваться им из Claude Code через
  CLI вызов безопасно.
- **«Not being actively developed»** — но релиз был 2026-03-28, минорные
  фиксы есть. Mark из README — больше предупреждение «не ждите фич», чем
  «брошен».
- Требует одинаковую размерность страниц у обоих PDF (иначе много шумных
  диффов).
- На Windows может конфликтовать с другими установками wxWidgets, если
  компилировать из исходников. Chocolatey-бинарь работает «из коробки».

## Вердикт
ВЗЯТЬ как внешний CLI (subprocess). Лучший в нише визуального PDF diff,
есть Chocolatey-пакет под Windows, активен. GPL допустим при вызове как
process, **не копировать код** в наши скиллы.
