## Где сломался / уроки
- **★ КРИТИЧЕСКАЯ ошибка id: перепутал RCU с лампой.** При «исправлении краевых» взял id
  811228/811229 как лампы боксов 028/029, но **811228 = RCU 6.08.028** (принятый в v7!), а лампа
  028 — это 811227. Сдвинул принятый RCU на позицию/высоту лампы (в коридор, Z2300). Поймал
  ФИНАЛЬНЫМ программным аудитом наложений (<120мм) → 811227 LUM ↔ 811228 RCU = 85мм. Восстановил
  RCU 028 на v7-координаты (зеркально к целому RCU 029 811230). **Урок: финальный аудит наложений
  + Z-аномалий ОБЯЗАТЕЛЕН — он один поймал перепутанный id; визуальные кадры мелкого масштаба
  ошибку не показали.** Связь id→тип брать СТРОГО из placed_actual_v3 (там type), id соседних
  приборов одного помещения идут парами LUM/RCU — легко перепутать.
- **Лампа: купол зависит от ряда.** Эталон блока3: верхний ряд fo(0,1) купол вниз (коридор южнее),
  нижний fo(0,−1) купол вверх (коридор севернее) — купол ВСЕГДА в коридор. Дефолт fo(0,1) верен
  только для верхнего ряда; нижний надо вращать 180°. (Память ранее «не поворачивать» — неверно
  для зеркального нижнего ряда.)
- **Диагноз ошибки v4 ламп:** лампа брала X двери СВОЕЙ ПАЛАТЫ и опускалась в коридор → стояла над
  дверью палаты, а не над дверью БОКСА. Правильно — группировка ламп бокса по двери тамбура в коридор.
- **BOM в json из IronPython** → читать `codecs.open(...,'utf-8-sig')`; `json.loads(open().read())`
  падает «No JSON object».
- **Структура бокса блок1:** палаты(север) → двери → санузлы(Уборная+Душевая) → Тамбур → дверь в
  Коридор 6.00.0xx. Лампы на стене коридора (граница y≈±83612/94757).
