# session-reports/

Наши отчёты о реальных рабочих сессиях Claude Code. Каждая закончившаяся
содержательная сессия может оставить здесь подпапку с разбором.

> **Важно про имя.** Папка раньше называлась `sessions/`, но Claude Code
> сам использует `~/.claude/sessions/` для своих transient JSON-файлов
> (session state). Переименовали в `session-reports/` чтобы не пересекалось.

## Формат

```
session-reports/
└── <YYYY-MM-DD>_<краткое-имя-задачи>/
    ├── report.md            # что делал, какие источники, где сломал, итерации
    ├── harvested/
    │   ├── <repo-name>.md   # оценка одного внешнего репо
    │   └── ...
    └── artifacts/           # обезличенные образцы (опционально)
```

## Дисциплина

- Имя подпапки в kebab-case, без ФИО / шифров / адресов проектов.
- В `report.md` — обезличенные ссылки на источники (`<шифр проекта>`, `<строительный объект>`).
- В `artifacts/` — **только** обезличенные образцы. Если артефакт нельзя обезличить — не кладём.
- Push после завершения сессии (через auto-push hook или вручную).

## Зачем

База для аналитической работы пользователя на ноутбуке: разбираем что
Claude делал на других ПК, какие инструменты находил, что работало/не
работало, какие уроки извлечь для агентской системы.
