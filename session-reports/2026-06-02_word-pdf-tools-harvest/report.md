## Арка 1 — PRO context budget (CLAUDE.md mojibake)

**Проблема:** PRO-юзерам (окно 200K) контекста хватало на ~2 промпта.
**Корень (доказан):** `~/.claude/CLAUDE.md` хранился как double-encoded mojibake
(UTF8→CP1251→UTF8). Кириллица-mojibake токенизировалась в 3-4× хуже. Файл = ~34k
токенов = **17% PRO-окна** одним файлом. memory/agents/skills были чистыми.
**Фикс:** переписан в чистом UTF-8 + де-водизация (64KB→10KB, 489→111 строк),
reference-блоки → `memory/*.md` (cascade). Устранено противоречие 8/9 MCP.
**Доказательство:** before/after на рабочем ПК (Sonnet, окно 200K): 60.6k/30% →
29.4k/15%, освободилось ~31k = 15% окна. Коммит 53657e7. Лечение команде = `/sync-base`.
**Урок:** auto-push сработал сам (раньше думал — придержу push; hook запушил).

## Арка 2 — PDF-вектор (Inkscape) — ВЕРИФИЦИРОВАНО

**Контекст:** все прошлые подходы к правке штампов чертежей проваливались (surgery,
белые заливки; AutoCAD — отдельный проект, пользователь его вынес из скоупа).
**Harvest:** turnkey свободного нет; PDFDancer/Nutrient = cloud/платно (off-limit).
Выбран Inkscape (GPL, локально).
**Валидация на РЕАЛЬНОМ чертеже К-7:** «Внутренний импорт» падал на плотном векторе
(mesh-штриховка = тысячи объектов) — но открылся по одной странице; объекты (заливки/
текст/линии) живые, удаляются и двигаются. Экспорт PDF: рендером подтверждено —
**штамп удалён начисто (не белая заливка), чертёж цел, 0 растра, не раздулось.**
**Вшито:** pdf-helper (метод Inkscape, c3e72e5) + установка в /sync-base предупреждение.
**Открыто:** батч/многостраничность (CLI inkex), генерализация.
**Урок:** не капать непроверенные решения. Доказал примитив рендером до внедрения.

## Арка 3 — Word docx

**Harvest:** дыры в инструментах НЕТ (docx редактируем). Кандидаты: docx-plus
(style-cascade), loadfix/python-docx, OfficeIMO (.NET, 461★). Все MIT.
**Реальный кейс:** отчёт 2026-06-01_docx-blanks-failures — корни совпали с жалобами:
пересборка с нуля → шапка исчезла/вниз («переворачивает»); переписывание абзацев →
затёрло табы/подчёркивания («underline сбивает»); сдача без рендера («сказал исправил»).
**Вшито:** word-helper (секция «Правка существующего docx» + предупреждение у вредного
паттерна) + memory/reference_docx_editing_failures.md. Коммит 68b079a.
**Вывод:** фикс методологический + verify-гейт, не новый инструмент.
