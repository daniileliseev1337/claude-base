## Где сломался / грабли
- **«PDF file not found» на живом файле с «ё» в имени** («…счёта.pdf», Glob файл видит).
  Первичный диагноз «pdf-mcp падает на ё» ОПРОВЕРГНУТ экспериментом (пары NFC/NFD
  тест-PDF): pdf-mcp открывает NFC-«ё»/«Ё»/«й» без проблем; корень — **NFD-имя из
  внешнего источника** (Сбер-онлайн отдал `е`+U+0308), модель эмитит только NFC,
  NTFS не нормализует → путь честно не существует. Задокументировано: anti-patterns
  A3.10 + doc-extract/pdf-edit «Ловушки»; исходный файл переименован в NFC.
  Мета-урок: LLM физически не может напечатать NFD-литерал — только wildcard/chr().
- pdf_search: параметр называется `path`, не `pdf_path` (стоило 1 ретрая).
- word-MCP кириллические пути F:\ читает без проблем.
- **Ложный дедуп lxml в собственной read-back-верификации** (2-я итерация): «голый»
  `seen.add(id(cell._tc))` без удержания ссылок — GC освобождает эфемерный прокси,
  id переиспользуется, живые ячейки отсеиваются как «дубли» → verify «терял» строки
  таблицы, файл при этом был цел. Диагноз — по артефакту (диагностический дамп),
  не по коду. Фикс: для проверки наличия подстрок дедуп не нужен. Ловушка дописана
  в word-helper §Ловушки-8 (дедуп корректен только с `uniq.append(cell)`).
- **PermissionError при пересохранении**: артефакт был открыт в Word у пользователя.
  Фикс в генераторе: fallback-сохранение « v2» + автоудаление устаревшего v2 при
  успешном каноническом сейве. Пользователя не дёргали «закрой файл».

## Уроки
- Источники противоречили друг другу в 3 местах (отчество исполнителя в обосновании
  vs паспорт; срок «5 недель» в тексте vs «3 недели» в таблице того же документа;
  «около 20 листов» vs арифметическая сумма таблицы 22). Verifiable-first + грилинг
  вскрыли все три ДО генерации; в артефакт ушли подтверждённые значения, владельцу
  перечислены ошибки исходника для правки.

  другого объекта. Расхождения источников решать по свежести/официальности, отличия
  от образца — явным списком в ТЗ ревьюеров (иначе ложные замечания).
- Спавн ревьюеров с ТЗ-ограничениями и путями источников отработал чисто: 0 ложных
  замечаний, оба гейта пройдены с первого прогона.
