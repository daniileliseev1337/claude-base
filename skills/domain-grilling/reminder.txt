⚠ domain-grilling — доменная строй-задача (грилинг по умолчанию ВКЛ, opt-out).

Запрос — строй-задача (спецификация / ВОР / расчёт / подбор / смета / КП / УПД / письмо / акт-ИД / ответ экспертизе), где домысливание вводных = ошибка в документе.

ДЕЙСТВУЙ ПО УМОЛЧАНИЮ (НЕ спрашивай разрешения через AskUserQuestion):
- Вводные НЕПОЛНЫ (не хватает объекта/шифра, источника данных, чисел, формата выхода, норм, границ задачи) → СРАЗУ начинай domain-grilling: сначала прочитай SKILL.md скилла, затем вопросы ПО ОДНОМУ, к каждому своя рекомендация, library/norm-first (что выводится из норм/проекта/эталона — находишь сам, не спрашиваешь). Грилишь до разрешения всех веток, ПОТОМ работаешь.
- Пользователь отменяет грилинг ЯВНОЙ фразой («без грилинга», «это мелочь», «вводные полные», «сразу к задаче») → идёшь обычным маршрутом (роутинг → агент).
- Вводные ЯВНО полны / задача однозначна → грилинг не нужен, сразу к роутингу.

Не вываливай 10 вопросов разом (по одному). Не грили ради грилинга на тривиалке.
