# Извлечение из PDF: маршрутизация по типу листа

**Лидера-инструмента НЕТ. Выбор — по типу страницы. Проверяй текст-слой ПЕРВЫМ (дёшево, локально, без облака).**

## Дерево решений
1. **Есть текст-слой?** `len(fitz.open(pdf)[i].get_text().strip()) > 50`
   - **ДА, обычный документ** → **pymupdf** (`fitz.get_text()`): ~0.03 с, чисто, локально, 0 облака.
     НЕ гнать текст-слой в MinerU-облако/docling — в сотни раз медленнее без выгоды.
   - **ДА, вектор-чертёж (CAD-экспорт)** → **pymupdf** берёт выноски/экспликацию/штамп текстом.
     ⚠ «битый cmap»: часть выносок встроенным чертёжным шрифтом → каша. Если каша — добить OCR (docling-OCR-ветка чище текст-слоя на таких).
   - **НЕТ (скан)** → OCR (п.2).
2. **Скан без текст-слоя** → OCR, но чистого победителя НЕТ:
   - **MinerU cloud** (pipeline+ru): то отлично (декларация), то теряет тело таблицы (сертификат). Латинизирует латинские бренды (GEBERIT→GЕВЕRТ).
   - **docling** (локально, EasyOCR ru): берёт таблицы, но систематические русские OCR-ошибки (и↔н). Надёжнее по ПОКРЫТИЮ таблиц.
   - **обязательна ручная сверка** количеств/артикулов после OCR.
3. **Рукопись** → провал у ВСЕХ OCR (кириллический курсив не по зубам ни EasyOCR, ни MinerU). Только ручной ввод / спец-HTR. Даты обычно распознаются.
4. **Геометрия чертежа** (стены/оси/линии/взаимное расположение оборудования) → документ-парсеры НЕ берут (сваливают в одну картинку-подложку). Нужен CAD-слой: `autocad-mcp` / скилл `cad-reader`.
5. **Мета чертежа-PDF** (штамп, текстовые блоки с bbox, слои `/OCProperties`, помещения по легенде) → `tools/extract_drawing_meta.py` (`extract_text_blocks`, `find_stamp_data`) + справка `references/drawings.md`.

## Инструменты (tools)
- **pymupdf (fitz)** — рабочая лошадка текст-слоя. Локально, всегда. ~90% текста ИД сюда.
- **MinerU cloud** — `python ~/.claude/skills/doc-extract/tools/mineru_extract.py <pdf> --lang ru` (pipeline+ru — **НЕ vlm**, vlm калечит кириллицу; авто-сплит томов >180 стр на батчи + склейка; токен из `~/.claude.json`). MCP-сервер `mineru` (дефолт pipeline).
  **Приватность:** облако mineru.net (КНР) — НЕ для конфиденциального ИД без явного разрешения владельца.
- **docling** — локально, без лимитов/токена (подстраховка для неавторизованных / больших томов). ⚠ НЕ drop-in на Win+кириллица: нужен junction для ASCII-пути Python + transformers≥5.
- **EasyOCR (ru+en, torch) — рабочий локальный OCR-baseline** (torch переваривает кириллицу в пути). Для сканов на этой машине — брать ЕГО. ⚠ **paddleocr на Win+кириллица НЕ работает:** 3.5 = PIR-краш, 2.9 = C++ paddle не открывает модель под не-ASCII путём `~/.paddleocr` (то же, что ломало docling); лечится только ASCII-junction для model-home. Вывод: OCR-baseline = **easyocr**, не paddleocr.

## Лимиты MinerU cloud
Precision (с бесплатным токеном): ≤200 МБ, ~200–600 стр/файл, ~1000–2000 стр/день. Без токена (Flash): 10 МБ / 20 стр. Большие тома → хелпер режет на батчи автоматически.
