## Статус приёмки (revit-testbed, 2026-07-03 — ФИНАЛ: ПРИНЯТЬ ✓)

Живой прогон на стенде (Revit 2025 + pyRevit Routes MCP). **ВЕРДИКТ: ПРИНЯТЬ.**
Рабочая связка базы: **скилл (JSON по подписке) → validate_family.py → наш MCP-executor**
(IronPython через Revit-Connector). **Аддон FamFab НЕ нужен** — наш executor его заменил
и превзошёл (снял оба ограничения оригинала: не нужен API-ключ, достижима настоящая
параметричность).

Чек-лист BIM-менеджера К7 закрыт **8/8** на эталоне `cube_etalon.rfa` — вся лестница
доказана целиком: живые габариты (NewAlignment + NewDimension/FamilyLabel, EQ-симметрия) →
формулы (Base_Area, Cube_Volume) → материал по параметру → видимость Yes/No + детализация →
lookup-CSV (S/M/L) → shared-параметры по GUID из корпоративного ФОП2021_ADSK_К7 → каталог
типоразмеров → **программная смена категории** (Generic→Оборудование, начинка пережила) +
Room Calculation Point + **MEP трубный коннектор** (CreatePipeConnector, диаметр от параметра).

⚠ Честные caveat'ы (не «всё идеально»): (1) `.adsklib` библиотека материалов К7 подключается
ТОЛЬКО через UI Диспетчера материалов — API загрузки нет; после подключения те же параметры
принимают К7-материалы. (2) Coarse-вариант геометрии статичен (alignments конфликтуют с
кубовыми, 0/4) — вариантная геометрия по детализации работает, но её Coarse-бокс не
параметричен. Открытый вопрос владельцу: перевод шаблона на К7-ФОП (roadmap В7).

### Уроки executor (для развития — сверх 5+3 ловушек ниже)
- `Regenerate()` — только внутри транзакции.
- Revit лочит CSV после `ImportSizeTable` (реимпорт — под новым именем или после закрытия).
- `size_lookup`: ключ поиска — в управляющем столбце №2+ (НЕ первый!); канонические типы
  заголовков (`##LENGTH##MILLIMETERS`, `##OTHER##`, `##NUMBER##GENERAL`); дефолт длины с
  единицами («100 мм»).
- Модальные диалоги Revit (вкл. забытое окно типоразмеров) блокируют MCP-мост → таймаут 60с,
  транзакция откатывается целиком.

## Известные ловушки (из полного чтения кода, 2026-07-02)

1. **Кириллица в именах параметров ломает формулы** — regex обоих парсеров латиница-only.
   Геометрические параметры именовать латиницей (Width/Depth/Height); русские ADSK — справочным слоем.
2. **`constraints` мертва в executor** — генерятся и валидируются, но NewAlignment не создаются;
   параметричность держится только на expressions в offset'ах.
3. **Ошибка выражения → молча 0** (C#-executor) — опечатка в имени параметра = тихо неверная
   геометрия (см. anti-patterns.md «молчаливый 0»). Проверять размеры результата глазами.
4. **Нет:** shared parameters (ADSK — только постпроцессом), MEP-коннекторов, назначения materials;
   `l_shape` не реализован в executor (только в схеме/viewer).
5. API-ключ аддона хранится plain-text в `%AppData%\ArchSmarter\FamFab\FamFab.json`.

Дополнено живой приёмкой 2026-07-03 (сверх чтения кода):

6. **BYOK-гейт блокирует даже офлайн-путь Load JSON** — аддон проверяет ключ ДО окна
   (дефект UX; обход — фиктивный ключ). С нашим MCP-executor неактуально.
7. **«Параметры-визитки»:** v1 не создаёт ни alignment, ни labeled dimensions — параметры
   НЕ ведут геометрию (constraints мертвы + выражения вычислены в числа при генерации).
   README-обещание «parametric family» фактически = «family с параметрами». Настоящая
   параметричность — только нашим v1.1-путём (alignment + labeled dims).
8. **Нормаль стандартной RP «Центр (Спереди/Сзади)» = −Y** — экструзия «вперёд» требует
   отрицательного EndOffset (флип). Ловушка для любого executor.
