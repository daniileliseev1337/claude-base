# pnr-vor-helper

## Когда подключаться

Когда нужно собрать для раздела ПД/РД комплект **ПНР + ВОР**: `ПНР и ВОРы <раздел>.xlsx`
(лист ВОР + лист ПНР) + `Обоснование функций ПНР <раздел>.docx`, с валидацией перед
сдачей (в т.ч. в экспертизу/МГЭ). Источник методологии — серия из 15 комплектов
одного гос-объекта (стадия П), 2026-06-01.

**Не путать:** [[spec-writer]] делает спецификации оборудования (.СО). Здесь — ПНР+ВОР.
Старые доменные кейсы пусконаладки: [[2026-05-07-pnr-ventilation]],
[[2026-05-08_pnr-cooling-9vs8]].

## Структура таблиц (см. `tools/table_structure.md`)

- **ВОР** (9 колонок): `Поз. | № в ЛСР | Наименование работ | Ед.изм | Кол-во | Ссылка |
  Вес одной ед., кг | Высота установки, м | Тип монтажа`.
- **ПНР** (3 колонки): `Функции | Кол-во функций на единицу | Кол-во функций итого`.
- **Формула ПНР:** `Кол-во итого = Кол-во на ед. × Кол-во устройств этого типа из ВОР`.
- **Группа-заголовок** = строка, где заполнена только колонка C (ВОР) / A (ПНР),
  остальные пусты. Иерархия: 1 уровень (по типам оборудования) или 2 уровня
  (помещение → подсистема).

## 5 методических профилей ПНР (см. `tools/pnr_profiles.md`)

**Профиль ПНР для каждого типа систем РАЗНЫЙ — не выдумывать, согласовывать.**

1. **Слаботочка** (СКС, ОС, СОТ, СКУД, СПС, СОУЭ, АУГПТ) — единица = адресное
   устройство, функции поштучно (2–12 ф/устр.). ЗИП: в ВОР с пометкой, в ПНР без.
2. **Инженерные ОВ/ВК** — единица = система/этаж, не устройство. Вентиляция ~8 ф/система
   (СП 73.13330.2016) + клапаны; отопление этаж+блоки; ВК по системам.
3. **Электрика (ЭОМ)** — единица = ЛИНИЯ и АППАРАТ (НЕ «щит»). Число линий — точно из
   кабельного журнала. ГЭСНп 81-05-01.
4. **Водяное пожаротушение (ВПВ/АУВПТ)** — испытания + наладка полевых устройств.
   ГЭСНп 81-05-02 Сб.2.
5. **Оснащение (ТХ)** — ВОР на всё, ПНР только на **активное** оборудование (мебель/
   ИТ за ДИТ пусконаладки не имеют).

## Pipeline (9 шагов — см. `tools/pipeline_checklist.md`)

1. **Разведка** папки (`Get-ChildItem`) + `pdf_info`. «(выверенная спека)» → готовый
   xlsx-источник DataLink; «(для загрузки в МГЭ)» → спека в PDF.
2. **Чтение источника:** готовый xlsx → **openpyxl**; спека в PDF → **PyMuPDF
   `find_tables()`** (НЕ pdf-mcp — глючит на граничных страницах; НЕ excel MCP на
   больших — token-лимит, см. [[excel-helper]] ловушка 9).
3. **Подсчёт элементов** по типам/этажам/системам (для удельных чисел ПНР).
4. **AskUserQuestion** — 4 вопроса: охват (ВОР+ПНР или только ПНР) + принцип подсчёта +
   удельные числа функций + ЗИП/специфика. Профиль зависит от типа системы.
5. **norm-lookup ДО генерации docx** (CLAUDE.md правило #7). Не цитировать СП/ГОСТ/ГЭСНп
   по памяти. Локальная библиотека норм богаче, чем кажется.
6. **Сборка xlsx:** копия шаблона-эталона → парс источника + маппинг работ +
   наследование стилей + перезапись (openpyxl). Стиль нейтральный, без синего ([[excel-helper]]).
7. **Сборка docx обоснования** (python-docx): 5 разделов — общие сведения / нормативная
   база / методика / сводная таблица с ИТОГО / исключения + подписи.
8. **Parallel-валидация:** `excel-validator` + `word-checker` через Task tool в одном
   сообщении. До PASSED — не двигаться. Prompt валидатору про ЗИП (см. `tools/validator_prompts.md`).
9. **Memory** — обновить если узнал новое, без дублей.

## Связь с DWG (если нужны чертежи ПНР/ВОР)

Таблицы в DWG — **AcDbTable + `ACAD_DATALINK`** на xlsx-источник. Правка только в xlsx,
DWG обновляется `DATALINKUPDATE`. Перепривязку DataLink оставлять пользователю в AutoCAD UI
(через MCP создавать DataLink — багоопасно). AutoCAD-работа — см. [[2026-05-21_acad-com-cookbook]].

⚠ **Блокер DataLink** (наблюдался): AutoCAD не видит Excel из-за пустого
`HKLM\…\Office\16.0\Excel\InstallRoot\Path` (Office LTSC 2024 + AutoCAD 2024). Фикс
реестра требует прав админа.

## Что НЕ делать (сводные уроки)

1. Не цитировать нормы/числа функций «по памяти» — norm-lookup + аналог из похожего раздела.
2. Не считать ПНР по количеству С ЗИП; не вычитать ЗИП из ВОР.
3. Не объединять дубли позиций спецификации без сверки с проектом (бывают разные места установки).
4. Не дублировать в ПНР строки разных исполнений с идентичным составом функций.
5. Не выдумывать профиль ПНР нового типа систем — согласовывать через AskUserQuestion.
6. При сомнении в объёме — анализ + расширение, не защита первой версии (кейс ОВ2: 216→401).
