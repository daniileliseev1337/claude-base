# Чек-лист pipeline ПНР+ВОР (9 шагов)

Последний пункт почти всегда — верификация (Karpathy #4). До PASSED не двигаться.

- [ ] **1. Разведка.** `Get-ChildItem -Recurse` папки раздела + `pdf_info`/`pdf_get_toc`.
      «(выверенная спека)» → искать готовый xlsx-источник DataLink;
      «(для загрузки в МГЭ)» → спека в PDF.
- [ ] **2. Чтение источника.** Готовый xlsx → **openpyxl**. Спека в PDF → **PyMuPDF
      `find_tables()`** (НЕ pdf-mcp на граничных страницах; НЕ excel MCP на больших —
      token-лимит). Обрабатывать ВСЕ таблицы страницы (штамп-таблица бывает крупнее).
- [ ] **3. Подсчёт элементов** по типам/этажам/системам (для удельных чисел ПНР).
      Парсинг .СО: наименование = первая непустая ячейка после кода (не max-len);
      включать единицы м.п.; стоп-лист поставщиков.
- [ ] **4. AskUserQuestion** (4 вопроса):
      1) охват — только ПНР или ВОР+ПНР в одной книге?
      2) принцип подсчёта ПНР (по типам / по помещениям-подсистемам)?
      3) удельные числа функций для каждого типа (профиль из `pnr_profiles.md`)?
      4) ЗИП / специфика раздела?
- [ ] **5. norm-lookup ДО docx** (CLAUDE.md #7). СП/ГОСТ/ГЭСНп — только через агента
      norm-lookup, не по памяти. Локальная библиотека норм (Я.Диск) — основной путь.
- [ ] **6. Сборка xlsx** (openpyxl): копия шаблона-эталона → парс источника + маппинг
      работ + наследование стилей + перезапись. Стиль нейтральный (excel-helper).
      Read-back verification после save.
- [ ] **7. Сборка docx обоснования** (python-docx): 5 разделов — общие сведения /
      нормативная база / методика / сводная таблица с ИТОГО / исключения + подписи.
      ⚠ Heading-стили (не Normal) — иначе автооглавление не строится. Метаданные docx
      (author/дата) заполнить, не оставлять `python-docx`/2013-12-23.
- [ ] **8. Parallel-валидация** — `excel-validator` + `word-checker` в ОДНОМ сообщении
      (Task tool). Prompt про ЗИП — см. `validator_prompts.md`. До PASSED не двигаться.
- [ ] **9. Memory** — обновить, если узнал новое; без дублей.

## Что НЕ делать

1. Нормы/числа функций «по памяти» — нет (norm-lookup + аналог).
2. ПНР по количеству С ЗИП — нет; ЗИП из ВОР не вычитать.
3. Объединять дубли позиций без сверки с проектом — нет (бывают разные места установки).
4. Дублировать в ПНР строки разных исполнений с идентичным составом — нет.
5. excel MCP на больших xlsx / pdf-mcp на граничных страницах — нет (openpyxl / PyMuPDF).
