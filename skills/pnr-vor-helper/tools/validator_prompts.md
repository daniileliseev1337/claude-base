# Prompt'ы валидаторам (excel-validator + word-checker)

Спавнить параллельно (одно сообщение, два Task-вызова). Прямая реализация
anti-patterns A1.1 (делегация с verify-критерием) и A9.1 (строгий failure-mode).

## excel-validator (для `ПНР и ВОРы <раздел>.xlsx`)

```
Read-only валидация <путь>.xlsx. Лист ВОР (9 кол.) и лист ПНР (3 кол.). Проверь:
1) Структуру: ВОР = Поз/№ЛСР/Наименование/Ед.изм/Кол-во/Ссылка/Вес/Высота/Тип монтажа;
   ПНР = Функции/Кол-во на ед./Кол-во итого.
2) Арифметику ПНР: для каждой строки  C = B × (кол-во устройств типа из ВОР).
   ВАЖНО (методическая оговорка): ПНР считается БЕЗ ЗИП. Если в ВОР указано
   «X к монтажу + Y ЗИП», эталон для ПНР = X (к монтажу), НЕ (X+Y). Расхождение
   C = B×X — это НЕ ошибка, это правило. NOT PASSED ставить только если C ≠ B×(к монтажу).
3) Группы-заголовки: только колонка A (ПНР) / C (ВОР) заполнена, прочие пусты.
4) Типы данных в колонках количеств — числа, не текст. Нет #REF!/#DIV/0!.
5) Кодировка кириллицы.
Верни PASSED/NOT PASSED + явный список проверенных пунктов с CRITICAL/WARN/INFO.
```

## word-checker (для `Обоснование функций ПНР <раздел>.docx`)

```
Read-only ревью <путь>.docx. Проверь:
1) Структуру: 5 разделов — общие сведения / нормативная база / методика подсчёта /
   сводная таблица функций с ИТОГО / исключения.
2) Сводная таблица: суммы по группам сходятся с ИТОГО (арифметика счётчиков).
3) Нормативные ссылки (СП/ГОСТ/ГЭСНп) — НЕ проверять номер «на глаз», но отметить
   любые подозрительные склейки названий/номеров для перепроверки через norm-lookup
   (был кейс склейки ГОСТ Р 59636/59638 по памяти).
4) Незаполненные плейсхолдеры ({{}}, <впишите>, подписанты-заглушки).
5) Heading-стили (не Normal) для автооглавления; метаданные (не python-docx/2013).
Верни PASSED/NOT PASSED + явный список проверенных пунктов с CRITICAL/WARN/INFO.
```

## Известные «ложные» вердикты (методически разрешены)

- **ЗИП-развилка:** excel-validator ждёт `C = B × N(с ЗИП)` → формальный NOT PASSED.
  Это не ошибка файла, ПНР без ЗИП. Разрешается уточнением prompt'а (выше).
- **Дубль позиции:** может оказаться не дублем, а разными местами установки — сверять
  с PDF проекта перед объединением.
