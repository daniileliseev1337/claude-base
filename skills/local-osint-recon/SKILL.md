# local-osint-recon

Обёртка над OSINT/pentest-арсеналом в **Kali WSL** для рабочих задач фирмы:
**due-diligence контрагентов** и **анализ присланных документов** (пассив), плюс доступ ко
всему арсеналу для авторизованных active/offensive-задач.

> Актуальный инвентарь: `C:\Users\Даниил\Desktop\osint-arsenal-revizia_2026-07-01.md` (вне git).
> Этот скилл — ЛИЧНЫЙ (`local-`), в claude-base не переносить.

## Граница (жёстко, но не блокирует)

- **Пассив** (due-diligence, поддомены passive, метаданные, ники/почты по открытым данным) — гейта НЕТ.
- **Active/offensive** (сканы портов, перебор путей, эксплуатация, брутфорс, фишинг-прокси) —
  `recon.py` печатает предупреждение про **ст. 272–274 УК РФ** и НЕ запускает без флага `--authorized`.
  **Порядок для Claude:** показать пользователю предупреждение + цель → получить явное «да, цель моя/авторизована»
  → только тогда повторить вызов с `--authorized`. Не запускать по чужим целям молча.
- Не писать пошаговых инструкций атак на конкретные чужие цели; не помогать с mass-targeting / DoS.

## Как звать

```
python "C:\Users\Даниил\.claude\skills\local-osint-recon\tools\recon.py" <команда> [аргументы]
```

Диспетчер сам решает: user vs root (go-инструменты — только `-u root`), пассив vs гейт, конвертит
Windows-путь файла в `/mnt/c/...`, отдаёт результат. Требует запущенного WSL (поднимется сам при первом вызове).

### Команды-рецепты (пассив, без гейта)

| Команда | Что делает | Инструменты |
|---|---|---|
| `dd-domain <домен>` | Due-diligence по домену контрагента: поддомены + фейк-домены + DNS | dnstwist, sublist3r, dnsrecon, subfinder(root) |
| `dd-email <email>` | Где засветилась почта / регистрации | holehe, socialscan |
| `dd-user <ник>` | Аккаунты по нику в соцсетях | maigret |
| `meta <путь-к-файлу>` | Метаданные документа (антифрод сканов паспортов/сертификатов) | exiftool |
| `list` | Показать реестр инструментов по категориям | — |

### Произвольный инструмент

```
recon.py run <tool> [-- args...]          # пассив запустится; active/offensive потребует --authorized
recon.py run nmap --authorized -- -sV 10.0.0.1
```

## Ловушки (проверено ревизией 2026-07-01)

1. **go-инструменты только от root** (`subfinder`, `nuclei`, `gobuster`, `ffuf`, `amass`, `waybackurls`,
   `naabu`, `gospider`, `gowitness`, `Modlishka` …) — `recon.py` зовёт их `wsl -d kali -u root -- /root/go/bin/<t>`.
2. **`theHarvester` НЕ установлен** — для почт/сотрудников использовать `bbot`, `osrframework-cli`, `crosslinked`.
3. **API-ключей нет** — `shodan`/`censys`/`greynoise`/`zoomeye` вернут пусто; `recon.py` их помечает `needs_key` и пропускает с сообщением.
4. **Egress = Cloudflare WARP (иностранный).** Для стыка с росс. госреестрами (ЕГРЮЛ/АРШИН/pub.fsa
   в связке с due-diligence) — либо выключить WARP, либо скилл [[ru-gov-access]]. Иностранные источники — напрямую.
5. **Кодировка** — вывод читать UTF-8 (кириллица в предупреждениях); `recon.py` уже ставит `PYTHONIOENCODING`.
