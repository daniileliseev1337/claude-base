# local-video-digest

Claude не «смотрит» видео нативно. Этот скилл даёт два канала контекста сразу:
**кадры** (контактный лист с таймкодами) + **речь** (транскрипт по таймкодам) →
читаешь оба и кросс-ссылаешься по времени. Закадровое пояснение часто важнее кадров.

## Как использовать

1. Запустить инструмент:
   ```
   python ~/.claude/skills/local-video-digest/tools/video_digest.py "<видео>"
   ```
   Опции: `--model small|medium|large-v3` (RU точнее на medium/large-v3, дольше),
   `--frames N`, `--lang ru`, `--device auto|cuda|cpu`, `--no-audio`.
2. Прочитать выход в `<видео>_digest/`:
   - `digest.md` — сводка + полная речь;
   - `transcript.md` — речь с таймкодами `[mm:ss]`;
   - `contact_sheet.jpg` — кадры, на каждом таймкод (Read tool).
3. Кросс-ссылка: нашёл в речи «здесь двутавр 25» на `[00:19]` → смотри кадр с меткой 00:19.

## Слой Tools

`tools/video_digest.py` — детерминированный конвейер (Karpathy «код в tools/»):
ffmpeg извлекает кадры по интервалу (с таймкодами) → Pillow собирает контактный лист;
ffmpeg извлекает аудио 16к моно → faster-whisper транскрибирует.

**Зависимости** (поставлены 2026-06-23): ffmpeg в `C:\ProgramData\K7-tools\ffmpeg\bin`
(или PATH), `faster-whisper`, `Pillow`. Модель Whisper качается с HF Hub при первом
запуске (корп-прокси проходит, как у paddleocr).

## Заметки / грабли

- `device=cpu` — фоллбэк по умолчанию (для GPU нужны CUDA12+cuDNN9; на RTX 3060 ставится
  отдельно, тогда `--device cuda` быстрее). На ~2 мин видео CPU+small ~1-2 мин.
- Точность RU: `small` норм для смысла; для терминов/цифр (диаметры, марки) — `medium`/`large-v3`.
- Речь с объекта бывает с матом — **передавать технический смысл нейтрально**, не цитировать брань.
- Будущее (v2, при касании): кадры по детекту смены сцены (`select='gt(scene,0.3)'`) вместо интервала.
