# handoff-to-new-chat — Codex

Адаптация методологии этого скилла под сессионную механику Codex CLI/desktop.
Формат пакета и шаги сборки не меняются — см. основной `SKILL.md`; здесь
только то, что отличается на стороне Codex.

## Индикатор перегруза

Codex показывает прямое число, а не эвристику: доля контекста в футере
(`N% context left`) и команда `/status` с разбивкой по токенам. Ориентир для
handoff — это число, а не proxy-метрики основного скилла (счётчики
Read/Agent/turns калиброваны под Claude Code, где такого прямого индикатора
модели не видно, и на Codex не переносятся).

Пороги:

| Уровень | Заполнение контекста | Действие |
|---|---|---|
| Предупреждение | от ~60% | дисциплина контекста (файлы вместо истории чата), лёгкий путь при удобном моменте |
| Обязательный handoff | от ~80% | собрать полный пакет — сессия приближается к порогу авто-компакции (270000 токенов из ~272K эффективного окна) |

Запас между 80% (≈218K) и порогом авто-компакции (270K) нужен для того,
чтобы handoff состоялся по решению, а не потому что авто-компакция уже
сжала историю и часть ранних решений стала недоступна.

## Лёгкий путь

`/compact` сжимает контекст на месте, без смены чата. Команда доступна в
CLI; в desktop-приложении Codex ручной команды нет — там работает только
автоматическая компакция по достижении порога 270K.

`/fork` (либо двойной Esc в интерфейсе) открывает ветку от текущей или
более ранней точки истории — инструмент для исследования альтернативного
решения без потери текущей версии, а не для разгрузки контекста: история в
новой ветке та же, объём не сокращается.

Лёгкий путь предпочтителен на уровне предупреждения, когда сессия идёт в
CLI и задача не подошла к естественной границе шага.

## Полный handoff

Формат пакета не меняется — TL;DR, состояние артефактов, открытые вопросы,
промпт для нового чата (см. основной `SKILL.md`, шаги 1-5). Меняется только
механизм переноса: `/new` открывает чистый чат (в отличие от `codex resume`,
который продолжает тот же тред с уже накопленным контекстом), туда
вставляется собранный пакет первым сообщением.

Размер пакета зависит от подписки, тем же принципом, что FULL/LITE у Claude:

- **Plus** (меньшее окно, рутинные задачи на Luna, см.
  `codex-layer/AGENTS.codex.md` §«Профили лимитов») — короткий пакет,
  детали остаются в `report.md` на диске, подтягиваются по необходимости.
- **Pro x5** (эффективное окно ~272K) — пакет полнее, но всё равно без
  построчных листингов и истории итераций: потолок ниже, чем у Claude MAX
  (до 1M), экономия актуальна и здесь.

Подписку уточнять так же, как в шаге 1b основного скилла, если она не
известна из контекста сессии.

## Прямого аналога нет

Ни одна штатная команда Codex не собирает такой пакет самостоятельно:
`codex resume` продолжает тред без сокращения, `/fork` ветвит с той же
историей, `/compact` сжимает на месте. Комбинация «собрать пакет → `/new` →
вставить» — единственный способ дать сессии Codex прицельный чистый старт,
поэтому методология этого скилла остаётся нужной и здесь.

## Соответствие понятий

| Наше понятие | Codex |
|---|---|
| Proxy-эвристики перегруза (Read/Agent/turns) | прямое число: футер `N% context left`, `/status` |
| Порог "отметить про себя" (~60%, эвристика) | ~60% заполнения (прямое число) |
| Порог proactive handoff (~70%, эвристика) | ~80% заполнения — обязательный handoff |
| `/compact` Claude Code | `/compact`, доступно только в CLI |
| Продолжение сессии в этом же чате | `codex resume [--last]` |
| Ветвление без потери текущей версии | `/fork` / двойной Esc |
| Смена темы → новый чат | `/new` |
| Полный handoff-пакет (TL;DR + промпт + report.md) | не встроено — пакет вставляется вручную после `/new` |
| Подписка PRO → пакет LITE | Plus → короткий пакет |
| Подписка MAX x5/x20 → пакет FULL | Pro x5 → пакет полнее (потолок ~272K, не 1M) |
| `session-reports/.../report.md` на диске | тот же путь, тот же принцип cascade loading |
