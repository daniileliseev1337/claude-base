# Пример: сверка комплекта ВОР слаботочки против СО (12 подразделов)

> Обезличено. Кейс 2026-06-23. Объект → «<объект, больница>». ФИО → роли.

## Задача
Перепроверить комплект из 12 ВОР (ведомостей объёмов работ, по подразделам ИОС5.1–5.12)
против исходных спецификаций оборудования (СО) в проектных PDF. Прошлая сессия собирала
ВОР, читая СО ВИЗУАЛЬНЫМ РЕНДЕРОМ — и пропустила однотипные ошибки количеств.

## Что сделал движок
1. `co_verify --pdf <СО.pdf> --json <data.json>` по каждому из 12.
   Результат: 11 подразделов — 0 расхождений; 1 подраздел (ЛВС/СКС, 546 позиций) —
   **9 расхождений кол-ва**: патч-корды F/O и модули SFP+ 10G в нескольких узлах
   стояли «4», в СО = «2».
2. Каждое из 9 расхождений **адъюдицировано визуально** по конкретной странице PDF
   (рендер только спорной строки) — все 9 подтвердились (PDF=2).
3. Правка данных (4→2), пересборка xlsx, повторный `co_verify` → 0. Замкнул цепочку
   PDF → json → xlsx (отдельная сверка xlsx==json подтвердила генератор).

## Уроки кейса
- **9/9 расхождений нашёл text-layer, не глаз.** Более того: при ручной сверке я сам
  один раз прочитал «16» и «25» там, где text-layer (и два независимых ревьюера) дали
  «10» и «20». Для векторного PDF извлечение из text-layer авторитетнее визуального чтения.
- **Опечатка в самом PDF:** в одном узле патч-корд напечатан с чужим номером позиции —
  экстрактор честно прочитал кривой ярлык и дал ложный «дубль». Лечится гибридным
  матчингом (позиция + название), а не доверием к номеру.
- **Безномерные строки:** реальная позиция (монтажный шнур, 128 шт) шла в PDF с пустым
  номером — нельзя фильтровать строки по наличию номера, фильтруй по наличию количества.
- **Расширение на штамп/мету** ловит другой класс: geo-сверка текста штампа показала,
  что старая редакция ПД содержала ошибочный объект «Балашиха» (исправлен на «Северодонецк»
  в новой) — это уже не СО, а сверка текста штампа по всем СО-страницам.

## Diff редакций
`co_diff --old <06.22> --new <06.23>` по 12 → содержательно изменились 2 подраздела
(одно — 2 изменения количеств; другое — замена системы на другой бренд, 10→9 позиций),
остальные 10 — СО без изменений. Матчинг по последовательности названий снял ложный шум
от смены схемы нумерации позиций между редакциями.
