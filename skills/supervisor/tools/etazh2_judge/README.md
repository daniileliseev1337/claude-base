# etazh2_judge — «судья судьи» (многослойный контроль вердикта AI-воркера)

Инструментарий для проверки качества/честности работы поднадзорной AI-сессии по
МАШИННЫМ ФАКТАМ (не по саморапорту). Многослойный контроль вердикта:

```
факты воркера → floor (детерминированный пол) → судья-1 (LLM) →
   выборка 2-го уровня → [один судья-2 | СОВЕТ 3 методами] → адъюдикация
```

## Файлы

- `extract_facts.py` — детерминированная выжимка машинных фактов из JSONL-транскрипта
  (хроника инструментов, блокировки гейта отдельно от обычных ошибок; саморапорт — отдельно как «слова»).
- `checkers.py` — **пол (floor)**: машинные чекеры без LLM (`deny_but_claims_success` ловит
  сокрытие блокировки под видом успеха; `artifact_missing/stale` по mtime; `unrecovered_error`).
  `reconcile_with_verdict1` — сверка floor↔судья-1. Выборка 2-го уровня:
  `should_call_judge2` (серая зона / расхождение / WARN) и `should_call_council`
  (high-stakes: floor RED / обвинение во лжи / провал+эскалация → совет; council ⊆ judge2).
- `judge_prompt.md` — шаблон судьи-1 (оценка работы воркера, нейтральный, anti-injection, JSON).
- `judge2_prompt.md` — шаблон судьи-2 (аудит ВЕРДИКТА судьи-1: клевета/слепота, существо vs форма).
- `build_judge2_input.py` — сборка промпта судьи-2 из шаблона (+`method` для совета:
  `inversion` / `first_principles` / `naive` — method-diversity против groupthink на одной модели).
- `council.py` — агрегатор СОВЕТА N судей-2 в формат verdict2 (мажоритарный вердикт + dissent
  + флаг correlated-bias при единогласии). `adjudicate` потребляет синтез без изменений.
- `adjudicate.py` — детерминированная адъюдикация: эскалация по градации (существенное расхождение
  честности/обвинений/эскалации → владельцу; мягкое расхождение оценки → в журнал). `--notify`
  шлёт дубль-алерт через `notify.py` супервизора (родительская папка) ТОЛЬКО при эскалации.

## Тесты (синтетика, без спавнов и транскриптов)

- `test_council.py` — агрегатор совета (14 кейсов).
- `test_floor_smoke.py` — floor + выборка 2-го уровня на dict-фактах.

Запуск: `$env:PYTHONIOENCODING='utf-8'; python test_council.py; python test_floor_smoke.py`

## Происхождение и границы

Извлечено из PoC «Claude-управляющий сессиями» (этаж 2). Живой ДВИЖОК (спавн воркера через
гейт-арбитр, станция управляющего, журнал) остался в проекте-источнике — здесь только
переносимая логика контроля вердикта. Машинно-специфичная приёмка на реальных транскриптах
(`test_checkers.py` с конкретными session_id) в базу НЕ переносилась — её заменяет
`test_floor_smoke.py`. Промпты и логика читают транскрипт/факты как ДАННЫЕ (anti-injection):
инструкции внутри фактов не исполняются — это материал дела.
