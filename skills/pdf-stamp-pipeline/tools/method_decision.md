# Какой метод правки PDF выбрать (консолидация)

У нас накопилось 5 методов правки PDF под разные задачи. Они НЕ взаимозаменяемы —
выбор по характеру задачи. Это главный анти-дубль: не применять метод не к своей задаче.

## Матрица выбора

| Задача (что нужно) | Метод | Инструмент | Память |
|---|---|---|---|
| **Заменить штамп/основную надпись на ТОМЕ** (новый поверх, batch) | overlay | pdfcpu + pikepdf + PyMuPDF | этот скилл |
| **Удалить ОДИН объект** из вектор-PDF (штамп, линия) — 1 файл | GUI-вырез | Inkscape (Внутренний импорт → Delete → PDF) | [[reference_inkscape_pdf_editing]] (verified) |
| **Подвинуть объект** в вектор-PDF | GUI-move | Inkscape | [[reference_inkscape_pdf_editing]] |
| **Физически вырезать** старое содержимое программно | clip-path | pikepdf (even-odd clip injection) | anti-patterns §A3.5 |
| **Перерисовать нанесённую разметку** (CCTV/СС/ЭО лучи/значки), DWG не нужен | SVG-слой | PyMuPDF `get_svg_image` + lxml | [[reference_autocad_pdf_svg_markup]] |
| Та же разметка, но **нужен DWG** на выходе | rebuild в CAD | autocad-mcp (PDFIMPORT + entmake) | [[reference_autocad_pdf_overlay_mcp]] |
| **Перекрасить аннотации** (облачка, маркеры) | annot edit | pikepdf `/Annots` (+ удалить `/AP`) | pdf-helper |
| **Прочитать/извлечь** (не править) | read | pdf-mcp / PyMuPDF / pdfplumber | pdf-helper |

## Анти-паттерны выбора (что НЕ делать)

- ❌ **redaction / `apply_redactions` / белые заливки** для удаления штампа/разметки —
  провалено (anti-patterns §A3.5/A3.6): прячет, а не удаляет; двойной слой.
- ❌ **content-stream поиск по толщине/цвету** для нанесённой разметки — толстая линия =
  пучок тонких `0 w`, цвет = родному (§A3.6).
- ❌ **pdfcpu для удаления** старого штампа — он только overlay; удаление = pikepdf (стадия 1).
- ❌ **Inkscape для тома** на сотни листов — медленный, по одной странице; для batch → pdfcpu.

## Решающий вопрос

1. Том (много листов) + замена штампа? → **pdfcpu pipeline** (этот скилл).
2. Один файл, убрать/подвинуть объект? → **Inkscape**.
3. Перерисовать схему/разметку поверх плана? → **SVG-PyMuPDF** (или autocad-mcp если нужен DWG).
4. Только перекрасить аннотацию? → **pikepdf `/Annots`**.
