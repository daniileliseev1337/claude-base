# ТН-приёмка тома ИД (постраничный аудит через Workflow)

Реализует **детальнейший аудит** собранного тома исполнительной документации в роли
жёсткого, придирчивого технадзора. Прошёл боевую проверку: на томе 395 стр выявил
9 CRITICAL + 22 MAJOR (чужой приказ, реклама-вкладка, неподписанные акты, штамп
АОСР↔АОМР, водяные знаки «ОБРАЗЕЦ», расхождение марок пены), отсеяв ~14 ложных тревог.

## Когда применять
Том собран (PDF, обычно 100–400 стр: титул → ведомости/реестры → акты → исп.схемы →
документы качества → ОЖР). Нужно проверить КАЖДУЮ страницу на соответствие, корректность
оформления и отсутствие постороннего/устаревшего. Особенно — финальная проверка перед сдачей.

## Процедура

### 1. Построить эталонную карту (страница → что должно быть)
Карта — основа строгости: агент знает, что ОЖИДАЕТСЯ на каждой странице.
- Если есть лог сборки (`assemble_log.txt` от assemble_pdf) — прогнать через `tools/build_map.py`
  (накопление offset по блокам `(N)` → диапазоны страниц с разделами).
- Дополнить из реестров/описи (что за документы, сколько позиций, какой акт).
- Зафиксировать **список ЗАПРЕЩЁННОГО** (то, чего быть НЕ должно): старое/заменённое
  оборудование, удалённые сертификаты/позиции, неверные привязки актов. Это из контекста
  замен по объекту (FACTS / свод факта монтажа). Без этого списка аудит проверит только
  форму, а не «свежесть» содержания.

### 2. Запустить Workflow `tools/priemka_workflow.js`
Передать через `args`: `{pdf, map, forbidden, rules?, batch?, model?, verifyModel?}`.
- **Фаза 1 (fan-out):** N агентов по `batch` (по умолчанию 10) страниц — рендерят и
  придирчиво смотрят КАЖДУЮ страницу, возвращают находки по схеме (page/severity/category/issue/detail).
- **Фаза 2 (верификация):** каждую CRITICAL/MAJOR находку перепроверяет второй, более
  скептичный агент (рендерит ту же страницу сам) — отсев ложных тревог.
- **Синтез:** confirmed по severity + false_alarms.

**Модель субагентов** — параметр `model`/`verifyModel` (по умолчанию `sonnet`; для
рукописи/сложных сканов или максимальной строгости можно `opus`). ⚠ ТОЧКА РАСШИРЕНИЯ:
довести до явного выбора пользователем перед запуском (сколько агентов × какая модель =
прогноз токенов), по правилу токен-дисциплины CLAUDE.md.

### 3. Чек-лист технадзора (в RULES каждого агента)
ЦЕЛОСТНОСТЬ (пустая/чёрная/перевёрнутая/дубль/обрезана) · ОРИЕНТАЦИЯ (навигаторы portrait)
· ОФОРМЛЕНИЕ (штамп на месте, ФИО не обрезаны, нет цветной заливки/рекламных хвостов) ·
СООТВЕТСТВИЕ РАЗДЕЛУ (по карте) · ЗАПРЕЩЁННОЕ (старое оборуд/удалённое/чужой раздел) ·
РЕЕСТРЫ (нумерация без разрывов, листаж, жирность) · КОНСИСТЕНТНОСТЬ (даты/номера/шифр).
severity: CRITICAL (старое/удалённое/чужое/битое/неверный раздел) · MAJOR (обрезка,
кривой штамп, неверная ориентация, разрыв нумерации, неподписанный акт, просроченный
документ) · MINOR (косметика) · INFO.

### 4. Сформировать ведомость замечаний
Свести confirmed в файл `ВЕДОМОСТЬ ЗАМЕЧАНИЙ ТН (<дата>).md` рядом с томом, с разбивкой
**кто исправляет**: сборка (Claude — удалить/заменить страницы) / DWG (перепубликация
схем) / подрядчик (подписи, реквизиты, досканирование) / снабжение (замена сканов). +
отдельным блоком — отсеянные ложные тревоги (чтобы не возвращались).

## Ловушки (проверено боем)
- **pdf_render_pages**: параметр `path` (не file_path), `pages` — строка одной страницы;
  oversized → `clip=[x0,y0,x1,y1]` (доли 0..1) или Read PNG с диска. Сканы — vision (sonnet+,
  не haiku: врёт в реквизитах).
- **mtime на облачных дисках ненадёжен** — сверять СОДЕРЖАНИЕ, не дату.
- **Карта сборки ≠ ожиданиям**: проверять ВСТАВЛЕННОЕ в том, не источник (старые схемы со
  старым оборудованием выглядят как новые по структуре).
- **Рукопись (ОЖР)** даёт ложные хроно-разрывы при беглом OCR (08↔11, 24↔23) — верификация
  обязательна, второй агент читает дату внимательно.
- **Многомодельные паспорта/сертификаты** (Dyson WD-04 «вкл. WD05/WD06»; VALTEC многосерийный)
  — НЕ дефект, занести в forbidden-исключения.

## Уроки боя 2026-06-22 (Музей Р-4-ВиВ, обкатка №2)
- **pdf-mcp лежит → PNG-режим (надёжнее даже когда работает).** Заранее отрендерить ВСЕ
  страницы в PNG (fitz, 150 dpi → A4 читаемо, A3-схемы ок) в одну папку `pNNN.png` (3 цифры),
  агентам в RULES дать абсолютный путь и «Read pNNN.png, НЕ pdf_render_pages». 40 агентов не
  спотыкаются о холодный кэш. Шаблон: `<том>\_Claude_сборка\priemka_r4_png.js` (PNG-режим, opus).
- **Workflow на image-heavy opus роняет ~10% батчей** («Connection closed mid-response» + stalls
  с ретраями). ОБЯЗАТЕЛЬНО сверять `pages_checked` с total; упавшие батчи добрать отдельным
  проходом (прямые `Agent(model:opus)` по 10 стр) — иначе тихий пробел покрытия. «No silent caps».
- **opus оправдал себя против галлюцинаций.** Адверсариальная верификация на opus КОРРЕКТНО
  отсеяла ложные: «дубль 108/109» (разные листы инструкции), «старый код РИДАН на стр.293/296»
  (там кода нет — только тип «BVR модификация R»). sonnet исторически врал в реквизитах сканов.
  Для томов с обилием паспортов/сканов — opus на обеих фазах.
- **Стоимость факт:** полный opus-прогон ~354 стр = 50 агентов, **~3.4M токенов, 18 мин**
  (не 8-12M — прогноз завышен). На X20 — рядовой прогон.
- **Порядок листов И.С — сверять ВИЗУАЛЬНО, не текстом.** Текст-слой А3-векторных схем
  конкатенируется криво и врёт (даёт ложный PASS на размещении). Рендерить заголовок
  «Приложение к АО_-N» + штамп-подзаголовок ГЛАЗАМИ. Признак scramble: листы одного акта
  разнесены по тому. См. [[muzey_r4_viv_pdf_assembly]].
- **Тип ≠ код.** «BVR модификация R» / «BVR-R» — это ТИП крана, НЕ «код без RG». Флагать только
  реальную строку «Код материала: …R». Занести в forbidden-исключения, иначе ложные MAJOR.
