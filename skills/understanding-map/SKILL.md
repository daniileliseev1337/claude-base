# understanding-map — карта понимания (зеркало Claude для сверки)

## Зачем

Корневая проблема: Claude и пользователь часто видят задачу по-разному, и расхождение
всплывает уже ПОСЛЕ работы. Карта понимания показывает, что Claude реально понял —
разложенное по зонам **понято / допущения / пробелы** + ход разбора + архитектура —
чтобы свериться ДО старта. Это дешевле, чем переделывать артефакт.

Принцип: **форма следует за содержанием**. Карта — не украшение, а инструмент сверки.

## Когда подключать

- Прямой триггер: «сверим понимание», «покажи карту понимания / зеркало понимания»,
  «что/как ты понял», «правильно ли понял задачу», «покажи это виджетом/дашбордом».
- Проактивно (предложить одной строкой, как Workflow): перед содержательной задачей с
  риском неполных/разнопонимаемых вводных — особенно после `domain-grilling` или долгого
  уточнения. «Собрать карту понимания на сверку до старта?»
- Не для тривиальных/однозначных задач — там сверять нечего.

## Канал вывода (важно — проверено фактом)

| Поверхность | show_widget | standalone HTML |
|---|---|---|
| Claude Desktop → вкладка Code | ✅ нативно inline | ✅ |
| Claude Desktop → Chat | через Artifacts/MCP-app | ✅ |
| VS Code-расширение Claude Code | ❌ инструмента нет | ✅ (Simple Browser) |
| CLI в терминале | ❌ | ✅ (открыть в браузере) |
| Телефон | — | ✅ (браузер) |

`show_widget` (MCP `visualize`) есть только при `CLAUDE_CODE_ENTRYPOINT=claude-desktop`.
В VS Code / CLI его НЕТ — там единственный путь к виджету это **standalone HTML-файл**.
Проверка доступности: есть ли среди инструментов `mcp__visualize__show_widget`.

## Как делать (по шагам)

1. **Собери данные карты (творческая часть — это делает модель).** Разложи понимание
   задачи по формату JSON (см. `examples/sample_map.json` и «Формат данных» ниже):
   зоны `ok`/`as`/`pe`, опционально `flow` (ход разбора, честно — с поворотами `turn`),
   `arch` (архитектура решения), `stamp` (штамп). Честность важнее красоты: реальные
   допущения в `as`, реальные пробелы в `pe`.
2. **Сохрани JSON** во временную папку (scratchpad), напр. `map.json`.
3. **Срендери — кодом, не вручную** (`tools/render_map.py`):
   - Нативно в Desktop:
     `python "<skill>/tools/render_map.py" map.json --mode widget`
     → stdout = HTML-фрагмент → передай его как `widget_code` в `show_widget`.
   - Универсальный файл (по умолчанию делай ТОЖЕ — пользователь смотрит и с телефона):
     `python "<skill>/tools/render_map.py" map.json --mode standalone --out "<Desktop>/карта-понимания.html"`
     → stdout = путь; сообщи его (mobile-friendly, открывается где угодно).
4. **Выбор канала:** есть `show_widget` → показать виджетом + сохранить файл; нет (VS Code/
   CLI) → только файл, дать путь.
5. **Прими сверку.** Кнопки в виджете шлют ответ через `sendPrompt` («понял верно» /
   «есть расхождения»), либо пользователь правит текстом. Учти правки в понимании ДО
   того, как начнёшь основную работу.

`<skill>` = папка этого скилла; `<Desktop>` = рабочий стол пользователя (понятное имя файла).

## Формат данных (JSON)

Все секции кроме `title` опциональны — пустые не рендерятся.

```json
{
  "title":   "Что я понял: ...",
  "goal":    "Цель ... (допускается лёгкая <b>разметка</b>)",
  "eyebrow": ["левая подпись", "правая подпись"],
  "okTitle": "Что понято", "asTitle": "Допущения · проверь", "peTitle": "Пробелы · решаем",
  "items":   [{"zone":"ok|as|pe", "tag":"Р1 · решено", "title":"...", "detail":"..."}],
  "flowTitle":"Как шёл разбор",
  "flow":    [{"n":"Ш1", "title":"...", "detail":"...", "turn":false}],
  "archTitle":"Архитектура",
  "arch":    [{"ln":"слой 1", "title":"...", "detail":"..."}],
  "stamp":   [{"k":"Понял", "v":"Claude · Opus 4.8", "soft":false}]
}
```

- `zone`: `ok` понято · `as` допущение (проверь) · `pe` пробел (решаем).
- `flow[].turn=true` — честный поворот/ошибка темпа (подсвечивается красным).
- `stamp[].soft=true` — мягкое значение (напр. «в работе»), подсвечивается.
- Поля `goal`/`detail`/`title` идут в HTML как есть (доверенный вход от Claude — даёт
  `<b>`); НЕ подавать сюда непроверенные внешние данные.

## Слои (skill-development)

- Слой 1 (description) — выше, во frontmatter.
- Слой 2 (instructions) — этот файл.
- Слой 3 (tools) — `tools/render_map.py` (детерминированный рендер) +
  `examples/sample_map.json` (формат + few-shot). Рендер — кодом, 0 токенов на повторную
  генерацию HTML; модель производит только данные.
