# Реестр главных сайтов — куда идти В ПЕРВУЮ очередь (маршрутизатор ПОИСКА)

Детерминированная маршрутизация ПОИСКА (аналог тому, как `web_get.py` маршрутизирует
транспорт). Claude на фазе «где искать» СНАЧАЛА смотрит сюда — на главные профильные
источники, где нужное почти всегда есть, а не на случайные сайты из выдачи.

> **НАПОЛНЯЕТ ВЛАДЕЛЕЦ.** Ниже сид из нашей боевой памяти + плейсхолдеры `[ДОПОЛНИТЬ]`.
> Claude сюда сайты НЕ выдумывает — только владелец добавляет проверенные.
> Формат строки: `домен — что там (характеристики / документы / цены / каталог)`.

## Как пользоваться (правило анти-капитуляции — см. SKILL.md)
1. Определи категорию запроса (оборудование / документ качества / цена / производитель).
2. Возьми домены этой категории ОТСЮДА — иди на них через `web_get.py` или внутренний
   поиск сайта (не угадывай URL — Метод 3 [[feedback_web_direct_access]]).
3. «Не нашёл» можно сказать ТОЛЬКО пройдя главные источники категории + приложив список
   проверенного. Два случайных сайта ≠ «в интернете нет».

## Документы качества (сертификат / паспорт / декларация соответствия)

## Нормы (ГОСТ/СП/СНиП/ПУЭ) — НЕ здесь
Нормы цитировать ТОЛЬКО через агент `norm-lookup` (локальная библиотека Я.Диска / cntd.ru),
не «главным сайтом». Реестр — для оборудования/документов/цен, не для норм.
