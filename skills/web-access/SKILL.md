# web-access — единый вход веб-доступа (лестница в коде)

## Зачем этот скилл
Раньше лестница веб-доступа жила ТЕКСТОМ (правило в CLAUDE.md): модель каждый раз
вспоминала порядок exa → fetch → playwright → WebFetch → r.jina и вручную дёргала
инструменты по очереди. Ступени забывались, падение одной читалось как «в интернете
нет». Здесь детерминированная часть перебора — в коде: `tools/web_get.py`.

**Одна команда закрывает ~80% «сходи в сеть»:**
```
python ~/.claude/skills/web-access/tools/web_get.py <URL> [--kind page|file|auto] [-o out] [--json]
```
- Без `--json` — человекочитаемый лог (какие ступени пробованы, что сработало).
- С `--json` — машиночитаемый результат (парси и решай следующий ход).
- `--kind auto` (дефолт): `file` если задан `-o` или URL кончается на `.pdf/.zip/...`, иначе `page`.

## Что делает КОД (детерминированные ступени)
1. Определяет РЕАЛЬНОЕ гео egress (`ipinfo`, мимо прокси окна — env-проверки мало,
   системный VPN в переменных не виден; [[web_access_survey_2026_06]]).
2. Классифицирует цель: RU-хост (TLD `.ru/.su/.рф`) или зарубеж; страница или файл.
3. Строит лестницу под (тип, RU-цель, egress) и идёт по ней ДО первого
   верифицированного успеха:
   - **direct** — обычный curl с браузерным UA;
   - **noproxy** — `curl --noproxy "*"` (обход корп-прокси окна; на RU-egress берёт RU-сайты);
   - **jina** — `r.jina.ai/<URL>`, облачный reader (антибот/JS-сайты → markdown; без ключа; только ТЕКСТ страниц);
   - **ru** — делегирует [[ru-gov-access]] `ru_fetch.py` (RU exit-IP; RU-слой НЕ дублируется — один дом).
4. **Верифицирует** каждый ответ, а не доверяет HTTP 200:
   - страница: 2xx, непустая, без Cloudflare/DDoS-Guard challenge-маркеров;
   - файл: сигнатура (`%PDF`, `PK` office/zip, PNG/JPEG…), НЕ `<!DOCTYPE` html-заглушка,
     `--kind file` + `.pdf` → строго `%PDF`. Ловит скачанную заглушку, которую старый
     ручной подход часто сохранял как «документ».

## Правило анти-капитуляции (лечит «сдался рано / искал не там»)
Транспорт (web_get) — половина дела. Вторая боль ПОВЕДЕНЧЕСКАЯ: сдаться после 1-2
неудачных сайтов и сказать «в интернете нет», или искать на случайных сайтах вместо
главных. Это логическая подмена: «не смог зайти на 2 сайта» ≠ «этого не существует».

**Жёсткое правило вывода:**
- **«Не нашёл / такого нет» ЗАПРЕЩЕНО**, пока не пройдены ГЛАВНЫЕ профильные источники
  категории из `tools/main_sites.md` И отрицательный вывод не сопровождается СПИСКОМ
  проверенного (какие сайты/запросы, какой результат каждого). Без списка — это не
  «нет», а «я сдался».
- **Сначала — реестр, потом выдача.** На фазе «где искать» СНАЧАЛА смотри
  `tools/main_sites.md` (главные сайты по категории: оборудование / документ качества /
  цена / производитель), потом уже общий поиск. Не идти на случайный первый сайт из выдачи.
- **Падение ступени/сайта ≠ конец.** web_get вернул fail → взять его `next` (MCP-ступень).
  Сайт не открылся → другой главный сайт категории / внутренний поиск сайта / скриншот
  глазами (ШАГ 0). Минимум усилий до вывода «нет»: главные источники категории пройдены.
- **Домен не угадывать** — брать из реестра / карточки товара (Метод 3 [[feedback_web_direct_access]]).

Это дёшево берёт ЦЕННУЮ идею deep-research (не сдаваться + проверять) без его 5-6М-токенного
харнесса. Тяжёлый deep-research — только по ЯВНОМУ запросу с оценкой цены (см. token_economy).

## Где КОД останавливается — честная граница (дальше Claude, MCP не из питона)
Скрипт покрывает curl-семейство + jina. MCP-ступени из питона недёргаемы — это
инструменты Claude. Когда кодовые ступени исчерпаны, `web_get` печатает `next` —
какую ступень взять руками:
- **exa** (`web_search_exa` / `web_fetch_exa`) — semantic-поиск и чтение, хорош на антиботе;
- **firecrawl** (`firecrawl_scrape` / `firecrawl_search`) — скрейп/поиск с извлечением;
- **playwright** — JS/SPA, антибот-cookies, DATA-слой реестров (`--proxy-server=socks5://<RU>` на иностр. egress);
- для ФАЙЛА по прямому URL, что не дался: playwright на КАРТОЧКУ товара → снять cookies →
  `curl -b` (Метод 2, [[feedback_web_direct_access]]).

Это не слабость, а разделение: детерминированное перебор+верификация — в коде;
браузер/семантика — за моделью. **ШАГ 0 всегда в силе:** при сомнении «сайта/документа
нет» — сперва `playwright browser_take_screenshot`, посмотреть глазами, потом вывод.

## Порядок ступеней (маршрутизация в коде)
| egress | цель | page | file |
|---|---|---|---|
| RU | любая | noproxy* → direct → jina | noproxy* → direct |
| ≠RU | RU-хост | noproxy* → direct → jina | noproxy* → direct |
| ≠RU | зарубеж | direct → noproxy* → jina | direct → noproxy* |

`*` — **noproxy в лестнице ТОЛЬКО если обход прокси жив** (детект в egress-пробе: на
WARP-машине `--noproxy` мёртв → ступень висела бы 15с по таймауту; тогда исключается).

**Только БЫСТРЫЕ ступени в авто-переборе** (боевые уроки 2026-07-08): (1) `palerom.ru/...png`
взялся прямым curl с NL-egress за секунды — НЕ все RU-сайты гео-блокированы, прямой первым;
(2) медленный RU exit-IP (`ru-gov-access`, ищет бесплатный прокси десятки секунд) УБРАН из
авто-лестницы — реально гео-блокированные RU-цели (в основном госсайты) уходят в `next_hint`
как осознанный шаг. jina — на гео-блок/антибот страниц (consultant-класс); для ФАЙЛА jina
бесполезна (только текст). **Egress кешируется на сессию** (TTL 20 мин) — убирает 12-40с
детекта на каждый запуск (замер: lunda.ru 40с → 5с).

## Как встроено в правило CLAUDE.md
CLAUDE.md §«Веб-доступ» теперь указывает СНАЧАЛА на эту команду (детерминированный
перебор), а текстовая лестница exa/fetch/playwright осталась как СЕМАНТИКА для
MCP-остатка. Полная гео-модель 3 состояний — [[feedback_web_direct_access]] (ШАГ −1).

## Границы (не для этого скилла)
- RU-госреестры pub.fsa/ЕГРЮЛ/АРШИН как самостоятельная задача → [[ru-gov-access]]
  (web_get вызывает его ru-слоем, но глубокая работа с SPA-API — там).
- Поиск/добыча документа качества по артикулу (карточки, dorks) → `doc-finder`.
- Извлечение текста/таблиц из уже скачанного PDF → `doc-extract`.

## Файлы скилла
- `tools/web_get.py` — транспорт (лестница ступеней + верификация).
- `tools/main_sites.md` — **реестр главных сайтов** (маршрутизатор ПОИСКА; наполняет владелец).
- `tests/test_web_get.py` — юнит-тесты транспорта.

## Тесты
- `tests/test_web_get.py` — 27 юнит-проверок чистой логики (классификация, маршрутизация,
  верификация page/file, next_hint), без сети. Прогон: `python tests/test_web_get.py`.
- Живые smoke (2026-07-08, egress=NL): зарубежная страница → direct; ridan.ru →
  jina после падения ru; мёртвый URL → все пали + next_hint; arxiv PDF → direct+%PDF;
  W3C 403 → заглушка `<!DO` пойма верификацией (не выдана за успех).

## Раздача
Скилл целиком в `~/.claude/skills/web-access/` (SKILL.md + tools/web_get.py + tests/) —
штатный sync-base, без ключей. Зависит от соседнего `ru-gov-access/tools/ru_fetch.py`
(тоже в базе). `$RU_PROXY` (per-machine, `CLAUDE.user.md`) усиливает ru-слой.
