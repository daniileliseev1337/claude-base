# doc-finder — поиск и добыча документа качества на оборудование

## Когда подключаться
Нужен официальный документ качества (для ИД, проверки поставщика, спецификации) по
артикулу/бренду оборудования. НЕ для произвольного веб-поиска — именно документы качества.

## Workflow

### Шаг 1 — Сгенерировать поисковые запросы (СКРИПТ)
```bash
python ~/.claude/skills/doc-finder/tools/gen_dorks.py --article "<артикул>" --brand "<бренд>" --type сертификат [--site <домен>]
```
`--type`: `сертификат` / `паспорт` / `декларация` / `любой`. Скрипт строит Google-dork
запросы (filetype:pdf, intitle:"index of", site:). Сети не требует.

### Шаг 2 — Выполнить запросы через web
Каждый запрос прогнать через web-лестницу (CLAUDE.md): **exa** → **fetch** → **playwright**.
Цель — найти карточку товара актуального артикула на сайте производителя/B2B-поставщика.

### Шаг 3 — Добыть официальный PDF
По методам [[feedback_web_direct_access]]:
- зайти в карточку товара → раздел «Документация / Сертификаты», брать ссылку ОТТУДА
  (не конструировать URL, не из Google — часто битый);
- за антиботом/сессией: playwright cookies → `curl --noproxy -b cookies`;
- проверить сигнатуру `%PDF` (не заглушка-HTML);
- последний код артикула: полный комплект на странице актуального кода (Метод 4).

### Шаг 4 — Верификация добытого
- Открыть/рендерить 1-ю страницу — это нужный документ на нужный артикул (не заглушка).
- При сомнении в подлинности — [[supplier-due-diligence]] `verify_pdf_metadata.py`.
- **Verify:** PDF официальный, на нужный артикул, читается. «Документа нет» — только после
  визуального просмотра правильной карточки на правильном домене.

## Tools (слой 3)
- `tools/gen_dorks.py` — генератор поисковых запросов (offline). Протестирован.

## Связь с базой
- Использует [[feedback_web_direct_access]] (обход прокси/антибота, curl cookies) как методы добычи.
- Питает `снабженец` и `id-engineer` (документы качества для ИД).
- Подлинность добытого — [[supplier-due-diligence]].

## Ловушки
- Скриншот документа ≠ документ качества — для ИД нужен официальный PDF.
- Прямая ссылка из Google/exa часто битая — брать из карточки товара (Метод 3).
- Госреестры (pub.fsa) за корп-прокси недоступны — документы есть на B2B/у производителя.
