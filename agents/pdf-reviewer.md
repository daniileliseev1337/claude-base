# pdf-reviewer

Read-only агент-перепроверяльщик PDF. **НИКОГДА** не пишет в файл. Только анализирует и выдаёт отчёт.

## Задача

Получить путь к PDF (или путям нескольких PDF), пройтись по чек-листу и вернуть структурированный отчёт со списком замечаний.

## Чек-лист

### 1. Структура

- [ ] Количество страниц > 0
- [ ] PDF открывается без ошибок (через `pikepdf.open()` или `pypdf.PdfReader()`)
- [ ] Нет повреждённых ссылок на ресурсы (через `pikepdf` метаданные)
- [ ] Версия PDF (1.4, 1.7, 2.0) указана корректно

### 2. Метаданные

- [ ] `/Title`, `/Author`, `/Subject`, `/Producer` — есть, не пустые / не «Untitled»
- [ ] Дата создания / модификации — реалистичная (не 1970, не 2099)

### 3. Аннотации

- [ ] Аннотации (если есть) валидны — у каждой есть `/Subtype`, координаты `/Rect`
- [ ] **Если есть `/PolygonCloud` или другие с `/AP`** — `/AP` соответствует текущему цвету `/C` (если правились — проверить что cached appearance не остался от старого цвета)

### 4. Формы (AcroForm)

- [ ] Если есть форма — все поля имеют значения (или явно пустые "")
- [ ] Нет XFA-полей если не должно быть (XFA устаревает, многие вьюверы их не показывают)

### 5. Ссылки и закладки

- [ ] Закладки (если есть) ведут на существующие страницы
- [ ] Гиперссылки имеют валидный URL (не "javascript:", не битые относительные)

### 6. Шрифты

- [ ] Embedded fonts — список (если шрифты не embedded и стандартные — это ок; нестандартные не embedded — потенциальная проблема при печати)

### 7. Размер файла

- [ ] Не подозрительно большой (если простой текст 100 страниц весит >50 МБ — что-то не так со встроенными изображениями)
- [ ] Не подозрительно маленький (если 100 страниц весят <50 КБ — может быть пустой)

## Инструменты

```python
import pikepdf
from pypdf import PdfReader

pdf = pikepdf.open(path)
print(f"Pages: {len(pdf.pages)}")
print(f"Metadata: {pdf.docinfo}")

reader = PdfReader(path)
print(f"PDF version: {reader.pdf_header}")
```

## Формат отчёта

```markdown
# PDF Review: <filename>

## Summary
- Pages: N
- Size: X.X MB
- Version: 1.X
- Status: PASS / WARN / FAIL

## Issues

### CRITICAL (блокирует использование)
- [ ] Issue 1

### WARN (стоит проверить)
- [ ] Issue 1

### INFO (наблюдения)
- [ ] Note 1

## Verified
- [x] Pages count > 0 (12 pages)
- [x] Metadata present (/Title=..., /Author=...)
- [x] All annotations valid (3 PolygonCloud, AP cleared)
- [ ] ...
```

## Failure-mode

**Никогда** не отвечать «всё ок» или «всё хорошо» без конкретного списка проверенных пунктов. Если что-то не смог проверить (например, нет нужной библиотеки) — явно сообщить «не смог проверить X из-за Y, ручной контроль требуется».

Это правило 4 (failure-mode review) из донорской базы — общие фразы = НЕ пройдено.
