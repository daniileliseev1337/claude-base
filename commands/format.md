# /format — применить стиль оформления к документу

Slash-команда для **явного** запуска оформления. Использует скилл
`stroy-formatting` ([`~/.claude/skills/stroy-formatting/SKILL.md`](../skills/stroy-formatting/SKILL.md)).

## Когда использовать

- Пользователь сам хочет применить стиль к уже сгенерированному
  markdown / json / тексту контенту.
- Нужно пересохранить документ в другом стиле (например, был
  `plain-clean`, теперь надо `gost-report-full` для экспертизы).
- Перед выдачей финального документа — пользователь хочет видеть
  выбор стилей явно, не дожидаясь автоматического вопроса.

## Алгоритм

1. **Прочитать `~/.claude/skills/stroy-formatting/SKILL.md`** — там
   полная методика и параметры всех 4 стилей.

2. **Узнать у пользователя через `AskUserQuestion`** — какой стиль
   применить:
   - ГОСТ полный (без рамки)
   - ГОСТ упрощённый
   - ГОСТ с рамкой (ВКР)
   - Простой аккуратный

3. **Узнать** (если ещё не понятно из контекста):
   - **Что оформлять** — содержимое в markdown/тексте, или копировать
     из готового `.md` файла? Или Claude собирает с нуля по заданию?
   - **Имя итогового файла** (по умолчанию — по теме задачи,
     `kebab-case`)
   - **Куда сохранить** (по умолчанию — cwd сессии)

4. **Применить шаблон**:
   - Скопировать `~/.claude/formatting-templates/<style>.docx` →
     `<cwd>/<name>.docx`
   - Через python-docx + word-MCP добавить контент:
     - Заголовки → `add_heading` (уровни 1/2/3)
     - Абзацы → `add_paragraph` (со стилем `Body Text` / `000. ПЗ`
       для ГОСТ-вариантов)
     - Таблицы → `add_table` или `add_table_*` из word-MCP
     - Изображения → `add_picture` если нужны
   - Сохранить

5. **Опционально — конвертация в PDF**:
   - Спросить пользователя: «Нужен также PDF?»
   - Если да — `mcp__word__convert_to_pdf` или fallback

6. **Доложить результат**:
   - Какой стиль применён
   - Путь к финальному DOCX (и PDF, если был)
   - Что внутри (количество страниц / разделов, если уместно)

## Параметры команды (опционально)

Команда может принимать аргументы в форме `/format <стиль> [<имя>]`:

- `/format full` — без вопроса, стиль `gost-report-full`
- `/format light` — `gost-report-light`
- `/format border` — `gost-report-with-border`
- `/format plain` — `plain-clean`
- `/format full ПЗ-ОВ-блок-А` — стиль и имя файла

Если аргумент не передан — Claude спрашивает через `AskUserQuestion`.

## Связанные

- Скилл с методикой: `~/.claude/skills/stroy-formatting/SKILL.md`
- Шаблоны: `~/.claude/formatting-templates/*.docx`
- Word-MCP для манипуляций: `mcp__word__*`
- Похожие команды для других форматов:
  - Спецификация по ГОСТ Р 21.110: см. скилл `stroy-specs-xlsx`
    (если установлен) — отдельная категория документов со штампом
