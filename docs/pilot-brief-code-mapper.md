# Бриф пилота: code-mapper (отдельная сессия)

Источник решения: session-reports/2026-06-10_repo-harvest/report.md (добивка, ПОПРОБОВАТЬ-гибрид).
Репо: github.com/damien220/code-mapper (public domain, чистый Python 17 КБ, без зависимостей).

## Что это
CLI: генерит компактный PROJECT_CONTEXT.md — дерево файлов + Mermaid-диаграммы
классов/зависимостей + индекс сигнатур (~78% сжатия против чтения кода подряд).

## Позиционирование у нас (НЕ конкурент graphify)
- graphify — queryable-граф для md-базы (query дёшев, инкрементальные апдейты) — остаётся.
- code-mapper — статичный контекст за 0 токенов построения. Ниша: Python-скрипты
  в tools/ скиллов (image-text-replace, spec-writer и др.) + onboarding-материалы.

## Цель пилота
1. Прогнать на 2-3 папках tools/ наших скиллов.
2. Оценить: PROJECT_CONTEXT.md в Required reading скилла экономит ли чтение кода субагентом.
3. Решение: шаг в skill-development (генерить карту при создании tools/) или «не надо».

## Критерии успеха
- Карта строится без ошибок на наших tools/ (кириллические пути!);
- субагент по карте отвечает на «что делает скрипт X» без чтения исходника;
- размер карты ≤ 25% исходников.

Модель: sonnet достаточно. Скрипт перед запуском прочитать (17 КБ, быстро).
