## Что это
Git-hook: при коммите сворачивает детали в ~300-токеновую запись ledger;
восстановление деталей — через git show. Обещает 28× до auto-compact (маркетинг — проверить).

## Зачем нам
Кандидат в memory/context_discipline.md: дисциплина длинных dev-сессий хаба.
ВАЖНО: с 2026-06-11 уже есть гейт сессий в CLAUDE.md + хук-счётчик — пилот имеет смысл,
только если они не закроют проблему (смотреть телеметрию tool-usage.jsonl за неделю).

## Предусловия и риски
- Git Bash есть; совместимость с Claude Code Desktop НЕ проверена;
- хук в .git/hooks может конфликтовать с auto-pull/auto-push хуками базы — проверить
  на тестовом репо, НЕ на claude-base.

## Критерии успеха
- Замер: одинаковая задача с ledger и без — разница в cache-read токенах (ccusage);
- нет конфликтов с auto-sync. Иначе — вердикт «не надо», идею паттерна в context_discipline.
