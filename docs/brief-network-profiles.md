# Бриф: сетевые профили ПК и медленные RU-сайты под VPN (отдельная сессия)

Источник: наблюдение пользователя 2026-06-11 (dev-ПК DANIIL). Симптом: страницы,
открываемые Claude в браузере, грузятся медленно; обновление/новая вкладка у
пользователя — быстро.

## Диагноз (проверен в обсуждении, доказать замером в сессии)

1. Холодная загрузка (Claude) против кэша браузера (пользователь) — базовая разница.
2. Усилитель: на DANIIL системный VPN на весь ПК → RU-сайты отвечают с зарубежного
   IP медленно/капчой (гео-фильтры, антибот). На корп-ПК ситуация другая: корп-прокси.
3. Лестница веба неоднородна: exa/firecrawl — облачные (VPN/прокси ПК НЕ влияет);
   fetch MCP / playwright / Chrome-расширение / WebFetch — локальные (влияет).

## Что сделать в сессии

1. **Замер для подтверждения**: один RU-сайт (например, карточка товара поставщика)
   через fetch/playwright с VPN и через прокси — сравнить время/блокировки.
2. **Сетевые профили в базу** (memory/network_profiles.md или дополнить proxy_github.md):
   - corp-ПК: корп-прокси + GitHub-bypass (уже описано);
   - DANIIL: системный VPN, личный RU-прокси в `~/.claude-proxy.json` (host:port там,
     в базу значения не класть — обезличивание).
3. **ВАЖНО (поправка пользователя 11.06)**: личный прокси — ЗАРУБЕЖНЫЙ exit (для
   Claude/GitHub), НЕ русский. «Chrome (with proxy)» RU-сайты НЕ лечит — выход всё
   равно за рубежом. Для RU-сайтов под системным VPN нужен **split tunneling**:
   выяснить VPN-клиент пользователя и его возможности (исключения по приложению /
   домену / IP-спискам РФ), настроить прямой маршрут для RU-доменов мимо туннеля.
   Fallback, если split tunneling недоступен: r.jina.ai-префикс (memory) или
   облачные exa/firecrawl (на них VPN не влияет).
4. **playwright MCP**: после настройки split tunneling проверить, что его Chromium
   тоже ходит на RU-сайты напрямую (он берёт системный маршрут). Прокси-вариант
   `--proxy-server` держать в уме для обратной задачи — доступ к заблокированным
   из РФ ресурсам без системного VPN.
5. **Обновить лестницу веба в CLAUDE.md одной строкой**, если появится новая ступень
   (например: «RU-сайт тормозит/капча на VPN-ПК → playwright-ru / Chrome with proxy»).

## Критерии успеха
- Замер до/после: RU-сайт открывается через прокси-маршрут заметно быстрее или без капчи.
- Профили зафиксированы в памяти базы (обезличенно), правило в лестнице — если нужно.
- Ничего не сломано для corp-ПК (их маршрут не трогаем).

Модель: обычная (sonnet/opus), Fable не нужна. Субагенты — по матрице CLAUDE.md.
