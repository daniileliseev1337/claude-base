## Профили лимитов

План Plus — рутинные задачи на Luna (или встроенной лёгкой модели вроде 5.4-mini), Sol — точечно, на шагах, которые того стоят; следи за недельным лимитом через `/usage`, не доводи до полного исчерпания.

План Pro x5 — Sol с уровнем рассуждений high используется по умолчанию для содержательной работы; уровни xhigh/max/ultra — только на крупную ставку (деньги, экспертиза, сдача, критичные нормы) и после явного вопроса владельцу, не как дефолт.

## Контекст

Авто-компакция включена, порог 270000 токенов; зона свыше ~272K входных токенов дороже (двойной тариф на инпут) — не разгоняй диалог до неё без нужды.

Длинную многошаговую работу веди через файлы (план, журнал, промежуточные артефакты), а не через раздувание истории чата — тот же принцип, что и в остальной базе.

## Сессии

Продолжение прерванной сессии — `codex resume`, не новый чат с пересказом контекста заново.

Ветвление на развилке решения — `/fork`, чтобы не терять рабочую версию при исследовании альтернативного пути.

Смена темы задачи — новый чат, а не продолжение прежнего треда другим содержанием.

## Субагенты

Независимые подзадачи — параллельным fan-out через spawn_agents, не последовательно одним потоком без причины.

Воркеры — модели Terra или Luna по сложности подзадачи; оркестратор координирующего уровня — Sol.

Ревьюер-гейт обязателен: перед выдачей содержательного артефакта — отдельная роль auditor из `agents/`, та же архитектура, что и в `~/.claude`. Пропуск проверки ради скорости — нарушение, не оптимизация.

Нормо-проверку делегируй роли `norm-lookup` из `agents/` — она возвращает дословную цитату пункта из источника; по памяти нормы не цитируй.

## Инструменты — маршрутизация (предварительная, до тест-заезда)

`.xlsx` → плагин spreadsheets; `.docx` → плагин documents; `.pdf` → плагин pdf; работа в браузере → плагин browser.

Уникальные доменные задачи (Revit, AutoCAD, время, поиск через exa) — через MCP из белого списка `codex-layer/mcp-whitelist.json`, не через общие плагины.

Таблица маршрутизации предварительная: после тестового прогона (после Задачи 11) уточняется по фактическому поведению плагинов на реальных файлах.

## Комментарии в коде

Комментарии в коде — по-русски. Это явное разрешение кириллицы для этой среды; ASCII-дефолт Codex для комментариев в этой базе отменён данным правилом.

## Известные грабли

`/review` иногда отвечает по-английски независимо от языка запроса — это особенность самого Codex, не повод менять правило языка ответа здесь или в ядре.

## MCP-серверы

Не подключай новый MCP-сервер без необходимости: каждый подключённый сервер расходует часть лимита контекста в каждом сообщении, даже когда не используется в этом шаге.

Состав белого списка — файл `codex-layer/mcp-whitelist.json`; правь список там, не подключай сервер напрямую в обход него.

## Оформление деловых документов

Деловые `.docx`/`.xlsx` (акты, письма, спецификации, сметы) — нейтральное оформление или оформление по образцу документа-источника; без декоративных цветных шапок и заливок по умолчанию.

Цвет в таком документе — только функциональный (разметка диффа, подсветка ошибки) или по явной просьбе пользователя, не украшение.
