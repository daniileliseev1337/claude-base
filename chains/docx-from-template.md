# chain:docx-from-template

## Назначение

Генерация Word-документа на основе готового шаблона: чтение шаблона,
поиск плейсхолдеров, сбор значений у пользователя, рендер,
форматирование, сохранение в рабочий каталог.

## Шаги

### Stage 1 — read template (collect)

Прочитать шаблон через `mcp__word__get_document_info` +
`mcp__word__find_text_in_document` чтобы найти плейсхолдеры формата
`{{name}}` или `[[name]]`.

**Verify:** список плейсхолдеров непуст. Если пуст — спросить
пользователя, тот ли шаблон.

### Stage 2 — collect values (extract)

Для каждого плейсхолдера спросить пользователя через `AskUserQuestion`
(батч 1-4 вопроса за раз). Если плейсхолдеров > 4 — разбить на серии.

**Verify:** для каждого плейсхолдера есть значение или явное «оставить
пустым».

### Stage 3 — render (act)

Через `mcp__word__search_and_replace` подставить значения в копию
шаблона в рабочем каталоге пользователя.

**Verify:** в финальном документе нет строк, совпадающих с шаблонными
скобками `{{...}}` / `[[...]]`. Если есть — Stage 2 был неполный,
вернуться к нему.

### Stage 4 — format + save

Если документ финальный (для сдачи) — применить `stroy-formatting`
скилл, спросив стиль через `AskUserQuestion` (если не указан явно).
Сохранить в cwd пользователя.

**Verify:** `mcp__word__get_document_info` возвращает документ без
ошибок. Прогнать `word-checker` агент для финальной проверки;
требуется PASSED.

## Когда НЕ использовать

- Если шаблон без плейсхолдеров — это просто копирование, не chain.
- Если шаблона нет вовсе — это `chain:docx-from-scratch`
  (не реализован).
- Если плейсхолдеров > 20 — спросить пользователя, не лучше ли
  передать таблицу значений (CSV/JSON), чем серию AskUserQuestion.
