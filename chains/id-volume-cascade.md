# Chain: id-volume-cascade — каскадная правка состава тома ИД

Назначение: единый пайплайн «изменился состав → провести правку по всему тому без разрыва инварианта».
Применять, когда в ИД добавляется/убирается/меняется позиция оборудования или материала (часто — закрытие
замечания ТН вида «в ИД не включено …»). Обезличено (методология). Связано: skill `id-volume-graph`,
методичка ИД §26, `chains/project-doc-pack.md`.

## Когда запускать
- Замечание ТН «не включено / привести в соответствие» по составу.
- Любая правка позиции после того, как том уже собран (ретро-правка состава).
- Инвариант **ПВ ≡ ВСО ≡ ВОР ≡ ИС** где-то нарушен (разрыв).

## Граница каскада (ВАЖНО — это «каскад СОСТАВА», ничего больше)
Этот каскад строго про **состав и его согласованность**:
1. выровнять **ПВ ≡ ВСО ≡ ВОР** (одна номенклатура / количества / привязка по помещениям во всех трёх ведомостях);
2. привести состав в соответствие конкретному замечанию ТН (добавить / убрать / исправить позиции по эталону);
3. проверить **правильность заполнения** этих ведомостей (нумерация, merge, суммы, инвариант, наименования из источника,
   №акта/дата — от соседей или по самому акту, одна верная редакция ПВ).

**НЕ входит в этот каскад** (отдельные каскады, передаются дальше В ЗАВИСИМОСТИ от разбираемого замечания):
- **документы качества** под позиции (паспорта / СС / декларации / отказные / гарантийные талоны → Реестр №X → опись) — это docs-каскад;
- **полнота исп. схем** (длины участков, изменение диаметра, арматура, трассировка) — каскад схем;
- нормоконтроль / приведение документов к факту и т.п.
Нормативные ссылки (через `norm-lookup`) и сбор паспортов — это уже НЕ состав. Хэндофф наружу оформлять строкой
«**→ <имя>-каскад**» в реестре замечаний / журнале каскада.

## Порядок (граф «Каскад изменений»)
**Акт (АОСР/АОМР) → Реестр №X → ВОР → ВСО → ПВ → Реестр-опись → таблицы/выноски в DWG (вручную)**.
На практике один из документов уже правлен (эталон) — тянем от него к остальным, сохраняя инвариант.

## Шаги
0. **Контекст из базы, не из памяти.** `id-volume-graph` (explain «Каскад изменений», query инвариант).
   Факт прошлой сессии перепроверить по источнику (часто расходится).
1. **Ориентировка — факт первичен.** Открыть РЕАЛЬНЫЕ файлы и замапить роли с нуля.
   ⚠ ПВ/ВОР могут лежать НЕ в папке тома (в родительской), в одной книге с другой системой; листов много.
   Реестр-опись — отдельный файл, перечисляет ТОЛЬКО документы тома (ПВ/ВОР туда не входят).
   Большие .xlsx не читать MCP-методами (лимит токенов) — openpyxl-скриптом с выводом в файл UTF-8.
   Зафиксировать контрольные суммы из эталона (напр. сифоны=N, подводки=M).
2. **Бэкап** всех затрагиваемых файлов ДО правок (откат надёжнее починки поверх).
3. **Каскад (tool `id-cascade`).** Вставлять позиции в КОНЕЦ блока каждого помещения, по тем же помещениям и
   количествам, что в эталоне. Дисциплина merge: снять → вставка СНИЗУ-ВВЕРХ → восстановить со сдвигом.
   Перенумерация **по прогонам** (reset на строке-заголовке таблицы), не сквозняком — заодно чистит старые дубли.
   №Акта/Дата/Ед.изм — от соседних строк, НЕ выдумывать. Идемпотентность: если позиция уже есть — стоп.
4. **Учёт изменённых листов.** Артефакт «Изменённые листы (для пересборки)»: что перепечатать, что НЕ трогается.
   Листаж: **1 лист = 2 стр PDF (двусторонняя)**; добор строк БЕЗ добора листов → опись НЕ меняется.
   Пересчитать опись только если у документа-строки описи изменилось число ЛИСТОВ.
   DWG: таблицы вставлены статично (Data Link нет) → пометить к ручной перевставке.
5. **QA (оба, строгий failure-mode).** `excel-validator` (merge целы, нумерация без дыр/дублей, суммы, типы,
   целостность книги) + `auditor` (инвариант сумм во ВСЕХ проекциях, наименования из источника, №акта/дата от
   соседей, инженерная осмысленность, нет двойного учёта). Оба должны дать PASS/PASSED со списком пунктов.
6. **Учётные артефакты.** Реестр замечаний (статус), журнал каскадных правок (строка: что→каскад→листы→статус),
   ответы ТН (формулировка).
7. **Хвосты пользователю.** Открыть правленую книгу в Excel и пересохранить (openpyxl не кэширует значения
   формул). Перевставить таблицы в DWG. Финальная пересборка PDF тома — отдельным шагом.

## Грабли (преднабор)
- openpyxl save убирает кэш значений формул во всей книге → открыть в Excel и пересохранить.
- не называть скрипты именами stdlib-модулей (`inspect.py` → circular import в openpyxl).
- ВОР с несколькими прогонами нумерации + предсуществующими дублями → перенумеровывать по прогонам.
- ПВ должна быть ОДНА (могут быть разные РЕДАКЦИИ, но не две живущие копии). Если в книге два листа ПВ —
  это рассинхрон редакций = ДЕФЕКТ: свести к одной верной (сверять каждую ось — состав и даты — с источниками:
  ИС/трассы для наличия позиции; дату акта — по САМОМУ акту, опись лишь для перекрёстной сверки), а не править обе параллельно.
- ПДЦ (договорная цена) — коммерческий слой ВНЕ тома ИД, проектный комплект ≠ as-built — в каскад не входит.
