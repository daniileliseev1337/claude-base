# chains/ — именованные цепочки

## Что такое chain

**Chain** — формализованный многошаговый сценарий, оркестрируемый
main-агентом. Это first-class сущность нашей базы:

- имя в формате `chain:<verb-noun>`;
- описание в `chains/<verb-noun>.md` с YAML frontmatter;
- триггеры для распознавания запроса;
- последовательность шагов с verify-критериями на каждом.

## Зачем

- Стандартизация типовых рабочих pipeline'ов (ПЗ к проекту, обработка
  скан-PDF, генерация комплекта документов).
- Явная декомпозиция многошаговых задач на проверяемые шаги
  (Karpathy §4).
- Защита от «забыл сделать stage X» при ручном повторении сценария.

## Когда использовать chain vs прямой вызов агента

- **Chain** — задача с 3+ шагами и зависимостями между шагами.
- **Прямой вызов агента** — одношаговая задача или ad-hoc запрос.

См. также скилл [[chains-pattern]] — методология «когда и как создавать
новые chains».

## Правила оркестратора

1. **При получении запроса** main-агент проверяет таблицу триггеров
   chains. Если совпадает — запускает chain.
2. **При запуске chain** main читает `chains/<name>.md` целиком, потом
   выполняет шаги последовательно.
3. **Verify-критерий обязателен на каждом шаге** — без него шаг
   не считается завершённым (см. [[karpathy-guidelines]] §4).
4. **На любом шаге chain может прервать flow** — если verify не пройден,
   main останавливается и эскалирует пользователю.
