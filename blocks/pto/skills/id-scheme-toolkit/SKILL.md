# id-scheme-toolkit — xlsx-ВОР → нативная ACAD_TABLE на исп. схеме

Инструментарий **W-слоя** блока ГЧ: заменяет статичные OLE-таблицы Excel в DWG
исполнительных схем на **нативные ACAD_TABLE**, генерируемые из xlsx-ведомости.
Обезличено (логика, не данные; пути/листы — аргументы). Обкатан на томе Р-4-ВиВ:
«Об 1» — 15 поз single (4 кол), «Об -1» — 88 поз multi (4 кол), «М т -1» — 129 поз
1-колонкой (5 кол с «Примечание», узкая свободная полоса) — все verify PASSED.

## Зачем
OLE-объекты Excel в DWG программно НЕ редактируются (отсюда правило каскадов «таблицы
перевставлять вручную»). Этот toolkit делает перевставку АВТОМАТИЧЕСКОЙ и нативной:
таблица становится живым ACAD_TABLE (правится поячеечно, как штамп), вмещается в лист
целиком, проверяется на совпадение значений с источником.

## Ключевое требование (жёсткое, от пользователя)
Таблица показывается **ЦЕЛИКОМ и вмещается в лист**. Большая → авто-масштаб шрифта,
не лезет в одну колонку → разбивка на несколько колонок рядом (с балансировкой и
повтором шапки). **ACAD_TABLE ПРОЗРАЧНА** (в отличие от OLE-картинки с белым фоном) →
ставить только в СВОБОДНУЮ зону, иначе план/выноски просвечивают = каша. Поэтому
автопоиск зоны — не опция, а часть пайплайна.

## Правило размещения (от пользователя, 2026-06-18)
1. **Основной режим — в свободном месте текущего листа**, играя размером самой таблицы
   (шрифт/колонки), как раньше двигали OLE. Автозона + auto-fit это и делают.
2. **Fallback — если на листе совсем нет места:** перенести таблицу на РОДСТВЕННЫЙ лист
   той же группы работ (план -1 / план 1 / аксонометрия — это комплект одного вида работ),
   обычно на аксонометрию, а на исходном листе оставить текст-ссылку «см. объёмы на листе N».
   (Группа = общий префикс layout: «М т -1/М т 1/М т акс» = монтаж трубопровода.)
3. Перекрывать план белым фоном (имитировать OLE) — НЕ делаем по умолчанию; только если
   пользователь явно попросит для конкретного листа.

## Пайплайн (из любого cwd; `TOOLS` = папка этого скилла/tools)
Бэкенд autocad-mcp должен быть **file_ipc** (живой AutoCAD): `system status` → если `ezdxf`,
сделать `system init`. Вывод LISP с кириллицей приходит cp1251 → при дампах конвертировать.

**0. Бэкап DWG** (откат надёжнее починки). Открыть целевой DWG, перейти на нужный layout.

**1. Автопоиск свободной зоны** (растровый — надёжнее векторного):
   - LISP: `(setvar "CTAB" "<layout>")(command "_.ZOOM" "_E")` → прочитать
     `(getvar "VIEWCTR")` и `(getvar "VIEWSIZE")` → `PNGOUT C:/temp/clean.png`
     (лист БЕЗ будущей таблицы; если заменяем OLE — сначала удалить OLE2Frame, потом рендер).
   - `python TOOLS/find_free_zone.py C:/temp/clean.png <ctrx> <ctry> <vsize> <colw> <gap>`
     → пишет `C:/temp/free_zone.txt` (XRIGHT/YTOP/WIDTH/HEIGHT/MAXCOLS) + печатает зону.

**2. xlsx → данные:** `python TOOLS/xlsx2acadtable.py <xlsx> <sheet> <tidx> <layout>
   <ins_x> <ins_y> <cols_csv> <style> <thmax> <margin> <target_h> <renumber>
   [<gap> <xstart> <ytop> <maxcols>]` → `C:/temp/table_data.txt` (cp1251).
   - single-column: 12 аргументов; multi-column: +4 (gap/xstart/ytop/maxcols из free_zone).
   - `xstart` = XRIGHT (правый край зоны, колонки растут влево), `target_h` = HEIGHT зоны.
   - `cols_csv` = ширины подколонок мм (напр. `6,96,12,12` = №/Наим/Ед.изм/Всего ≈126).
   - `renumber 1` = шапка №, позиции 1..N сквозной нумерацией (выносок типовых нет).

**3. Построение в AutoCAD:** скопировать `TOOLS/gen_table.lsp` → `C:/temp/` (load по
   ASCII-пути, в пути базы кириллица ломает `load`), `(load "C:/temp/gen_table.lsp")` →
   `(c7:build)` (single, auto-fit высоты) или `(c7:build-multi)` (multi, баланс колонок).
   Заменяя OLE — сначала удалить: перебрать block layout, `AcDbOle2Frame` → `vla-Delete`.

**4. Верификация (ревьюер графики):** `(c7:verify-dump)` → `C:/temp/acad_pairs.txt`
   (пары № : последняя колонка из построенных таблиц) → `python TOOLS/verify_table.py
   <xlsx> <sheet> <tidx> <renumber>` → PASSED/FAILED (сверка всех позиций vs xlsx,
   reuse `parse_table`). + рендер `PNGOUT` полного листа и крупный зум — глазами проверить
   вмещение, отсутствие перекрытий (план/выноски/угловые штампы вроде «Приложение к АОСР»).

**5. Сохранить DWG** (`drawing save`) — только после PASSED и подтверждения пользователя.

## Tools (слой 3 — детерминированная логика в коде)
- `find_free_zone.py` — растровый детектор: бинаризация PNG → downsample-сетка (+clearance)
  → max-empty-rectangle (гистограммный) → перевод пикс→коорды листа (mmpp=VIEWSIZE/px_h).
- `xlsx2acadtable.py` — резка листа на ВОР-таблицы, классификация строк (title/header/
  section/data), нормализация нумерации → cp1251 data-файл. **N колонок** (cols_csv задаёт
  число и ширины: 4 кол «№/Наим/Ед/Всего» или 5 «…/Примечание»). Авто-определение колонки
  «Всего»/«Объём»/«Кол-во» (`detect_qty_col`, пишется QTYCOL в data — может быть НЕ последней).
  `sanitize()` — символы вне cp1251 → ACAD-коды (∅/⌀/Ø → `%%c` = ⌀; узкие пробелы → space),
  иначе диаметры теряются как «?». `parse_table()` и `detect_qty_col()` импортируемы (verify).
- `gen_table.lsp` — `(c7:build)` single auto-fit | `(c7:build-multi)` multi: измерение высот
  пробной таблицей → жадная раскладка по колонкам (повтор title+header, перенос висячих
  секций, балансировка target=total/K) → бинпоиск шрифта чтобы колонок ≤ MAXCOLS → K таблиц,
  прижатых к правому краю. `(c7:verify-dump)` — пары для сверки.
- `verify_table.py` — сверка `acad_pairs.txt` vs xlsx (reuse parse_table), exit 0/1.

## Грабли (накоплено боями)
- `execute_lisp` таймаутит на тяжёлых однопроходных скриптах → дробить; `RegenerateTableSuppressed`
  на время заполнения (уже в build-list); дамп больших данных — в файл, не в payload.
- Координаты прокси-MText роняют LISP «bad list» через `vla` (bbox/InsertionPoint/entget-via-vla)
  → брать через `ssget "_X"` + DXF (assoc 10/1/40). Так найдена рамка «Приложение к АОСР».
- `load` с кириллическим путём ненадёжен → класть `gen_table.lsp` в `C:/temp/` (ASCII).
- Большие xlsx читать openpyxl-скриптом (MCP excel переполняет токены; консоль cp866 ломает).
- data-файл в cp1251 → символы вне неё (∅ ⌀ Ø, узкие пробелы) теряются как «?» → `sanitize()`
  (∅→`%%c`). Если на листе всплыл новый «?» — добавить замену в `_SUBS`.
- Колонка «Всего» может быть НЕ последней (5-кол с «Примечание») → verify по QTYCOL, не по last.
- Автопоиск: рендерить ЧИСТЫЙ лист (без будущей таблицы), иначе зона «съедается» старой таблицей.
- ZOOM "_W" центрирует окно под аспект картинки → для точного кадра считать аспект или зумить по bbox.

## Границы
- Это **W-слой** (таблицы ВОР на схемах). R-слой (чтение/сверка ИС: длины DIMENSION,
  номенклатура выносок) — отдельные прототипы; количество приборов с выносок НЕ берётся
  (выноска = типовой узел, кратит) — см. [[gch-is-anatomy]].
- Значения/наименования берём ИЗ xlsx как есть (факт первичен), toolkit не сочиняет данные.
- Сопоставление наименований ИС↔ВОР (формулировки разные) — зона `auditor`/модели, не регэкспа.
