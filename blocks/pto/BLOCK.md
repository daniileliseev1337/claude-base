# Блок ПТО — каскады производственно-технического отдела

```yaml
name: pto
status: experimental          # → stable решением хаба + CHANGELOG
head_agent: pto-engineer      # глава семьи (существующий агент общего слоя)
pilot_machines: [DANIIL]      # hostname машин пилота (хаб, 2026-06-10)
roles: [ПТО, инженер ПТО]     # кому предлагать при status: stable
```

## Зона блока

Каскады задач ПТО: объёмы → ВОР → спецификации → исполнительная документация → том ИД.
Поэлементная проработка каскадов: где скрипт, где агент, где субагент — на боевых
задачах хаба, затем шлифовка через feedback.

| Новые агенты семьи | в разработке | → `blocks/pto/agents/` (класть сразу сюда) |

⚠ Скиллы блока НЕ попадают в авто-реестр скиллов Claude Code (он сканирует только
`~/.claude/skills/`) — это и есть «0 токенов у неактивных». Доступ: id-engineer и каскад
читают файлы блока по абсолютным путям (Required reading). Если понадобится авто-триггер
скилла у пилотов — доработать активацию /sync-base (копировать и skills, не только agents).

Агенты общего слоя `pto-engineer` (глава) и `id-engineer` НЕ переезжают — они нужны
и вне каскадов. Блок их **дополняет** семьёй узких агентов.

## Разграничение триггеров (защита от роутинг-коллизий)

- «посчитай объёмы / ВОР / спецификация» → `pto-engineer` (как сейчас, вне блока).
- «собери том ИД / каскад ИД / прогони каскад» → каскады блока (через выбор пользователя).
- «измени состав / приведи в соответствие / инвариант ПВ≡ВСО≡ВОР» → `id-volume-cascade`.
- «таблицу ВОР в чертёж / перевставить ведомость в DWG / OLE не правится / не вмещается в лист»
  → `id-scheme-cascade` (W-слой, skill `id-scheme-toolkit`).
- АОСР/журналы/реестры поштучно → `id-engineer` (как сейчас).

## Активация и UX

По правилам `blocks/README.md`: триггер темы → main предлагает выбор свободной формой
(каскад блока / Workflow с предупреждением о токенах / индивидуально) + рекомендация.
Вопрос задаёт **оркестратор до спавна** (субагент не может спрашивать пользователя).

## Feedback

Блок использовался в сессии → в feedback-отчёт абзац `## блок pto`: какой каскад,
что сработало, где сломалось, сколько шагов потребовало ручного вмешательства.
