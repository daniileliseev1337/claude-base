## Зона блока

Каскады задач ПТО: объёмы → ВОР → спецификации → исполнительная документация → том ИД.
Поэлементная проработка каскадов: где скрипт, где агент, где субагент — на боевых
задачах хаба, затем шлифовка через feedback.

Над каскадами — **governance-слой осей** (`axes/`): зоны ответственности и порядок
доведения тома ИД до сдачи (Р→Б→1-6), с протоколом возврата через кросс-блокеры.
См. секцию «Слой осей» ниже.

## Слой осей (governance процесса) — `axes/`

Над каскадами стоит **governance-слой осей** — КАК блок доводит том ИД до сдачи: зоны ответственности, порядок, протокол возврата. Канон правил: `axes/ОСИ ПТО — стратегия (зоны, порядок, протокол возврата).md`; навигатор — `axes/Граф осей ПТО (узлы · рёбра · гейты).md` + queryable graphify-граф `axes/graphify-out/` (15 узлов; открыть `graph.html` или `graphify query`).

**Оси:** `Р` ТН-замечания (рамка) → `Б` независимый аудит оригинала → `1` Состав → `2` Документы качества → `3` Графика → `4` Акты → `5` Пересборка PDF → `6` Приёмка → выдача. Каждая ось работает только в своей зоне; чужое расхождение → **кросс-блокер** в `02_СОСТАВ <том>`, переход к следующей оси — при нуле открытых блокеров (гейт перед осью 5 — самый жёсткий).

**Связь осей с активами блока:**
- Ось 1 (Состав) → каскад `chains/id-volume-cascade.md`.
- Ось 3 (Графика) → каскад `chains/id-scheme-cascade.md` + skill `id-scheme-toolkit`.
- Оси Б и 6 (аудит/приёмка) → skill `id-tom-priemka` (Workflow постранично, opus, PNG).
- Граф **осей** (`axes/graphify-out/`) = процесс «как доводим»; граф **устройства тома** (`skills/id-volume-graph/`) = предметка «как устроен том» (акты/ведомости/инвариант/документы качества), на которую оси опираются. Два графа комплементарны.

Снимок статуса конкретного тома по осям — в `01_СТАТУС <том>.md` объекта, не здесь.

## Разграничение триггеров (защита от роутинг-коллизий)

- «посчитай объёмы / ВОР / спецификация» → `pto-engineer` (как сейчас, вне блока).
- «собери том ИД / каскад ИД / прогони каскад» → каскады блока (через выбор пользователя).
- «измени состав / приведи в соответствие / инвариант ПВ≡ВСО≡ВОР» → `id-volume-cascade`.
- «таблицу ВОР в чертёж / перевставить ведомость в DWG / OLE не правится / не вмещается в лист»
  → `id-scheme-cascade` (W-слой, skill `id-scheme-toolkit`).
- АОСР/журналы/реестры поштучно → `id-engineer` (как сейчас).
- «прими том / приёмка тома / проверь постранично / независимый аудит оригинала» → skill `id-tom-priemka` (оси Б и 6; Workflow, opus, PNG).
