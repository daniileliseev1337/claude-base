# Chain: id-scheme-cascade — каскад исполнительных схем (DWG) тома ИД

Назначение: пайплайн правки графической части (исп. схемы в DWG). Текущий охват — **W-слой**:
замена статичных OLE-таблиц ВОР на нативные ACAD_TABLE из xlsx (toolkit `id-scheme-toolkit`).
Обезличено (методология). Связано: skill `id-scheme-toolkit`, skill `id-volume-graph`,
chain `id-volume-cascade` (каскад состава — сюда приходит хэндофф «→ схемы-каскад»),
skill `acad-recreation` (autocad-mcp toolkit, cp1251, PNGOUT).

## Когда запускать
- Замечание ТН / правка по графике: «таблицу ВОР в схему», «перевставить ведомость в DWG»,
  «OLE-таблица не правится», «таблица не вмещается в лист».
- После `id-volume-cascade` (изменился состав ВОР) → таблицы в DWG помечены к перевставке →
  сюда (нативная замена вместо ручной).
- Сборка/актуализация исп. схем тома по листам.

## Граница каскада (что входит)
**Входит (W-слой, автоматизировано):** замена/вставка таблиц ВОР как нативных ACAD_TABLE —
вмещение целиком, автопоиск свободной зоны, авто-разбивка/баланс колонок, стиль под штамп,
верификация значений ACAD↔xlsx.
**R-слой (полнота самой схемы) — осваивается, методика ниже:** длины участков, переходы
диаметров ►◄, арматура/ЗРА, трассировка, УГО; правка выносок (MULTILEADER) — **ОСВОЕНО**;
правка штампа листа. См. раздел «R-слой — методика оформления схемы ИС».
Состав ведомостей (ПВ≡ВСО≡ВОР≡ИС) — это `id-volume-cascade`, не сюда.

## R-слой — методика оформления схемы ИС (длины / размеры / переходы / арматура / выноски)

> **Канон (источник — практик; НЕ копировать готовые тома как «эталон»):** исп. схема (ИС)
> рисуется ПО ФАКТУ монтажа (как реально смонтировано) → отличается от РД/ПД. Сначала смотрят
> факт на объекте → рисуют как смонтировано → берут РД и правят под факт.

**Длины и размеры:**
1. **Длины меряются ПО DWG** — подложка/чертёж сделаны 1к1, размеры в DWG = реально проложенные.
   НЕ требовать внешних «фактических замеров монтажника» — DWG и ЕСТЬ факт. (Ловушка: счесть
   «длин нет в источниках» и объявить пункт невыполнимым — ошибка.)
2. **Размеры/длины — ТОЛЬКО на ПЛАНАХ.** Аксонометрия НЕ источник горизонтальных длин (схематична).
   На аксонометрии: отметки высот + наглядность опусков/подъёмов + подсчёт фасонки (опуск «Г» =
   2 отвода данного Ду + опуск N м — чего на плане сверху не видно). Размеры на аксонометрию НЕ ставить.
3. **По аксонометрии — только ВЫСОТНЫЕ длины:** верх.отметка − ниж.отметка = длина вертикального
   участка трубы. Горизонтальные — только с плана.
4. **Σ(длин по Ду) ≈ метраж ВОР по Ду — с ДОПУСКОМ.** Идеально отрисовать/измерить нельзя →
   погрешность норма. Кол-во в ВОР/ПВ иногда задаётся иначе (прораб «купили X / надо закрыть X
   неважно что по факту проложено Y»). Расхождение Σдлин vs ВОР — **НЕ чинить молча**, это тонкий
   момент согласования (уточнять). Это допуск к инварианту ПВ≡ВСО≡ВОР≡ИС по длинам.

**Переходы Ø и арматура:** места изменения диаметра ►◄ — по точкам смены проставленного Ду;
недостающая запорная арматура — по ВОР (краны/задвижки/обр.клапаны/воздухоотводчики/компенсаторы),
размещение по факту узлов. Кол-во приборов/арматуры — из ВОР, НЕ с выносок (типовой узел кратит).

**Выноски оборудования (MULTILEADER) — ОСВОЕНО:** текст в коде DXF 304; читать/писать через
`vla-get/put-TextString` по handle (НЕ вешает мост, в отличие от vla-чтения ACAD_TABLE).
Актуализация под ВОР/ВСО = точечная замена устаревшего ФРАГМЕНТА наименования (regex, устойчивый
к `\P`-переносам MTEXT), данные через cp1251-файл (не argv). Менять только наименования факта-замен;
проектные позиции, НЕ видные на листах тома (заморожены в VP), — не трогать.

**Простановка на тяжёлом DWG (десятки тыс. объектов):** MCP идёт «вслепую» (без живого курсора) →
размещение в точках узлов рискованно; делать ПОЭЛЕМЕНТНО с рендер-контролем (PNGOUT зоны), начинать
с пилота на одном узле. Бэкап DWG до правок; save после verify + ревьюер графики PASSED.

## Порядок (W-слой, на лист = на таблицу)
**Бэкап → автозона → данные → построение → верификация → рендер-сверка → сохранение.**
Источник истины значений — xlsx-ВОР (факт первичен), toolkit ничего не сочиняет.

## Шаги (детали — в SKILL.md `id-scheme-toolkit`, раздел «Пайплайн»)
0. **Контекст из базы.** SKILL.md `id-scheme-toolkit`; при нужде устройства тома — `id-volume-graph`.
   Бэкенд autocad-mcp = file_ipc (`system status`→`init` при ezdxf). Открыть DWG, перейти на layout.
1. **Бэкап DWG** до правок.
2. **Удалить старые OLE** (AcDbOle2Frame) на листе, если заменяем.
3. **Автопоиск зоны** (`find_free_zone.py` по чистому PNGOUT) — растровый, обходит план/выноски/
   угловые штампы. Прозрачность ACAD_TABLE делает этот шаг обязательным.
4. **Данные** (`xlsx2acadtable.py`) — резка/классификация/нумерация → cp1251 data-файл.
   Зону подставить: xstart=XRIGHT, ytop=YTOP, target_h=HEIGHT, maxcols=MAXCOLS.
5. **Построение** (`gen_table.lsp`: `c7:build` | `c7:build-multi`) в AutoCAD.
6. **QA (ревьюер графики, строгий failure-mode):**
   - значения: `c7:verify-dump` + `verify_table.py` → PASSED (все позиции сходятся с xlsx);
   - визуал: рендер PNGOUT полного листа + крупный зум — вмещение целиком, НЕТ перекрытий
     (план, выноски, «Приложение к АОСР», штамп), читаемость, шапка/секции/нумерация.
   Не PASSED или есть перекрытие → откат к бэкапу, скорректировать зону/параметры, повтор.
7. **Хвосты пользователю.** Показать рендер, по подтверждению `drawing save`. Зафиксировать в
   журнале каскадов (лист → таблица заменена → verify статус).

## Грабли (преднабор; полный список — SKILL.md)
- ACAD_TABLE прозрачна (OLE были непрозрачны) → строго в свободную зону, иначе нечитаемо.
- auto-fit, максимизирующий шрифт, раздувает таблицу на чертёж → для схем компактный режим
  (thmax ≤1.5), прижать к свободному углу.
- Координаты прокси-MText (штампы/приложения) — через `ssget`+DXF, не vla (роняет «bad list»).
- `load` LISP — по ASCII-пути (копировать в C:/temp), кириллица в пути базы ломает.
- Рендерить ЧИСТЫЙ лист для автозоны (без будущей таблицы).
- Количество приборов НЕ брать с выносок ИС (типовой узел кратит) — только из ВОР/xlsx.
