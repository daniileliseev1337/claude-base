# blocks/ — крупные доменные блоки с активацией по роли

Блок = самодостаточный доменный комплект (семья агентов + скиллы + каскады + память),
который нужен НЕ всем сотрудникам. Живёт в общем репо (один sync, один граф, один
feedback), но **активируется по профилю ПК** — у неактивных пользователей не стоит
ни одного токена контекста, кроме одной роутинг-строки в CLAUDE.md.

## Устройство

```
blocks/<имя>/
├── BLOCK.md       # манифест: статус, состав, триггеры, активация, feedback
├── agents/        # семья агентов блока (регистрируются только при активации)
├── skills/        # скиллы блока
├── chains/        # каскады (последовательности агентов/скриптов)
└── memory/        # уроки и справки блока
```

## Правила

1. **Активация** — только через `/sync-base` (профиль ПК в `.local-state/blocks.json`).
   Активация = копирование `blocks/<имя>/agents/*` в `~/.claude/agents/` с префиксом
   `block-<имя>-` (копии gitignored). Деактивация удаляет копии. После — restart Claude Code.
2. **Статус** в BLOCK.md: `experimental` — /sync-base предлагает только машинам из
   `pilot_machines`; `stable` — предлагается всем по описанию роли. Перевод в stable —
   решение хаба + запись в CHANGELOG.
3. **Роутинг** — в CLAUDE.md (роутинг-гейт) одна строка на блок. Триггер темы блока →
   main предлагает пользователю выбор СВОБОДНОЙ формой (AskUserQuestion): каскад блока
   (если активен; что сделает, плюсы/минусы) / Workflow (предупредить про токены) /
   индивидуально под задачу — и даёт свою рекомендацию. Блок не активен → одна строка:
   «есть блок <имя>, включается /sync-base».
4. **Feedback** — если блок использовался в сессии, в feedback-отчёт добавляется
   абзац `## блок <имя>` (что запускалось, что сработало/сломалось).
5. **Переиспользование** — query-before-build и «гравитация блоков»
   (см. `skills/skill-development`): инструмент, нужный второму блоку, переезжает
   в общий слой.
6. **Граф** — единый по всему репо, включая blocks/. Хаб актуализирует при правках
   (CLAUDE.md, секция graphify).
