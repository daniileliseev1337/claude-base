⚠ РОУТИНГ-ГЕЙТ (похоже на доменную задачу).

Запрос напоминает содержательную доменную задачу — генерацию/правку артефакта: спецификация / ВОР / расчёт / подбор оборудования / смета / КП / УПД / письмо / акт-ИД / ответ экспертизе / раздел ПД-РД.

ДО начала работы выполни роутинг-гейт (CLAUDE.md, «Архитектура: main → domain agent → auditor»):
1. Напиши видимой ПЕРВОЙ строкой: «Роутинг: <домен> → агент <имя>» ЛИБО «Роутинг: инлайн — <почему ни один из доменных агентов не подходит>».
2. Есть доменный агент — спавни ЕГО (Task, subagent_type = имя файла агента), НЕ делай доменную работу сам и НЕ через general-purpose. Агента выбирай по frontmatter-description (там живые фразы пользователя).
3. Перед спавном субагента — спроси у пользователя модель (Токен-дисциплина).
4. Перед выдачей артефакта — ОБЯЗАТЕЛЬНО ревьюер (auditor / профильный валидатор).

Если это НЕ задача-генерация (просто вопрос или диалог) — игнорируй это напоминание.
