**Правило для STOP-процедуры (добавление к шагам 2 и 5):**

- **Шаг 2.5 (новый): Agents-аудит явный.** Сверить available agents
  (видны Claude в system prompt) с эталоном **16 наших агентов** выше.
  Если каких-то нет в available list — **первой строкой** ответа
  предупредить пользователя в формате:
  `⚠ Агенты не подгружены: <имена>. Restart Claude Code (hot-reload отсутствует).`
  Если все 16 на месте — отметить это в подтверждении (шаг 5).
- **Шаг 5 (расширение):** строка подтверждения теперь содержит **оба**
  счётчика:
  `✓ прочитан CLAUDE.md (MCP: X/11, agents: Y/16)`
  где X — MCP из шага 2, Y — агенты из шага 2.5.

**Почему отдельный шаг:** Claude Code загружает агентов **один раз
при старте сессии** (hot-reload отсутствует). Если пользователь только
что сделал `git pull` новых агентов в работающей сессии — они НЕ
подхватятся. Этот шаг ловит ситуацию «обновил базу, забыл перезапустить»
и явно подсказывает пользователю restart.

**Что считается «not loaded»:** агент есть в `~/.claude/agents/<имя>.md`,
но при попытке `Agent(subagent_type='<имя>')` Claude Code вернёт
`Agent type '<имя>' not found`. Эталон сверяется со списком в system
prompt `Available agent types for the Agent tool`.
