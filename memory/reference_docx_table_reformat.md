# Переформат docx-таблиц (акты ВСО/ИД) под новую шапку — метод + грабли

Загружать по триггерам: «переделай N актов docx под новую шапку», «переформат таблиц
актов», «разнести наименование на колонки», «серийники с чертежей».

## Контекст

Массовое переформатирование актов (ВСО / АОСР / ИД) под новую N-колоночную шапку
заказчика. Детерминированный pipeline на python-docx + выверенный словарь (json),
verify-скрипт. Кейс: 9 актов, раздел `<раздел>`, 5 зданий, шапка 6 колонок.

**Решение по инфраструктуре (инкремент):** пока — этот memory-кейс. Скилл
`vso-reformatter` поднять при **2-м** кейсе переформата актов (словарь маппинга
специфичен для заказчика/проекта — рано обобщать в скилл).

## Метод (детерминированный трансформер)

1. **Чтение таблицы:** python-docx, но **таблица может быть вложена в ячейку** —
   `doc.tables` отдаёт только верхний уровень. Если «таблица не найдена», а визуально
   она есть → рекурсивно обходить `cell.tables`. (Симптом: акт «пустой» в скрипте, но
   у пользователя заполнен N строк — см. также A8.8 про online-only файл.)
2. **Разнесение свободного текста** наименования на колонки (Наименование / Тип-марка /
   Завод) — **выверенным словарём json**, НЕ регэкспами. Словарь редактируемый, ~100
   позиций, проверяется человеком.
3. **Завод-изготовитель — только из факта** (бренд в тексте / паспорт / торговая марка),
   иначе `—`. Пример сопоставлений марка→завод: SANICUBIC=SFA, Duofix=Geberit,
   насосы SSP/VSL/SK-712=Wilo. Не угадывать бренд.
4. **Новые колонки + границы** добавлять программно; сохранять стиль исходной таблицы.
5. **Verify-скрипт** (обязательно): проверить новую шапку, число строк (== исходному),
   опечатки, отсутствие «утечки брендов» (завод подставлен где не должен).

## Грабли (специфичные)

- **Серийники смонтированного оборудования** живут только на **исполнительных чертежах
  (рукопись)**, не в реестрах/паспортах (там «б/н»). На фото низкого разрешения цифры
  неоднозначны (3↔5, 7↔1, 4↔9) — **не угадывать**, помечать `?`/`(проверить)`, отдавать
  на сверку с бумагой. (Правило CLAUDE.md #1.)
- **PowerShell `>` пишет UTF-16** → Python пишет файл сам в utf-8 (anti-patterns A8.7).
- **Yandex.Disk online-only** файл читается пустым (anti-patterns A8.8).
