## Рабочий шаблон ФТ (функциональных требований) к залу — переиспользуемый

На зал/зону единый блок:

1. **Назначение** — что зал делает, какой опыт у посетителя.
2. **Сценарий** — пользовательский флоу (вошёл → триггер → реакция системы).
3. **FR-X.Y** (Functional Requirements) — функции системы, нумерованные.
4. **ТТ-таблица** (технические требования) — параметры (яркость, разрешение, SPL, угол...).
5. **Конструктив** — крепления, ниши, фальшпол, кабельные трассы.
6. **Контент** — что показывается/звучит; кто разрабатывает (в поставке интегратора или
   отдельный подряд — частый открытый вопрос).

Сводные разделы тома ФТ: инфраструктура (электрика, СКС, климат, безопасность, МГН) +
таблица мощности по залам; технологический стек; **раздел открытых вопросов** (бюджет,
типы спорных решений — фиксировать ОВ, не выдумывать).

## Ловушки домена

- ⚠ **ЦЕНЫ И АРТИКУЛЫ — НЕ ПО ПАМЯТИ.** Самооценка автора по ценам «по памяти» — ±50%,
  модели могли быть сняты/переименованы. Перед сдачей спеки — **harvest актуальных
  прайсов** ([[harvest_workflow]]) или явный вопрос пользователю/поставщику. Помечать
  «Ориент.» в спеке и выносить в открытые вопросы. (Правило CLAUDE.md #1 «не выдумывать».)
- **Количества из концепта** («планшеты» во мн.ч. без числа, «аудиосистема» без конфигурации)
  — не додумывать молча; разумный дефолт (напр. Atmos 7.1.4) фиксировать как открытый вопрос.
- **Excel-спека** — формулы через `apply_formula`, RU-запятая ломает формулу, считать строки
  при блочной записи (см. [[excel-helper]] ловушки 10-13).
- **Ревью бинарных артефактов:** наши ревьюеры (`word-checker`/`excel-validator`) уже имеют
  свои `mcp__*` (fix `6d79b4f`) — работают из коробки; на старой несинхронизированной базе
  субагент MCP не видел → обновить базу (`/sync-base`).
