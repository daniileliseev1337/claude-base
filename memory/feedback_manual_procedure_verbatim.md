При переносе технической ПРОЦЕДУРЫ из руководства/мануала (порядок коммутации, шаги
пуска/останова, диагностика, последовательность отключения автоматов) в свой документ —
переносить **дословно по нумерации источника**, не пересказывать «своими словами».

**Why:** перестановка шагов при пересказе — типовая ловушка. Реальный кейс (Eaton 93T,
ошибка 85): в артефакте порядок отключения автоматов был «байпас→выход», в руководстве —
«выход→байпас»; поймал `auditor`. Неверный порядок коммутации питания = риск для
оборудования и персонала.

**How to apply:**
1. Держать перед глазами выдержку источника (раздел, страницы); шаги копировать по номерам.
2. Единицы и термины — как в источнике (кейс: кВА↔кВт перепутаны; руководство оперирует кВт).
3. Не уточнять числа, которых нет в источнике («2–4 шкафа» → «несколько», если не подтверждено).
4. Ревью бинарного docx: `auditor`/ревьюер без office-MCP не читает docx — передавать ему
   ИЗВЛЕЧЁННЫЙ текст + дословные выдержки первоисточника в промпте (внешние сайты вроде
   Scribd/eaton.com недоступны из среды субагента → иначе честный FAIL «не подтверждено»).
   См. правило фасировки артефакта в [[CLAUDE]] (секция Ревьюеры) и [[auditor]].
