# Named chains — `~/.claude/chains/` (вынесено из CLAUDE.md CORE)

Загружать по триггерам: «составь pipeline», «многошаговая задача», «собери комплект
документов», «весь процесс от ТЗ до сдачи». Методология создания — скилл `chains-pattern`.
Полный референс — `~/.claude/chains/README.md`.

Для типовых многошаговых сценариев (3+ stage с зависимостями) — именованные цепочки
в `~/.claude/chains/`. Первичная сущность оркестратора, рядом с доменными агентами.

| Chain | Когда срабатывает | Status |
|-------|-------------------|--------|
| `chain:docx-from-template` | «сгенерируй документ по шаблону», «заполни шаблон» | ✅ |
| `chain:pdf-scan-extract` | «разбери скан-PDF», «извлеки данные из скана» | ✅ |
| `chain:project-doc-pack` | «собери комплект ПЗ + спецификация + ведомости» | 🚧 stub (зависит от stage-decomposition `designer`) |
| `chain:design-stamp-corrections` | «правки по замечаниям», «исправить замечания», «обновить спецификацию по экспертизе», «синхронизировать штамп» | ✅ |
| `chain:upd-to-spec-reconcile` | «пришёл УПД, обнови реестр», «применить УПД к Спец», «обнови S по УПД», «сверь УПД с реестром» | ✅ |

**Логика оркестратора:**
1. При получении запроса — проверить таблицу триггеров chains. Если совпадает —
   запустить chain (прочитать `chains/<name>.md` целиком, выполнить шаги последовательно).
2. Если не совпадает — стандартный flow main → доменный агент → ревьюер.
3. **На каждом шаге chain — обязателен verify-критерий** (Karpathy §4).
4. При провале verify — chain останавливается, эскалация пользователю.
