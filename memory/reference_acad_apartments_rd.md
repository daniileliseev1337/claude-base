# РД квартир через AutoCAD MCP — накопитель метода и граблей

> **Это СКЕЛЕТ.** Цель — копить сюда находки по мере обучения Claude делать рабочую
> документацию квартир через AutoCAD MCP. Когда этап стабилизируется — переносим в
> скилл/агента (см. «Фазность» ниже). Не теория — складывать ТОЛЬКО проверенное на деле.

## Цель и инкремент (план развития)

Обучаем Claude делать РД квартир (планы/узлы/ведомости по ГОСТ СПДС) через AutoCAD MCP,
поэтапно. Развитие — строго инкрементом (как PyRevit/AV):

- **Фаза 1 (сейчас):** отладить каждый этап end-to-end до **воспроизводимости**.
  Грабли и рабочие приёмы — в этот файл. Критерий перехода: этап стабильно повторяется.
- **Фаза 2:** стабильный этап → **скилл** с детерминированными LISP-скриптами и шаблонами
  (блоки/слои/штамп) в `tools/` (skill-development: код в tools/, не генерить LISP заново).
- **Фаза 3:** 3-5 стабильных этапов → **агенты-этапы** + оркестратор (chain). Лимит глубины 2:
  `main (оркестратор) → агент-этап → ревьюер` (нормоконтроль площадей/проходов/ГОСТ).

**Не строить агентов раньше Фазы 3** — спекулятивная команда закрепит хрупкий процесс.

## Общие AutoCAD MCP грабли (НЕ дублировать — ссылки)

База уже накоплена, при работе свериться:
- [[2026-05-21_acad-com-cookbook]] — COM-приёмы AutoCAD.
- [[reference_autocad_pdf_overlay_mcp]] §«Главные уроки» — backend `file_ipc` (`system init`),
  `execute_lisp` не возвращает stdout (писать в файл, читать `Get-Content -Encoding default`,
  AutoLISP пишет CP1251), `vla-Open` + отдельный `vla-Activate`, модальные диалоги вешают IPC,
  печать только `vla-PlotToFile`, `entmake`/`entmod` для блоков.

Складывать сюда рабочие, проверенные артефакты (когда появятся):
- LISP-скрипты этапов (детерминированные, с записью результата в файл).
- Библиотека блоков (двери/окна/сантехника/мебель) + слои по ГОСТ СПДС.
- Шаблон штампа (форма 3/5), экспликации, ведомости отделки.
- Verify-скрипты (площади сходятся, слои корректны, штамп заполнен).
