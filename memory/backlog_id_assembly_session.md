## Что за кейс

Связка из 3 этапов вокруг **сборки и проверки тома ИД** инженерных систем (тома ВиВ,
объект `<объект>`):
1. **Переформат актов** (ВСО) под новую шапку — уже частично в [[reference_docx_table_reformat]].
2. **Устранение замечаний техзаказчика/технадзора** — реестры замечаний (№/суть/решение/статус/
   тип работы), категоризация (наши/не наши; правит Claude vs сбор документов), материалы
   замечаний внутрь папок ИД.
3. **Реверс-инжиниринг устройства тома ИД** (НОВОЕ, ценное): как сшит том — комплекты вокруг
   актов + 3 прошивки; типы актов АОСР/АОМР/**АОУСИТО**; цепочка журнал→акт→материалы/Реестр→
   паспорта→схемы→ВСО/ПВ; нумерация листов = 2 стр PDF. + метод Workflow-картирования эталонного
   тома (постраничная карта).

## Почему важно

Переход от «делаем по образцу» к **формализованной методике сборки/проверки ИД** — основа:
- прокачать существующий агент `id-engineer` (агент) (сейчас формы РД 11-02/11-05; добавить устройство
  тома, реестр замечаний, картирование);
- возможно новые инструменты: распознавание замечаний ТН с фото/видео, реестр-артефакт, чек-листы.
