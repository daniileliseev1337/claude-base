## Failure-modes (что именно ломалось)

1. **Пересборка с нуля вместо правки живого файла** → фирменная шапка с логотипом
   **исчезла / встала вниз** («переворачивает»). НЕ пересобирать — резать/править оригинал.
2. **Плавающие (заякоренные) логотипы** (`w:drawing`, привязка к абзацам + section
   breaks). python-docx их не понимает: при нарезке якорь рвётся → логотип всплывает
   непредсказуемо (то вверх, то вниз).
3. **Переписывание абзацев целиком** → затёрло **табы и подчёркивания** полей
   «(ФИО)/(подпись)», линии `_____` разъехались. Это и есть «underline сбивает».
4. **Пачки пустых абзацев** (9-28) после блоков подписи: текст длиннее заглушки →
   выталкивает подвал на лишние страницы («всё уплыло»).
5. **Путаница начала/конца диапазона** удаления (`Range(0,start)` vs `Range(start,end)`)
   → сносил лишнее / оставлял хвост чужого документа.
6. **Слепая замена аббревиатур** (ПФР→СФР) → «Пенсионный фонд РФ (СФР)» франкенштейн.
7. **Сдавал не глядя** — «готово, проверено» без PDF-рендера и взгляда на вёрстку =
   ЛОЖНЫЙ РАПОРТ. Пользователь проверял за меня.
8. **Кириллица в пути** через bash heredoc → искажение cp1251/utf-8 → `PackageNotFoundError`.
9. **е/ё** («Отчет»/«Отчёт») → файл то находится, то нет.

## ВЕРИФИЦИРОВАННЫЙ рабочий метод

1. **СНАЧАЛА полный дамп структуры** (НЕ заполнять вслепую): разделы (`doc.sections`),
   header/footer, якоря `w:drawing`, индексы всех таблиц, стили. Понять материал → потом метод.
2. **Плавающую шапку проще убить и поставить inline-блоком в самый ВЕРХ** каждого
   документа: таблица 2 колонки (логотип | название), без границ + линия-разделитель +
   реквизиты. Логотип вытащить **настоящий** из `word/media/imageN.jpeg` оригинала.
   Inline не плавает → всегда сверху, предсказуемо.
3. **Точечная замена внутри нужного run, НЕ трогая табы/подчёркивания** шаблона.
   Заполнять ЗНАЧЕНИЕ, не разрушая выравнивание поля. НЕ переписывать абзац целиком.
4. **Учитывать пустые абзацы** после подписей (могут вытолкнуть подвал).
5. **Кириллический путь:** скрипт через Write-tool (гарантир. UTF-8); файл искать
   `os.listdir()` по ASCII-подстроке или `Get-ChildItem -Filter '*_02.docx'`; НЕ
   передавать кириллицу как bash-аргумент.
6. **Замена аббревиатур → охранять контексты** (полное имя vs аббревиатура, «4-ФСС»).
7. **VERIFY-ГЕЙТ (обязателен):** каждый «готово» = **PDF-рендер постранично + взгляд
   глазами** + read-back (нет `{{...}}`, ожидаемые значения на месте) + агент
   `word-checker`. Никаких «проверено» на веру. Это прямое нарушение Karpathy #4, если пропустить.

## Опциональный assist (не обязателен — фикс выше методологический)

- **docx-plus** (Python, MIT) — style-cascade resolver: читает «почему этот абзац
  underline/14pt» и применяет стили Word-native. Для случаев, где форматирование
  плывёт из-за direct formatting. НОВЫЙ (1★) → валидировать на реальном файле.
- **loadfix/python-docx** (форк, MIT) — `append_document`, границы таблиц cross-doc.
- **OfficeIMO** (.NET, MIT, 461★, PowerShell) — зрелый движок если нужен .NET.
