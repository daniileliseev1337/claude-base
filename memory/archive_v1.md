# Архив старой базы v1 — НЕ источник действующих правил (вынесено из CLAUDE.md)

Загружать по триггерам: упоминание папки «Обучение и развитие Claude под наши задачи»,
«claude-stroy-base», «block-behavior», «CLAUDE-core.md», «правило 8/11/14», «v1».

На ноутбуке владельца (`C:\Users\Даниил\`) может находиться папка
`Desktop\_ARCHIVE_Обучение и развитие Claude под наши задачи\` — это **архив старой базы
`claude-stroy-base` v1** (тег `archive/v1-legacy`). **Действующая база — только
`~/.claude/`**, синхронизируется через `claude-base` репо.

**При работе в любой сессии:**
- **Не читать** в STOP-процедуре файлы из v1-папки как источник правил:
  `CLAUDE-core.md`, `block-behavior.md`, `block-design.md`, `block-pto.md`,
  `block-id.md`, `block-bim.md`, `block-docs.md`, `block-tools.md`, `block-infra.md`,
  `block-session-end.md`, `STATE-current.md`. Устаревший слой методики.
- **Не цитировать** «правило 8/11/14» и т.п. из v1 `CLAUDE-core.md`. Действующие
  правила пронумерованы 1-7 в секции «Универсальные правила работы» CLAUDE.md.
- **Не использовать** скиллы `stroy-*` и агенты из v1-папки. Нужное — переносится
  явно в `~/.claude/skills/` и `~/.claude/agents/` по запросу пользователя.

**Если функциональность из v1 нужна** — обсудить и **перенести явно** в текущую базу,
не цитировать как «правило» из v1.

В v1-папке лежит маркер `_ARCHIVE_v1_DO_NOT_USE.md` с тем же содержанием.
