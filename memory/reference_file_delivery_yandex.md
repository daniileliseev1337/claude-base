# Reference — доставка файлов владельцу на телефон (через синк Я.Диска)

Как отдать сгенерированный файл (HTML/виджет/документ) так, чтобы владелец забрал с телефона.
Проверено 2026-06-30 на ПК-хабе (Windows). Тип: reference.

## Рабочий канал: синкаемые папки Я.Диска (ExternalDirs)

На ПК включена синхронизация ВНЕШНИХ папок Я.Диска (реестр
`HKCU\Software\Yandex\Yandex.Disk.2` → `ExternalDirs`). У владельца туда добавлены Рабочий стол
(`<USERPROFILE>\Desktop`) и пара рабочих папок на отдельном диске.

Кладёшь файл в любую из синкаемых папок → клиент `YandexDisk2` (должен быть запущен) синкает
своим каналом → владелец забирает с телефона. Реальные пути смотри в `ExternalDirs` на конкретном ПК.

НЕ класть в корень `<USERPROFILE>\YandexDisk\` — он локально пуст (online-only), сбивает.

## ВАЖНО (iOS): чем открывать интерактивный HTML на iPhone

- Открывать через приложение **Documents (Readdle)** — у него полноценный WebView, JS/фильтры/
  тапы работают. Documents цепляется к облакам (Я.Диск/Dropbox/Google), открывает файл из
  синк-папки напрямую.
- **НЕ через Safari/браузер** — из облака HTML либо не открывается, либо теряет интерактив (JS
  не исполняется, виджет «мёртвый»). Проверено владельцем 2026-06-30.

## Что НЕ работает (проверено фактом)

- `SendUserFile` (вложение в чат Claude Desktop) — у владельца не скачивается.
- LAN `http.server` — только если телефон в той же сети (дома); на работе бесполезно.
- cloudflared quick-tunnel и upload на catbox С ПК — системный VPN-адаптер рвёт исходящее
  (QUIC к Cloudflare таймаутит, curl timeout). Туннель/облако-upload с такого ПК не работают.
