Правила работы по документам качества ИД, заданные пользователем (объект «<объект>», ИД ВиВ, 2026-06-09). Применять во всех каскадах документов качества.

**Why:** без них ходим кругами — берём неактуальные версии, ищем не там, нарушаем логику приёмки ТН.

**How to apply:**
1. **Сначала PDF тома, потом инвентаризация.** После сверки с замечаниями/хвостами — открыть собранный PDF тома и посмотреть, какие паспорта/сертификаты ВЛОЖЕНЫ по факту (что есть/чего нет). Только потом инвентаризировать. Сверяться И с PDF, И с реестрами (опись, Реестры №X).
2. **Правило дат документ↔акт:** дата окончания действия документа НЕ раньше начальной даты акта; дата начала действия документа НЕ позже последней (главной) даты акта. Даты «с-по» акта — на 2-й странице акта. НЕ привязываться к формулировке: бывает «с/до», «с/по» и др. вариации.
3. **Последовательность поиска документа:** (а) обыскать весь интернет, много сайтов, в т.ч. зарубежные; (б) если не скачать, но есть ссылка — прислать ссылку; (в) искать в похожих разделах (другие объекты/тома — реально сработало для Dyson через Ручку/Треугольник); (г) спросить пользователя; (д) письмо снабжению/поставщику. Гарантийные талоны — могут не найтись (исключение). Метод запросов — [[feedback_id_doc_search_method]].
4. **Складирование найденных документов:** в папку «Паспорта и сертификаты» (создать если нет) → подпапка с названием того, ПОД ЧТО прикладывается → сортировать по реестру (как пойдут в ИД) и по реестрам. Имя файла = материал/изделие + тип документа (как в «…\Оборудование\14. Трубный сифон Geberit …»).
5. **Не можешь найти/сомневаешься в файле (проект/ИД/реестры/акты) — спроси, где искать.** Иначе возьмёшь неактуальную версию (чуть не ушёл в чертёж `МТМ-1. Р_ВиВ. ВиВ.pdf` вместо тома `ИД ВиВ стр 1 02.03.2026.pdf`).
6. **Итоговая папка устранённых замечаний — копия со штемпелем даты**, замены делать в копии, оригинал не трогать.
7. **Затронуло позицию ВОР/ПВ из другого блока — НЕ устранять на месте.** Если в ходе текущего блока выяснилось, что могла поменяться позиция в ВОР/ПВ (наименование, состав, количество) — устранение перенаправить в профильный блок (каскад состава) и зарегистрировать СОБСТВЕННОЕ ЗАМЕЧАНИЕ-БЛОКЕР в журнале каскадных правок (секция «Собственные замечания — блокеры пересборки»): без его устранения сбор/пересборку ИД не выполнять. Разведка факта (grep по книге) — допустима и нужна, чтобы замечание было точным; правка — нет.
