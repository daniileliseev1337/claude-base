# Приём: добывание документов (паспорта/СС/datasheet) под антиботом — браузер пробивает

> Источник: пользователь, 2026-06-09, docs-каскад ИД (кейс LUNDA). Это **приёмы, НЕ жёсткие правила**.

## Лестница добывания (эскалация, не догма)
1. Обычный fetch: `WebFetch` / `fetch` MCP / `exa` / `firecrawl`.
2. **Антибот (401/403, пусто, Cloudflare/CAPTCHA) → НЕ сдаваться. Реальный браузер `playwright`:**
   открыть карточку товара у поставщика → найти **прямую CDN-ссылку** (напр. `cdn.lunda.ru/...`) →
   скачать документ. Браузерные инструменты пробивают там, где HTTP-fetch/Firecrawl ловят 401.
3. `r.jina.ai` — дополнительный обход чтения (см. [[web_access_r_jina_fallback]]).

## Приёмы поиска (диагностика «почему не нашёл» — кейс LUNDA)
- Идти к **B2B-поставщикам** (lunda, сантехкомплект и т.п.), а не в гос-реестры/розницу.
- Запрос держать **товарным** (бренд+артикул), не юридически-перегруженным: «декларация/сертификат на …»
  уводит в реестры и розницу.
- **Watermark на документе = источник.** ©LUNDA на PDF → искать напрямую у LUNDA.
- На первом антиботе **не сдаваться** — это сигнал сменить инструмент (браузер), а не «документа нет».

## Без категоричности
Форму документа (декларация / сертификат / паспорт / инф.письмо) НЕ фиксируем — что подходит, решается
по позиции (типы взаимозаменяемы). Это приёмы под конкретный поиск, а не обязательная процедура.
