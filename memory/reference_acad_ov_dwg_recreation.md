# Воссоздание ОВ-проекта квартиры из PDF в DWG через autocad-mcp

> Накопитель метода и граблей. Цель — научить Claude **полностью пересоздавать ОВ-проект
> квартиры из PDF в DWG по эталону фирмы** (слои, динблоки, листы, штампы — «лицо фирмы»).
> Развитие — инкрементом: memory → скилл (tools/ LISP) → агенты (Фаза 3, см. ниже).
> Не теория — складывать только проверенное на деле.

## Задача и эталон

Из PDF-подложки + растровых схем (вентиляция/кондиционирование/решётки/люки/электрика)
воссоздать ОВ-DWG по эталону фирмы: 24 слоя (префикс `!`), динблоки воздуховодов/фасонок/
оборудования, 11 листов, рамка-штамп (динблок). Стиль аккуратный, «без неточностей».
Эталон: `<эталон>.dwg` (готовая квартира — стиль/динблоки) + `<библиотека динблоков>.dwg`.

## Pipeline (8 этапов) — статус-трекер

Статусы: `⬜ не начат` / `🔄 отлаживается` / `✅ доказан`.

- **✅ Э0. Каркас** — копия эталона КАК ЕСТЬ в ASCII-путь `C:\temp\*.dwg` (модель не чистить).
- **✅ Э1. Подложка** — PDFIMPORT → калибровка → чистка → монохром+блок на `!Подложка`.
- **🔄 Э3. Вентиляция** — воздуховоды приток(кр)/вытяжка(син)/рециркуляция, фасонка, вентоборуд.
  (механизм разводки динблоком доказан; разводка по растру — приблизительная, согласовано).
- **⬜ Э4. Кондиционирование** — трассы, внутр./внеш. блоки, дренаж.
- **⬜ Э5. Воздухораспределители и люки** — диффузоры, решётки, люки.
- **⬜ Э6. Аннотации** — тексты (СТИЛЬ ФИРМЫ, не свой), размеры, выводы питания, подписи ИС.
- **⬜ Э7. Компоновка листов** — видовые, масштаб под А3, штампы; УДАЛИТЬ старую квартиру, навести видовые ТОЛЬКО В ФИНАЛЕ.
- **⬜ Э8. Верификация** — печать PDF, сверка с исходником, нормоконтроль.

**Рабочий приём:** новую квартиру строить РЯДОМ (свободная зона X≈750000), старую держать
референсом, удалять и наводить видовые только в финале.

## Ключевые рецепты (отлажены; детали — в отчёте-источнике)

- **Кириллица autocad-mcp** — вынесено в общий [[reference_autocad_mcp_cyrillic]] (вход Unicode
  в коде / выход cp1251 через файл; `drawing open` MCP врёт → COM `vla-Open`+`put-ActiveDocument`).
- **PDFIMPORT вектора:** `FILEDIA 0`, `PDFIMPORTMODE 7`, `-PDFIMPORT _F <pdf> 1 750000,0 1 0`;
  ложится одним блоком → Explode+Delete; OCG-слои PDF → `PDF_<имя>` (чистка фильтром).
- **Калибровка масштаба:** круглого масштаба нет (печать «по размеру листа»); коэффициент по
  2 одинаковым размерным подписям соседних равных сегментов (units_per_mm). На объекте было 0.9644.
- **Чистка подложки (НЕ огулом):** слои-в-выброс (размеры/DIST/NET/вспомог/плитка/LINE); двери —
  оставить чёрные полотна, удалить ОРАНЖЕВЫЕ (color 30) распашонки; PDF_0 — оставить дуги/эллипсы
  (сантехника), удалить 2-вершинные полилинии; текст — удалить размерные числа/экспликацию.
  **⚠ Фильтр по слою НЕДОСТАТОЧЕН** (дизайнер кладёт размеры на слои архитектуры) → +позиция
  (вне контура стен) +геометрия (длинные 2-точечные) +ОБЯЗАТЕЛЬНЫЙ визуальный обход скрином по зонам.
  **⚠ Чистить ДО слияния в `!Подложка`.**
- **Финал подложки:** сначала собрать в ОДИН блок (отдельное имя), ПОТОМ масштабировать блок
  (`vla-ScaleEntity`). НЕ масштабировать россыпь `ssget "_C"` — берёт только видимое → рассинхрон.
- **Динблок «Воздуховод»:** dyn-props напрямую (`Диаметр`/`Радиус`/`Расстояние1` double), НЕ через
  lookup-таблицу (не резолвится). Слой — `CLAYER` перед вставкой. Ориентация в РАДИАНАХ.
- **Рендер для проверки:** `plot_pdf`/`PlotToFile` дают пустой PDF → использовать `PNGOUT` зоны
  (`ZOOM _W` + `PNGOUT`), читать Read. Скрин ЗОНЫ, не весь план (мельчит).
- **Зависание (таймаут на тривиальном = модальный диалог):** PowerShell `AppActivate($pid)`+`SendKeys("{ESC}")`.
  Перед деструктивом — save; проверять охват выборки (`sslength`); откат `(command "_.U")`.

## Phase 3 — предложенная архитектура агентов (BACKLOG, НЕ строить сейчас)

Строить **только когда 3-5 этапов доказаны** (сейчас готовы Э0-Э1, Э3 в работе). Из отчёта:
- **Оркестратор `dwg-recreator`** — pipeline 8 этапов, чтение FACTS/схем, делегация. Depth-лимит 2.
- **tools/ (детерминированный LISP, не модель):** `cyrillic_dump.lsp`, `pdf_import_calibrate`,
  `backdrop_cleanup` (правила чистки параметрически), `block_and_scale`, `place_duct`
  (Воздуховод: диаметр/длина/ориентация/слой), `pngout_zone`. + Python: PyMuPDF probe (OCG/вектор-растр), cp1251-reader.
- **Ревьюер `dwg-backdrop-reviewer`** (read-only) — после чистки обязательный визуальный обход по
  зонам (PNGOUT): нет размеров/засечек/экспликации/распашонок; есть стены/окна/двери/мебель/сантех;
  вне контура стен пусто. Failure-mode строгий (без списка проверенных зон — НЕ пройдено).
- **Инварианты-гварды (в код):** вход кириллица в коде/выход cp1251 файлом; ASCII рабочий путь;
  фильтр+позиция+геометрия+визуальный обход; блок→масштаб; скрин зоны; save+сверка после деструктива;
  ничего не удалять огулом.
- **Отдавать человеку (не автоматизировать слепо):** финальная приёмка стиля, разводка по растру
  без вектора (приблизительно), спорные «оставить/удалить» (подсветка цветом → решение человека).
