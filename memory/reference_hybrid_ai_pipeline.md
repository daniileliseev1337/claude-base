# Гибридный AI-конвейер — методичка проектирования batch-AI-систем

**Формула: «код держит порядок, модель держит насмотренность».**
Детерминированный движок (парсинг, партии, запуск, учёт расходов — покрыт тестами)
+ AI только на творческих шагах (понять вход, подобрать решение, написать промпт,
оценить результат). Применимо к любым партиям: заявки, документы, чертежи,
семейства Revit, изображения — не только картинки.

## Каноническая схема

```
вход → [AI: понимание/творчество] → ИНСПЕКТИРУЕМЫЙ АРТЕФАКТ (JSON по схеме / промпт+эталоны)
     → [код: валидация] → [превью/ревью: человек] → [код: детерминированное исполнение]
     → [итерация ТОЛЬКО брака] → (выборы человека → улучшение промптов, ОБРАТИМО)
```

Это расширение нашего «Deterministic-first» (CLAUDE.md) до полного жизненного цикла:
+ возобновляемость, + точечная переделка, + самообучение с откатом.

## Чек-лист приёмов (проверять при проектировании ЛЮБОЙ batch-AI-системы)

| Приём | Суть | Вопрос себе |
|---|---|---|
| **Инспектируемый артефакт** | между моделью и исполнителем — артефакт по схеме; executor всеяден (не знает про AI) | где точка проверки ДО необратимого? (anti-pattern A11.1) |
| **Возобновляемость партии** | журнал состояния: перезапуск продолжает с места остановки | что будет при обрыве на позиции 730/1000? |
| **Точечная догенерация** | перегенерация ТОЛЬКО бракованных единиц | платная генерация → повтор всей партии = прямые деньги |
| **Ревью-UI** | один объект на экран + варианты + зум + хоткеи; отбор в 2 клика | человек — финальный судья; сколько кликов на решение? |
| **Самообучение на отборе** | выборы человека копятся и кормят промпты следующих партий; ОБРАТИМО (откат к проверенной версии промпта) | версионируются ли промпты? есть ли откат? |
| **Честная роль AI-QC** | авто-проверка только ПОМЕЧАЕТ брак (vision слабее глаза), не решает | не отдали ли AI финальное «годен»? |
| **Бюджет-контроль per-call** | учёт расходов на каждый вызов + конфиг per-клиент | платформенность вместо одноразового скрипта |
| **Fail fast, не молчаливый дефолт** | ошибка вычисления = исключение/маркер, не тихий 0 | anti-pattern A11.2 |

## Зрелые промпт-практики (из SKILL.md FamFab — в копилку)

- Ограничение вопросов пользователю (max 3–4), запрет спрашивать видимое на картинке;
- Уровни детализации Quick/Standard/Detailed с целевыми числами элементов;
- Математика спейсинга с обязательной самопроверкой суммы;
- «Пересобирай артефакт целиком, не патчь» — единый источник правды;
- v1-ограничения объявлены явно + `_TODO`-массив в самом артефакте.
