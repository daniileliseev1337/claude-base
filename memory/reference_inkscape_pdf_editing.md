# Редактирование вектор-PDF через Inkscape — ВЕРИФИЦИРОВАННЫЙ метод

Загружать по триггерам: «редактировать чертёж PDF», «убрать/подвинуть штамп»,
«удалить объект из вектор-PDF», «заменить штамп», «vector pdf edit», когда
provered surgery/белые заливки.

## Контекст: почему этот файл существует

До 2026-06-02 правка векторных чертежей-PDF проваливалась десятки итераций
(АХП Балашиха, см. session-reports 2026-05-22/27): PyMuPDF `apply_redactions` не
пробивает Form XObjects, pikepdf clip прячет а не удаляет, модель клала белые
заливки поверх. **2026-06-02 найден и проверен рабочий метод — Inkscape.**

## ВЕРИФИЦИРОВАННЫЙ метод (GUI)

Доказано на реальном К-7 чертеже (`ПСИ-158-06-01-СТМ.ЭТ`, «Архитектурное
освещение фасадов», плотный план с mesh-штриховкой). Сверка рендером до/после:
штамп удалён **начисто** (нет белой заливки, нет призрака), остальной чертёж цел,
**0 растеризации** (pages_with_raster_images 1→0), размер ~тот же (2.22→2.27 MB),
текст сохранён как текст (10388→6912 симв = ушёл только текст штампа). Перемещение
объектов тоже работает (подтверждено пользователем).

1. **Открыть** PDF в Inkscape (`Файл → Открыть`).
2. В окне импорта — вкладка **«Внутренний импорт»** (Internal), НЕ Cairo.
   Internal сохраняет **текст как текст** (редактируемый). Cairo превращает текст
   в кривые — хуже для правки.
   - Шрифты: «Нарисовать отсутствующие шрифты» — ок.
3. **Если падает/виснет на плотном чертеже** (тысячи объектов mesh-штриховки):
   импортировать **по одной странице** (вписать номер в «Страницы», не `all`).
   Краш на `all` — известная слабость Inkscape (грузит всё в DOM).
4. **Выделить** объекты инструментом «Выделение» (S). По штампу: клик; если
   выделяется группа — двойной клик внутрь или `Ctrl+Shift+G` разгруппировать.
   На проверенном файле выделялись и заливки, и текст, и линии.
5. **Delete** для удаления / тащить мышью для перемещения.
6. **Файл → Сохранить как → тип `PDF`**, в **новый** файл (оригинал не трогать).
   - Предупреждение «формат с потерей данных, сохранить как Inkscape SVG?» —
     **это норма**, «потеря» = внутренние SVG-фишки Inkscape, не чертёж. Нам нужен
     PDF → жмём сохранить PDF (НЕ «как Inkscape SVG»).
7. **ВЕРИФИКАЦИЯ объективная** (не на глаз): отрендерить результат (`pdf-mcp
   pdf_render_pages`) и сверить с оригиналом. Проверить `pdf_info`:
   `pages_with_raster_images` должно остаться 0 (если стало >0 — Inkscape
   растеризовал, плохо); размер не должен взлететь; дельта текста = только удалённое.

## Ловушки и ограничения

- **Производительность.** Inkscape **медленно** открывает/сохраняет большие
  чертежи, особенно на слабом ноуте без GPU. На томах это ощутимо — учитывать
  при батч-стратегии.
- **Краш на плотных `all`** → строго по одной странице.
- **«Потеря данных» при Save** → норма для финального PDF, жать PDF.
- **Не возвращаться к белым заливкам / content-stream surgery** — провалено,
  см. `anti-patterns.md` §A3.5. Inkscape — рабочая замена.

## Открыто (будущая работа)

- **Батч/многостраничность** через CLI `inkscape --actions` / `inkex` (Python) —
  не проверено. Для томов на N листов нужен постраничный заход без GUI.
- **Генерализация** — проверено на 1 чертеже; на других проверять рендером.
- **Вставка «правильного» штампа** — перемещение доказано, вставка готового
  блока-штампа end-to-end не догнана (не было готового штампа под рукой).

## Альтернативы (если Inkscape не тянет конкретный файл)

- **AutoCAD round-trip** (`PDFIMPORT → ssget _C → erase → PLOT`) — для файлов,
  которые Inkscape не открывает даже постранично. Отдельный проект (освоить
  autocad-mcp под наши штампы).
- **Коммерческий on-prem SDK** (Nutrient / PDFDancer Enterprise) — если нужен
  programmatic batch без GUI. Платно, procurement. См. harvest
  `session-reports/2026-06-02_word-pdf-tools-harvest/harvested/pdf-vector-editors.md`.
