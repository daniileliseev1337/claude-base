# Правка нанесённой графики на AutoCAD-PDF — чистый PyMuPDF через SVG-слой (БЕЗ AutoCAD)

Загружать по триггерам: «перерисовать разметку на чертеже-PDF», «починить
наложение CCTV/СС/ЭО поверх плана», «заменить нанесённую графику в PDF», когда
чертёж пришёл **только в PDF** (DWG нет) и DWG на выходе **не нужен**.

## Контекст: почему этот файл существует

Класс задачи: AutoCAD-PDF (печать через `pdfplotNN.hdi`, A1, вектор), поверх плана
— халтурная разметка (напр. видеонаблюдение: толстые красные лучи углов обзора,
кружки фикс. камер, квадраты поворотных). Требуется перерисовать нанесённое
(лучи → тонкие синие секторы, кружки/квадраты → значки камер с разворотом по
обзору), **базовый чертёж не трогать, без белых заливок**.

**ВЕРИФИЦИРОВАНО 2026-06-03** (`<объект>` ПЗУ/СПОЗУ): сделано полностью в Python,
векторное качество подложки сохранено (34764 пути, A1), заказчик доволен. Это путь,
который НЕ требует AutoCAD вообще — альтернатива [[reference_autocad_pdf_overlay_mcp]]
(тот через живой AutoCAD/PDFIMPORT, когда нужен DWG).

## Почему НЕ content-stream surgery / redaction (тупики)

Подтверждает [[anti-patterns]] §A3.5:
- сырая `w` (толщина) ≠ page-толщине: мастер-масштаб `matrix(.12,0,0,-.12,0,1684)`
  + Y-flip; «толстая» линия в потоке = пучок тонких штрихов `0 w` → поиск «6 w»
  даёт 0 совпадений;
- цвет нанесённого = родному красному плана (граница участка, кресты сноса) →
  по цвету нанесённое от родного НЕ отделить в потоке;
- `apply_redactions(REMOVE_IF_COVERED)` на плотном чертеже сносит базу под
  диагональными bbox, а саму разметку не убирает.

## Рабочий метод (SVG-слой)

1. **Опознать AutoCAD-PDF:** `pikepdf` docinfo → Producer `pdfplotNN.hdi`; вектор —
   `len(page.get_drawings())` большой, мало растровых картинок.
2. **Маска «нанесённого» из `get_drawings()`** — классификация примитивов:
   `'l'`=линия / `'re'`=прямоугольник / `≥3 'c'`=окружность; фильтр по
   **цвет+толщина в page-space** (напр. дал 25 лучей + 19 кружков + 4 квадрата).
3. **`page.get_svg_image(text_as_path=True)`** — чистая геометрия с явными
   `transform="matrix(.12,...)"` и presentation-атрибутами `stroke="#ff0000"`.
   Правится как XML (lxml). **Удаление узла = физическое удаление** (без белых
   заливок). Для каждого красного SVG-штриха середину `d` переводим матрицей в
   page-space (`x'=a·x+c·y+e`) и сверяем с маской из п.2.
   - Валидация маски: перекрасить найденное в синий → посинеть должно ТОЛЬКО
     нанесённое (если посинела база — маска неточна).
4. **Новый слой** в page-координатах (origin сверху-слева, y вниз, без transform):
   линии/секторы/значки; поворот — `<g transform="translate(...) rotate(...)">`.
5. **Обратно в вектор:** `fitz.open(svg).convert_to_pdf()`.
6. **Кириллица — ПОСЛЕ** SVG→PDF через `fitz insert_text(fontfile=arial.ttf)`
   (SVG `<text>` для кириллицы ненадёжен; `text_as_path=True` обязателен — иначе
   MuPDF подменяет шрифты AutoCAD и текст съезжает; минус — теряется текстовый слой).
7. **Метаданные** SVG→PDF обнуляет → восстановить `pikepdf.String(str(src.docinfo[k]))`.
8. **Проверка объективная:** diff-рендер src vs final (изменения только на разметке +
   лёгкий антиалиас текста), `len(get_drawings)` (вектор цел), агент `pdf-reviewer`.

## Геометрия камер (специфика этой задачи, переиспользуемо)

- Направление камеры = биссектриса = центр **минимальной дуги**, охватывающей все
  её лучи (НЕ среднее — ломается на веере).
- Привязка лучей к камерам — **эксклюзивная** (каждый луч → одна ближайшая камера);
  неэксклюзивная + усреднение → «камеры смотрят не туда».
- Камера с 1 лучом → дефолт-сектор 90° по лучу. Камера без лучей → только значок,
  направление **не выдумывать**.

## Ловушки (для базы)

- `fitz.get_text_length` НЕ принимает `fontfile` → `fitz.Font(fontfile=...).text_length(...)`.
- `page.new_shape()` требует `sh.commit()`.
- pikepdf docinfo: только `pikepdf.String(str(value))` (копирование чужого
  объекта → `ForeignObjectError`).
- Кириллица в путях через PowerShell→stdin ломается → работать через `.py` файл
  или ASCII-копию исходника (см. [[anti-patterns]] §A8.1).

## Когда какой метод

- **DWG на выходе НЕ нужен** → этот метод (SVG/PyMuPDF), легче и без AutoCAD.
- **Нужен DWG** или правка должна вернуться в чертёж → [[reference_autocad_pdf_overlay_mcp]].
- **Просто удалить/подвинуть объект** (штамп) в вектор-PDF → [[reference_inkscape_pdf_editing]].
