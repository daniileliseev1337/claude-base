# Урок: «у нас уже есть» ≠ «у нас работает» — verify before dismissing

> **ПОГЛОЩЕНО по веб-доступу:** актуальная механика веб-доступа — [[feedback_web_direct_access]]. Здесь оставлен только урок «verify before dismissing» (история harvest 2026-06).

> Источник: пользователь, 2026-06-01, серия harvest-анализов
> (Exa+Firecrawl, Higgsfield).

## Общий принцип
**Не закрывать harvest вердиктом ⚫ «у нас уже есть базовый стек»**
без верификации что этот стек реально работает на нужных задачах.

Аргументы «cloud dependency» / «metadata leak» / «overlap с X» применимы
ко множеству инструментов которые мы и так используем (WebFetch, MCP-серверы)
— они **не являются** основанием против нового инструмента.

## Подтверждённые failure-кейсы (обновлять)

### 1. WebFetch / WebSearch — 80-90% fail на сайтах
- WebFetch парсит **static HTML** → на SPA-сайтах с JS-рендером
  (производители оборудования, тендерные площадки) ловит пустоту.
- Антиботы (Cloudflare, CAPTCHA) блокируют простые HTTP запросы.
- WebSearch работает через Google API → rate-limit + плохо на нишевых темах.
- **Замена/дополнение**: Firecrawl (scraping с headless browser) +
  Exa (semantic search).

### 2. Adobe Firefly — не работает на наших задачах эмпирически
- `create_firefly_board`, `image_generative_expand` — generation проваливается
  на задачах команды.
- Subjective сигнал от пользователя: «по ощущениям не работает ни у кого,
  результат плохой когда подобные задачи были».
- Image/video generation у <организация> **фактически не покрыто** — это дыра, не
  «80% уже закрыто».
- **Замена/дополнение**: Higgsfield (Veo + Soul + 30 моделей).
- **Открытый вопрос**: работают ли Adobe MCP **editing** инструменты
  (`image_apply_*`, `image_adjust_*`, `video_create_quick_cut`) — нужна
  оценка пользователя, чтобы решить судьбу Adobe MCP в эталоне 9.

## Применение в будущих сессиях
1. **Перед вердиктом ⚫ через «у нас уже есть X»** — спросить пользователя:
   «X у вас работает на этих задачах? есть конкретные success/fail кейсы?»
2. Если ответ «не работает / не пробовали» — **ставить новый инструмент**,
   не пропускать. Параллельное существование лучше отсутствия.
3. Если ответ «работает» — попросить пример задачи где работает,
   и сравнить с тем что предлагает новый инструмент.
4. **Анти-паттерн**: домысливать «basic stack покрывает» из общих соображений
   совпадения категорий. Категория совпала ≠ задача решается.
