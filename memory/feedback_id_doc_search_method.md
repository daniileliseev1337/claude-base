## Порядок поиска (правило пользователя, не нарушать)
1. **Весь интернет, мультисайтово** — включая зарубежные источники, не зацикливаться на одном сайте.
2. **Ссылка, если не скачать** — если файл за сессией/ботозащитой и не качается, сохранить прямую ссылку.
3. **Похожие разделы / аналоги** — если точного документа нет, искать на серию/семейство/аналог.
4. **Спросить пользователя** — прежде чем сдаваться.
5. **Запросить у поставщика** — последняя ступень.
Домен НЕ угадывать; ссылку на документ брать из карточки товара (раздел «Документы/Сертификаты»),
не конструировать URL. «Документа нет» — только после визуального просмотра нужной карточки на
правильном домене (см. ШАГ 0 — скриншот, в [[feedback_web_direct_access]]).

## Приём 1 — Google Dorking (основной метод добычи)
Операторы загоняют поиск прямо в публичный индекс PDF/техлистов, минуя навигацию по сайту:
- `filetype:pdf site:<производитель>.ru "паспорт изделия" <артикул>` — паспорт на конкретный артикул.
- `filetype:pdf ("сертификат соответствия" OR "декларация") <бренд> <модель>` — документ качества.
- `intitle:"index of" <бренд> pdf` — открытые директории с пачкой документов на сайте.
- `site:<производитель>.ru (ext:pdf OR ext:doc) (РЭ OR "руководство по эксплуатации")` — РЭ/КС.
Зарубежный производитель — те же дорки на англ.: `datasheet`, `installation manual`, `certificate`.
Это методология (не установка) — работает из любого поисковика, усиливает каскад добычи.

## Приём 2 — инструменты добычи и верификации источника (легальный OSINT-сабсет)
Раздаётся всем как справочник; CLI-инструменты — на dev-окружении (WSL/Kali), web — в браузере.

| Инструмент | Что делает в нашей задаче | Доступ |
|---|---|---|
| **Photon** | Краулинг сайта производителя/поставщика — собрать все PDF/техлисты/сертификаты с публичных страниц батчем | CLI |
| **Wayback Machine** | Архивная версия карточки/техлиста (производитель обновил или удалил документ — берём из архива) | web, free |
| **Intelligence X** | Поиск по архивам/пастам/снапшотам — историческая версия страницы поставщика | web, freemium |
| **VirusTotal** | Безопасное открытие PDF/DOCX-вложения от поставщика + репутация домена из КП до клика | web, freemium |
| **DNSDumpster** | Экспресс-карта DNS контрагента (поддомены, MX) без регистрации — найти реальный портал документов | web, free |
| **Hunter.io** | Контакты/формат корпоративной почты поставщика по домену — кому слать запрос на документ | web, freemium |
| **httpx** | Batch-проверка живости доменов из списка контрагентов — отсев мёртвых до отправки запросов | CLI |
| **OSINT Industries** | Верификация контакта поставщика по email/телефону — связанные аккаунты/профили | web, freemium |
| **GetContact / Truecaller** | РФ: реальное имя организации/сотрудника по телефону из КП, «спам»-метка | web/app |
| **ТелПоиск (TelPoisk)** | РФ: поиск юрлица/ИП по телефону через агрегатор открытых баз | web |

Реестры реквизитов (ЕГРЮЛ/АРШИН/Росаккредитация) и санкции — отдельно в [[supplier-due-diligence]]
и `agents/auditor.md`. РФ-госсайты за иностранным egress — через скилл [[ru-gov-access]].

## Ловушки
- `<!DOCTYPE>`/HTML вместо `%PDF` в скачанном — ссылка из выдачи битая, брать из карточки товара.
- Скан-документ без текстового слоя проверять моделью `sonnet`, не haiku (haiku врёт в изготовителе/типе).
- Серия/семейство: один сертификат на линейку моделей — это НЕ «документ не на тот артикул».
- Чистые метаданные ≠ подлинность; при сомнении — ELA + сверка реквизитов в реестре ([[supplier-due-diligence]]).
