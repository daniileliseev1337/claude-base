# Auto-memory — политика (вынесено из CLAUDE.md CORE)

Загружать по триггерам: «запомни», «сохрани в память», «auto-memory», когда решаешь
что записать в `~/.claude/projects/<id>/memory/`.

Claude Code автоматически ведёт память на каждый проект в
`~/.claude/projects/<project-id>/memory/MEMORY.md`. **Это не CLAUDE.md** — две независимые вещи:

| | CLAUDE.md | auto-memory |
|---|---|---|
| Что | Статические правила | Накопленные факты |
| Кто пишет | Пользователь / installer | Сам Claude |
| Где | `~/.claude/CLAUDE.md` | `~/.claude/projects/<id>/memory/MEMORY.md` |
| Перенос между ПК | Через installer | Локально, не переносится |

**Когда сохранять в auto-memory:**
- **user**: роль, окружение, специфика работы («работает в ПТО», «1С УНФ», «ПК без админ-прав»).
- **feedback**: пользователь скорректировал подход (с **Why** и **How to apply**).
- **project**: текущие задачи, дедлайны, мотивации решений (относительные даты → абсолютные).
- **reference**: где живут внешние ресурсы («сметы — в \\\\srv\\smeta\\», «нормативы — в C:\\norms\\»).

**Не записывать:**
- Пароли, API-ключи, ПДн, банковские данные.
- Дублирование того что уже в CLAUDE.md.
- Эфемерное («сейчас работаю над X»).
