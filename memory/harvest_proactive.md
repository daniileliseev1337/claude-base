## Harvest-workflow — проактивные триггеры (дополнение к CORE)

CORE-секция `Harvest-workflow` описывает **что и где** искать, фильтры
кандидатов и адаптацию. Этот раздел — **когда** проактивно запускать.

### Триггеры

- **Первое появление нового типа файла в сессии** (по расширению или
  типу задачи). Дедупликация: повтор того же типа в той же сессии —
  пропустить.
- **Старт новой задачи** (пользователь сказал «новая задача:», «давай
  делать X», и т.п.).

### Алгоритм

1. **Проверить свою базу** — `~/.claude/skills/` и `~/.claude/agents/`.
   Если есть подходящий инструмент — работать им, harvest не нужен.
2. **Если нет** — **НЕ молча искать.** Спросить через `AskUserQuestion`:
   > «Новая задача типа <X>. Поискать готовые инструменты (2-5
   > кандидатов на GitHub / MCP registry / Anthropic skills) или
   > работать сразу своими средствами?»
3. Если пользователь сказал «найди» — запустить логику `/harvest <query>`
   (см. `commands/harvest.md`).
4. Если «работай сразу» — пропустить harvest для этого типа задачи в
   этой сессии, запомнить решение в контексте.

### Источники

Порядок и список источников — **источник истины `commands/harvest.md`** (действующий скилл):
pre-check встроенных skills (anthropic-skills + наши, 0 установки) → **GitHub + skills.sh равноправно**
→ MCP registry. Не дублировать здесь.

> Устаревшее «GitHub первым временно, перевернём при >5 скиллов» — **снято**: условие давно
> выполнено (25+ скиллов в базе), действующий порядок — выше (по `commands/harvest.md`).

### Объём кандидатов

**2-5**. Если фильтрация даёт меньше 2 — не застревать, выдать что
есть или сказать «не нашёл подходящего, работаю своими средствами».
Никаких «100 кандидатов» — это переусложнение и риск безопасности.

### Фильтр кандидатов

Источник истины — `commands/harvest.md` §«Фильтр кандидатов» (<12 мес / ≥50 stars / лицензия:
MIT-семейство зелёный, GPL — флаг, нет LICENSE — код не копируем). Не дублировать критерии здесь.

### Куда сохранять заметки

- **Per-session** (текущие находки этой сессии):
  `session-reports/<YYYY-MM-DD>_<тема>/harvested/<repo>.md`
- **Накопленная база** (общая для всех ПК, через auto-sync):
  `~/.claude/harvested/<repo>.md`

При завершении сессии релевантные заметки переносятся / копируются в
общую базу.

### Когда НЕ применяется

- Уже есть свой скилл/агент в `~/.claude/skills/` или `~/.claude/agents/`
  под задачу — используем его.
- Рутинная задача (просто чтение, простое редактирование) — инструмент
  не нужен.
- В сессии уже спрашивали про этот тип задачи (дедупликация).
- Тривиальный one-off — написать самим быстрее чем искать.
