# Правило формирования ИД: даты документов качества vs дата акта

ФУНДАМЕНТАЛЬНОЕ правило формирования ИД (того же ранга, что инвариант ПВ≡ВСО≡ВОР и принцип каскада изменений).

## Правило
Действительность документа качества (сертификат соответствия / декларация / паспорт / отказное письмо) проверяется
на **ДАТУ АКТА** (АОСР/АОМР/АОУСИТО), к которому он приложен, — а НЕ на сегодняшнюю дату. Разбивка по типу документа:

1. **Документ с периодом «с … по …»** (декларации, сертификаты соответствия): дата акта должна попадать
   **ВНУТРЬ периода** — дата «с» ≤ дата акта ≤ дата «по».
2. **Документ с одной датой** (паспорт, сертификат/паспорт качества партии «от …», информационное письмо):
   срока действия НЕТ → проверяется только **дата выдачи ≤ дата акта** (документ не может быть выдан позже работ).
3. **Отказное письмо** — частный случай (2): НЕ имеет срока действия (подтверждает статус продукции, который
   не «истекает»). Даже если в тексте письма указан «действителен 1 год» — для ИД оно НЕ считается просроченным;
   важна только дата выдачи ≤ дата акта.

## Why
Документ подтверждает качество материала на момент монтажа/приёмки, зафиксированный актом. Сертификат, истёкший
сегодня, но действовавший на дату акта, — ВАЛИДЕН в томе. И наоборот: документ, просроченный уже на дату акта, —
нарушение, даже если «свежий» по другим меркам. Реальные промахи (исправлены): (а) СС материала с действующим на
дату акта сроком ошибочно помечен «просрочен» — оценивал на сегодня; (б) к отказному письму применили срок «1 год»,
хотя к отказным правило срока не относится.

## How to apply
Взять дату акта-источника (у каждого акта своя). Для каждого приложенного документа: определить тип (период «с-по» /
одна дата / отказное) → проверить по соответствующему пункту. «Просрочки» сортировать ОТНОСИТЕЛЬНО ДАТЫ АКТА, не today.
Тот же принцип — у приказов о назначении ответственных (дата приказа ≤ дата акта; при нескольких на лицо — последний
по дате ≤ акта). Связано: [[id-doc-dates-rule]], [[feedback_id_doc_search_method]], [[feedback_web_direct_access]].
